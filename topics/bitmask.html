Bitmask Techniques

Phase 14 Medium

📖 What are Bitmask Techniques?

Bitmask techniques use binary representations of integers to efficiently encode sets, flags, and states. Common operations include checking, setting, clearing, and toggling individual bits — all in O(1) time.

💡 Key Insight

An integer with n bits can represent a set of n elements. Bit operations (AND, OR, XOR, NOT, shifts) replace loops and conditionals with single CPU instructions — making code faster and more compact.

🌍 Real-Life Analogy

🎛️ Control Panel Switches

A bitmask is like a row of switches — each switch represents a feature or flag. You can flip any switch independently, check which switches are on, or toggle groups of switches simultaneously. Hardware engineers use this exact concept for control registers.

JavaScript — Common Bitmask Patterns
// Count number of 1 bits (Hamming Weight)
function hammingWeight(n) {
    let count = 0;
    while (n) {
        n &= (n - 1); // Clear lowest set bit
        count++;
    }
    return count;
}
// hammingWeight(11) → 11 = 1011 → 3 bits

// Check if n is a power of 2
function isPowerOfTwo(n) {
    return n > 0 && (n & (n - 1)) === 0;
}
// 8(1000) & 7(0111) = 0 → true. 6(110) & 5(101) = 4 → false

// Generate all subsets of a bitmask
function allSubsets(mask) {
    const subsets = [];
    for (let sub = mask; sub > 0; sub = (sub - 1) & mask) {
        subsets.push(sub);
    }
    subsets.push(0); // Empty subset
    return subsets;
}

// Use bitmask as a set (e.g., track seen lowercase letters)
function hasAllUniqueChars(str) {
    let seen = 0;
    for (const ch of str) {
        const bit = 1 << (ch.charCodeAt(0) - 97); // 'a'=bit 0, 'b'=bit 1, ...
        if (seen & bit) return false; // Already seen!
        seen |= bit;
    }
    return true;
}

// Swap two numbers without temp variable
function swap(a, b) {
    a ^= b;
    b ^= a;
    a ^= b;
    return [a, b];
}

⚠️ Common Mistakes

❌ JavaScript 32-bit limitation

Bitwise operators in JS work on 32-bit signed integers. For bits > 31, use BigInt. Also, 1 << 31 gives a negative number (sign bit).

❌ Confusing & with &&

& is bitwise AND (operates on bits). && is logical AND (operates on truthiness). 5 & 3 = 1 but 5 && 3 = 3.

❌ Forgetting operator precedence

Bitwise operators have LOW precedence. if (n & (1 << i) !== 0) is wrong! Use if ((n & (1 << i)) !== 0) with extra parentheses.

🃏 Flash Cards

What does n & (n-1) do?
It clears the lowest set bit of n. Example: 12 (1100) & 11 (1011) = 8 (1000). Used for: counting set bits (repeat until 0), checking power of 2 (result is 0 for powers of 2).
How do you check if the i-th bit is set?
if (n & (1 << i)). This creates a mask with only bit i set, then ANDs with n. If the result is non-zero, bit i is set. Remember to add parentheses due to operator precedence!
