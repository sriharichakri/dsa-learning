Hashing & HashMaps

Phase 2 Beginner Friendly

📖 What is Hashing?

Hashing is the process of converting data (a key) into a fixed-size number (hash code) using a hash function. This number tells us exactly where to store or find the data — giving us near-instant O(1) access.

A HashMap (also called dictionary, hash table, or object) stores key-value pairs and uses hashing to make lookups blazingly fast.

💡 Key Insight

Hashing is the single most powerful optimization technique in interviews. Whenever you see an O(n²) brute force that searches for a complement/match, a HashMap can almost always bring it down to O(n). This one trick solves dozens of LeetCode problems.

🌍 Real-Life Analogy

📚 Library Card Catalog

In a library, you don't search every shelf for a book. You look up the title in the card catalog, which tells you exactly which shelf and position (Aisle 3, Shelf B, Position 7). The catalog is the hash function — it converts a book title (key) into a location (index). The shelves are the hash table.

  • Phone book: Look up a name (key) → get a phone number (value). You go directly to the "S" section for "Smith", not page 1.
  • Coat check: Give your coat, get a numbered ticket (hash). Later, show the ticket → get your coat instantly. No searching!
  • Locker at the gym: You pick locker #42 (hash of your membership). You know exactly where to go every time.

⚙️ How Hashing Works

Hash Function: Key → Index
Key (input) Hash Function Index (output) "apple" ──→ hash("apple") ──→ 3 "banana" ──→ hash("banana") ──→ 7 "cherry" ──→ hash("cherry") ──→ 1 Hash Table (array of buckets): Index: 0 1 2 3 4 ... 7 +-----+---------+-----+---------+-----+----+---------+ | | "cherry"| | "apple" | | | "banana"| +-----+---------+-----+---------+-----+----+---------+

A simple hash function for strings:

JavaScript
// Simple hash: sum of char codes, modulo table size
function hash(key, tableSize) {
    let total = 0;
    for (let char of key) {
        total += char.charCodeAt(0);
    }
    return total % tableSize;
}

hash("apple", 10);  // Sum of codes % 10 → some index 0-9

💥 Collision Handling

A collision happens when two different keys produce the same hash index. This is inevitable — there are infinite possible keys but finite slots.

1. Chaining (Linked List at each slot)

Index 3 → ["apple", 10] → ["grape", 25] → null (Both "apple" and "grape" hash to index 3)

2. Open Addressing (find next empty slot)

"apple" hashes to 3 → slot 3 taken → try slot 4 → empty → store here! Index: 0 1 2 3 4 5 +-----+-----+-----+--------+--------+-----+ | | | | "grape"| "apple"| | +-----+-----+-----+--------+--------+-----+
💡 In Practice

You don't need to implement hash tables from scratch in interviews. JavaScript has Map and {}, Python has dict, Java has HashMap. But understanding how they work helps you analyze their complexity.

🔧 Using HashMap & HashSet

HashMap (key → value)

JavaScript
// Using a Map
const map = new Map();
map.set("apple", 3);      // Insert
map.set("banana", 5);
map.get("apple");          // 3 — O(1) lookup!
map.has("cherry");         // false — O(1) check
map.delete("banana");      // Remove — O(1)
map.size;                  // 1

// Using plain object (common in interviews)
const freq = {};
freq["apple"] = 3;
freq["banana"] = 5;
console.log(freq["apple"]); // 3
"apple" in freq;             // true

HashSet (unique values only)

JavaScript
const set = new Set();
set.add(1);     // {1}
set.add(2);     // {1, 2}
set.add(1);     // {1, 2} — duplicate ignored!
set.has(1);     // true — O(1) lookup
set.delete(2);  // {1}
set.size;       // 1

// Quick way to remove duplicates from an array:
let arr = [1, 2, 3, 2, 1, 4];
let unique = [...new Set(arr)]; // [1, 2, 3, 4]

⭐ Classic: Two Sum with HashMap

The most famous interview problem — solved optimally with a HashMap.

JavaScript
// Find two numbers that add up to target
function twoSum(nums, target) {
    const map = new Map(); // value → index
    
    for (let i = 0; i < nums.length; i++) {
        let complement = target - nums[i];
        
        if (map.has(complement)) {
            return [map.get(complement), i];
        }
        
        map.set(nums[i], i);
    }
    return []; // No solution found
}

// twoSum([2, 7, 11, 15], 9) → [0, 1]
// Because 2 + 7 = 9
1
i=0: num=2, complement=9-2=7. Map has 7? No. Store {2→0}.
Map: {2:0}
2
i=1: num=7, complement=9-7=2. Map has 2? Yes! Return [0, 1].
Answer: [0, 1] ✅ (nums[0]+nums[1] = 2+7 = 9)
🎯 The HashMap Pattern

Brute force: check every pair → O(n²). With HashMap: for each element, check if its complement exists in the map → O(n). This "store and look up complement" pattern appears in dozens of problems.

📊 Pattern: Frequency Counter

JavaScript
// Count frequency of each element
function buildFrequencyMap(arr) {
    const freq = {};
    for (let item of arr) {
        freq[item] = (freq[item] || 0) + 1;
    }
    return freq;
}

buildFrequencyMap([1, 2, 2, 3, 3, 3]);
// { 1: 1, 2: 2, 3: 3 }

// Use case: Find the most frequent element
function mostFrequent(arr) {
    const freq = buildFrequencyMap(arr);
    let maxCount = 0, result;
    for (let [key, count] of Object.entries(freq)) {
        if (count > maxCount) {
            maxCount = count;
            result = key;
        }
    }
    return result;
}

⏱️ Complexity Analysis

Insert (set)O(1)O(n)Worst: all keys collide

DeleteO(1)O(n)Same reason as lookup
SpaceO(n)Stores n key-value pairs
💡 Average O(1) is what matters

The worst case O(n) happens only with a terrible hash function where all keys collide. In practice (and in interviews), always assume O(1) average for HashMap operations.

⚠️ Common Mistakes

❌ Using objects as keys in plain JS objects
// ❌ JS objects convert keys to strings!
let obj = {};
obj[[1,2]] = "array";
obj[[1,2]];  // "array" — but ONLY because "1,2" === "1,2"

// ✅ Use Map for non-string keys
let map = new Map();
map.set([1,2], "array"); // Works correctly
❌ Forgetting to check if key exists before accessing
// ❌ Accessing undefined key
freq[item]++;  // NaN if item doesn't exist!

// ✅ Initialize with 0 first
freq[item] = (freq[item] || 0) + 1;
❌ Not considering the space trade-off

HashMaps trade space for time. If you use a HashMap to speed up from O(n²) to O(n), you're using O(n) extra space. Interviewers may ask: "Can you do it without extra space?"

💼 Interview Perspective

  • HashMap is your best friend. If brute force is O(n²), ask yourself: "Can I use a HashMap to make it O(n)?"
  • Know the difference between HashMap (key→value) and HashSet (unique values).
  • Common use cases: frequency counting, complement lookup, grouping, deduplication.
  • Be ready to explain how hash tables work: hash function, buckets, collision resolution.
  • Remember: O(1) is amortized average. Worst case is O(n) due to collisions.

📝 Quick Revision Notes

  • Hash function converts key → index. HashMap stores key-value pairs.
  • Average: O(1) insert, lookup, delete. Worst case: O(n) if all keys collide.
  • Collisions handled by chaining (linked lists) or open addressing (probing).
  • Two Sum pattern: store elements in map, look up complement. O(n²) → O(n).
  • Frequency counter: use map to count occurrences of each element.
  • HashSet = HashMap without values. Used for deduplication and O(1) membership checks.

🃏 Flash Cards

What is the average time complexity of HashMap lookup?
O(1) amortized. The hash function computes the index directly, giving constant-time access. Worst case is O(n) if all keys collide into one bucket.
What is a hash collision and how is it handled?
A collision occurs when two different keys produce the same hash index. Handled by (1) Chaining: store multiple items in a linked list at that index, or (2) Open Addressing: probe for the next empty slot.
How does a HashMap optimize Two Sum from O(n²) to O(n)?
Instead of checking every pair (nested loop), store each number in a map as you iterate. For each number, check if its complement (target - num) exists in the map. One pass = O(n).
What is the difference between HashMap and HashSet?
HashMap stores key-value pairs (e.g., name→phone). HashSet stores only keys (unique values). HashSet is like a HashMap where you only care about existence, not associated values.
