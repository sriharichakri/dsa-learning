Tree DP

Phase 12 Hard

📖 What is Tree DP?

Tree DP applies dynamic programming on tree structures. Each node computes its answer based on answers from its children (post-order DFS). Common problems include finding diameter, maximum path sum, and house robber on trees.

💡 Key Insight

The key pattern: DFS returns information from children, and the parent combines it. Often you need to track two values per node — e.g., the best path ending at this node vs. the best path through this node (which can't be extended further).

🌍 Real-Life Analogy

🏢 Corporate Reporting

Each department reports its results to its manager. The manager combines reports from all departments and passes a summary to their boss. The CEO gets the final combined result. Tree DP works the same way — leaves report to parents, who combine and pass up.

⚙️ Tree DP Problems

1. Binary Tree Maximum Path Sum

JavaScript — Max Path Sum (LC 124)
function maxPathSum(root) {
    let maxSum = -Infinity;

    function dfs(node) {
        if (!node) return 0;

        // Max gain from left and right (ignore negative paths)
        const leftGain = Math.max(0, dfs(node.left));
        const rightGain = Math.max(0, dfs(node.right));

        // Path through this node (can't be extended further up)
        const pathThroughNode = node.val + leftGain + rightGain;
        maxSum = Math.max(maxSum, pathThroughNode);

        // Return max extendable path (can only go through one child)
        return node.val + Math.max(leftGain, rightGain);
    }

    dfs(root);
    return maxSum;
}

2. Diameter of Binary Tree

JavaScript — Diameter (LC 543)
function diameterOfBinaryTree(root) {
    let diameter = 0;

    function depth(node) {
        if (!node) return 0;
        const left = depth(node.left);
        const right = depth(node.right);

        // Diameter through this node = left depth + right depth
        diameter = Math.max(diameter, left + right);

        return 1 + Math.max(left, right); // Return depth
    }

    depth(root);
    return diameter;
}

3. House Robber III (Tree)

JavaScript — House Robber III (LC 337)
function rob(root) {
    // Returns [robThis, skipThis]
    function dfs(node) {
        if (!node) return [0, 0];

        const left = dfs(node.left);
        const right = dfs(node.right);

        // Rob this node: can't rob children
        const robThis = node.val + left[1] + right[1];
        // Skip this node: take best of each child
        const skipThis = Math.max(...left) + Math.max(...right);

        return [robThis, skipThis];
    }

    return Math.max(...dfs(root));
}

⚠️ Common Mistakes

❌ Confusing "path through node" vs "path from node"

A path THROUGH a node goes left→node→right (can't extend further up). A path FROM a node goes node→one child (can extend to parent). Return the FROM value, update global max with THROUGH value.

❌ Not handling negative values

Use Math.max(0, childGain) to ignore negative paths. A negative child path makes things worse — better to not include it.

💼 Interview Perspective

  • Max Path Sum (LC 124) is one of the hardest tree problems — and very commonly asked.
  • The pattern: DFS returns one thing (extendable path), updates global with another (full path through node).
  • Diameter is the same pattern but simpler — good warm-up before max path sum.
  • House Robber III combines tree DFS with the take/skip DP pattern.

📝 Quick Revision Notes

  • Tree DP: DFS returns info from children, parent combines.
  • Max Path Sum: return extendable gain, update global with through-path.
  • Diameter = max(left_depth + right_depth) across all nodes.
  • House Robber III: return [robThis, skipThis] pair from each node.
  • All are O(n) time, O(h) space. Single DFS traversal.

🃏 Flash Cards

In Max Path Sum, why do we return one value but update global max with another?
We return the extendable path (node + one child) because the parent can only extend through one branch. But the best path through the node (left + node + right) might be the global answer — it just can't be extended further. So we update global max with it.
How does House Robber III use tree DP?
Each node returns a pair: [rob_this, skip_this]. If we rob this node, we must skip both children (take their skip values). If we skip this node, we take the best (max of rob/skip) from each child. Final answer: max of root's pair.
