Fast & Slow Pointer

Phase 5 Medium

📖 What is the Fast & Slow Pointer Technique?

The Fast & Slow Pointer (also called Floyd's Tortoise and Hare) technique uses two pointers that move at different speeds. The slow pointer moves one step at a time, while the fast pointer moves two steps. This simple idea solves cycle detection, finding middle elements, and more.

🌍 Real-Life Analogy

Imagine two runners on a circular track. One runs at twice the speed of the other. If the track is circular, the fast runner will eventually lap the slow runner — they'll meet! If it's a straight road, the fast runner just reaches the end first. This "meeting" or "reaching the end" tells us about the structure of the path (circular or linear).

⚙️ How It Works

1. Cycle Detection in Linked List

JavaScript
function hasCycle(head) {
    let slow = head, fast = head;
    
    while (fast && fast.next) {
        slow = slow.next;         // 1 step
        fast = fast.next.next;     // 2 steps
        if (slow === fast) return true;  // They met = cycle!
    }
    return false;  // Fast reached end = no cycle
}

2. Find Middle of Linked List

JavaScript
function findMiddle(head) {
    let slow = head, fast = head;
    
    while (fast && fast.next) {
        slow = slow.next;
        fast = fast.next.next;
    }
    return slow;  // When fast reaches end, slow is at middle!
}
// [1→2→3→4→5] → slow=3 (middle)
// [1→2→3→4→5→6] → slow=4 (second middle)

3. Find Cycle Start (Floyd's Algorithm)

JavaScript
function detectCycleStart(head) {
    let slow = head, fast = head;
    
    // Phase 1: Detect cycle
    while (fast && fast.next) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow === fast) break;
    }
    
    if (!fast || !fast.next) return null; // No cycle
    
    // Phase 2: Find start of cycle
    slow = head;  // Reset slow to head
    while (slow !== fast) {
        slow = slow.next;  // Both move 1 step
        fast = fast.next;  // Both move 1 step
    }
    return slow;  // Meeting point = cycle start!
}

4. Happy Number (LC 202)

JavaScript
function isHappy(n) {
    function getNext(num) {
        let sum = 0;
        while (num > 0) {
            let d = num % 10;
            sum += d * d;
            num = Math.floor(num / 10);
        }
        return sum;
    }
    
    let slow = n, fast = getNext(n);
    while (fast !== 1 && slow !== fast) {
        slow = getNext(slow);         // 1 step
        fast = getNext(getNext(fast)); // 2 steps
    }
    return fast === 1;
}

💡 Why O(1) space?

Without this technique, cycle detection requires a hash set (O(n) space) to track visited nodes. Fast & slow pointers achieve the same result with just two pointer variables!

⚠️ Common Mistakes

❌ Not checking fast AND fast.next
// ❌ Crashes on odd-length lists!
while (fast.next) { ... }

// ✅ Check both to avoid null pointer
while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
}
❌ Confusing cycle detection with cycle start

The meeting point of fast & slow is NOT the cycle start! After detection, reset slow to head, then move both one step at a time until they meet again — THAT is the cycle start.

💼 Interview Perspective

  • This technique is a must-know for linked list problems.
  • Cycle detection without extra space → Floyd's algorithm (this technique).
  • Finding middle node in one pass → use this instead of counting length first.
  • "Happy Number" is a disguised cycle detection problem — the sequence either reaches 1 or cycles.
  • Can also find the kth node from end: start fast k steps ahead, then move both until fast reaches end.

📝 Quick Revision Notes

  • Slow moves 1 step, fast moves 2 steps. If they meet → cycle exists.
  • When fast reaches end, slow is at the middle.
  • Cycle start: after meeting, reset slow to head. Move both 1 step until they meet.
  • All solutions are O(n) time, O(1) space.

🃏 Flash Cards

Why does the fast pointer meet the slow pointer if there's a cycle?
In the cycle, fast gains 1 node per step on slow. If the cycle length is C, fast will close the gap in C steps. They must meet within one full cycle — the gap decreases by 1 each step until it's 0.
How do you find the START of a cycle using Floyd's algorithm?
After fast and slow meet: (1) Reset slow to head. (2) Move BOTH slow and fast one step at a time. (3) Where they meet again is the cycle start. This works due to the mathematical relationship between distances.
