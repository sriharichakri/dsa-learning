Next Greater Element

Phase 7 Medium

📖 What is the Next Greater Element?

For each element in an array, the Next Greater Element (NGE) is the first element to its right that is larger than it. If no such element exists, the answer is -1.

🌍 Real-Life Analogy

Imagine standing in a crowd and looking to your right for the first person taller than you. That's your "next greater element." If everyone to your right is shorter, there's no next greater (answer = -1). A tall person at the back may never find someone taller!

⚙️ How It Works

Basic NGE (right to left scan)

JavaScript
function nextGreaterElement(arr) {
    const n = arr.length;
    const result = new Array(n).fill(-1);
    const stack = []; // Monotonic decreasing stack
    
    // Scan right to left
    for (let i = n - 1; i >= 0; i--) {
        // Pop elements smaller than current
        while (stack.length && stack[stack.length - 1] <= arr[i]) {
            stack.pop();
        }
        // Top of stack is the next greater (if exists)
        if (stack.length) result[i] = stack[stack.length - 1];
        stack.push(arr[i]);
    }
    return result;
}

// [4, 5, 2, 25] → [5, 25, 25, -1]

NGE I — Two arrays (LC 496)

JavaScript
function nextGreaterElementI(nums1, nums2) {
    const map = new Map();
    const stack = [];
    
    // Find NGE for all elements in nums2
    for (let num of nums2) {
        while (stack.length && stack[stack.length - 1] < num) {
            map.set(stack.pop(), num);
        }
        stack.push(num);
    }
    
    // Look up results for nums1
    return nums1.map(num => map.get(num) ?? -1);
}

NGE II — Circular array (LC 503)

JavaScript
function nextGreaterElements(nums) {
    const n = nums.length;
    const result = new Array(n).fill(-1);
    const stack = [];
    
    // Loop through array twice to handle circular
    for (let i = 2 * n - 1; i >= 0; i--) {
        const idx = i % n;
        while (stack.length && stack[stack.length - 1] <= nums[idx]) {
            stack.pop();
        }
        if (stack.length) result[idx] = stack[stack.length - 1];
        stack.push(nums[idx]);
    }
    return result;
}

// [1, 2, 1] → [2, -1, 2]  (circular: after 1, wraps to 2)

⚠️ Common Mistakes

❌ Using strict less-than instead of less-than-or-equal

When popping from the stack, use <= to handle equal elements. For "strictly greater," equal elements should also be popped since they're not greater.

❌ Forgetting circular handling in NGE II

For circular arrays, iterate from 2*n-1 to 0 and use i % n as the index. This simulates wrapping around the array.

💼 Interview Perspective

  • NGE is the gateway problem for monotonic stack — master it first.
  • Two approaches: left-to-right (pop when found answer) or right-to-left (maintain potential answers).
  • "Daily Temperatures" (LC 739) is NGE in disguise — find the next warmer day.
  • For circular variant, iterate 2×n with modulo. This is a standard trick.
  • Variation: Next Smaller Element (use increasing stack instead).

📝 Quick Revision Notes

  • NGE = first element to the right that is larger than current.
  • Use monotonic decreasing stack. Pop elements ≤ current.
  • Right-to-left: stack holds "potential next greaters." Top = answer.
  • Circular: iterate 2×n with i % n. Time still O(n).

🃏 Flash Cards

How do you handle circular arrays for Next Greater Element?
Iterate from 2*n - 1 down to 0. Use i % n as the actual index. This effectively processes the array twice, allowing elements to "wrap around" and find their next greater on the other side.
What's the difference between left-to-right and right-to-left monotonic stack approaches?
Left-to-right: Stack holds elements "waiting" for their answer. When current element is the answer, pop and assign. Right-to-left: Stack holds "potential answers." Top of stack is the answer for current element. Both are O(n).
