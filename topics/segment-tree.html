Segment Tree

Phase 9 Hard

📖 What is a Segment Tree?

A Segment Tree is a binary tree data structure used for efficiently performing range queries (like sum, min, max over a subarray) and point updates on an array. Both operations run in O(log n) time.

Each node in the tree stores the result of a query (e.g., sum) for a specific segment (range) of the array. The root stores the result for the entire array, and leaves store individual elements.

💡 Key Insight

Without a segment tree, range sum is O(n) per query (or O(1) with prefix sums but then updates are O(n)). Segment trees give you O(log n) for both queries AND updates — the best of both worlds.

🌍 Real-Life Analogy

🏆 Tournament Bracket

Think of a sports tournament bracket. Each match determines the winner (max/min) of two players. Quarter-finals combine pairs, semi-finals combine quarters, and the final gives the overall champion. If a player's score changes, you only need to re-evaluate the matches they participated in — not the entire tournament. That's exactly how a segment tree works!

  • Corporate hierarchy: Each manager knows the total salary of their department. Change one employee's salary → only update managers up to the CEO.
  • School grading: Class averages are precomputed. One student's grade changes → update section → department → school average.

🔧 Segment Tree Implementation

JavaScript — Segment Tree (Range Sum)
class SegmentTree {
    constructor(arr) {
        this.n = arr.length;
        this.tree = new Array(4 * this.n).fill(0); // 4n is safe upper bound
        this.build(arr, 1, 0, this.n - 1);
    }

    // Build the tree — O(n)
    build(arr, node, start, end) {
        if (start === end) {
            this.tree[node] = arr[start]; // Leaf node
            return;
        }
        const mid = Math.floor((start + end) / 2);
        this.build(arr, 2 * node, start, mid);       // Left child
        this.build(arr, 2 * node + 1, mid + 1, end); // Right child
        this.tree[node] = this.tree[2 * node] + this.tree[2 * node + 1]; // Merge
    }

    // Point Update — O(log n)
    update(idx, val, node = 1, start = 0, end = this.n - 1) {
        if (start === end) {
            this.tree[node] = val; // Update leaf
            return;
        }
        const mid = Math.floor((start + end) / 2);
        if (idx <= mid) {
            this.update(idx, val, 2 * node, start, mid);
        } else {
            this.update(idx, val, 2 * node + 1, mid + 1, end);
        }
        this.tree[node] = this.tree[2 * node] + this.tree[2 * node + 1]; // Re-merge
    }

    // Range Query — O(log n)
    query(l, r, node = 1, start = 0, end = this.n - 1) {
        if (r < start || end < l) return 0;       // No overlap
        if (l <= start && end <= r) return this.tree[node]; // Total overlap
        const mid = Math.floor((start + end) / 2);
        return this.query(l, r, 2 * node, start, mid) +
               this.query(l, r, 2 * node + 1, mid + 1, end); // Partial overlap
    }
}

// Usage
const st = new SegmentTree([1, 3, 5, 7, 9, 11]);
console.log(st.query(1, 4)); // 3+5+7+9 = 24
st.update(2, 10);            // arr[2] = 10 (was 5)
console.log(st.query(1, 4)); // 3+10+7+9 = 29

Use prefix sums for static arrays (no updates). Use segment trees when you need both queries and updates.

⚠️ Common Mistakes

❌ Array size too small

The segment tree array needs 4 × n space, not 2 × n. Using 2n can cause index out of bounds for non-power-of-2 array sizes.

❌ Wrong overlap check in query

Three cases: no overlap (return identity), total overlap (return node value), partial overlap (recurse both). Getting the boundary conditions wrong leads to incorrect results.

❌ Forgetting to re-merge after update

After updating a leaf, you must propagate the change upward by re-computing parent nodes. Missing this leaves the tree inconsistent.

💼 Interview Perspective

  • Segment trees appear in "Range Query + Update" problems. If you see both operations required, think segment tree or BIT.
  • For range sum + point update, a Fenwick tree (BIT) is simpler to code. Use segment tree for range min/max or range updates.
  • Advanced: Lazy Propagation enables range updates in O(log n) — asked in competitive programming more than interviews.
  • Practice implementing from scratch — interviewers may ask you to build one.
  • Key pattern: if a brute-force approach does O(n) per query and there are Q queries, segment tree brings it to O(Q log n).

📝 Quick Revision Notes

  • Segment Tree: binary tree where each node stores info about a range (segment) of the array.
  • Build O(n), Query O(log n), Update O(log n), Space O(4n).
  • Three cases in query: no overlap → return identity; total overlap → return node; partial → recurse.
  • Children of node i: left = 2i, right = 2i+1.
  • Use Fenwick Tree (BIT) for simpler range sum problems; Segment Tree for min/max/complex queries.

🃏 Flash Cards

What are the three overlap cases in a segment tree query?
1. No overlap: query range doesn't intersect node range → return identity (0 for sum). 2. Total overlap: node range is fully inside query → return node value. 3. Partial overlap: recurse on both children and merge results.
Why is the segment tree array size 4n?
A segment tree is a complete binary tree with n leaves. The number of nodes can be up to 2×(next power of 2 ≥ n) − 1. Using 4n guarantees enough space for any n, even when n is not a power of 2.
When to use Segment Tree vs Prefix Sum vs BIT?
Prefix Sum: static array, O(1) queries, no updates. BIT (Fenwick): point updates + prefix/range sum, simpler code. Segment Tree: point/range updates + any associative query (sum, min, max, GCD).
