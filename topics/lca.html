Lowest Common Ancestor (LCA)

Phase 9 Hard

📖 What is the Lowest Common Ancestor?

The Lowest Common Ancestor (LCA) of two nodes p and q in a tree is the deepest node that is an ancestor of both p and q. A node is also considered an ancestor of itself.

There are multiple approaches: simple recursion for binary trees, parent pointer traversal, and the advanced Binary Lifting technique for efficient repeated LCA queries on general trees.

💡 Key Insight

For a single LCA query on a binary tree, a simple DFS works in O(n). But if you have Q queries on a tree of n nodes, binary lifting preprocesses in O(n log n) and answers each query in O(log n), making Q queries cost O(n log n + Q log n) instead of O(Qn).

🌍 Real-Life Analogy

👨‍👩‍👧‍👦 Family Tree

In a family tree, the LCA of two people is their most recent common ancestor. For example, if Alice and Bob are cousins, their LCA is their shared grandparent. The LCA is the "lowest" (closest to them) person who appears in both of their lineages.

  • File system: The LCA of two files is their closest common directory.
  • Organizational chart: The LCA of two employees is their closest shared manager.
  • Version control: The merge-base of two git branches is their LCA in the commit graph.

⚙️ Approach 1: Recursive LCA (Binary Tree)

JavaScript — Recursive LCA (Binary Tree)
function lowestCommonAncestor(root, p, q) {
    // Base case: reached null or found one of the targets
    if (!root || root === p || root === q) return root;

    // Search in both subtrees
    const left = lowestCommonAncestor(root.left, p, q);
    const right = lowestCommonAncestor(root.right, p, q);

    // If both sides return non-null, current node is the LCA
    if (left && right) return root;

    // Otherwise, return whichever side found something
    return left || right;
}
// Time: O(n), Space: O(h) where h is height

🚀 Approach 2: Binary Lifting (for Multiple Queries)

Binary Lifting precomputes the 2^k-th ancestor of every node. To find LCA, lift both nodes to the same depth, then lift both together until they meet.

JavaScript — Binary Lifting LCA
class LCABinaryLifting {
    constructor(n, edges, root = 0) {
        this.n = n;
        this.LOG = Math.ceil(Math.log2(n)) + 1;
        this.depth = new Array(n).fill(0);
        this.up = Array.from({length: n}, () => new Array(this.LOG).fill(-1));

        // Build adjacency list
        const adj = Array.from({length: n}, () => []);
        for (const [u, v] of edges) {
            adj[u].push(v);
            adj[v].push(u);
        }

        // BFS to compute depth and up[node][0] (direct parent)
        const visited = new Array(n).fill(false);
        const queue = [root];
        visited[root] = true;
        while (queue.length) {
            const u = queue.shift();
            for (const v of adj[u]) {
                if (!visited[v]) {
                    visited[v] = true;
                    this.depth[v] = this.depth[u] + 1;
                    this.up[v][0] = u;
                    queue.push(v);
                }
            }
        }

        // Fill up table: up[v][k] = 2^k-th ancestor of v
        for (let k = 1; k < this.LOG; k++) {
            for (let v = 0; v < n; v++) {
                if (this.up[v][k - 1] !== -1) {
                    this.up[v][k] = this.up[this.up[v][k - 1]][k - 1];
                }
            }
        }
    }

    query(u, v) {
        // Step 1: Bring to same depth
        if (this.depth[u] < this.depth[v]) [u, v] = [v, u];

        let diff = this.depth[u] - this.depth[v];
        for (let k = 0; k < this.LOG; k++) {
            if ((diff >> k) & 1) u = this.up[u][k];
        }

        if (u === v) return u;

        // Step 2: Lift both until they meet
        for (let k = this.LOG - 1; k >= 0; k--) {
            if (this.up[u][k] !== this.up[v][k]) {
                u = this.up[u][k];
                v = this.up[v][k];
            }
        }
        return this.up[u][0]; // Parent is the LCA
    }
}
// Preprocessing: O(n log n), Each query: O(log n)

⚠️ Common Mistakes

❌ Forgetting that a node is its own ancestor

If p is an ancestor of q, then LCA(p, q) = p. Don't assume the LCA must be a separate node above both.

❌ Confusing Binary Tree LCA with BST LCA

In a BST, you can use the ordering property to find LCA more efficiently. In a general binary tree, you must search both subtrees.

❌ Wrong base case in recursion

The base case checks root === p || root === q. If you only check for null, you'll miss the case where one node is the ancestor of the other.

💼 Interview Perspective

  • The recursive approach is the most common interview question — know it cold. LC 236.
  • For BSTs, use the BST property: if both nodes are smaller, go left; both larger, go right; split means current is LCA.
  • Binary lifting is for competitive programming or system design with many queries.
  • Follow-up: "What if nodes have parent pointers?" → Treat as finding intersection of two linked lists.
  • Follow-up: "What if one node might not exist in the tree?" → Do full traversal and check both found.

📝 Quick Revision Notes

  • LCA = deepest node that is ancestor of both p and q.
  • Recursive: if both subtrees return non-null → current node is LCA. O(n) per query.
  • Binary Lifting: precompute 2^k-th ancestors. O(n log n) preprocessing, O(log n) per query.
  • BST LCA: use ordering property for O(h) solution.
  • A node is considered an ancestor of itself.

🃏 Flash Cards

What does the recursive LCA function return at each node?
It returns: (1) null if neither p nor q is found in the subtree, (2) the node itself if it's p or q, (3) the node if both children return non-null (it's the LCA). Otherwise returns whichever child's result is non-null.
How does binary lifting work for LCA?
1. Precompute up[v][k] = 2^k-th ancestor of v. 2. To find LCA(u,v): bring both to the same depth using binary jumps. 3. If they're the same node, that's the LCA. 4. Otherwise, jump both up together with decreasing powers of 2 until their parents match. Return the parent.
How does LCA differ in a BST vs a general binary tree?
In a BST: start at root. If both p,q < node, go left. If both > node, go right. Otherwise, current node is LCA. O(h) time. In a general binary tree, you must search both subtrees since there's no ordering property.
