Monotonic Stack

Phase 7 Hard

📖 What is a Monotonic Stack?

A Monotonic Stack is a stack that maintains its elements in strictly increasing or decreasing order. When pushing a new element, you pop all elements that violate the monotonic property. This seemingly simple idea solves many "next greater/smaller element" problems in O(n).

🌍 Real-Life Analogy

Imagine a line of people where only increasingly taller people can stay. When a new person arrives, everyone shorter than them must leave the line (from the back). The line always goes from shortest (bottom) to tallest (top). When someone gets "popped out," the new person is their "next taller person" — the answer we're looking for!

⚙️ How It Works

Monotonic Decreasing Stack (for Next Greater Element)

JavaScript
function nextGreaterElements(arr) {
    const n = arr.length;
    const result = new Array(n).fill(-1);
    const stack = []; // Stores INDICES, not values
    
    for (let i = 0; i < n; i++) {
        // Pop all elements smaller than current
        while (stack.length && arr[stack[stack.length - 1]] < arr[i]) {
            const idx = stack.pop();
            result[idx] = arr[i]; // arr[i] is the next greater for arr[idx]
        }
        stack.push(i);
    }
    return result;
}

// [2, 1, 2, 4, 3] → [4, 2, 4, -1, -1]
// 2's next greater = 4
// 1's next greater = 2
// No next greater for 4 and 3

Monotonic Increasing Stack (for Next Smaller Element)

JavaScript
function nextSmallerElements(arr) {
    const n = arr.length;
    const result = new Array(n).fill(-1);
    const stack = [];
    
    for (let i = 0; i < n; i++) {
        while (stack.length && arr[stack[stack.length - 1]] > arr[i]) {
            const idx = stack.pop();
            result[idx] = arr[i];
        }
        stack.push(i);
    }
    return result;
}

Largest Rectangle in Histogram (Classic)

JavaScript
function largestRectangleArea(heights) {
    const stack = [];
    let maxArea = 0;
    
    for (let i = 0; i <= heights.length; i++) {
        const h = i === heights.length ? 0 : heights[i];
        
        while (stack.length && heights[stack[stack.length - 1]] > h) {
            const height = heights[stack.pop()];
            const width = stack.length === 0 ? i : i - stack[stack.length - 1] - 1;
            maxArea = Math.max(maxArea, height * width);
        }
        stack.push(i);
    }
    return maxArea;
}

💡 Why O(n) and not O(n²)?

Each element is pushed once and popped at most once. So across all iterations, there are at most 2n operations total = O(n).

⚠️ Common Mistakes

❌ Storing values instead of indices

Always store indices in the stack, not values. You need the index to set the result at the correct position. Access values with arr[stack[top]].

❌ Wrong monotonic direction

For next greater: maintain decreasing stack (pop smaller). For next smaller: maintain increasing stack (pop larger). Getting this backwards gives wrong answers.

❌ Forgetting remaining stack elements

Elements left in the stack at the end have no next greater/smaller. Initialize result with -1 to handle this automatically.

📝 Quick Revision Notes

  • Monotonic stack: elements maintain increasing or decreasing order.
  • Next greater → decreasing stack. Next smaller → increasing stack.
  • Store indices, not values. Each element pushed/popped at most once → O(n).

🃏 Flash Cards

Why is a monotonic stack O(n) even though it has nested while loops?
Each element is pushed exactly once and popped at most once. Across all iterations: at most n pushes + n pops = 2n total operations. The inner while loop doesn't re-process elements — each pop is final.
For "next greater element," do you use an increasing or decreasing stack?
Decreasing stack (from bottom to top). When a new element is larger than the top, you pop — the new element IS the next greater for the popped elements. The stack keeps elements "waiting" for their next greater.
