Tabulation (Bottom-Up DP)

Phase 12 Medium

📖 What is Tabulation?

Tabulation (bottom-up DP) solves problems by filling a DP table iteratively, starting from the smallest subproblems and building up to the final answer. Unlike memoization, it's non-recursive and computes all subproblems in a predefined order.

💡 Key Insight

Tabulation avoids recursion overhead and stack overflow risks. It also makes space optimization easier — if dp[i] only depends on dp[i-1] and dp[i-2], you can replace the entire array with just two variables.

🌍 Real-Life Analogy

🧱 Building a Wall Brick by Brick

You can't build the top row of a brick wall without finishing the bottom rows first. Tabulation works the same way — solve small subproblems first (bottom rows), then use those results to build larger solutions (upper rows), until you reach the final answer (the roof).

⚙️ Tabulation Examples

Unique Paths (Grid DP)

JavaScript — Unique Paths (LC 62)
function uniquePaths(m, n) {
    const dp = Array.from({length: m}, () => new Array(n).fill(1));

    for (let i = 1; i < m; i++) {
        for (let j = 1; j < n; j++) {
            dp[i][j] = dp[i - 1][j] + dp[i][j - 1]; // From top + from left
        }
    }
    return dp[m - 1][n - 1];
}

// Space-optimized: O(n) instead of O(m×n)
function uniquePathsOptimized(m, n) {
    let prev = new Array(n).fill(1);

    for (let i = 1; i < m; i++) {
        const curr = new Array(n).fill(1);
        for (let j = 1; j < n; j++) {
            curr[j] = prev[j] + curr[j - 1];
        }
        prev = curr;
    }
    return prev[n - 1];
}

Minimum Path Sum

JavaScript — Minimum Path Sum (LC 64)
function minPathSum(grid) {
    const m = grid.length, n = grid[0].length;

    // Fill first row
    for (let j = 1; j < n; j++) grid[0][j] += grid[0][j - 1];
    // Fill first column
    for (let i = 1; i < m; i++) grid[i][0] += grid[i - 1][0];

    // Fill rest
    for (let i = 1; i < m; i++) {
        for (let j = 1; j < n; j++) {
            grid[i][j] += Math.min(grid[i - 1][j], grid[i][j - 1]);
        }
    }
    return grid[m - 1][n - 1];
}

Coin Change (Bottom-Up)

JavaScript — Coin Change Tabulation
function coinChange(coins, amount) {
    const dp = new Array(amount + 1).fill(Infinity);
    dp[0] = 0;

    for (let i = 1; i <= amount; i++) {
        for (const coin of coins) {
            if (coin <= i && dp[i - coin] + 1 < dp[i]) {
                dp[i] = dp[i - coin] + 1;
            }
        }
    }
    return dp[amount] === Infinity ? -1 : dp[amount];
}

⚠️ Common Mistakes

❌ Wrong iteration order

Tabulation requires filling the table in the correct order. If dp[i] depends on dp[i-1] and dp[i-2], iterate left to right. If it depends on dp[i+1], iterate right to left.

❌ Forgetting base cases in the table

Unlike memoization where base cases are in the recursion, tabulation requires explicitly initializing dp[0], dp[1], etc. Missing these leads to incorrect propagation.

❌ Off-by-one in table size

For amount=5, you need dp[0..5] — that's 6 entries, not 5. Create new Array(amount + 1).

💼 Interview Perspective

  • Start with memoization (easier), then convert to tabulation if asked to optimize space.
  • Space optimization: if dp[i] depends on last row only, use a 1D array. If on last 2 values, use 2 variables.
  • For 2D DP, can often reduce O(m×n) space to O(n) by processing row by row.
  • Tabulation is generally faster in practice (no recursion overhead).
  • In-place modification of input grid saves space but modifies input — mention the tradeoff.

📝 Quick Revision Notes

  • Tabulation = iterative DP. Fill table from smallest subproblem to largest.
  • No recursion → no stack overflow. Easier space optimization.
  • Space tricks: 2D→1D (row-by-row), 1D→O(1) (rolling variables).
  • Must determine correct iteration order based on dependencies.
  • Always initialize base cases explicitly in the DP table.

🃏 Flash Cards

How do you optimize a 2D DP table to O(n) space?
If each row only depends on the previous row, keep only two 1D arrays: prev and curr. After processing each row, set prev = curr. This reduces O(m×n) space to O(n).
When should you choose tabulation over memoization?
Choose tabulation when: (1) you need space optimization (rolling arrays/variables), (2) the input is large (avoid stack overflow), (3) all subproblems are needed (tabulation doesn't waste effort). Choose memoization when only a subset of subproblems is needed.
