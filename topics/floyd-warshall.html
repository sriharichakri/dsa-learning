Floyd-Warshall Algorithm

Phase 10 Hard

📖 What is the Floyd-Warshall Algorithm?

Floyd-Warshall computes the shortest paths between ALL pairs of vertices in a weighted graph. It works with negative edge weights and can detect negative cycles.

The key idea: for each pair (i, j), check whether going through an intermediate vertex k gives a shorter path: dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]).

💡 Key Insight

Floyd-Warshall considers each vertex as a potential "stepping stone" in a path. It iterates through ALL possible intermediate vertices k, and for each k, updates ALL pairs (i, j). Three nested loops — simple but O(V³). Best for small dense graphs where you need all-pairs shortest paths.

🌍 Real-Life Analogy

✈️ Flight Connection Optimizer

Imagine you're building a travel price table for all city pairs. Initially, you only know direct flight prices. Then you ask: "What if we route through City A? City B? City C?" For each stopover city, you update all city pairs with cheaper routes. After considering ALL cities as potential stopovers, you have the cheapest routes between every pair.

  • Road distance tables: The distance charts on road maps show all-pairs shortest distances.
  • Social network: Finding the shortest connection path between any two people.

JavaScript — Floyd-Warshall Algorithm
function floydWarshall(n, edges) {
    // Initialize distance matrix
    const dist = Array.from({length: n}, (_, i) =>
        Array.from({length: n}, (_, j) => i === j ? 0 : Infinity)
    );

    // Fill direct edges
    for (const [u, v, w] of edges) {
        dist[u][v] = w;
    }

    // The core: try each vertex k as intermediate
    for (let k = 0; k < n; k++) {
        for (let i = 0; i < n; i++) {
            for (let j = 0; j < n; j++) {
                if (dist[i][k] + dist[k][j] < dist[i][j]) {
                    dist[i][j] = dist[i][k] + dist[k][j];
                }
            }
        }
    }

    // Check for negative cycles: if dist[i][i] < 0
    for (let i = 0; i < n; i++) {
        if (dist[i][i] < 0) return null; // Negative cycle!
    }

    return dist;
}

// Network Delay Time variant
function networkDelayTime(times, n, k) {
    const dist = floydWarshall(n, times.map(([u,v,w]) => [u-1,v-1,w]));
    if (!dist) return -1;
    let maxTime = 0;
    for (let i = 0; i < n; i++) {
        if (dist[k-1][i] === Infinity) return -1;
        maxTime = Math.max(maxTime, dist[k-1][i]);
    }
    return maxTime;
}

⚠️ Common Mistakes

❌ Wrong loop order (k must be outermost)

The k-loop MUST be the outermost loop. for k → for i → for j. If k is inner, you'll use intermediate vertices that haven't been fully processed yet.

❌ Integer overflow with Infinity + weight

In languages with fixed-size integers, Infinity + weight can overflow. Always check if (dist[i][k] !== Infinity) before adding.

❌ Forgetting diagonal initialization

dist[i][i] should be 0 (distance from a node to itself). If you initialize the entire matrix to Infinity, the algorithm breaks.

💼 Interview Perspective

  • Floyd-Warshall appears in problems asking for all-pairs shortest paths with small n (n ≤ 200).
  • "Network Delay Time" can be solved with Floyd-Warshall, Dijkstra, or Bellman-Ford — know trade-offs.
  • The triple loop is easy to code but hard to optimize — O(V³) is fixed.
  • For transitive closure (reachability), replace min/+ with OR/AND operations.
  • Can reconstruct actual paths by maintaining a "next" matrix alongside distances.

📝 Quick Revision Notes

  • Floyd-Warshall: all-pairs shortest path using DP. dist[i][j] = min(dist[i][j], dist[i][k]+dist[k][j]).
  • Three nested loops: k (intermediate) must be OUTERMOST.
  • Time O(V³), Space O(V²). Best for small dense graphs.
  • Handles negative edges. Detects negative cycles (dist[i][i] < 0).
  • For sparse graphs with non-negative edges, running Dijkstra V times is faster.

🃏 Flash Cards

What is the recurrence relation for Floyd-Warshall?
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
For each intermediate vertex k, check if routing i→k→j is shorter than the current i→j path.
Why must k be the outermost loop?
The DP state is: "shortest path from i to j using only vertices {0, 1, ..., k} as intermediates." We build up from using no intermediates to using all vertices. If k isn't outermost, we use intermediate vertices before they're properly considered.
