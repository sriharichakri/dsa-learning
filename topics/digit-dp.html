Digit DP

Phase 12 Hard

📖 What is Digit DP?

Digit DP is a technique for counting numbers in a range [L, R] that satisfy certain digit-based constraints (e.g., digits sum to X, no adjacent same digits, contain digit 7, etc.).

Instead of checking each number individually (which is infeasible for large ranges), Digit DP builds numbers digit by digit, tracking constraints via DP states.

💡 Key Insight

Count(L, R) = Count(0, R) - Count(0, L-1). Convert to "count numbers ≤ N with property P." Process digits from most significant to least. The key state is a tight flag: are we still bounded by N's digits, or have we already placed a smaller digit (and can now place 0-9 freely)?

🌍 Real-Life Analogy

🔢 Combination Lock

Imagine a combination lock where you set digits one by one from left to right. If your limit is 357, and you've set the first digit to 2, you're "free" — the remaining digits can be anything 0-9. But if the first digit is 3, the second can only go up to 5. This "tight" constraint is exactly how digit DP works.

// Example: Count numbers ≤ n where digit sum = target
function countWithDigitSum(n, target) {
    const digits = String(n).split('').map(Number);
    const len = digits.length;
    const memo = new Map();

    function dp(pos, tight, sum) {
        if (sum > target) return 0; // Pruning
        if (pos === len) return sum === target ? 1 : 0;

        const key = `${pos},${tight},${sum}`;
        if (memo.has(key)) return memo.get(key);

        const limit = tight ? digits[pos] : 9;
        let count = 0;

        for (let d = 0; d <= limit; d++) {
            count += dp(pos + 1, tight && (d === limit), sum + d);
        }

        memo.set(key, count);
        return count;
    }

    return dp(0, true, 0);
}

⚠️ Common Mistakes

❌ Not handling leading zeros

The number "007" = 7, not 007. Use a "started" flag to distinguish leading zeros from actual zeros in the number.

❌ Wrong tight propagation

newTight = tight AND (d === digits[pos]). If previous digit was already below the limit, tight becomes permanently false for all subsequent positions.

🃏 Flash Cards

What does the "tight" flag mean in digit DP?
The tight flag indicates whether the digits placed so far exactly match the upper bound N. If tight=true, the current digit can only go up to N's digit at this position. If tight=false (we've already placed a smaller digit), the current digit can be anything 0-9.
How do you count numbers in [L, R] using digit DP?
Use the formula: count(L, R) = f(R) - f(L-1), where f(N) counts numbers from 0 to N satisfying the property. This reduces a range problem to two "count up to N" problems, which digit DP handles directly.
