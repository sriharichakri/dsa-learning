N-Queens Problem

Phase 13 Hard

📖 What is the N-Queens Problem?

Place N queens on an N×N chessboard such that no two queens attack each other. Queens attack along rows, columns, and both diagonals. This is the classic constraint satisfaction backtracking problem.

💡 Key Insight

Place one queen per row (eliminating row conflicts). For each row, try each column. Check column conflicts with a set, and diagonal conflicts using the property: cells on the same diagonal have the same row-col (main diagonal) or row+col (anti-diagonal).

🌍 Real-Life Analogy

♟️ Chess Tournament Seating

Imagine seating N VIPs in an N×N grid such that no two can see each other horizontally, vertically, or diagonally. You try placing one per row, and if a conflict arises, you move the last placed person and try the next seat.

JavaScript — N-Queens (LC 51)
function solveNQueens(n) {
    const result = [];
    const cols = new Set();
    const diag1 = new Set(); // row - col (main diagonal)
    const diag2 = new Set(); // row + col (anti-diagonal)
    const board = Array.from({length: n}, () => '.'.repeat(n));

    function backtrack(row) {
        if (row === n) {
            result.push([...board]);
            return;
        }

        for (let col = 0; col < n; col++) {
            if (cols.has(col) || diag1.has(row - col) || diag2.has(row + col)) {
                continue; // Conflict — prune!
            }

            // Place queen
            cols.add(col);
            diag1.add(row - col);
            diag2.add(row + col);
            board[row] = '.'.repeat(col) + 'Q' + '.'.repeat(n - col - 1);

            backtrack(row + 1);

            // Remove queen (backtrack)
            cols.delete(col);
            diag1.delete(row - col);
            diag2.delete(row + col);
            board[row] = '.'.repeat(n);
        }
    }

    backtrack(0);
    return result;
}

// N-Queens II: just count solutions
function totalNQueens(n) {
    let count = 0;
    const cols = new Set(), d1 = new Set(), d2 = new Set();

    function backtrack(row) {
        if (row === n) { count++; return; }
        for (let col = 0; col < n; col++) {
            if (cols.has(col) || d1.has(row-col) || d2.has(row+col)) continue;
            cols.add(col); d1.add(row-col); d2.add(row+col);
            backtrack(row + 1);
            cols.delete(col); d1.delete(row-col); d2.delete(row+col);
        }
    }

    backtrack(0);
    return count;
}

⚠️ Common Mistakes

❌ Forgetting diagonal checks

Checking only row and column conflicts misses diagonal attacks. Use row-col for main diagonals and row+col for anti-diagonals.

❌ Not restoring board state on backtrack

Must remove the queen from sets AND reset the board string when backtracking.

💼 Interview Perspective

  • N-Queens is THE classic backtracking interview problem. Know it well.
  • The diagonal trick (row-col, row+col) is elegant — explain it clearly.
  • Using Sets for O(1) conflict checking is better than scanning the board.
  • N-Queens II (just count) is simpler — no board construction needed.

📝 Quick Revision Notes

  • Place one queen per row. Try each column. Check column + 2 diagonals.
  • Main diagonal: same row-col. Anti-diagonal: same row+col.
  • Use Sets for O(1) conflict checking.
  • Time: O(n!) with pruning. Space: O(n).

🃏 Flash Cards

How do you check diagonal conflicts in N-Queens?
Two cells (r1,c1) and (r2,c2) are on the same main diagonal if r1-c1 === r2-c2, and on the same anti-diagonal if r1+c1 === r2+c2. Track used values in Sets for O(1) lookup.
Why place one queen per row in N-Queens?
Since no two queens can share a row, there must be exactly one queen per row. This reduces the problem from placing N queens on N² cells to choosing one column per row — a much smaller search space (N^N worst case, N! with pruning).
