Tree Traversal (Inorder, Preorder, Postorder, Level Order)

Phase 9 Medium

📖 What is Tree Traversal?

Tree Traversal is the process of visiting every node in a tree exactly once, in a specific order. There are four main traversals:

  • Inorder (LNR): Left → Node → Right — gives sorted order in BST
  • Preorder (NLR): Node → Left → Right — used to serialize/copy trees
  • Postorder (LRN): Left → Right → Node — used to delete trees
  • Level Order (BFS): Level by level, left to right — uses a queue
🌍 Real-Life Analogy

Think of visiting rooms in a building. Preorder = enter and mark a room, then visit its sub-rooms. Inorder = visit left wing, mark current room, then right wing. Postorder = visit all sub-rooms first, then mark the current room. Level order = visit all rooms on floor 1, then floor 2, etc.

⚙️ How It Works

Example tree for all traversals
1 / \ 2 3 / \ \ 4 5 6 Inorder (LNR): 4, 2, 5, 1, 3, 6 Preorder (NLR): 1, 2, 4, 5, 3, 6 Postorder (LRN): 4, 5, 2, 6, 3, 1 Level Order: 1, 2, 3, 4, 5, 6

Recursive Implementations

JavaScript
function inorder(root, result = []) {
    if (!root) return result;
    inorder(root.left, result);    // L
    result.push(root.val);          // N
    inorder(root.right, result);   // R
    return result;
}

function preorder(root, result = []) {
    if (!root) return result;
    result.push(root.val);          // N
    preorder(root.left, result);   // L
    preorder(root.right, result);  // R
    return result;
}

function postorder(root, result = []) {
    if (!root) return result;
    postorder(root.left, result);  // L
    postorder(root.right, result); // R
    result.push(root.val);          // N
    return result;
}

Iterative Implementations

JavaScript
// Iterative Inorder — uses stack
function inorderIterative(root) {
    const result = [], stack = [];
    let curr = root;
    
    while (curr || stack.length) {
        while (curr) {
            stack.push(curr);
            curr = curr.left;  // Go as left as possible
        }
        curr = stack.pop();
        result.push(curr.val);  // Visit node
        curr = curr.right;       // Go right
    }
    return result;
}

// Iterative Preorder — straightforward with stack
function preorderIterative(root) {
    if (!root) return [];
    const result = [], stack = [root];
    
    while (stack.length) {
        const node = stack.pop();
        result.push(node.val);
        if (node.right) stack.push(node.right); // Right first!
        if (node.left) stack.push(node.left);   // Left on top
    }
    return result;
}

// Level Order — BFS with queue
function levelOrder(root) {
    if (!root) return [];
    const result = [], queue = [root];
    
    while (queue.length) {
        const levelSize = queue.length;
        const level = [];
        
        for (let i = 0; i < levelSize; i++) {
            const node = queue.shift();
            level.push(node.val);
            if (node.left) queue.push(node.left);
            if (node.right) queue.push(node.right);
        }
        result.push(level);
    }
    return result;  // [[1], [2,3], [4,5,6]]
}

⚠️ Common Mistakes

❌ Mixing up the order of L, N, R

Inorder = LNR, Preorder = NLR, Postorder = LRN. The name tells you when the Node is processed: in-between, pre (before), post (after) the children.

❌ Pushing left before right in iterative preorder
// ❌ Left pushed first = left popped last = right processed first!
stack.push(node.left);
stack.push(node.right);

// ✅ Push RIGHT first, then LEFT (stack is LIFO!)
if (node.right) stack.push(node.right);
if (node.left) stack.push(node.left);

💼 Interview Perspective

  • Inorder of BST gives sorted order — use this to validate BSTs.
  • Preorder + Inorder can reconstruct a unique binary tree.
  • Level order is BFS — used for zigzag, right side view, level averages.
  • Know both recursive and iterative versions — iterative shows deeper understanding.
  • Morris Traversal achieves O(1) space inorder — bonus points in interviews.

📝 Quick Revision Notes

  • Inorder (LNR): sorted order in BST. Preorder (NLR): copy/serialize. Postorder (LRN): delete.
  • Level order: BFS with queue, process level by level.
  • All traversals: O(n) time. Space: O(h) for DFS, O(w) for BFS.
  • Iterative preorder: stack, push right before left. Iterative inorder: go left, pop, go right.

🃏 Flash Cards

What does inorder traversal of a BST give you?
The nodes in sorted (ascending) order. This is because in a BST, left < root < right. Inorder visits left first, then root, then right — producing a sorted sequence.
How do you remember the order: Inorder, Preorder, Postorder?
The name tells you when the Node (N) is processed relative to children (L, R):
• Preorder: N first → NLR
• Inorder: N in middle → LNR
• Postorder: N last → LRN
