Linear Search

Phase 3 Easy

📖 What is Linear Search?

Linear Search is the simplest searching algorithm. You start from the first element and check every single element one by one until you find what you're looking for — or until you've checked everything.

It works on any array — sorted or unsorted. No preconditions needed. It's the "brute force" approach to searching.

🌍 Real-Life Analogy

Imagine you lost your keys in a messy room. You don't have any system — you just start looking at the first spot, then the next, then the next, until you find them. That's linear search! You check every possible location one at a time. No shortcuts, no assumptions about where they might be.

⚙️ How It Works

  1. Start from the first element (index 0).
  2. Compare the current element with the target.
  3. If it matches → return the index. Found it!
  4. If it doesn't → move to the next element.
  5. If you've checked all elements and none matched → return -1 (not found).
JavaScript
function linearSearch(arr, target) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) {
            return i;  // Found! Return the index
        }
    }
    return -1;  // Not found
}

// Example
linearSearch([10, 23, 45, 70, 11, 15], 70);  // → 3
linearSearch([10, 23, 45, 70, 11, 15], 99);  // → -1

Variations

JavaScript
// Find ALL occurrences
function linearSearchAll(arr, target) {
    const indices = [];
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) indices.push(i);
    }
    return indices;
}

// Search in a string
function linearSearchString(str, char) {
    for (let i = 0; i < str.length; i++) {
        if (str[i] === char) return i;
    }
    return -1;
}

// Find minimum element (linear scan)
function findMin(arr) {
    let min = arr[0];
    for (let i = 1; i < arr.length; i++) {
        if (arr[i] < min) min = arr[i];
    }
    return min;
}

⚠️ Common Mistakes

❌ Using linear search on sorted data

If the array is sorted, use binary search (O(log n)) instead. Linear search wastes the sorted property and is unnecessarily slow.

❌ Forgetting to return -1
// ❌ Returns undefined if not found
function search(arr, target) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) return i;
    }
    // Missing return -1!
}

// ✅ Always handle the "not found" case
function search(arr, target) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) return i;
    }
    return -1;
}
❌ Off-by-one in loop bounds

Use i < arr.length, not i <= arr.length. The last valid index is arr.length - 1.

💼 Interview Perspective

  • Linear search is rarely asked directly — but it's the baseline that interviewers expect you to improve upon.
  • When you propose linear search, immediately ask: "Is the array sorted?" If yes, switch to binary search.
  • Linear search is used inside many algorithms: finding min/max, checking existence, counting occurrences.
  • For unsorted data with frequent searches, suggest using a hash set for O(1) lookups instead.
  • Know when linear search is optimal: unsorted data, one-time search, small arrays, linked lists.

📝 Quick Revision Notes

  • Linear search: check every element one by one. O(n) time, O(1) space.
  • Works on sorted AND unsorted arrays. No prerequisites.
  • Best for small arrays, unsorted data, or one-time searches.
  • For sorted arrays → use binary search. For frequent lookups → use hash set.
  • Always return -1 (or null) when element is not found.

🃏 Flash Cards

What is the time complexity of linear search?
O(n) in the worst and average case. O(1) best case (target is first element). It must potentially check every element.
When should you NOT use linear search?
When the data is sorted (use binary search O(log n) instead) or when you need frequent lookups (use a hash set for O(1)).
Does linear search require the array to be sorted?
No! That's its main advantage — it works on any array, sorted or unsorted. It's the only option for unsorted data (without preprocessing).
