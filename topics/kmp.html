KMP Algorithm

Phase 15 Hard

📖 What is the KMP Algorithm?

The Knuth-Morris-Pratt (KMP) algorithm is a string pattern matching algorithm that finds all occurrences of a pattern in a text in O(n + m) time, where n is the text length and m is the pattern length. It avoids the worst-case O(n × m) of the naive approach by never re-scanning characters in the text.

The key innovation is the failure function (prefix table / LPS array), which tells the algorithm how far to jump when a mismatch occurs, based on the longest proper prefix of the pattern that is also a suffix.

🔍 Real-Life Analogy: Remembering Partial Matches

Imagine you're searching for the word "ABCABD" in a long text. You've matched "ABCAB" and then hit a mismatch at 'D'. A naive approach restarts from scratch. But you notice the matched part "ABCAB" ends with "AB" which is also how your pattern starts! So instead of starting over, you slide forward, keeping the "AB" prefix matched, and continue from the 3rd character. KMP remembers what you've already matched and never goes backward in the text.

⚙️ The Failure Function (LPS Array)

LPS Array for pattern "ABABCABAB"
Pattern: A B A B C A B A B Index: 0 1 2 3 4 5 6 7 8 LPS[i] = length of longest proper prefix of pattern[0..i] that is also a suffix of pattern[0..i] i=0: "A" → no proper prefix = suffix → LPS[0] = 0 i=1: "AB" → no match → LPS[1] = 0 i=2: "ABA" → "A" = "A" (prefix=suffix) → LPS[2] = 1 i=3: "ABAB" → "AB" = "AB" → LPS[3] = 2 i=4: "ABABC" → no match → LPS[4] = 0 i=5: "ABABCA" → "A" = "A" → LPS[5] = 1 i=6: "ABABCAB" → "AB" = "AB" → LPS[6] = 2 i=7: "ABABCABA" → "ABA" = "ABA" → LPS[7] = 3 i=8: "ABABCABAB" → "ABAB" = "ABAB" → LPS[8] = 4 LPS = [0, 0, 1, 2, 0, 1, 2, 3, 4] On mismatch at position i in pattern: Jump to LPS[i-1] instead of restarting from 0.

🔧 Implementation

JavaScript — KMP Pattern Matching
// Build the LPS (Longest Proper Prefix which is also Suffix) array
function buildLPS(pattern) {
    const m = pattern.length;
    const lps = new Array(m).fill(0);
    let len = 0; // Length of previous longest prefix-suffix
    let i = 1;

    while (i < m) {
        if (pattern[i] === pattern[len]) {
            len++;
            lps[i] = len;
            i++;
        } else {
            if (len !== 0) {
                len = lps[len - 1]; // Fall back, don't increment i
            } else {
                lps[i] = 0;
                i++;
            }
        }
    }
    return lps;
}

// KMP Search — O(n + m)
function kmpSearch(text, pattern) {
    const n = text.length, m = pattern.length;
    if (m === 0) return [];

    const lps = buildLPS(pattern);
    const results = [];
    let i = 0; // Index in text
    let j = 0; // Index in pattern

    while (i < n) {
        if (text[i] === pattern[j]) {
            i++;
            j++;
        }

        if (j === m) {
            results.push(i - m); // Found match at index i - m
            j = lps[j - 1];     // Continue searching
        } else if (i < n && text[i] !== pattern[j]) {
            if (j !== 0) {
                j = lps[j - 1]; // Use failure function
            } else {
                i++;             // No prefix match, advance text
            }
        }
    }
    return results;
}

// Usage
console.log(kmpSearch("ABABDABACDABABCABAB", "ABABCABAB"));
// [9] — pattern found at index 9

console.log(kmpSearch("AAAAAA", "AAA"));
// [0, 1, 2, 3] — overlapping matches

KMP's guarantee is that the text pointer i never moves backward. Each character in the text is compared at most twice.

⚠️ Common Mistakes

❌ Confusing LPS with "longest prefix that equals suffix" (including entire string)

LPS uses proper prefix/suffix — the entire string doesn't count. "ABAB": the proper prefix-suffix is "AB" (length 2), not "ABAB" (length 4).

❌ Incrementing i on mismatch when j ≠ 0

When there's a mismatch and j > 0, only update j = lps[j-1]. Do NOT increment i — you need to re-compare the current text character with the new pattern position.

❌ Not continuing search after finding a match

After j reaches m (full match found), set j = lps[j-1] to find overlapping matches. Don't reset j to 0 — that misses overlaps like "AAA" in "AAAA".

📝 Quick Revision Notes

  • KMP: pattern matching in O(n + m) — text pointer never goes backward.
  • LPS[i] = length of longest proper prefix of pattern[0..i] that equals a suffix.
  • On mismatch at pattern[j]: if j > 0, jump j = LPS[j-1]; else advance i.
  • On full match (j = m): record position, set j = LPS[m-1] for overlapping matches.
  • Repeated Substring Pattern check: m % (m - LPS[m-1]) === 0.

🃏 Flash Cards

What does LPS[i] represent?
The length of the longest proper prefix of pattern[0..i] that is also a suffix of pattern[0..i]. "Proper" means the prefix can't be the whole string. Example: for "ABCAB", LPS[4] = 2 because "AB" is both a prefix and suffix.
Why does the text pointer never go backward in KMP?
On mismatch, instead of restarting, KMP uses the LPS array to know how much of the pattern is still valid. It slides the pattern forward, keeping the already-matched prefix. The text pointer only advances or stays — it never retreats. This guarantees O(n) comparisons.
How to check if a string is a repeated pattern using KMP?
Build the LPS array. If LPS[m-1] > 0 and m % (m - LPS[m-1]) === 0, then the string is made of repeated copies of pattern[0..m-LPS[m-1]-1]. Example: "abcabc" → LPS[5]=3, period = 6-3 = 3, 6%3=0 ✓ → repeated "abc".
