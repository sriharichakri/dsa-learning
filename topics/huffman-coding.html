Huffman Coding

Phase 11 Hard

📖 What is Huffman Coding?

Huffman Coding is a greedy algorithm for lossless data compression. It assigns variable-length binary codes to characters, where frequent characters get shorter codes and rare characters get longer codes.

The result is a prefix-free code — no code is a prefix of another — ensuring unambiguous decoding. This is the foundation of compression formats like ZIP, GZIP, JPEG, and MP3.

💡 Key Insight

Fixed-length encoding (like ASCII with 8 bits per character) wastes bits on common characters. Huffman coding uses a greedy tree-building approach: repeatedly merge the two least frequent nodes into a new parent. The resulting binary tree assigns optimal prefix codes — provably the best possible for character-by-character encoding.

🌍 Real-Life Analogy

📱 Morse Code

Morse code assigns shorter codes to common letters: E is "." (1 symbol) while Q is "--.-" (4 symbols). This is because E appears far more often in English. Huffman coding does the same thing optimally — it's like creating the perfect Morse code for any given text, guaranteed to use the fewest total symbols.

  • Speed dial: Frequent contacts get 1-digit shortcuts, rare ones get longer numbers.
  • Abbreviations: Common words get short abbreviations (Dr., St.) while rare words are spelled out.

🔧 Huffman Coding Implementation

JavaScript — Huffman Coding
class HuffmanNode {
    constructor(char, freq) {
        this.char = char;
        this.freq = freq;
        this.left = null;
        this.right = null;
    }
}

function buildHuffmanTree(text) {
    // Step 1: Count frequencies
    const freq = {};
    for (const ch of text) freq[ch] = (freq[ch] || 0) + 1;

    // Step 2: Create priority queue (min-heap by frequency)
    const pq = Object.entries(freq)
        .map(([char, f]) => new HuffmanNode(char, f));

    // Step 3: Build tree — merge two smallest nodes repeatedly
    while (pq.length > 1) {
        pq.sort((a, b) => a.freq - b.freq); // Simple sort as min-heap
        const left = pq.shift();
        const right = pq.shift();

        const parent = new HuffmanNode(null, left.freq + right.freq);
        parent.left = left;
        parent.right = right;
        pq.push(parent);
    }

    return pq[0]; // Root of Huffman tree
}

function generateCodes(root, prefix = '', codes = {}) {
    if (!root) return codes;

    if (root.char !== null) {
        codes[root.char] = prefix || '0'; // Handle single-character case
        return codes;
    }

    generateCodes(root.left, prefix + '0', codes);
    generateCodes(root.right, prefix + '1', codes);
    return codes;
}

function encode(text) {
    const root = buildHuffmanTree(text);
    const codes = generateCodes(root);

    const encoded = text.split('').map(ch => codes[ch]).join('');
    return { encoded, codes, root };
}

function decode(encoded, root) {
    let result = '';
    let node = root;

    for (const bit of encoded) {
        node = bit === '0' ? node.left : node.right;
        if (node.char !== null) {
            result += node.char;
            node = root; // Reset to root for next character
        }
    }
    return result;
}

// Usage
const { encoded, codes } = encode("ABRACADABRA");
console.log(codes);   // { A: '0', B: '100', R: '101', C: '110', D: '111' }
console.log(encoded); // "010010101101110100101​0"

⚠️ Common Mistakes

❌ Not using a min-heap

Using a sorted array and searching for minimums is O(k²). A proper min-heap reduces tree construction to O(k log k).

❌ Confusing prefix-free with prefix codes

Huffman codes are prefix-free: no code is a prefix of another. This ensures unambiguous decoding. If 'A' = '0' and 'B' = '01', the bitstring '01' is ambiguous.

❌ Forgetting the single-character edge case

If the text has only one unique character (e.g., "aaaa"), the tree has only one leaf. Assign it code '0' — don't leave it empty.

💼 Interview Perspective

  • Huffman coding demonstrates greedy optimality — a favorite proof question in algorithm courses.
  • The key operation is repeated extraction of the two minimums — a classic min-heap application.
  • Real-world usage: GZIP, DEFLATE (HTTP compression), JPEG, MP3 all use Huffman or similar.
  • Huffman is optimal for symbol-by-symbol encoding but not for overall compression (arithmetic coding is better).
  • Related problem: "Minimum Cost to Merge Stones" uses a similar heap-based approach.

📝 Quick Revision Notes

  • Huffman Coding: variable-length prefix-free codes. Frequent chars → short codes.
  • Build tree: repeatedly merge two least-frequent nodes (greedy with min-heap).
  • Codes: traverse tree — left=0, right=1. Leaf node = character code.
  • Prefix-free: no code is a prefix of another → unambiguous decoding.
  • Optimal for character-by-character encoding. O(k log k) to build, O(n) to encode/decode.

🃏 Flash Cards

How is a Huffman tree built?
1. Count character frequencies. 2. Create a leaf node for each character. 3. Put all nodes in a min-heap. 4. Repeatedly extract two minimum nodes, create a parent with their combined frequency, and insert it back. 5. The last remaining node is the root.
Why are Huffman codes prefix-free?
Because all characters are at leaf nodes of the binary tree. Since no leaf is an ancestor of another leaf, no code (root-to-leaf path) can be a prefix of another code. This ensures any encoded bitstring can be decoded unambiguously.
