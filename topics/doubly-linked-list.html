Doubly Linked List

Phase 6 Medium

📖 What is a Doubly Linked List?

A Doubly Linked List (DLL) is a linked list where each node has two pointers: one to the next node and one to the previous node. This allows traversal in both directions — forward and backward.

🌍 Real-Life Analogy

Think of a music playlist with "Previous" and "Next" buttons. A singly linked list is like a playlist with only "Next" — you can't go back. A doubly linked list is like a full playlist where you can skip forward AND backward. Each song knows the song before it and the song after it.

⚙️ How It Works

Doubly Linked List Structure
null ← [prev|10|next] ⇌ [prev|20|next] ⇌ [prev|30|next] → null head tail Each node: { prev, data, next } Singly: A → B → C → D (one-way street) Doubly: A ⇌ B ⇌ C ⇌ D (two-way street)
JavaScript
class DLLNode {
    constructor(data) {
        this.data = data;
        this.prev = null;
        this.next = null;
    }
}

class DoublyLinkedList {
    constructor() {
        this.head = null;
        this.tail = null;
        this.size = 0;
    }
    
    // Insert at front — O(1)
    insertFront(data) {
        const node = new DLLNode(data);
        if (!this.head) {
            this.head = this.tail = node;
        } else {
            node.next = this.head;
            this.head.prev = node;
            this.head = node;
        }
        this.size++;
    }
    
    // Insert at end — O(1) with tail pointer
    insertEnd(data) {
        const node = new DLLNode(data);
        if (!this.tail) {
            this.head = this.tail = node;
        } else {
            node.prev = this.tail;
            this.tail.next = node;
            this.tail = node;
        }
        this.size++;
    }
    
    // Delete a node — O(1) given the node reference
    deleteNode(node) {
        if (node.prev) node.prev.next = node.next;
        else this.head = node.next;
        
        if (node.next) node.next.prev = node.prev;
        else this.tail = node.prev;
        
        this.size--;
    }
    
    // Traverse forward
    printForward() {
        let curr = this.head;
        const result = [];
        while (curr) { result.push(curr.data); curr = curr.next; }
        return result.join(' ⇌ ');
    }
    
    // Traverse backward
    printBackward() {
        let curr = this.tail;
        const result = [];
        while (curr) { result.push(curr.data); curr = curr.prev; }
        return result.join(' ⇌ ');
    }
}

🔍 Dry Run

Insert 10, 20, 30 then delete 20

1
insertEnd(10): Create node, head=tail=node
null ← [10] → null
2
insertEnd(20): Link 10↔20, tail=20
null ← [10] ⇌ [20] → null
3
insertEnd(30): Link 20↔30, tail=30
null ← [10] ⇌ [20] ⇌ [30] → null
4
delete(20): Set 10.next=30, 30.prev=10
null ← [10] ⇌ [30] → null ✅

⚠️ Common Mistakes

❌ Forgetting to update BOTH prev and next
// ❌ Only updating next pointer
node.prev.next = node.next;
// Missing: node.next.prev = node.prev;

// ✅ Always update both directions
if (node.prev) node.prev.next = node.next;
if (node.next) node.next.prev = node.prev;
❌ Not handling head/tail edge cases

When deleting the head, update head = head.next. When deleting the tail, update tail = tail.prev. Always check for null before accessing .prev or .next.

💼 Interview Perspective

  • DLL is the backbone of LRU Cache (LC 146) — O(1) deletion with node reference.
  • Used in browser history (back/forward), text editor undo/redo.
  • When you need O(1) deletion of a known node → DLL.
  • Trade-off: DLL uses more memory per node (extra prev pointer).
  • Common interview pattern: DLL + HashMap = O(1) for both access and deletion.

📝 Quick Revision Notes

  • Each node has prev + next pointers. Can traverse both directions.
  • O(1) insertion at head/tail. O(1) deletion given node reference.
  • Used in LRU Cache, browser history, text editors.
  • Always update BOTH prev and next pointers when inserting/deleting.

🃏 Flash Cards

What is the main advantage of DLL over singly linked list?
O(1) deletion given a node reference. In a singly linked list, you need O(n) to find the previous node. DLL already has the prev pointer, so deletion is instant.
Why is DLL used in LRU Cache?
LRU Cache needs: (1) O(1) access (HashMap), (2) O(1) removal of any node, (3) O(1) insertion at front. DLL provides O(1) removal and insertion. HashMap maps keys to DLL nodes for O(1) access.
