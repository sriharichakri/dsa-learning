Insertion Sort

Phase 4 Easy

📖 What is Insertion Sort?

Insertion Sort builds the sorted array one element at a time. It picks each element and inserts it into its correct position among the already-sorted elements to its left.

It's the most natural sorting algorithm — it's how most people sort playing cards in their hand.

🌍 Real-Life Analogy

You're holding a hand of cards, sorted left to right. You draw a new card. You scan from right to left in your hand to find where it fits, shift the bigger cards right, and insert the new card. That's exactly insertion sort! Each new card is "inserted" into the correct spot in the already-sorted portion.

⚙️ How It Works

JavaScript
function insertionSort(arr) {
    const n = arr.length;
    
    for (let i = 1; i < n; i++) {
        let key = arr[i];       // Element to insert
        let j = i - 1;
        
        // Shift elements that are greater than key
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j]; // Shift right
            j--;
        }
        arr[j + 1] = key;       // Insert at correct position
    }
    return arr;
}

// insertionSort([5, 2, 4, 6, 1, 3]) → [1, 2, 3, 4, 5, 6]

💡 Why Insertion Sort Matters

Despite being O(n²), insertion sort is optimal for nearly-sorted data and small arrays. In fact, languages like V8 (JavaScript engine) use insertion sort for arrays with ≤ 10 elements inside their Timsort implementation!

⚠️ Common Mistakes

❌ Starting loop at i=0 instead of i=1

The first element (i=0) is already "sorted" by itself. Start from i=1 and insert each element into the sorted portion [0..i-1].

❌ Forgetting to save the key
// ❌ key gets overwritten during shifting!
for (let j = i - 1; j >= 0 && arr[j] > arr[i]; j--) {
    arr[j + 1] = arr[j]; // arr[i] was overwritten!
}

// ✅ Save key BEFORE shifting
let key = arr[i];
let j = i - 1;
while (j >= 0 && arr[j] > key) {
    arr[j + 1] = arr[j];
    j--;
}
arr[j + 1] = key;

💼 Interview Perspective

  • Insertion sort is stable (equal elements maintain their order) — unlike selection sort.
  • It's adaptive — O(n) for nearly sorted data. Best among O(n²) sorts for real-world use.
  • It's online — can sort data as it arrives (streaming).
  • Used as a subroutine in Timsort and for small partitions in quicksort.
  • Compare with selection sort: insertion is better for nearly sorted; selection does fewer swaps.

📝 Quick Revision Notes

  • Pick each element, insert into correct position in the sorted left portion.
  • Best: O(n) when sorted. Worst/Avg: O(n²). Space: O(1).
  • Stable, adaptive, online. Best O(n²) sort for practical use.
  • Used inside Timsort for small subarrays.

🃏 Flash Cards

Why is insertion sort preferred over selection sort for nearly-sorted data?
Insertion sort is adaptive — it does O(n) work on nearly-sorted data because the inner while loop barely runs. Selection sort always does O(n²) comparisons regardless of input order.
Is insertion sort stable? What about selection sort?
Insertion sort is stable — it shifts elements, never jumping over equal ones. Selection sort is NOT stable — swapping can displace equal elements.
