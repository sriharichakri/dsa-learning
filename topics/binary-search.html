Binary Search

Phase 3 Medium

📖 What is Binary Search?

Binary Search is a divide and conquer algorithm that finds a target value in a sorted collection by repeatedly cutting the search space in half.

Instead of checking every element (O(n)), you eliminate half the remaining elements with each comparison — making it O(log n).

💡 The Power of Halving

In an array of 1,000,000 elements, linear search might need 1,000,000 checks. Binary search needs at most 20 checks. That's the power of O(log n)!

🌍 Real-Life Analogy

📚 Finding a Word in a Dictionary

You want to find "Mango" in a dictionary. Do you start at page 1? No! You open somewhere in the middle. If you see "Paper" — too far right. You go to the middle of the left half. See "Lemon" — too far left. You go right. A few flips and you find "Mango"!

  • Number guessing game: "Think of a number 1-100. I'll guess it in 7 tries." Always guess the middle!
  • Phone book: Looking up a name by opening to the middle and narrowing down
  • Library bookshelf: Books sorted by call number, jump to approximate section

⚙️ How Binary Search Works

JavaScript
function binarySearch(arr, target) {
    let left = 0;
    let right = arr.length - 1;
    
    while (left <= right) {
        let mid = Math.floor((left + right) / 2);
        
        if (arr[mid] === target) {
            return mid;           // Found it!
        } else if (arr[mid] < target) {
            left = mid + 1;       // Target is in right half
        } else {
            right = mid - 1;      // Target is in left half
        }
    }
    return -1; // Not found
}

⭐ Binary Search Variants

Lower Bound (First occurrence ≥ target)

JavaScript
function lowerBound(arr, target) {
    let left = 0, right = arr.length;
    while (left < right) {
        let mid = Math.floor((left + right) / 2);
        if (arr[mid] < target) left = mid + 1;
        else right = mid;
    }
    return left;
}

Upper Bound (First occurrence > target)

JavaScript
function upperBound(arr, target) {
    let left = 0, right = arr.length;
    while (left < right) {
        let mid = Math.floor((left + right) / 2);
        if (arr[mid] <= target) left = mid + 1;
        else right = mid;
    }
    return left;
}

Binary Search on Answer

Sometimes you don't search an array — you search a range of possible answers.

Example: Koko Eating Bananas
// Find minimum eating speed to finish all piles in h hours
function minEatingSpeed(piles, h) {
    let left = 1;
    let right = Math.max(...piles);
    
    while (left < right) {
        let mid = Math.floor((left + right) / 2);
        let hours = piles.reduce((sum, p) => sum + Math.ceil(p / mid), 0);
        if (hours <= h) right = mid;    // Can go slower
        else left = mid + 1;            // Too slow, eat faster
    }
    return left;
}

⏱️ Complexity

AspectComplexityWhy?
Time (worst/avg)O(log n)Halving the search space each step: n → n/2 → n/4 → ... → 1
Time (best)O(1)Target is at the middle on first check
Space (iterative)O(1)Just a few variables
Space (recursive)O(log n)log n stack frames
💡 Why O(log n)?

How many times can you halve n before reaching 1? That's log₂(n). For n=1,000,000: log₂(1,000,000) ≈ 20. So at most 20 comparisons!

⚠️ Common Mistakes

❌ Integer Overflow in Mid Calculation
// ❌ Can overflow for large left + right
let mid = (left + right) / 2;

// ✅ Safe calculation
let mid = left + Math.floor((right - left) / 2);
❌ Wrong Loop Condition

left <= right (inclusive) vs left < right (exclusive). Using the wrong one causes off-by-one errors or infinite loops.

❌ Forgetting the Sorted Prerequisite

Binary search only works on sorted data. If the array isn't sorted, sort it first (O(n log n)) or use linear search.

💼 Interview Tips

  • When to use: "sorted array," "find minimum/maximum that satisfies condition," "search in rotated array"
  • Pattern recognition: If the problem has a monotonic condition (all false then all true), binary search works
  • Always clarify: Is the array sorted? Ascending or descending? Duplicates?
  • Binary Search on Answer: When the answer lies in a range and you can check if a value is valid → binary search!

📝 Quick Revision

  • Binary search = halve the search space each step. O(log n).
  • Requires sorted input or a monotonic condition.
  • Use left + (right - left) / 2 to avoid overflow.
  • left <= right for exact match, left < right for bounds.
  • "Binary Search on Answer" searches a range of possible solutions.
