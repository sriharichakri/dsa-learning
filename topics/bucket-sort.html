Bucket Sort

Phase 4 Medium

📖 What is Bucket Sort?

Bucket Sort distributes elements into several "buckets" (sub-arrays), sorts each bucket individually (often with insertion sort), and then concatenates them. It's especially efficient for uniformly distributed data.

Average case: O(n + k) where k is the number of buckets. It works beautifully for floating-point numbers in a known range.

🌍 Real-Life Analogy

Imagine sorting mail by ZIP code. You set up bins labeled 00000-09999, 10000-19999, etc. Drop each letter into its bin. Then sort within each bin (small pile = fast). Stack all bins together. The mail is sorted! Each "bin" is a bucket, and sorting small buckets is much faster than sorting everything at once.

⚙️ How It Works

JavaScript
function bucketSort(arr, bucketCount = arr.length) {
    if (arr.length <= 1) return arr;
    
    const min = Math.min(...arr);
    const max = Math.max(...arr);
    const range = max - min;
    
    // Create buckets
    const buckets = Array.from({ length: bucketCount }, () => []);
    
    // Distribute elements into buckets
    for (let num of arr) {
        const idx = range === 0 ? 0 : 
            Math.min(Math.floor(((num - min) / range) * (bucketCount - 1)), bucketCount - 1);
        buckets[idx].push(num);
    }
    
    // Sort each bucket (insertion sort for small buckets)
    let result = [];
    for (let bucket of buckets) {
        bucket.sort((a, b) => a - b);  // Or use insertion sort
        result.push(...bucket);
    }
    return result;
}

// Sort floats in [0, 1)
function bucketSortFloats(arr) {
    const n = arr.length;
    const buckets = Array.from({ length: n }, () => []);
    
    for (let num of arr) {
        const idx = Math.floor(num * n);
        buckets[Math.min(idx, n - 1)].push(num);
    }
    
    let result = [];
    for (let bucket of buckets) {
        bucket.sort((a, b) => a - b);
        result.push(...bucket);
    }
    return result;
}

// bucketSortFloats([0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21, 0.12, 0.23, 0.68])
// → [0.12, 0.17, 0.21, 0.23, 0.26, 0.39, 0.68, 0.72, 0.78, 0.94]

⚠️ Common Mistakes

❌ Using bucket sort on non-uniform data

If all elements fall into one bucket, you're just doing insertion sort on the full array → O(n²). Bucket sort assumes roughly uniform distribution.

❌ Wrong bucket index calculation

For floats in [0,1): bucket index = floor(value × n). For arbitrary ranges, normalize: floor((value - min) / range × (k-1)). Always clamp to valid bucket indices.

💼 Interview Perspective

  • Bucket sort shines for uniformly distributed floating-point numbers.
  • Average O(n+k), but O(n²) worst case if distribution is skewed.
  • Not in-place (needs extra space for buckets). Can be stable.
  • Good follow-up: "How would you sort 1 million floats in [0,1)?" → Bucket sort!
  • Combined with counting sort (integer buckets) or insertion sort (small buckets).

📝 Quick Revision Notes

  • Distribute into buckets → sort each bucket → concatenate.
  • Average: O(n+k). Worst: O(n²). Space: O(n+k).
  • Best for uniformly distributed data, especially floats.
  • Number of buckets ≈ n for best performance.

🃏 Flash Cards

When does bucket sort degrade to O(n²)?
When all elements land in the same bucket (highly skewed data). Then sorting that one bucket is O(n²) with insertion sort. Bucket sort requires uniform distribution for O(n) performance.
How is bucket sort different from counting sort?
Counting sort counts exact values (integers only). Bucket sort groups into ranges (works for floats). Counting sort is O(n+k) always; bucket sort averages O(n+k) but can degrade.
