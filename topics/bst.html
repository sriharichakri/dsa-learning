Binary Search Tree (BST)

Phase 9 Medium

📖 What is a Binary Search Tree?

A Binary Search Tree (BST) is a binary tree where every node follows the BST property:

  • All values in the left subtree are less than the node
  • All values in the right subtree are greater than the node

This ordering makes search, insert, and delete operations O(log n) on average — like doing binary search on a dynamic data set.

💡 Key Insight

An inorder traversal of a BST always produces elements in sorted order. This is the most important property to remember — it comes up in many interview problems!

🌍 Real-Life Analogy

🗄️ Organized Filing Cabinet

Imagine a filing cabinet where every drawer has a label. When you look for a file, you check the current drawer's label. If your file comes before alphabetically, go to the left section. If it comes after, go right. Each step cuts your search in half — just like a BST!

  • Library book organization: Books ordered by call number with a hierarchical structure
  • Tournament bracket: Each match narrows down — winners go one way, losers go another
  • Binary decision tree: Yes/no questions that narrow down possibilities

🔧 BST Operations

JavaScript — BST Implementation
class TreeNode {
    constructor(val) {
        this.val = val;
        this.left = null;
        this.right = null;
    }
}

class BST {
    constructor() {
        this.root = null;
    }
    
    // Insert — O(log n) average, O(n) worst
    insert(val) {
        this.root = this._insertRec(this.root, val);
    }
    _insertRec(node, val) {
        if (!node) return new TreeNode(val);
        if (val < node.val) node.left = this._insertRec(node.left, val);
        else if (val > node.val) node.right = this._insertRec(node.right, val);
        return node;
    }
    
    // Search — O(log n) average
    search(val) {
        return this._searchRec(this.root, val);
    }
    _searchRec(node, val) {
        if (!node) return false;
        if (val === node.val) return true;
        if (val < node.val) return this._searchRec(node.left, val);
        return this._searchRec(node.right, val);
    }
    
    // Delete — O(log n) average
    delete(val) {
        this.root = this._deleteRec(this.root, val);
    }
    _deleteRec(node, val) {
        if (!node) return null;
        if (val < node.val) {
            node.left = this._deleteRec(node.left, val);
        } else if (val > node.val) {
            node.right = this._deleteRec(node.right, val);
        } else {
            // Node found! Handle 3 cases:
            if (!node.left) return node.right;     // Case 1 & 2
            if (!node.right) return node.left;     // Case 2
            // Case 3: Two children — replace with inorder successor
            let successor = this._findMin(node.right);
            node.val = successor.val;
            node.right = this._deleteRec(node.right, successor.val);
        }
        return node;
    }
    _findMin(node) {
        while (node.left) node = node.left;
        return node;
    }
    
    // Inorder traversal — gives sorted order
    inorder() {
        const result = [];
        this._inorderRec(this.root, result);
        return result;
    }
    _inorderRec(node, result) {
        if (!node) return;
        this._inorderRec(node.left, result);
        result.push(node.val);
        this._inorderRec(node.right, result);
    }
}

💡 Skewed BST = Linked List!

Inserting sorted data (1, 2, 3, 4, 5) creates a skewed BST — effectively a linked list with O(n) operations. Self-balancing trees (AVL, Red-Black) prevent this.

⚠️ Common Mistakes

❌ Validating BST by Checking Only Parent-Child
// ❌ WRONG: Only checks immediate parent-child
if (node.left.val < node.val) // Not enough!

// ✅ RIGHT: Check against the FULL valid range
function isValid(node, min, max) {
    if (!node) return true;
    if (node.val <= min || node.val >= max) return false;
    return isValid(node.left, min, node.val) &&
           isValid(node.right, node.val, max);
}
❌ Forgetting the Inorder Successor in Deletion

When deleting a node with two children, you must replace it with its inorder successor (smallest in right subtree) or inorder predecessor (largest in left subtree).

❌ Assuming BST is Always Balanced

A plain BST can degrade to O(n) with sorted insertions. For guaranteed O(log n), use AVL or Red-Black trees.

💼 Interview Tips

  • Inorder = sorted: If asked to validate BST, use inorder traversal or min/max range approach
  • Kth smallest: Inorder traversal, return the kth element. Or use augmented BST with subtree sizes.
  • LCA in BST: Exploit BST property — if both values < node, go left; both > node, go right; else current node is LCA
  • BST from preorder: Use min/max bounds to reconstruct the tree in O(n)
  • BST vs Heap: BST supports search for any element. Heap only gives min/max efficiently.

🏗️ Where BSTs Are Used

  • Database indexing: B-trees (generalized BSTs) power database index structures
  • File systems: Directory structures use tree-based organization
  • Symbol tables: Compilers use BSTs to look up variable names
  • Auto-balancing variants: Red-Black trees in Java TreeMap, C++ std::map
  • Priority scheduling: Self-balancing BSTs maintain sorted order dynamically

📝 Quick Revision Notes

  • BST property: left < node < right for ALL descendants, not just immediate children
  • Inorder traversal of BST = sorted order
  • Average: O(log n) for search/insert/delete | Worst: O(n) if skewed
  • Delete has 3 cases: leaf, one child, two children (use inorder successor)
  • Validate BST using min/max range, not just parent-child comparison

🃏 Flash Cards

What traversal of a BST gives sorted output?
Inorder traversal (Left → Node → Right). This visits nodes in ascending order because left subtree < node < right subtree.
How do you find the Lowest Common Ancestor (LCA) in a BST?
Start at root. If both values < node, go left. If both > node, go right. Otherwise, current node is the LCA (the split point). O(log n) for balanced BST.
What is the worst case for BST operations and when does it happen?
O(n) — when the tree is skewed (all nodes in one direction). Happens when inserting sorted data: 1, 2, 3, 4, 5 creates a chain. Use self-balancing trees to prevent this.
How do you delete a node with two children in a BST?
Replace it with its inorder successor (smallest node in right subtree). Copy the successor's value to the target node, then delete the successor (which has at most one child).
