Union Find / Disjoint Set

Phase 10 Medium

📖 What is Union Find?

Union Find (also called Disjoint Set Union or DSU) is a data structure that tracks a set of elements partitioned into non-overlapping groups. It supports two key operations:

  • Find(x): Which group does element x belong to? (Returns the group representative/root)
  • Union(x, y): Merge the groups containing x and y into one group
💡 Key Insight

With path compression and union by rank, both operations run in nearly O(1) time — technically O(α(n)) where α is the inverse Ackermann function, which is ≤ 4 for any practical input size.

🌍 Real-Life Analogy

👫 Friend Groups Merging

Imagine a classroom where students form friend groups. Initially, everyone is their own group. When Alice becomes friends with Bob, their groups merge. When Bob becomes friends with Charlie, Alice, Bob, and Charlie are now all in the same group. To check if two people are in the same group, you find the "group leader" of each — if they share a leader, they're in the same group!

  • Social networks: Merging friend circles when two people connect
  • Network connectivity: Are two computers on the same network?
  • Country alliances: When two countries form an alliance, their entire alliance groups merge

🔧 Implementation

JavaScript — Union Find with Path Compression + Union by Rank
class UnionFind {
    constructor(n) {
        this.parent = Array.from({ length: n }, (_, i) => i); // Each is its own parent
        this.rank = new Array(n).fill(0);                     // Rank (tree height)
        this.count = n;                                        // Number of groups
    }
    
    // Find root with PATH COMPRESSION
    find(x) {
        if (this.parent[x] !== x) {
            this.parent[x] = this.find(this.parent[x]); // Point directly to root
        }
        return this.parent[x];
    }
    
    // Union by RANK (attach shorter tree under taller)
    union(x, y) {
        const rootX = this.find(x);
        const rootY = this.find(y);
        
        if (rootX === rootY) return false; // Already in same group
        
        if (this.rank[rootX] < this.rank[rootY]) {
            this.parent[rootX] = rootY;
        } else if (this.rank[rootX] > this.rank[rootY]) {
            this.parent[rootY] = rootX;
        } else {
            this.parent[rootY] = rootX;
            this.rank[rootX]++;
        }
        
        this.count--;  // One fewer group
        return true;
    }
    
    // Check if two elements are in the same group
    connected(x, y) {
        return this.find(x) === this.find(y);
    }
    
    // Get number of distinct groups
    getCount() {
        return this.count;
    }
}

// Usage
const uf = new UnionFind(5);
uf.union(0, 1);
uf.union(2, 3);
uf.connected(0, 1);  // true
uf.connected(0, 2);  // false
uf.union(1, 3);
uf.connected(0, 2);  // true (0-1-3-2 all connected)
uf.getCount();        // 2 (one group of 4, one group of 1)

💡 What is α(n)?

The inverse Ackermann function α(n) grows incredibly slowly — α(n) ≤ 4 for n up to 10^(10^(10^(10^(10^10)))). For all practical purposes, Union Find operations are constant time.

⚠️ Common Mistakes

❌ Forgetting Path Compression

Without path compression, find can degrade to O(n) for a long chain. Always set parent[x] = find(parent[x]) to point directly to root.

❌ Comparing Elements Instead of Roots
// ❌ WRONG: Comparing elements directly
if (x === y) return true;

// ✅ RIGHT: Compare their roots
if (find(x) === find(y)) return true;
❌ Not Handling 0-indexed vs 1-indexed

Many problems use 1-indexed nodes. Make sure your parent array has size n+1 if nodes go from 1 to n.

🃏 Flash Cards

What are the two optimizations in Union Find and what do they achieve?
1) Path Compression: During find, point every node directly to the root → flattens the tree. 2) Union by Rank: Attach shorter tree under taller → prevents the tree from growing tall. Together: O(α(n)) ≈ O(1).
When should you use Union Find instead of DFS/BFS?
Use Union Find when edges are added incrementally (dynamic connectivity). DFS/BFS requires the full graph upfront. Union Find also has simpler code for "are X and Y connected?" queries.
How does Union Find detect cycles in an undirected graph?
For each edge (u, v): if find(u) === find(v) before union, adding this edge creates a cycle (they're already connected). If find(u) !== find(v), union them. This is the basis of Kruskal's MST.
