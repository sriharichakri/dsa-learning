Priority Queue

Phase 8 Medium

📖 What is a Priority Queue?

A Priority Queue is an abstract data type where each element has a priority. Elements with higher priority are dequeued before elements with lower priority, regardless of insertion order. It's typically implemented using a heap.

🌍 Real-Life Analogy

Think of an emergency room. Patients don't get treated in the order they arrive (that would be a regular queue). Instead, the most critical patient (highest priority) is treated first, even if they arrived last. A gunshot wound gets priority over a sprained ankle, regardless of who came in first.

⚙️ How It Works

JavaScript
class MinPriorityQueue {
    constructor() { this.heap = []; }
    
    push(val) {
        this.heap.push(val);
        this._bubbleUp(this.heap.length - 1);
    }
    
    pop() {
        const min = this.heap[0];
        const last = this.heap.pop();
        if (this.heap.length > 0) {
            this.heap[0] = last;
            this._sinkDown(0);
        }
        return min;
    }
    
    peek() { return this.heap[0]; }
    size() { return this.heap.length; }
    
    _bubbleUp(i) {
        while (i > 0) {
            const parent = Math.floor((i - 1) / 2);
            if (this.heap[parent] <= this.heap[i]) break;
            [this.heap[parent], this.heap[i]] = [this.heap[i], this.heap[parent]];
            i = parent;
        }
    }
    
    _sinkDown(i) {
        const n = this.heap.length;
        while (true) {
            let smallest = i;
            const left = 2 * i + 1, right = 2 * i + 2;
            if (left < n && this.heap[left] < this.heap[smallest]) smallest = left;
            if (right < n && this.heap[right] < this.heap[smallest]) smallest = right;
            if (smallest === i) break;
            [this.heap[i], this.heap[smallest]] = [this.heap[smallest], this.heap[i]];
            i = smallest;
        }
    }
}

// Usage: K Closest Points to Origin
function kClosest(points, k) {
    const pq = new MinPriorityQueue(); // Sort by distance
    for (let [x, y] of points) {
        pq.push({ dist: x*x + y*y, point: [x, y] });
    }
    const result = [];
    for (let i = 0; i < k; i++) result.push(pq.pop().point);
    return result;
}

⚠️ Common Mistakes

❌ Confusing min-heap and max-heap

Min priority queue uses a min-heap (smallest on top). Max priority queue uses a max-heap. For "K largest" problems, use a min-heap of size K (counterintuitive!).

❌ Using sorted array instead of heap

A sorted array gives O(1) peek but O(n) insert (need to shift elements). A heap gives O(log n) for both — much better for dynamic data.

💼 Interview Perspective

  • Priority queue = the go-to for "Top K", "Kth largest/smallest" problems.
  • For "K largest": use min-heap of size K. Pop when size > K.
  • For "K smallest": use max-heap of size K.
  • Priority queue is essential for Dijkstra's algorithm and Prim's algorithm.
  • JavaScript doesn't have a built-in PQ — know how to implement one with a heap.

📝 Quick Revision Notes

  • Priority queue: highest priority element dequeued first.
  • Implemented as heap: push O(log n), pop O(log n), peek O(1).
  • Min-heap: smallest on top. Max-heap: largest on top.
  • Top K largest → min-heap of size K. Top K smallest → max-heap of size K.

🃏 Flash Cards

Why use a min-heap (not max-heap) for "K largest elements"?
A min-heap of size K keeps the K largest elements seen so far. The root (min of the K) acts as a gatekeeper: if a new element is larger, pop the min and push the new element. At the end, the heap contains the K largest. This is O(n log k) instead of O(n log n).
What is the difference between a priority queue and a heap?
A priority queue is an abstract data type (ADT) — it defines operations (insert, extractMin). A heap is a specific data structure used to implement it. You could also implement a PQ with a sorted array or BST, but heap is the most efficient.
