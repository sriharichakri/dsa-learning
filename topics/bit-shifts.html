Bit Shifts & Operations

Phase 14 Medium

📖 What are Bit Shifts?

Bit shifting moves all bits of a number left or right by a specified number of positions. Left shift (<<) multiplies by 2, right shift (>>) divides by 2. These are the fastest arithmetic operations available — single CPU cycle.

💡 Key Insight

n << k = n × 2^k. n >> k = ⌊n / 2^k⌋. Shifts are much faster than multiplication/division. They're used in binary search (finding midpoint), hash functions, and efficient arithmetic.

🌍 Real-Life Analogy

🔢 Decimal Place Shifting

In decimal, adding a 0 to the right (123 → 1230) multiplies by 10. Removing from the right (1230 → 123) divides by 10. Bit shifting is the same concept in binary — left shift adds a 0 (multiply by 2), right shift removes a bit (divide by 2).

JavaScript — Bit Shift Patterns
// Multiply by power of 2
const times8 = n => n << 3;  // n * 8

// Divide by power of 2 (floor)
const divBy4 = n => n >> 2;  // Math.floor(n / 4)

// Safe midpoint (avoids integer overflow)
const mid = (lo, hi) => (lo + hi) >>> 1;
// Better than (lo + hi) / 2 which can overflow in other languages

// Check if number is even/odd
const isEven = n => (n & 1) === 0;  // Last bit = 0 means even
const isOdd = n => (n & 1) === 1;

// Reverse bits of a 32-bit integer
function reverseBits(n) {
    let result = 0;
    for (let i = 0; i < 32; i++) {
        result = (result << 1) | (n & 1);
        n >>>= 1;
    }
    return result >>> 0; // Convert to unsigned
}

// Add two integers without + operator
function getSum(a, b) {
    while (b !== 0) {
        const carry = (a & b) << 1;  // Carry bits
        a = a ^ b;                    // Sum without carry
        b = carry;
    }
    return a;
}

// Bitwise AND of range [left, right]
function rangeBitwiseAnd(left, right) {
    let shift = 0;
    while (left !== right) {
        left >>= 1;
        right >>= 1;
        shift++;
    }
    return left << shift; // Common prefix
}

⚠️ Common Mistakes

❌ Confusing >> and >>>

>> is arithmetic right shift (preserves sign bit). >>> is unsigned right shift (fills with 0s). For negative numbers, they give different results: -8 >> 1 = -4 but -8 >>> 1 = 2147483644.

❌ Shifting by more than 31 bits

In JavaScript, bit shifts use modulo 32: 1 << 32 equals 1 << 0 = 1, not 0! For shifts ≥ 32, the result wraps around.

❌ Negative number right shift for division

-7 >> 1 = -4 (rounds toward negative infinity), but Math.floor(-7/2) = -4 too. However, -7 / 2 = -3.5 truncates to -3 in some languages. Be aware of rounding behavior.

📝 Quick Revision Notes

  • n << k = n × 2^k. n >> k = ⌊n / 2^k⌋.
  • >> preserves sign, >>> fills with zeros.
  • Add without +: carry = (a & b) << 1, sum = a ^ b. Repeat until carry = 0.
  • Safe midpoint: (lo + hi) >>> 1.
  • Even/odd check: n & 1 === 0 (even) or 1 (odd).

🃏 Flash Cards

How do you add two numbers using only bit operations?
Use XOR for sum-without-carry and AND+left-shift for carry: sum = a ^ b, carry = (a & b) << 1. Set a=sum, b=carry, repeat until carry is 0. Works because XOR adds without carrying, and AND finds positions that generate carries.
What's the difference between >> and >>> in JavaScript?
>> (arithmetic shift) preserves the sign bit — negative numbers stay negative. >>> (unsigned shift) fills with 0s — treats the number as unsigned. Example: -1 >> 1 = -1, -1 >>> 1 = 2147483647.
