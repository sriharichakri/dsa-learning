Recursion

Phase 1 Medium

📖 What is Recursion?

Recursion is when a function calls itself to solve a smaller version of the same problem.

Think of it like Russian nesting dolls (Matryoshka): you open a doll, and inside there's a smaller doll. You keep opening until you reach the tiniest doll that can't be opened — that's your base case.

💡 Two Essential Parts

1. Base Case: The condition where recursion stops. Without it → infinite loop → stack overflow!
2. Recursive Case: The function calls itself with a smaller/simpler input, moving toward the base case.

🌍 Real-Life Analogies

🪞 Mirror Facing a Mirror

Stand between two mirrors. You see yourself reflected infinitely. Each reflection is a "recursive call." In programming, we need a base case to stop — otherwise, it's infinite reflections (stack overflow)!

  • Finding your way out of a maze: Try a path. Dead end? Go back and try the next path. This "try, backtrack, try again" IS recursion.
  • Counting people in a line: You ask the person in front: "How many people are in front of you?" They ask the next person. The person at the front says "0" (base case). Then answers bubble back: 0 → 1 → 2 → 3.
  • Google search "recursion": Google actually shows "Did you mean: recursion?" — a fun recursive joke!

⚙️ How Recursion Works: The Call Stack

Every time a function calls itself, a new frame is pushed onto the call stack. When the base case is reached, frames start popping off and returning values.

Example: Factorial (n!)

JavaScript
function factorial(n) {
    if (n <= 1) return 1;       // Base case
    return n * factorial(n - 1); // Recursive case
}

factorial(4); // → 24

🎯 Classic Recursion Problems

1. Fibonacci Numbers

JavaScript
// Naive recursive — O(2ⁿ) time, O(n) space
function fib(n) {
    if (n <= 1) return n;         // Base cases: fib(0)=0, fib(1)=1
    return fib(n-1) + fib(n-2);   // Recursive case
}

// Optimized with memoization — O(n) time, O(n) space
function fibMemo(n, memo = {}) {
    if (n <= 1) return n;
    if (memo[n]) return memo[n];
    memo[n] = fibMemo(n-1, memo) + fibMemo(n-2, memo);
    return memo[n];
}
⚡ Why naive Fibonacci is O(2ⁿ)

Each call branches into TWO more calls. fib(5) makes 15 calls, fib(10) makes 177, fib(30) makes over 2.6 million! The tree of calls doubles at every level.

2. Sum of Array (Recursive Thinking)

JavaScript
function sumArray(arr, i = 0) {
    if (i >= arr.length) return 0;      // Base: no elements left
    return arr[i] + sumArray(arr, i+1);  // Current + rest
}
// sumArray([3, 7, 2]) → 3 + 7 + 2 + 0 = 12

3. Power Function

JavaScript
// O(n) version
function power(base, exp) {
    if (exp === 0) return 1;
    return base * power(base, exp - 1);
}

// O(log n) version — fast exponentiation!
function fastPower(base, exp) {
    if (exp === 0) return 1;
    if (exp % 2 === 0) {
        let half = fastPower(base, exp / 2);
        return half * half;
    }
    return base * fastPower(base, exp - 1);
}

⚠️ Common Mistakes

❌ Missing or Wrong Base Case
// ❌ No base case → infinite recursion → CRASH!
function bad(n) { return n + bad(n - 1); }

// ✅ Always have a base case
function good(n) {
    if (n <= 0) return 0; // BASE CASE
    return n + good(n - 1);
}
❌ Not moving toward the base case
// ❌ n never decreases!
function broken(n) {
    if (n === 0) return 0;
    return broken(n); // should be broken(n-1)
}
❌ Forgetting that recursion uses stack space

A recursion going 100,000 levels deep will likely cause a stack overflow. For large inputs, consider converting to an iterative solution.

💼 Interview Perspective

  • Recursion is the foundation for Trees, Graphs, Backtracking, and DP
  • Interviewers love asking: "Can you convert this recursive solution to iterative?"
  • Always mention the space complexity of the call stack
  • Know when recursion is better (trees, backtracking) vs when iteration is better (simple loops)
  • Memoization turns O(2ⁿ) recursion into O(n) — this is the bridge to Dynamic Programming

📝 Quick Revision

  • Recursion = function calling itself. Needs base case + recursive case.
  • Call stack grows with each call. Space = O(depth).
  • Memoization = cache results to avoid repeated work.
  • Converting recursion to iteration usually uses an explicit stack.
  • Recursion is essential for trees, graphs, backtracking, DP.
