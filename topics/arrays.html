Arrays

Phase 2 Beginner Friendly

📖 What is an Array?

Imagine you have a row of lockers in a school hallway. Each locker has a number on it (0, 1, 2, 3...) and can store exactly one item inside. That's exactly what an array is in programming!

An array is a collection of elements stored in contiguous (side-by-side) memory locations. Each element can be accessed directly using its index (position number).

💡 Key Insight

Arrays are the most fundamental data structure. Almost every other data structure (stacks, queues, heaps, hash tables) can be built using arrays. Master arrays first — everything else builds on top.

🤔 Why Do We Need Arrays?

Let's say you want to store marks of 100 students. Without arrays, you'd need:

int marks1 = 85;
int marks2 = 92;
int marks3 = 78;
// ... 97 more variables! 😱

With an array, you just need one line:

int marks[100]; // Done! All 100 values stored.

Arrays solve the problem of managing large collections of similar data.

🌍 Real-Life Analogy

🏢 Hotel Rooms

Think of a hotel floor with rooms numbered 101, 102, 103... Each room holds one guest. You can go directly to room 105 without checking rooms 101-104 first. That's random access — the superpower of arrays!

More real-world examples:

  • Playlist on Spotify: Songs arranged in order — you can jump to song #5 directly
  • Seats in a cinema: Row of numbered seats, each holds one person
  • Egg carton: Fixed slots, each holds one egg
  • Parking lot: Numbered spots, each holds one car
  • Excel spreadsheet column: Cells A1, A2, A3... each stores a value

⚙️ How Arrays Work in Memory

When you create an array, the computer reserves a contiguous block of memory. If each integer takes 4 bytes:

Address: 1000 1004 1008 1012 1016 +-------+-------+-------+-------+-------+ Value: | 10 | 25 | 40 | 60 | 15 | +-------+-------+-------+-------+-------+ Index: 0 1 2 3 4

To find element at index i, the computer calculates:

address = base_address + (i × size_of_element)
// Element at index 3: 1000 + (3 × 4) = 1012 ✓
💡 This is why array access is O(1)

The computer doesn't need to "walk through" the array. It calculates the exact memory address using simple math. This makes array access instant regardless of array size!

🔧 Common Array Operations

Accessing an Element — O(1)

Jump directly to any position using the index.

JavaScript
let arr = [10, 25, 40, 60, 15];
console.log(arr[0]);  // 10  (first element)
console.log(arr[3]);  // 60  (fourth element)
console.log(arr[arr.length - 1]); // 15 (last element)

Inserting an Element

At the end — O(1): Just add to the next slot.

At the beginning or middle — O(n): Must shift all subsequent elements.

JavaScript
let arr = [10, 25, 40];

// Insert at end — O(1)
arr.push(60);  // [10, 25, 40, 60]

// Insert at beginning — O(n) ⚠️ shifts everything
arr.unshift(5); // [5, 10, 25, 40, 60]

// Insert at index 2 — O(n)
arr.splice(2, 0, 99); // [5, 10, 99, 25, 40, 60]

Deleting an Element

From the end — O(1): Just remove the last slot.

From beginning/middle — O(n): Must shift elements to fill the gap.

JavaScript
let arr = [10, 25, 40, 60, 15];

// Delete from end — O(1)
arr.pop();  // [10, 25, 40, 60]

// Delete from beginning — O(n) ⚠️
arr.shift(); // [25, 40, 60]

// Delete at index 1 — O(n)
arr.splice(1, 1); // [25, 60]

Searching — O(n)

In an unsorted array, you must check each element one by one.

JavaScript
function linearSearch(arr, target) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) return i;
    }
    return -1; // not found
}

linearSearch([10, 25, 40, 60], 40); // returns 2

For sorted arrays, use Binary Search — O(log n). We'll learn this in Phase 3!

Traversal — O(n)

Visit every element exactly once.

JavaScript
let arr = [10, 25, 40, 60, 15];

// Method 1: for loop
for (let i = 0; i < arr.length; i++) {
    console.log(arr[i]);
}

// Method 2: for...of
for (let val of arr) {
    console.log(val);
}

// Method 3: forEach
arr.forEach((val, index) => {
    console.log(`Index ${index}: ${val}`);
});

🔍 Dry Run: Finding Maximum Element

Let's trace through finding the maximum value in [3, 7, 2, 9, 4].

Algorithm
function findMax(arr) {
    let max = arr[0]; // Start with first element
    for (let i = 1; i < arr.length; i++) {
        if (arr[i] > max) {
            max = arr[i]; // Found a bigger one!
        }
    }
    return max;
}

⭐ Famous Pattern: Kadane's Algorithm

Problem: Find the maximum sum of a contiguous subarray.

Analogy: Imagine you're walking along a number line, collecting coins (+) or paying tolls (-). You want to find the stretch of road that gives you the most profit. If your running total drops below zero, it's better to start fresh!

JavaScript
function maxSubarraySum(arr) {
    let maxSum = arr[0];
    let currentSum = arr[0];
    
    for (let i = 1; i < arr.length; i++) {
        // Either extend the previous subarray or start fresh
        currentSum = Math.max(arr[i], currentSum + arr[i]);
        maxSum = Math.max(maxSum, currentSum);
    }
    return maxSum;
}

// Example: [-2, 1, -3, 4, -1, 2, 1, -5, 4]
// Answer: 6 (subarray [4, -1, 2, 1])
🎯 Why O(n)?

We visit each element exactly once. At each position, we make a simple decision: "Should I extend my current subarray, or start a new one here?" This single-pass approach is what makes it linear time.

⏱️ Time & Space Complexity

Operation Time Why?
Access by indexO(1)Direct address calculation
Search (unsorted)O(n)May need to check every element
Search (sorted)O(log n)Binary search halves the range
Insert at endO(1)No shifting needed
Insert at beginningO(n)Shift all n elements right
Delete from endO(1)No shifting needed
Delete from beginningO(n)Shift all n elements left

Space Complexity: O(n) — storing n elements requires n memory slots.

⚠️ Common Mistakes

❌ Off-by-One Error (Most Common!)

Arrays are 0-indexed. An array of size 5 has indices 0, 1, 2, 3, 4.

let arr = [10, 20, 30, 40, 50];
// ❌ WRONG: arr[5] → undefined (index out of bounds!)
// ✅ RIGHT: arr[4] → 50 (last element)
❌ Modifying Array While Iterating

Removing elements during a forward loop skips elements.

// ❌ WRONG: Skips elements!
for (let i = 0; i < arr.length; i++) {
    if (arr[i] === target) arr.splice(i, 1);
}

// ✅ RIGHT: Iterate backwards
for (let i = arr.length - 1; i >= 0; i--) {
    if (arr[i] === target) arr.splice(i, 1);
}
❌ Confusing length with last index

arr.length gives count of elements. Last index = arr.length - 1.

💼 Interview Perspective

  • Arrays are the #1 topic in coding interviews. About 30-40% of all interview questions involve arrays.
  • Always ask: Is the array sorted? Can it have duplicates? Can values be negative?
  • Think about edge cases: Empty array, single element, all same elements, already sorted.
  • Common patterns to know: Two pointers, sliding window, prefix sum, hash map complement.
  • Optimization path: Brute force O(n²) → Hash map O(n) → Two pointers O(n) with O(1) space.
🎯 Interviewer's Perspective

Interviewers want to see you recognize patterns quickly. When you see "find pair/triplet with target sum" → think Two Pointers or HashMap. When you see "contiguous subarray" → think Sliding Window or Prefix Sum.

  • Amazon: Shopping cart items stored as an array

  • Instagram: Feed posts stored in an array, loaded in chunks

📝 Quick Revision Notes

  • Array = contiguous memory, fixed-size (in most languages), 0-indexed
  • Access: O(1) | Search: O(n) unsorted, O(log n) sorted | Insert/Delete at ends: O(1)
  • Key pattern: "Can I solve this with two pointers or a hash map to avoid O(n²)?"
  • Always handle edge cases: empty array, single element, duplicates
  • Kadane's Algorithm = max subarray sum in O(n)

🃏 Flash Cards

What is the time complexity of accessing array[i]?
O(1) — constant time. The address is calculated directly: base + (i × element_size)
Why is inserting at the beginning of an array O(n)?
Every existing element must shift one position to the right to make room. n elements = n shifts.
What is Kadane's Algorithm used for?
Finding the maximum sum contiguous subarray in O(n) time. Key idea: extend current subarray or start fresh.
