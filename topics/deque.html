Double-ended Queue (Deque)

Phase 8 Medium

📖 What is a Deque?

A Deque (Double-Ended Queue, pronounced "deck") allows insertion and deletion at both ends — front and back. It's a hybrid of a stack and a queue, supporting all four operations in O(1).

🌍 Real-Life Analogy

Think of a tunnel with two openings. Cars can enter and exit from either end. A queue is a one-way tunnel (enter back, exit front). A stack is a dead-end tunnel (enter and exit same end). A deque is the most flexible — two-way traffic at both ends!

⚙️ How It Works

Deque Operations
Front Back addFront ↔ removeFront addBack ↔ removeBack ↓ ↓ [10, 20, 30, 40, 50] ↑ ↑ peekFront peekBack Stack behavior: addBack + removeBack (or addFront + removeFront) Queue behavior: addBack + removeFront
JavaScript
class Deque {
    constructor() {
        this.items = {};
        this.front = 0;
        this.back = 0;
    }
    
    addFront(val) {
        this.front--;
        this.items[this.front] = val;
    }
    
    addBack(val) {
        this.items[this.back] = val;
        this.back++;
    }
    
    removeFront() {
        if (this.isEmpty()) return undefined;
        const val = this.items[this.front];
        delete this.items[this.front];
        this.front++;
        return val;
    }
    
    removeBack() {
        if (this.isEmpty()) return undefined;
        this.back--;
        const val = this.items[this.back];
        delete this.items[this.back];
        return val;
    }
    
    peekFront() { return this.items[this.front]; }
    peekBack()  { return this.items[this.back - 1]; }
    isEmpty()   { return this.back - this.front === 0; }
    size()      { return this.back - this.front; }
}

// Sliding Window Maximum using deque
function maxSlidingWindow(nums, k) {
    const deque = []; // Stores indices
    const result = [];
    
    for (let i = 0; i < nums.length; i++) {
        // Remove indices out of window
        while (deque.length && deque[0] < i - k + 1) deque.shift();
        // Remove smaller elements (they'll never be the max)
        while (deque.length && nums[deque[deque.length - 1]] < nums[i]) deque.pop();
        deque.push(i);
        
        if (i >= k - 1) result.push(nums[deque[0]]);
    }
    return result;
}

// maxSlidingWindow([1,3,-1,-3,5,3,6,7], 3) → [3,3,5,5,6,7]

⚠️ Common Mistakes

❌ Using array shift() as deque front removal

JavaScript's Array.shift() is O(n) because it reindexes all elements. For a true O(1) deque, use a linked list or hash-based approach. For competitive programming, the O(n) shift is often acceptable for small inputs.

❌ Not removing out-of-window elements

In sliding window max, always check if deque[0] is still within the window bounds. Remove from front if deque[0] < i - k + 1.

💼 Interview Perspective

  • "Sliding Window Maximum" (LC 239) is THE classic deque problem — learn it cold.
  • A deque can act as both a stack AND a queue simultaneously.
  • In Python: collections.deque. In Java: ArrayDeque. In JS: implement manually.
  • Deque is key for monotonic queue problems (next topic).
  • Used in BFS with 0-1 weights (0-1 BFS): add 0-weight to front, 1-weight to back.

📝 Quick Revision Notes

  • Deque: insert/delete at BOTH ends in O(1).
  • Can simulate stack (one end) or queue (both ends).
  • Sliding Window Max: maintain decreasing deque of indices.
  • Remove from front if out of window, from back if smaller than current.

🃏 Flash Cards

How does a deque solve Sliding Window Maximum in O(n)?
Maintain a monotonically decreasing deque of indices. Front = max of current window. For each new element: remove from back anything smaller (they'll never be useful), remove from front if out of window. Each element enters and leaves the deque at most once → O(n).
How is a deque different from a regular queue?
A queue allows insert at back, remove at front (FIFO). A deque allows both insert and remove at both ends. It's strictly more powerful — it can simulate both a stack and a queue.
