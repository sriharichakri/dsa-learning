Grid DP

Phase 12 Medium

📖 What is Grid DP?

Grid DP solves optimization or counting problems on 2D grids. Typical problems: count paths from top-left to bottom-right, find minimum cost path, or determine if a path exists with certain constraints.

💡 Key Insight

In grid DP, each cell's value depends on its neighbors (usually top and left for top-to-bottom traversal). The recurrence is: dp[i][j] = f(dp[i-1][j], dp[i][j-1]) where f is sum (counting), min (optimization), or max. Always handle the first row and first column as base cases.

🌍 Real-Life Analogy

🗺️ City Block Navigation

Imagine walking from the top-left corner of a city grid to the bottom-right, only allowed to go right or down. How many different routes can you take? Each intersection's count equals the sum of routes from the intersection above and to the left.

⚙️ Grid DP Problems

1. Unique Paths with Obstacles

JavaScript — Unique Paths II (LC 63)
function uniquePathsWithObstacles(grid) {
    const m = grid.length, n = grid[0].length;
    if (grid[0][0] === 1) return 0;

    const dp = Array.from({length: m}, () => new Array(n).fill(0));
    dp[0][0] = 1;

    // First row: stop at first obstacle
    for (let j = 1; j < n; j++) dp[0][j] = grid[0][j] === 1 ? 0 : dp[0][j-1];
    // First column: stop at first obstacle
    for (let i = 1; i < m; i++) dp[i][0] = grid[i][0] === 1 ? 0 : dp[i-1][0];

    for (let i = 1; i < m; i++) {
        for (let j = 1; j < n; j++) {
            dp[i][j] = grid[i][j] === 1 ? 0 : dp[i-1][j] + dp[i][j-1];
        }
    }
    return dp[m-1][n-1];
}

2. Minimum Path Sum

JavaScript — Minimum Path Sum (LC 64)
function minPathSum(grid) {
    const m = grid.length, n = grid[0].length;
    for (let j = 1; j < n; j++) grid[0][j] += grid[0][j-1];
    for (let i = 1; i < m; i++) grid[i][0] += grid[i-1][0];

    for (let i = 1; i < m; i++) {
        for (let j = 1; j < n; j++) {
            grid[i][j] += Math.min(grid[i-1][j], grid[i][j-1]);
        }
    }
    return grid[m-1][n-1];
}

3. Dungeon Game (reverse DP)

JavaScript — Dungeon Game (LC 174)
function calculateMinimumHP(dungeon) {
    const m = dungeon.length, n = dungeon[0].length;
    const dp = Array.from({length: m + 1}, () => new Array(n + 1).fill(Infinity));
    dp[m][n-1] = dp[m-1][n] = 1; // Need at least 1 HP to survive

    for (let i = m - 1; i >= 0; i--) {
        for (let j = n - 1; j >= 0; j--) {
            const minHP = Math.min(dp[i+1][j], dp[i][j+1]) - dungeon[i][j];
            dp[i][j] = Math.max(1, minHP); // At least 1 HP needed
        }
    }
    return dp[0][0];
}

⚠️ Common Mistakes

❌ Wrong traversal direction

Most grid DP goes top-left to bottom-right. But Dungeon Game requires bottom-right to top-left because you need to know future damage to decide current HP.

❌ Forgetting obstacles block the entire row/column

In Unique Paths II, an obstacle in the first row blocks all cells to its right. An obstacle in the first column blocks all cells below it.

💼 Interview Perspective

  • Grid DP is one of the most common DP patterns — especially Unique Paths and Min Path Sum.
  • Always ask: can I modify the input grid in-place? This gives O(1) space.
  • Know when to traverse forward (counting/min cost) vs backward (Dungeon Game).
  • Space optimization: process row by row, keep only current + previous row.

📝 Quick Revision Notes

  • Grid DP: 2D table where dp[i][j] depends on top (dp[i-1][j]) and left (dp[i][j-1]).
  • Counting paths: dp[i][j] = dp[i-1][j] + dp[i][j-1].
  • Min cost: dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1]).
  • Handle first row and column separately as base cases.
  • Dungeon Game: reverse DP from bottom-right to top-left.

🃏 Flash Cards

Why does the Dungeon Game require reverse DP?
Because the minimum HP needed at each cell depends on future rooms (what damage/healing lies ahead). Forward DP can't determine the minimum starting HP since it doesn't know what's coming. Reverse DP starts at the destination and works backward.
How do you handle obstacles in grid DP?
Set dp[i][j] = 0 (or Infinity for min cost) when grid[i][j] is an obstacle. For the first row/column, once you hit an obstacle, all subsequent cells in that row/column are also 0 (unreachable).
