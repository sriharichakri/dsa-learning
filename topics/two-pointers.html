Two Pointers

Phase 5 Medium

📖 What is the Two Pointers Technique?

The Two Pointers technique uses two references (indices) that move through a data structure simultaneously, typically from different positions or at different speeds. This allows you to process pairs of elements efficiently without nested loops.

It reduces O(n²) brute force solutions to O(n) by cleverly eliminating unnecessary comparisons.

💡 Key Insight

Two Pointers works because of sorted order or structural properties. In a sorted array, if the sum of two elements is too large, moving the right pointer left decreases it. Too small? Move the left pointer right. This directed movement eliminates entire groups of impossible pairs.

🌍 Real-Life Analogy

🚶‍♂️ Two People Walking Toward Each Other

Imagine a long hallway with numbered tiles. Two people start at opposite ends and walk toward each other. At each step, they check the numbers on their tiles. If the sum is too small, the person on the left moves forward (to a bigger number). If too big, the person on the right moves backward. They keep adjusting until they find the target sum or meet in the middle.

  • Squeezing a tube of toothpaste: You press from both ends toward the middle — that's opposite-direction two pointers.
  • Tortoise and Hare: Slow and fast runners on a track — if there's a loop, the fast one catches up. That's fast-slow pointers!
  • Merging two sorted piles: Take the smaller top card from either pile — that's same-direction two pointers.

⚙️ Three Two-Pointer Patterns

1. Opposite Direction (Converging)

Left starts at beginning, right starts at end. They move toward each other.

Two Sum II: Find pair summing to 9 in [1, 2, 4, 6, 8, 9]
[1, 2, 4, 6, 8, 9] L R sum=1+9=10 > 9 → move R left L R sum=1+8=9 ✅ Found! [1, 3, 5, 7] L R sum=1+7=8 < target? → move L right L R sum=3+7=10 > target? → move R left L R sum=3+5=8 ✅
JavaScript
// Two Sum II — sorted array
function twoSumSorted(nums, target) {
    let left = 0, right = nums.length - 1;
    
    while (left < right) {
        let sum = nums[left] + nums[right];
        if (sum === target) return [left, right];
        else if (sum < target) left++;    // Need bigger sum
        else right--;                     // Need smaller sum
    }
    return [];
}

2. Same Direction (Fast-Slow)

Both pointers start at the beginning. One moves faster. Used for removing duplicates, partitioning.

JavaScript
// Remove duplicates from sorted array (in-place)
function removeDuplicates(nums) {
    if (nums.length === 0) return 0;
    let slow = 0;
    
    for (let fast = 1; fast < nums.length; fast++) {
        if (nums[fast] !== nums[slow]) {
            slow++;
            nums[slow] = nums[fast];
        }
    }
    return slow + 1; // Length of unique portion
}

// [1, 1, 2, 2, 3] → [1, 2, 3, _, _] returns 3
Remove Duplicates: [1, 1, 2, 2, 3]
[1, 1, 2, 2, 3] S F nums[F]=1 == nums[S]=1 → skip S F nums[F]=2 != nums[S]=1 → S++, copy S F nums[F]=2 == nums[S]=2 → skip S F nums[F]=3 != nums[S]=2 → S++, copy S F F reaches end. Answer = S+1 = 3 Result: [1, 2, 3, ...] (first 3 elements are unique)

3. Fast-Slow (Cycle Detection — Floyd's Algorithm)

JavaScript
// Detect cycle in linked list
function hasCycle(head) {
    let slow = head, fast = head;
    
    while (fast && fast.next) {
        slow = slow.next;        // 1 step
        fast = fast.next.next;   // 2 steps
        if (slow === fast) return true; // They met = cycle!
    }
    return false; // fast reached end = no cycle
}

🔍 Dry Run: Container With Most Water

Find two lines that together with x-axis form the container holding the most water.

JavaScript
function maxArea(height) {
    let left = 0, right = height.length - 1;
    let maxWater = 0;
    
    while (left < right) {
        let width = right - left;
        let h = Math.min(height[left], height[right]);
        maxWater = Math.max(maxWater, width * h);
        
        if (height[left] < height[right]) left++;
        else right--;
    }
    return maxWater;
}

⚠️ Common Mistakes

❌ Using two pointers on unsorted data

The opposite-direction technique relies on sorted order. If the array isn't sorted, sort it first (O(n log n)) or use a HashMap instead.

❌ Forgetting to skip duplicates in 3Sum
// ❌ Gets duplicate triplets
// ✅ Skip duplicates after finding a match
while (left < right && nums[left] === nums[left + 1]) left++;
while (left < right && nums[right] === nums[right - 1]) right--;
❌ Moving the wrong pointer

In opposite-direction: if sum is too small, move LEFT right (increase). If too big, move RIGHT left (decrease). Getting this backwards gives wrong answers.

💼 Interview Perspective

  • Sorted array + find pair? → Two pointers from both ends. O(n) time, O(1) space.
  • Remove/move elements in-place? → Slow-fast pointers from the same end.
  • Cycle detection? → Floyd's tortoise and hare (fast-slow pointers).
  • Two pointers gives O(1) space which HashMaps don't — mention this trade-off to impress interviewers.
  • For 3Sum: fix one element, use two pointers on the rest → O(n²) total.

📝 Quick Revision Notes

  • Opposite direction: L at start, R at end, move inward. Works on sorted arrays.
  • Same direction: slow/fast from start. Remove duplicates, partition, merge.
  • Fast-slow: cycle detection. Fast moves 2x, if they meet → cycle exists.
  • Two pointers typically give O(n) time and O(1) space.
  • Sort first if array is unsorted (adds O(n log n) but enables two pointers).

🃏 Flash Cards

What are the three two-pointer patterns?
1. Opposite direction (converging from both ends) — used for pair sums in sorted arrays. 2. Same direction (slow/fast) — used for removing duplicates. 3. Fast-slow (Floyd's) — used for cycle detection.
Why does the two-pointer approach work on sorted arrays?
Sorted order creates a monotonic property: moving left pointer right increases the sum, moving right pointer left decreases it. This eliminates large groups of impossible pairs in each step.
How does Floyd's cycle detection work?
Two pointers start at head. Slow moves 1 step, fast moves 2 steps per iteration. If there's a cycle, fast will eventually "lap" slow and they'll meet. If fast reaches null, there's no cycle.
