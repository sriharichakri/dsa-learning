Z Algorithm

Phase 15 Hard

📖 What is the Z Algorithm?

The Z Algorithm constructs a Z-array for a string in O(n) time. Z[i] is the length of the longest substring starting at position i that matches a prefix of the string. This array enables pattern matching, period detection, and many other string operations.

For pattern matching, concatenate pattern + "$" + text and build the Z-array. Any position where Z[i] = pattern.length indicates a match.

🔍 Real-Life Analogy: Comparing Book Openings

Imagine you have a long book and you want to find every paragraph that starts the same way as the book's opening line. For each paragraph, you compare it character by character with the start. But cleverly, if you already compared ahead and found a matching stretch, you reuse that information for nearby paragraphs. The Z-array is exactly this: for each position, how far does the text match the beginning?

🔧 Implementation

JavaScript — Z Algorithm
function zFunction(s) {
    const n = s.length;
    const z = new Array(n).fill(0);
    let l = 0, r = 0; // [l, r] = rightmost Z-box

    for (let i = 1; i < n; i++) {
        if (i < r) {
            // Reuse previously computed info
            z[i] = Math.min(r - i, z[i - l]);
        }
        // Extend as far as possible
        while (i + z[i] < n && s[z[i]] === s[i + z[i]]) {
            z[i]++;
        }
        // Update the Z-box if we extended past r
        if (i + z[i] > r) {
            l = i;
            r = i + z[i];
        }
    }
    return z;
}

// Pattern matching using Z-algorithm — O(n + m)
function zSearch(text, pattern) {
    const combined = pattern + "$" + text;
    const z = zFunction(combined);
    const results = [];
    const m = pattern.length;

    for (let i = m + 1; i < combined.length; i++) {
        if (z[i] === m) {
            results.push(i - m - 1); // Convert to text index
        }
    }
    return results;
}

// Usage
console.log(zFunction("aabxaab"));   // [0, 1, 0, 0, 3, 1, 0]
console.log(zSearch("xaabxaab", "aab")); // [1, 5]

The inner while loop is bounded by the total advancement of r, which goes from 0 to n. So total work is O(n).

⚠️ Common Mistakes

❌ Forgetting the separator character in pattern matching

When concatenating pattern + text, use a separator (like '$') that doesn't appear in either string. Without it, the Z-array may find matches that span across the boundary.

❌ Not capping z[i] at r - i before extending

When i is inside the current Z-box [l, r), z[i] should start at min(r - i, z[i - l]). Without the min, you might use a value that goes beyond r, which hasn't been verified.

❌ Confusing Z-array with LPS array (KMP)

Z[i] = length of longest match between s[0..] and s[i..]. LPS[i] = length of longest proper prefix of s[0..i] that equals a suffix. They encode similar but different information.

💼 Interview Perspective

  • Z-algorithm is a powerful alternative to KMP for pattern matching, often easier to understand and implement.
  • Can detect if a string is a rotation of another: check if pattern appears in text+text.
  • Period detection: the string has period p if z[p] ≥ n - p (and p divides n).
  • Less commonly asked than KMP in interviews, but equally valid as an answer.
  • The Z-box trick ([l, r] window) is what makes it linear — understand this to explain complexity.

📝 Quick Revision Notes

  • Z[i] = length of longest substring starting at i that matches a prefix of the string.
  • Built in O(n) using the Z-box optimization [l, r].
  • Pattern matching: build Z-array on "pattern$text", look for Z[i] = m.
  • Z-box: rightmost interval [l, r) where s[l..r) = s[0..r-l). Reuse info for i inside the box.

🃏 Flash Cards

What does Z[i] represent?
Z[i] is the length of the longest substring starting at position i in the string that is also a prefix of the string. For example, in "aabxaab", Z[4]=3 because s[4..6]="aab" matches the prefix "aab".
How does the Z-box make Z-algorithm O(n)?
The Z-box [l, r] tracks the rightmost interval matching a prefix. For i inside [l, r), we reuse z[i-l] (already computed). The explicit comparisons only happen when extending beyond r, and r only moves forward, so total comparisons are O(n).
