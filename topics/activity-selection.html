Activity Selection Problem

Phase 11 Medium

📖 What is the Activity Selection Problem?

The Activity Selection Problem is the foundational greedy algorithm problem: given a set of activities with start and finish times, select the maximum number of non-overlapping activities a single person can perform.

The greedy strategy is simple and provably optimal: always select the activity that finishes earliest among the remaining compatible activities.

💡 Key Insight

This was the first problem proven to have an optimal greedy solution. The proof uses an exchange argument: any other choice can be "exchanged" for the earliest-finishing activity without reducing the total count. This makes it the textbook example for proving greedy correctness.

🌍 Real-Life Analogy

🎭 Attending Festival Events

You're at a festival with many shows at different times. You want to attend as many shows as possible, but you can only be at one show at a time. Strategy: always pick the show that ends soonest — this gives you the most time to fit in more shows afterward.

  • Job scheduling: Complete the most jobs when each has a fixed time window.
  • Classroom usage: Schedule the most classes in one classroom.
  • TV recording: Record the most shows on a single-tuner DVR.

⚙️ Activity Selection Implementation

Activity Selection — Sort by finish time, pick greedily
Activities (sorted by finish time): A: [1, 3] ████ B: [2, 5] ██████ C: [4, 6] ████ D: [6, 8] ████ E: [5, 9] ████████ F: [8,10] ████ Select A(ends 3) → next must start >= 3 Select C(starts 4, ends 6) → next must start >= 6 Select D(starts 6, ends 8) → next must start >= 8 Select F(starts 8, ends 10) ✅ Maximum: 4 activities [A, C, D, F]
JavaScript — Activity Selection (Greedy)
function activitySelection(activities) {
    // Sort by finish time
    activities.sort((a, b) => a[1] - b[1]);

    const selected = [activities[0]];
    let lastFinish = activities[0][1];

    for (let i = 1; i < activities.length; i++) {
        if (activities[i][0] >= lastFinish) {
            // Compatible — no overlap with last selected
            selected.push(activities[i]);
            lastFinish = activities[i][1];
        }
    }
    return selected;
}

// Recursive version (textbook style)
function activitySelectionRecursive(activities, k = 0, lastFinish = 0) {
    // Find the first activity that starts after lastFinish
    let m = k;
    while (m < activities.length && activities[m][0] < lastFinish) {
        m++;
    }
    if (m >= activities.length) return [];
    return [activities[m], ...activitySelectionRecursive(activities, m + 1, activities[m][1])];
}

// Usage
const activities = [[1,3], [2,5], [4,6], [6,8], [5,9], [8,10]];
console.log(activitySelection(activities));
// [[1,3], [4,6], [6,8], [8,10]] — 4 activities

📐 Why Does Greedy Work Here?

Exchange Argument: Suppose the optimal solution picks activity X first (which doesn't end earliest). We can replace X with the earliest-finishing activity A without reducing the total count, because A ends no later than X, so all activities compatible with X are also compatible with A. Therefore, the greedy choice is at least as good as any other.

🎯 Interview Note

Understanding why greedy works (exchange argument) is more important than just knowing the algorithm. Interviewers may ask you to prove correctness.

⚠️ Common Mistakes

❌ Sorting by start time instead of finish time

Sorting by start time and picking the earliest start doesn't maximize selections. An early-starting but long activity blocks many others.

❌ Sorting by duration

Shortest activity first also doesn't work: a short activity in the middle might block two compatible long ones on either side.

❌ Confusing with weighted activity selection

If activities have different values/weights, greedy doesn't work — you need DP (weighted job scheduling).

💼 Interview Perspective

  • Activity Selection is identical to "Non-overlapping Intervals" (LC 435) — same algorithm, different framing.
  • Be ready to explain the greedy proof (exchange argument).
  • Variations: weighted version (use DP), multiple rooms (use heap), insert/merge intervals.
  • If the problem has weights or priorities, greedy likely won't work — consider DP.
  • Classic greedy problems often ask: "prove the greedy choice is safe."

📝 Quick Revision Notes

  • Activity Selection: maximize non-overlapping activities.
  • Greedy: sort by finish time, always pick earliest-finishing compatible activity.
  • Proof: exchange argument — greedy choice never worse than any alternative.
  • Time O(n log n), Space O(1). Identical to LC 435.
  • Doesn't work with weights — use DP for weighted version.

🃏 Flash Cards

What is the greedy strategy for the Activity Selection Problem?
Sort activities by finish time. Always select the next activity that starts after or when the last selected activity finishes. This maximizes the total number of non-overlapping activities.
What is the exchange argument for Activity Selection?
If the optimal solution doesn't start with the earliest-finishing activity, we can swap in the earliest-finishing one without reducing the count. Since it ends no later, all subsequent compatible activities remain compatible. So greedy is at least as good.
