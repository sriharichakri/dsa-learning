XOR Tricks

Phase 14 Medium

📖 What is XOR?

XOR (exclusive OR) is a bitwise operation that returns 1 when bits differ and 0 when they're the same. Its key properties make it incredibly useful for finding unique elements, swapping values, and solving parity problems.

💡 Key Properties

a ^ a = 0 (self-inverse), a ^ 0 = a (identity), commutative and associative. XORing all elements cancels out duplicates, leaving only the unique element. This is the basis of the famous "Single Number" problem.

🌍 Real-Life Analogy

💡 Light Switch Toggle

A light switch XORs its state: flip once → on. Flip again → off (back to original). Two flips cancel out. If you flip a switch an odd number of times, it's on; even times, it's off. XOR works the same way with bits.

⚙️ XOR Trick Problems

Single Number (every element appears twice except one)

JavaScript — Single Number (LC 136)
function singleNumber(nums) {
    let result = 0;
    for (const num of nums) {
        result ^= num; // Pairs cancel out: a^a=0, 0^b=b
    }
    return result;
}
// [4,1,2,1,2] → 4^1^2^1^2 = (1^1)^(2^2)^4 = 0^0^4 = 4

Single Number III (two unique elements)

JavaScript — Single Number III (LC 260)
function singleNumber3(nums) {
    // Step 1: XOR all → get xor of two unique numbers (a ^ b)
    let xorAll = 0;
    for (const num of nums) xorAll ^= num;

    // Step 2: Find a bit where a and b differ
    const diffBit = xorAll & (-xorAll); // Lowest set bit

    // Step 3: Split into two groups and XOR each
    let a = 0, b = 0;
    for (const num of nums) {
        if (num & diffBit) a ^= num;
        else b ^= num;
    }
    return [a, b];
}
// [1,2,1,3,2,5] → a=3, b=5

Missing Number

JavaScript — Missing Number (LC 268)
function missingNumber(nums) {
    let xor = nums.length; // Start with n
    for (let i = 0; i < nums.length; i++) {
        xor ^= i ^ nums[i]; // XOR index and value
    }
    return xor;
}
// [3,0,1] → 3^(0^3)^(1^0)^(2^1) = 3^3^0^1^2^1 = 0^0^2 = 2

Swap Without Temp

JavaScript — XOR Swap
function xorSwap(arr, i, j) {
    if (i === j) return; // Important: XOR swap fails if same index!
    arr[i] ^= arr[j]; // a = a^b
    arr[j] ^= arr[i]; // b = b^(a^b) = a
    arr[i] ^= arr[j]; // a = (a^b)^a = b
}

⚠️ Common Mistakes

❌ XOR swap on the same element

a ^= a gives 0, destroying the value. Always check if (i !== j) before XOR swapping array elements.

❌ Assuming XOR works for more than 2 duplicates

Single Number (XOR trick) only works when every element appears exactly twice except one. For three duplicates (Single Number II), you need a different approach (bit counting modulo 3).

💼 Interview Perspective

  • Single Number is asked frequently — know the XOR solution instantly.
  • Missing Number can use XOR or math (sum formula). Know both.
  • Single Number III is tricky — splitting by a differing bit is the key insight.
  • XOR is useful for parity, toggling, and finding differences between sets.

📝 Quick Revision Notes

  • XOR properties: a^a=0, a^0=a, commutative, associative.
  • Single Number: XOR all elements. Pairs cancel, unique remains.
  • Missing Number: XOR indices 0..n with all elements.
  • Two uniques: XOR all, find differing bit, split and XOR each group.
  • XOR swap: a^=b; b^=a; a^=b. Don't use if a and b are the same variable.

🃏 Flash Cards

Why does XORing all elements find the single number?
Because a ^ a = 0 (pairs cancel) and a ^ 0 = a (identity). When every number appears twice except one, all pairs cancel to 0, leaving only the unique number. XOR is commutative and associative, so order doesn't matter.
How does Single Number III find TWO unique elements?
1. XOR all → get a^b (the two uniques XORed). 2. Find any set bit in a^b (this bit differs between a and b). 3. Split all numbers into two groups by that bit. 4. XOR each group separately → one gives a, the other gives b. Pairs still cancel within each group.
