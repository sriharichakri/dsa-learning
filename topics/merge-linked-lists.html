Merge Linked Lists

Phase 6 Medium

📖 What is Merging Linked Lists?

Merging linked lists combines two (or more) sorted linked lists into one sorted list. It's the linked list version of the merge step in merge sort — comparing heads and always picking the smaller one.

🌍 Real-Life Analogy

Imagine two lines of people at an airport, both already sorted by boarding group number. You merge them into one line by always picking the person with the lower boarding number from the front of either line. You keep doing this until both lines are empty — now you have one perfectly merged, sorted line!

⚙️ How It Works

Merge Two Sorted Lists (Iterative)

JavaScript
function mergeTwoLists(l1, l2) {
    const dummy = { next: null };
    let tail = dummy;
    
    while (l1 && l2) {
        if (l1.val <= l2.val) {
            tail.next = l1;
            l1 = l1.next;
        } else {
            tail.next = l2;
            l2 = l2.next;
        }
        tail = tail.next;
    }
    
    tail.next = l1 || l2;  // Attach remaining
    return dummy.next;
}

Merge Two Sorted Lists (Recursive)

JavaScript
function mergeTwoListsRecursive(l1, l2) {
    if (!l1) return l2;
    if (!l2) return l1;
    
    if (l1.val <= l2.val) {
        l1.next = mergeTwoListsRecursive(l1.next, l2);
        return l1;
    } else {
        l2.next = mergeTwoListsRecursive(l1, l2.next);
        return l2;
    }
}

Merge K Sorted Lists (Divide & Conquer)

JavaScript
function mergeKLists(lists) {
    if (lists.length === 0) return null;
    
    while (lists.length > 1) {
        const merged = [];
        for (let i = 0; i < lists.length; i += 2) {
            const l1 = lists[i];
            const l2 = i + 1 < lists.length ? lists[i + 1] : null;
            merged.push(mergeTwoLists(l1, l2));
        }
        lists = merged;
    }
    return lists[0];
}
// Time: O(N log k) where N = total nodes, k = number of lists

⏱️ Complexity Analysis

OperationTimeSpace
Merge 2 lists (iterative)O(n + m)O(1)
Merge 2 lists (recursive)O(n + m)O(n + m) stack
Merge K lists (D&C)O(N log k)O(1)*
Merge K lists (heap)O(N log k)O(k)

N = total nodes across all lists, k = number of lists.

⚠️ Common Mistakes

❌ Forgetting the dummy node

Without a dummy node, you need special logic for the first element. Using dummy = { next: null } simplifies the code — just build from dummy.next and return dummy.next.

❌ Not attaching remaining nodes
// ❌ Forgets leftover nodes!
while (l1 && l2) { ... }
// Missing: tail.next = l1 || l2;

// ✅ Always attach the remaining list
tail.next = l1 || l2;

💼 Interview Perspective

  • "Merge Two Sorted Lists" is one of the most common easy interview questions.
  • "Merge K Sorted Lists" is a classic hard problem — know both heap and D&C approaches.
  • The dummy node trick is essential — use it for any list-building problem.
  • Iterative merge is preferred in interviews (no stack overflow risk).
  • This is the same merge operation used in merge sort for linked lists.

📝 Quick Revision Notes

  • Use dummy node, compare heads, pick smaller, advance that pointer.
  • Attach remaining list when one runs out: tail.next = l1 || l2.
  • Merge 2: O(n+m). Merge K: O(N log k) with divide & conquer or min-heap.
  • Iterative is O(1) space. Recursive uses O(n+m) stack space.

🃏 Flash Cards

What is the time complexity of merging K sorted lists using divide and conquer?
O(N log k) where N = total nodes and k = number of lists. We do log k rounds of merging, and each round processes all N nodes once.
Why use a dummy node when merging linked lists?
The dummy node eliminates the need for special handling of the first element. You build the list from dummy.next, and return dummy.next. No edge case for "is the result list empty?"
