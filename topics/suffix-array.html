Suffix Array

Phase 15 Hard

📖 What is a Suffix Array?

A Suffix Array is a sorted array of all suffixes of a string. More precisely, it stores the starting indices of all suffixes, sorted in lexicographic order. It's a space-efficient alternative to a Suffix Tree, used for pattern matching, finding longest repeated substrings, and many other string problems.

Building a suffix array naively takes O(n² log n), but efficient algorithms construct it in O(n log n) or even O(n) time.

📚 Real-Life Analogy: Book Index

Imagine creating an index for a book by listing every possible "reading start point" — every position from which you could begin reading until the end. Now sort all these alphabetically. That sorted list is like a suffix array. To find where a phrase appears, you binary search the index, just like flipping to the right section in a dictionary. It turns O(n) scanning into O(m log n) lookup.

🔧 Implementation

JavaScript — Suffix Array Construction (O(n log² n))
function buildSuffixArray(s) {
    const n = s.length;
    // Initial ranking based on single characters
    let sa = Array.from({ length: n }, (_, i) => i);
    let rank = Array.from(s, ch => ch.charCodeAt(0));
    let tmp = new Array(n);

    for (let gap = 1; gap < n; gap *= 2) {
        // Comparator: sort by (rank[i], rank[i + gap])
        const cmp = (a, b) => {
            if (rank[a] !== rank[b]) return rank[a] - rank[b];
            const ra = a + gap < n ? rank[a + gap] : -1;
            const rb = b + gap < n ? rank[b + gap] : -1;
            return ra - rb;
        };
        sa.sort(cmp);

        // Recompute ranks
        tmp[sa[0]] = 0;
        for (let i = 1; i < n; i++) {
            tmp[sa[i]] = tmp[sa[i - 1]] + (cmp(sa[i - 1], sa[i]) < 0 ? 1 : 0);
        }
        rank = [...tmp];

        // Early termination: all ranks are unique
        if (rank[sa[n - 1]] === n - 1) break;
    }
    return sa;
}

// Pattern search using suffix array — O(m log n)
function searchPattern(text, pattern, sa) {
    const n = text.length, m = pattern.length;
    // Binary search for first occurrence
    let lo = 0, hi = n - 1, first = -1;
    while (lo <= hi) {
        const mid = (lo + hi) >> 1;
        const suffix = text.substring(sa[mid], sa[mid] + m);
        if (suffix >= pattern) { first = mid; hi = mid - 1; }
        else lo = mid + 1;
    }
    if (first === -1 || text.substring(sa[first], sa[first] + m) !== pattern) return [];

    // Binary search for last occurrence
    lo = first; hi = n - 1; let last = first;
    while (lo <= hi) {
        const mid = (lo + hi) >> 1;
        const suffix = text.substring(sa[mid], sa[mid] + m);
        if (suffix <= pattern) { last = mid; lo = mid + 1; }
        else hi = mid - 1;
    }
    return sa.slice(first, last + 1);
}

// Usage
const text = "banana";
const sa = buildSuffixArray(text);
console.log("Suffix Array:", sa);  // [5, 3, 1, 0, 4, 2]
console.log(searchPattern(text, "ana", sa)); // [3, 1]

⚠️ Common Mistakes

❌ Off-by-one in gap comparisons

When comparing (rank[i], rank[i+gap]), handle i+gap ≥ n by returning -1 or -∞. Accessing out-of-bounds memory gives wrong results.

❌ Confusing suffix array with suffix tree

A suffix array is just a sorted array of indices. A suffix tree is a compressed trie. Suffix arrays use O(n) space; suffix trees use O(n) but with a larger constant.

❌ Not using LCP array for advanced queries

Many problems (longest repeated substring, number of distinct substrings) require the LCP array alongside the suffix array. Don't forget to build it when needed.

💼 Interview Perspective

  • Suffix arrays are advanced string structures — know the concept and when to apply it, even if you can't code the O(n) construction from scratch.
  • Common applications: pattern matching, longest common substring, number of distinct substrings.
  • Pair with LCP array (Longest Common Prefix between adjacent suffixes) for most applications.
  • In interviews, simpler approaches (KMP, rolling hash) usually suffice. Suffix arrays are more of a competitive programming tool.
  • Know trade-offs: Suffix Array (less space, harder to build) vs Suffix Tree (more space, easier queries).

📝 Quick Revision Notes

  • Suffix Array: sorted array of starting indices of all suffixes of a string.
  • Build: O(n log² n) with comparison sort, O(n log n) with radix sort, O(n) with SA-IS.
  • Pattern search: binary search on suffix array → O(m log n).
  • LCP array: stores longest common prefix between consecutive sorted suffixes. Built in O(n) using Kasai's algorithm.
  • Space: O(n) — much better than O(n) of suffix tree with large constant.

🃏 Flash Cards

What is the suffix array for "banana"?
[5, 3, 1, 0, 4, 2] — representing the sorted suffixes: "a" (5), "ana" (3), "anana" (1), "banana" (0), "na" (4), "nana" (2).
How does the O(n log² n) suffix array construction work?
Iteratively sort by longer prefixes: first by 1 character, then by 2, 4, 8... Using the previous iteration's ranks as a key, each sort compares 2^k length prefixes. After O(log n) iterations with O(n log n) sort each → O(n log² n). Use radix sort for O(n log n).
