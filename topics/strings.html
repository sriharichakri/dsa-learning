Strings

Phase 2 Beginner Friendly

📖 What Are Strings?

A string is a sequence of characters — letters, digits, symbols, or spaces. Under the hood, a string is essentially an array of characters with some special properties.

Strings are everywhere: usernames, passwords, URLs, search queries, chat messages, DNA sequences. About 20-25% of interview problems involve string manipulation.

💡 Key Insight

The most critical thing to understand about strings is immutability. In most languages (Java, Python, JavaScript), strings cannot be changed after creation. Every "modification" creates a brand new string. This has massive implications for time complexity.

🌍 Real-Life Analogy

📿 Necklace of Beads

Think of a string as a necklace of beads. Each bead is a character, and they're strung together in a specific order. You can look at any bead by position (index), but if you want to change one bead, you have to make an entirely new necklace — that's immutability!

  • Train of carriages: Each carriage (character) is connected in order. You can walk to carriage #5 directly.
  • A sentence on a whiteboard: Erasing one letter and rewriting means the whole sentence is rewritten internally.
  • DNA strand: A sequence of characters (A, T, G, C) — bioinformatics is essentially string algorithms!

⚙️ How Strings Work

String as Character Array

String "HELLO" in Memory
Index: 0 1 2 3 4 +-----+-----+-----+-----+-----+ Value: | 'H' | 'E' | 'L' | 'L' | 'O' | +-----+-----+-----+-----+-----+ str.length = 5 str[0] = 'H' str[4] = 'O'
JavaScript
let str = "HELLO";

// Accessing characters
str[0];          // 'H'
str.charAt(2);   // 'L'
str[str.length - 1]; // 'O' (last character)

// String length
str.length;      // 5

// Iterating through characters
for (let i = 0; i < str.length; i++) {
    console.log(str[i]);
}

// Or using for...of
for (let char of str) {
    console.log(char);
}

Immutability

JavaScript
let str = "Hello";
str[0] = 'J';     // ❌ Does nothing! Strings are immutable
console.log(str);  // Still "Hello"

// To "change" a string, create a new one:
str = 'J' + str.slice(1); // "Jello" — new string created
⚡ The Hidden O(n) Cost of String Concatenation
// ❌ O(n²) — each += creates a new string!
let result = "";
for (let i = 0; i < n; i++) {
    result += "a";  // Copies entire string each time!
}

// ✅ O(n) — use array and join
let parts = [];
for (let i = 0; i < n; i++) {
    parts.push("a");
}
let result = parts.join(""); // One copy at the end

🔧 Essential String Operations

JavaScript
let str = "Hello, World!";

// Searching
str.indexOf("World");     // 7 (first occurrence)
str.includes("Hello");    // true
str.startsWith("Hello");  // true
str.endsWith("!");        // true

// Extracting
str.slice(0, 5);          // "Hello"
str.substring(7, 12);     // "World"

// Transforming (returns NEW string)
str.toUpperCase();        // "HELLO, WORLD!"
str.toLowerCase();        // "hello, world!"
str.trim();               // Removes leading/trailing whitespace
str.replace("Hello", "Hi"); // "Hi, World!"

// Splitting and Joining
"a,b,c".split(",");      // ["a", "b", "c"]
["a","b","c"].join("-");  // "a-b-c"

// Converting
str.split("");            // ["H","e","l","l","o",",",...]
[..."Hello"];             // ["H","e","l","l","o"] (spread)
String(42);               // "42"
parseInt("42");           // 42

⭐ Key String Patterns for Interviews

1. Reverse a String

JavaScript
// Method 1: Built-in
function reverse(str) {
    return str.split("").reverse().join("");
}

// Method 2: Two pointers (in-place on array)
function reverseArray(chars) {
    let l = 0, r = chars.length - 1;
    while (l < r) {
        [chars[l], chars[r]] = [chars[r], chars[l]];
        l++; r--;
    }
}

2. Check Palindrome

JavaScript
function isPalindrome(str) {
    str = str.toLowerCase().replace(/[^a-z0-9]/g, "");
    let l = 0, r = str.length - 1;
    while (l < r) {
        if (str[l] !== str[r]) return false;
        l++; r--;
    }
    return true;
}
// "racecar" → true, "hello" → false

3. Check Anagram

JavaScript
function isAnagram(s, t) {
    if (s.length !== t.length) return false;
    let count = {};
    for (let c of s) count[c] = (count[c] || 0) + 1;
    for (let c of t) {
        if (!count[c]) return false;
        count[c]--;
    }
    return true;
}
// "listen", "silent" → true

⚠️ Common Mistakes

❌ String concatenation in a loop (O(n²))
// ❌ Each += copies the entire growing string
let s = "";
for (let i = 0; i < 10000; i++) s += "x"; // O(n²)!

// ✅ Use array + join
let arr = [];
for (let i = 0; i < 10000; i++) arr.push("x");
let s = arr.join(""); // O(n)
❌ Forgetting strings are immutable
let s = "hello";
s[0] = 'H'; // ❌ No effect! Strings can't be modified.

// ✅ Create a new string
s = 'H' + s.slice(1); // "Hello"
❌ Not normalizing before comparison
// ❌ Case-sensitive comparison
"Hello" === "hello"; // false

// ✅ Normalize first
s1.toLowerCase() === s2.toLowerCase(); // true

💼 Interview Perspective

  • Always ask: "Should I treat uppercase and lowercase as the same?"
  • Ask about character set: ASCII only? Unicode? This affects space complexity of frequency arrays.
  • Convert string to char array when you need to modify characters in-place.
  • Key patterns: two pointers (palindrome), hash map (anagram, frequency), sliding window (substring problems).
  • Know the difference between == and .equals() in Java for string comparison.

📝 Quick Revision Notes

  • Strings are immutable arrays of characters. Any "modification" creates a new string.
  • Access by index: O(1). Concatenation: O(n). Build strings with arrays + join().
  • Palindrome → two pointers. Anagram → frequency count. Substring → sliding window.
  • Always normalize (lowercase, trim) before comparisons.
  • Character codes: 'a'=97, 'z'=122, 'A'=65, '0'=48.

🃏 Flash Cards

Why is string concatenation in a loop O(n²)?
Strings are immutable. Each += creates a new string and copies all existing characters. After k iterations, you've copied 1+2+3+...+k = O(k²) characters total.
How do you check if two strings are anagrams?
Count character frequencies in both strings and compare. If lengths differ → not anagrams. Use a hash map or a 26-element array (for lowercase letters). O(n) time, O(1) space.
What does immutability mean for strings?
Once a string is created, its characters cannot be changed. Operations like replace, toUpperCase, or concatenation return a new string — the original is untouched.
