Bitmask DP

Phase 12 Hard

📖 What is Bitmask DP?

Bitmask DP uses a bitmask (integer) to represent a subset of elements as the DP state. Each bit position represents whether an element is included (1) or excluded (0). This allows tracking which items have been used/visited in O(1) per check.

💡 Key Insight

For n elements, there are 2ⁿ possible subsets. A bitmask integer from 0 to 2ⁿ-1 uniquely represents each subset. dp[mask] = answer when the elements corresponding to set bits in mask have been used. Works for n ≤ 20 (2²⁰ ≈ 1M states).

🌍 Real-Life Analogy

💡 Light Switches

Imagine a row of light switches. Each switch is either on (1) or off (0). The combination of all switches represents a state. With 4 switches, "1010" means switches 1 and 3 are on. Bitmask DP tries all combinations of switches and finds the optimal one.

⚙️ Bitmask DP Examples

Can I Win (LC 464)

JavaScript — Can I Win
function canIWin(maxChoosableInteger, desiredTotal) {
    if (maxChoosableInteger * (maxChoosableInteger + 1) / 2 < desiredTotal) return false;
    if (desiredTotal <= 0) return true;

    const memo = new Map();

    function dp(usedMask, remaining) {
        if (memo.has(usedMask)) return memo.get(usedMask);

        for (let i = 1; i <= maxChoosableInteger; i++) {
            const bit = 1 << i;
            if (usedMask & bit) continue; // Already used

            // If choosing i wins, or opponent loses after we choose i
            if (i >= remaining || !dp(usedMask | bit, remaining - i)) {
                memo.set(usedMask, true);
                return true;
            }
        }

        memo.set(usedMask, false);
        return false;
    }

    return dp(0, desiredTotal);
}

Partition to K Equal Sum Subsets

JavaScript — Partition to K Equal Sum Subsets (LC 698)
function canPartitionKSubsets(nums, k) {
    const total = nums.reduce((a, b) => a + b, 0);
    if (total % k !== 0) return false;
    const target = total / k;
    const n = nums.length;

    nums.sort((a, b) => b - a); // Optimization: try larger numbers first
    if (nums[0] > target) return false;

    const memo = new Map();

    function dp(mask, currentSum) {
        if (mask === (1 << n) - 1) return true; // All numbers used
        if (memo.has(mask)) return memo.get(mask);

        for (let i = 0; i < n; i++) {
            if (mask & (1 << i)) continue; // Already used
            const newSum = currentSum + nums[i];
            if (newSum > target) continue; // Exceeds target

            const newMask = mask | (1 << i);
            // If bucket full, start new bucket; else continue filling
            if (dp(newMask, newSum === target ? 0 : newSum)) {
                memo.set(mask, true);
                return true;
            }
        }

        memo.set(mask, false);
        return false;
    }

    return dp(0, 0);
}

⚠️ Common Mistakes

❌ Using bitmask DP for n > 20

2²⁰ ≈ 1 million states. 2²⁵ ≈ 33 million. Beyond n=20, bitmask DP is too slow/memory-intensive.

❌ Wrong bit operations

Check bit i: mask & (1 << i). Set bit i: mask | (1 << i). Clear bit i: mask & ~(1 << i). Using wrong operators corrupts the state.

💼 Interview Perspective

  • Bitmask DP appears when n is small (≤ 20) and you need to track which elements are used.
  • Classic problems: TSP (Traveling Salesman), partition into subsets, game theory.
  • Recognize the pattern: "partition n items into groups" or "assign n items optimally."
  • Often combined with memoization using the mask as the key.

🃏 Flash Cards

What does a bitmask represent in bitmask DP?
A bitmask is an integer where each bit represents whether an element is included (1) or excluded (0) from the current subset. For n=4, mask=1010₂=10 means elements at positions 1 and 3 are included. There are 2ⁿ possible masks.
When should you use bitmask DP?
When: (1) n is small (≤ 20), (2) you need to track which elements have been used/visited, (3) the problem involves partitioning, assignment, or subset selection. Examples: TSP, Can I Win, Partition to K Equal Sum Subsets.
