Difference Array

Phase 2 Medium

📖 What is a Difference Array?

A Difference Array is a technique for performing multiple range update operations efficiently. Instead of updating every element in a range, you mark only the start and end+1 of the range, then reconstruct the final array with a prefix sum.

It turns O(n) per range update into O(1) per update, with one O(n) pass at the end.

🌍 Real-Life Analogy

Imagine a bus route with stops 0 to n. When someone boards at stop 3 and exits at stop 7, instead of marking every stop 3-7 as "+1 passenger," you just note: "+1 at stop 3" and "-1 at stop 8." After processing all passengers, sweep through the notes to find the number at each stop. Much faster when there are thousands of passengers!

⚙️ How It Works

Difference Array Concept
Range Update: Add 5 to indices [1, 3] Brute Force: Difference Array: arr: [0, 0, 0, 0, 0] diff: [0, 0, 0, 0, 0, 0] [0, 5, 5, 5, 0] [0,+5, 0, 0,-5, 0] O(n) per update O(1) per update! Then prefix sum of diff → [0, 5, 5, 5, 0] ✅
JavaScript
// Range update: add val to arr[left..right]
function rangeUpdate(diff, left, right, val) {
    diff[left] += val;
    if (right + 1 < diff.length) {
        diff[right + 1] -= val;
    }
}

// Reconstruct final array from difference array
function reconstruct(diff) {
    const result = [diff[0]];
    for (let i = 1; i < diff.length; i++) {
        result.push(result[i - 1] + diff[i]);
    }
    return result;
}

// Example: Corporate Flight Bookings (LC 1109)
function corpFlightBookings(bookings, n) {
    const diff = new Array(n + 1).fill(0);
    
    for (let [first, last, seats] of bookings) {
        diff[first - 1] += seats;       // 1-indexed to 0-indexed
        if (last < n) diff[last] -= seats;
    }
    
    // Prefix sum to get final counts
    const result = [diff[0]];
    for (let i = 1; i < n; i++) {
        result.push(result[i - 1] + diff[i]);
    }
    return result;
}

// bookings = [[1,2,10],[2,3,20],[2,5,25]], n = 5
// → [10, 55, 45, 25, 25]

2D Difference Array

JavaScript
// Add val to submatrix [r1,c1] to [r2,c2]
function rangeUpdate2D(diff, r1, c1, r2, c2, val) {
    diff[r1][c1] += val;
    if (r2 + 1 < diff.length) diff[r2 + 1][c1] -= val;
    if (c2 + 1 < diff[0].length) diff[r1][c2 + 1] -= val;
    if (r2 + 1 < diff.length && c2 + 1 < diff[0].length) {
        diff[r2 + 1][c2 + 1] += val;
    }
}

Apply: add 3 to [1,3], add 2 to [2,4] on array of size 5

diff = [0, 0, 0, 0, 0, 0]
2
Add 3 to [1,3]: diff[1]+=3, diff[4]-=3
diff = [0, 3, 0, 0, -3, 0]
3
Add 2 to [2,4]: diff[2]+=2, diff[5]-=2
diff = [0, 3, 2, 0, -3, -2]
4
Prefix sum: 0, 0+3=3, 3+2=5, 5+0=5, 5-3=2
result = [0, 3, 5, 5, 2] ✅

⚠️ Common Mistakes

❌ Off-by-one: subtracting at right instead of right+1
// ❌ Subtracts one position too early
diff[right] -= val;  // This excludes the last index!

// ✅ Subtract at right + 1
diff[right + 1] -= val;  // Range is inclusive [left, right]
❌ Not handling boundary (right+1 out of bounds)

If right is the last index, right+1 is out of bounds. Either make the diff array one element larger (size n+1) or check bounds before subtracting.

💼 Interview Perspective

  • Difference array is the inverse of prefix sum. Prefix sum: range query. Difference array: range update.
  • Key signal: "multiple range updates, then query final state" → difference array.
  • Corporate Flight Bookings (LC 1109) is the classic problem.
  • Can be extended to 2D for matrix range updates.
  • For both range queries AND range updates, you need a Segment Tree or BIT.

📝 Quick Revision Notes

  • Range update [l, r] by val: diff[l] += val, diff[r+1] -= val. O(1) per update.
  • Reconstruct with prefix sum: O(n).
  • Total for Q updates: O(Q + n) vs brute force O(Q × n).
  • Inverse of prefix sum. Prefix sum = range query. Difference array = range update.

🃏 Flash Cards

How does a difference array turn O(n) range updates into O(1)?
Instead of updating every element in [l, r], mark only two points: diff[l] += val (start the effect) and diff[r+1] -= val (end the effect). When you take the prefix sum, the effect propagates correctly across the range.
What is the relationship between difference array and prefix sum?
They are inverses. Prefix sum of a difference array gives back the original array. Difference of a prefix sum array gives back the original. Prefix sum is for range QUERIES; difference array is for range UPDATES.
