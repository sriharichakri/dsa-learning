Queue Fundamentals

Phase 8 Beginner Friendly

рҹ“– What is a Queue?

A Queue is a linear data structure that follows the FIFO principle вҖ” First In, First Out. The first element added is the first one removed. You add at the rear (back) and remove from the front.

рҹ’Ў Key Insight

Queues maintain order of arrival. Wherever you need to process things in the order they came вҖ” task scheduling, BFS traversal, message processing вҖ” a queue is the right choice. Stack = LIFO (most recent first). Queue = FIFO (oldest first).

рҹҢҚ Real-Life Analogy

рҹҺ« Line at a Ticket Counter

People join at the back of the line and are served at the front. The first person who arrived gets served first вҖ” that's FIFO! You can't cut in line (no random access), and new people always join at the end (enqueue at rear).

  • Printer queue: Documents are printed in the order they were sent. First document sent = first printed.
  • Fast food drive-through: Cars enter at the back, get served at the front window.
  • Customer support call: "Your call will be answered in the order it was received."
  • Airport runway: Planes take off in the order they join the runway queue.

вҡҷпёҸ Implementation

JavaScript вҖ” Using Array
class Queue {
    constructor() {
        this.items = [];
    }
    
    enqueue(val) {
        this.items.push(val);        // Add to rear
    }
    
    dequeue() {
        if (this.isEmpty()) return undefined;
        return this.items.shift();    // Remove from front
    }
    
    front() {
        if (this.isEmpty()) return undefined;
        return this.items[0];         // Peek at front
    }
    
    isEmpty() {
        return this.items.length === 0;
    }
    
    size() {
        return this.items.length;
    }
}

const q = new Queue();
q.enqueue(10);  // [10]
q.enqueue(20);  // [10, 20]
q.enqueue(30);  // [10, 20, 30]
q.front();      // 10
q.dequeue();    // 10 (removed)
q.front();      // 20
вҡЎ Array.shift() is O(n)!

shift() removes the first element and shifts all others left вҖ” that's O(n). For performance-critical code, use a linked list or an object with head/tail indices for O(1) dequeue.

Efficient Queue (O(1) all operations)

JavaScript
class EfficientQueue {
    constructor() {
        this.items = {};
        this.head = 0;
        this.tail = 0;
    }
    
    enqueue(val) {
        this.items[this.tail] = val;
        this.tail++;
    }
    
    dequeue() {
        if (this.isEmpty()) return undefined;
        const val = this.items[this.head];
        delete this.items[this.head];
        this.head++;
        return val;
    }
    
    front() { return this.items[this.head]; }
    isEmpty() { return this.tail === this.head; }
    size() { return this.tail - this.head; }
}

вӯҗ Classic: Implement Queue using Two Stacks

A famous interview problem вҖ” simulate FIFO using two LIFO stacks.

JavaScript
class QueueUsingStacks {
    constructor() {
        this.inStack = [];   // For enqueue
        this.outStack = [];  // For dequeue
    }
    
    enqueue(val) {
        this.inStack.push(val);
    }
    
    dequeue() {
        if (this.outStack.length === 0) {
            // Move all from inStack to outStack (reverses order!)
            while (this.inStack.length > 0) {
                this.outStack.push(this.inStack.pop());
            }
        }
        return this.outStack.pop();
    }
}

// enqueue 1, 2, 3:  inStack = [1, 2, 3]
// dequeue:          outStack = [3, 2, 1] вҶ’ pop вҶ’ returns 1 вң… (FIFO!)

рҹҢҠ Where Queues Shine: BFS

Breadth-First Search (BFS) uses a queue to explore nodes level by level вҖ” close neighbors first, then their neighbors, and so on.

JavaScript
// BFS on a tree (level-order traversal)
function bfs(root) {
    if (!root) return [];
    const queue = [root];
    const result = [];
    
    while (queue.length > 0) {
        const node = queue.shift();  // Process front
        result.push(node.val);
        
        if (node.left) queue.push(node.left);   // Add children
        if (node.right) queue.push(node.right);
    }
    return result;
}

вҡ пёҸ Common Mistakes

вқҢ Using Array.shift() for high-performance queues

shift() is O(n) because it shifts all elements. For large queues, use a linked list or indexed object implementation.

вқҢ Confusing Stack and Queue

Stack: use when you need the most recent item (brackets, undo). Queue: use when you need the oldest item (scheduling, BFS).

вқҢ Forgetting to check empty before dequeue
// вқҢ Returns undefined or crashes
q.dequeue(); // Queue might be empty!

// вң… Check first
if (!q.isEmpty()) q.dequeue();

рҹ’ј Interview Perspective

  • BFS traversal always uses a queue. If they ask BFS, you need a queue.
  • Know how to implement a queue using two stacks (amortized O(1) dequeue).
  • Know how to implement a stack using two queues (less common but sometimes asked).
  • Circular queue (ring buffer) is used in system design for bounded buffers.
  • Queue + HashMap = LRU Cache (very popular design question).

рҹ“қ Quick Revision Notes

  • Queue = FIFO. Enqueue at rear, dequeue from front.
  • Array.shift() is O(n). Use linked list or object for O(1) dequeue.
  • Queue using two stacks: push to inStack, transfer to outStack for dequeue.
  • Main use cases: BFS, task scheduling, buffering, order processing.
  • Stack = DFS, Queue = BFS. Know which to use when.

рҹғҸ Flash Cards

What does FIFO stand for?
First In, First Out. The first element added is the first one removed. Like a line at a ticket counter вҖ” first person in line gets served first.
How do you implement a queue using two stacks?
Use inStack for enqueue (push). For dequeue: if outStack is empty, move all elements from inStack to outStack (reversing order), then pop from outStack. Amortized O(1) per operation.
Why is Array.shift() O(n) and why does it matter?
shift() removes the first element and shifts all remaining elements left by one position. For a queue with n elements, this requires n-1 shifts. Use an object-based queue or linked list for O(1) dequeue.
