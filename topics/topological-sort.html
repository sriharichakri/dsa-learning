Topological Sort

Phase 10 Medium

📖 What is Topological Sort?

Topological Sort is a linear ordering of vertices in a Directed Acyclic Graph (DAG) such that for every directed edge u → v, vertex u comes before v in the ordering.

It only works on DAGs — if the graph has a cycle, no topological ordering exists. Two main approaches: Kahn's algorithm (BFS) and DFS-based.

💡 Key Insight

Topological sort answers: "In what order should I do things if some things depend on others?" If the graph has a cycle (A depends on B, B depends on A), there's no valid order — Kahn's algorithm detects this by checking if all nodes were processed.

🌍 Real-Life Analogy

📚 Course Prerequisites

In college, some courses have prerequisites: you must take "Calculus I" before "Calculus II", and "Linear Algebra" before "Machine Learning". Topological sort gives you a valid order to take all courses such that every prerequisite is satisfied before you enroll in the dependent course.

  • Getting dressed: Underwear before pants, socks before shoes, shirt before jacket.
  • Build systems: Compile dependencies before the main project (like npm/Maven builds).
  • Task scheduling: Some tasks must finish before others can start.
  • Spreadsheet formulas: Cells that reference other cells must be computed in dependency order.

⚙️ How Topological Sort Works

Kahn's Algorithm (BFS) — Process nodes with in-degree 0
Graph (course prerequisites): A → C A: no prereqs A → D B: no prereqs B → D C: needs A C → E D: needs A, B D → E E: needs C, D In-degree: A:0 B:0 C:1 D:2 E:2 Step 1: Queue = [A, B] (in-degree 0) Step 2: Process A → reduce C(1→0), D(2→1). Queue = [B, C] Step 3: Process B → reduce D(1→0). Queue = [C, D] Step 4: Process C → reduce E(2→1). Queue = [D] Step 5: Process D → reduce E(1→0). Queue = [E] Step 6: Process E → done. Result: [A, B, C, D, E] ✅ (one valid topological order)

Kahn's Algorithm (BFS-based)

JavaScript — Kahn's Algorithm
function topologicalSortKahn(numNodes, edges) {
    const adj = Array.from({length: numNodes}, () => []);
    const inDegree = new Array(numNodes).fill(0);

    for (const [u, v] of edges) {
        adj[u].push(v);
        inDegree[v]++;
    }

    // Start with all nodes that have in-degree 0
    const queue = [];
    for (let i = 0; i < numNodes; i++) {
        if (inDegree[i] === 0) queue.push(i);
    }

    const result = [];
    while (queue.length > 0) {
        const node = queue.shift();
        result.push(node);

        for (const neighbor of adj[node]) {
            inDegree[neighbor]--;
            if (inDegree[neighbor] === 0) {
                queue.push(neighbor);
            }
        }
    }

    // If not all nodes processed → cycle exists!
    return result.length === numNodes ? result : []; // empty = has cycle
}

// Course Schedule (can finish all courses?)
function canFinish(numCourses, prerequisites) {
    const order = topologicalSortKahn(numCourses, prerequisites);
    return order.length === numCourses;
}

DFS-based Topological Sort

JavaScript — DFS Topological Sort
function topologicalSortDFS(numNodes, edges) {
    const adj = Array.from({length: numNodes}, () => []);
    for (const [u, v] of edges) adj[u].push(v);

    const visited = new Array(numNodes).fill(0); // 0:unvisited, 1:in-progress, 2:done
    const stack = [];
    let hasCycle = false;

    function dfs(node) {
        if (hasCycle) return;
        visited[node] = 1; // Mark in-progress

        for (const neighbor of adj[node]) {
            if (visited[neighbor] === 1) { hasCycle = true; return; } // Cycle!
            if (visited[neighbor] === 0) dfs(neighbor);
        }

        visited[node] = 2; // Mark done
        stack.push(node);  // Add to stack AFTER processing all neighbors
    }

    for (let i = 0; i < numNodes; i++) {
        if (visited[i] === 0) dfs(i);
    }

    return hasCycle ? [] : stack.reverse(); // Reverse gives topological order
}

⚠️ Common Mistakes

❌ Applying topological sort to cyclic graphs

Topological sort only works on DAGs. Always check: if result length < number of nodes, there's a cycle.

❌ Confusing edge direction in course schedule

In LC 207, prerequisite [a,b] means b→a (must take b before a). Getting the direction wrong reverses the dependency.

❌ DFS: not using 3-state visited array

Using a simple boolean visited misses cycle detection. You need 3 states: unvisited, in-progress (on current DFS path), and done.

💼 Interview Perspective

  • Course Schedule is one of the most frequently asked graph problems. Know both Kahn's and DFS approaches.
  • Kahn's is more intuitive and naturally gives the order. DFS requires reversing the result.
  • "Alien Dictionary" is a harder variation — build a graph from character ordering, then topological sort.
  • If the problem asks "is it possible?" → check for cycles. If it asks for "the order" → return topological order.
  • Multiple valid orderings exist. Unless the problem asks for lexicographic order (use min-heap instead of queue).

📝 Quick Revision Notes

  • Topological sort: linear ordering of DAG vertices where u comes before v for every edge u→v.
  • Kahn's (BFS): process in-degree 0 nodes first, reduce neighbors' in-degree.
  • DFS: post-order + reverse. Use 3-state visited for cycle detection.
  • Cycle check: result.length < numNodes means cycle exists.
  • Time: O(V+E) for both approaches.

🃏 Flash Cards

What is the key idea behind Kahn's Algorithm?
Repeatedly remove nodes with in-degree 0 (no dependencies). When you remove a node, decrement the in-degree of its neighbors. If a neighbor reaches 0, add it to the queue. If not all nodes are processed, a cycle exists.
How does DFS-based topological sort detect cycles?
Use a 3-state visited array: 0=unvisited, 1=in-progress, 2=done. If during DFS you reach a node marked "in-progress", you've found a back edge → cycle exists. Nodes are added to the result stack only after all descendants are processed (post-order).
Why must topological sort only work on DAGs?
If there's a cycle A→B→C→A, then A must come before B, B before C, and C before A — which is impossible. No linear ordering can satisfy circular dependencies. Topological sort requires a Directed Acyclic Graph.
