Expression Conversion (Infix/Postfix/Prefix)

Phase 7 Medium

📖 What are Expression Notations?

Mathematical expressions can be written in three forms:

  • Infix: Operator between operands → A + B (what humans use)
  • Postfix (RPN): Operator after operands → A B + (what computers prefer)
  • Prefix (Polish): Operator before operands → + A B
🌍 Real-Life Analogy

Think of giving directions: Infix is "take the road between the gas station and the mall." Postfix is "gas station, mall — take the road." Prefix is "take the road — gas station, mall." All say the same thing, just in different orders. Computers prefer postfix because it needs no parentheses and processes naturally with a stack!

⚙️ How It Works

Infix to Postfix (Shunting Yard Algorithm)

JavaScript
function infixToPostfix(expr) {
    const precedence = { '+': 1, '-': 1, '*': 2, '/': 2, '^': 3 };
    const rightAssoc = new Set(['^']);
    const stack = [];
    let output = '';
    
    for (let ch of expr) {
        if (/[A-Za-z0-9]/.test(ch)) {
            output += ch;  // Operand → output
        } else if (ch === '(') {
            stack.push(ch);
        } else if (ch === ')') {
            while (stack.length && stack[stack.length - 1] !== '(') {
                output += stack.pop();
            }
            stack.pop(); // Remove '('
        } else {
            // Operator
            while (
                stack.length &&
                stack[stack.length - 1] !== '(' &&
                (precedence[stack[stack.length - 1]] > precedence[ch] ||
                 (precedence[stack[stack.length - 1]] === precedence[ch] && !rightAssoc.has(ch)))
            ) {
                output += stack.pop();
            }
            stack.push(ch);
        }
    }
    while (stack.length) output += stack.pop();
    return output;
}

// infixToPostfix("A+B*C")     → "ABC*+"
// infixToPostfix("(A+B)*C")   → "AB+C*"
// infixToPostfix("A+B*C-D/E") → "ABC*+DE/-"

Evaluate Postfix Expression

JavaScript
function evalPostfix(expr) {
    const stack = [];
    
    for (let ch of expr.split(' ')) {
        if (!isNaN(ch)) {
            stack.push(Number(ch));
        } else {
            const b = stack.pop();
            const a = stack.pop();
            switch (ch) {
                case '+': stack.push(a + b); break;
                case '-': stack.push(a - b); break;
                case '*': stack.push(a * b); break;
                case '/': stack.push(Math.trunc(a / b)); break;
            }
        }
    }
    return stack[0];
}

// evalPostfix("2 1 + 3 *") → 9  // (2+1)*3

Infix to Prefix

JavaScript
function infixToPrefix(expr) {
    // 1. Reverse the expression (swap parentheses)
    let reversed = expr.split('').reverse().map(ch => {
        if (ch === '(') return ')';
        if (ch === ')') return '(';
        return ch;
    }).join('');
    
    // 2. Get postfix of reversed
    let postfix = infixToPostfix(reversed);
    
    // 3. Reverse the postfix
    return postfix.split('').reverse().join('');
}

// infixToPrefix("A+B*C") → "+A*BC"

⚠️ Common Mistakes

❌ Wrong operator precedence

Remember: ^ (3) > * / (2) > + - (1). Also, ^ is right-associative (2^3^4 = 2^(3^4)), while +, -, *, / are left-associative.

❌ Not handling parentheses correctly

When you see '(', push it. When you see ')', pop until '(' — but DON'T include '(' in the output. Forgetting to remove '(' from the stack is a common bug.

💼 Interview Perspective

  • The Shunting Yard algorithm (Dijkstra) is the standard for infix → postfix.
  • Postfix evaluation is straightforward: operand → push, operator → pop two, compute, push result.
  • "Basic Calculator" problems on LeetCode are variations of this concept.
  • Know the precedence table and associativity rules by heart.
  • Prefix → Postfix: evaluate right to left with a stack.

📝 Quick Revision Notes

  • Infix: A+B. Postfix: AB+. Prefix: +AB.
  • Infix → Postfix: operands to output, operators to stack (respect precedence).
  • Evaluate Postfix: operand → push, operator → pop 2, compute, push result.
  • Infix → Prefix: reverse, convert to postfix, reverse again.

🃏 Flash Cards

Why do computers prefer postfix notation over infix?
Postfix needs no parentheses and no precedence rules during evaluation. Just scan left to right: push operands, and when you see an operator, pop two and compute. Simple stack-based evaluation!
