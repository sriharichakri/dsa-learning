Dynamic Programming Introduction

Phase 12 Medium

📖 What is Dynamic Programming?

Dynamic Programming (DP) is an optimization technique that solves complex problems by breaking them into overlapping subproblems, solving each subproblem only once, and storing the results to avoid redundant computation.

It's essentially: recursion + memoization = DP. Or alternatively: smart brute force that remembers what it already computed.

💡 Key Insight

DP works when a problem has two properties: (1) Overlapping Subproblems — the same subproblems are solved multiple times, and (2) Optimal Substructure — the optimal solution can be built from optimal solutions of subproblems. If both are present, DP can transform exponential time into polynomial time.

🌍 Real-Life Analogy

📱 Remembering Phone Numbers

Imagine looking up a friend's phone number in a phone book. The first time, you flip through pages to find it. But instead of doing that every time, you memorize it (or save it in contacts). Next time you need it, you recall instantly. That's memoization! You solve the "find number" problem once and cache the result.

  • GPS navigation: If you've already computed the shortest route from A to B, and now need the route from A to C that passes through B, reuse the A→B result.
  • Studying for exams: You don't re-derive every formula from scratch. You learn it once and recall it — that's building on previously solved subproblems.
  • Fibonacci in nature: A sunflower's spiral follows Fibonacci numbers. Each number depends on the previous two — classic DP.

⚙️ The Fibonacci Example: Why DP Matters

Naive Recursion — O(2ⁿ) 💀

JavaScript
function fib(n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}
Call tree for fib(5) — notice the REPEATED work!
fib(5) / \ fib(4) fib(3) ← fib(3) computed TWICE / \ / \ fib(3) fib(2) fib(2) fib(1) ← fib(2) computed THREE times! / \ / \ / \ fib(2) fib(1) ... ... Total calls for fib(5): 15 Total calls for fib(30): 2,692,537 Total calls for fib(50): 40,730,022,147 💀

Top-Down DP (Memoization) — O(n)

JavaScript
function fibMemo(n, memo = {}) {
    if (n <= 1) return n;
    if (memo[n] !== undefined) return memo[n]; // Already computed!
    
    memo[n] = fibMemo(n - 1, memo) + fibMemo(n - 2, memo);
    return memo[n];
}
// fibMemo(50) → instant! Only 50 unique subproblems.

Bottom-Up DP (Tabulation) — O(n)

JavaScript
function fibTab(n) {
    if (n <= 1) return n;
    const dp = [0, 1];
    
    for (let i = 2; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    return dp[n];
}

// Space-optimized — O(1) space!
function fibOptimized(n) {
    if (n <= 1) return n;
    let prev2 = 0, prev1 = 1;
    
    for (let i = 2; i <= n; i++) {
        let curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

🎯 Interview Strategy

Start with top-down (memoization) — it's easier to write. If the interviewer asks for optimization, convert to bottom-up (tabulation). If they want even less space, optimize the DP array to only keep the values you need.

⭐ Classic: Climbing Stairs

You can climb 1 or 2 steps at a time. How many distinct ways can you climb n stairs?

JavaScript
// This IS Fibonacci! dp[i] = dp[i-1] + dp[i-2]
function climbStairs(n) {
    if (n <= 2) return n;
    let prev2 = 1, prev1 = 2;
    
    for (let i = 3; i <= n; i++) {
        let curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
// climbStairs(5) → 8 ways
1
n=5. Base cases: dp[1]=1 way, dp[2]=2 ways
prev2=1, prev1=2
2
i=3: dp[3] = dp[2]+dp[1] = 2+1 = 3
prev2=2, prev1=3
3
i=4: dp[4] = dp[3]+dp[2] = 3+2 = 5
prev2=3, prev1=5
4
i=5: dp[5] = dp[4]+dp[3] = 5+3 = 8
Answer: 8 ways ✅

🏠 Classic: House Robber

Rob houses along a street. Can't rob two adjacent houses. Maximize total loot.

JavaScript
function rob(nums) {
    if (nums.length === 0) return 0;
    if (nums.length === 1) return nums[0];
    
    let prev2 = 0;          // dp[i-2]
    let prev1 = 0;          // dp[i-1]
    
    for (let num of nums) {
        let curr = Math.max(
            prev1,           // Skip this house
            prev2 + num      // Rob this house + best from 2 ago
        );
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

// [2, 7, 9, 3, 1] → Rob houses with 2, 9, 1 = 12? 
// Or 7, 3 = 10? Answer: 12 (rob index 0, 2, 4)

📋 DP Problem-Solving Framework

  1. Define the state: What does dp[i] represent?
  2. Find the recurrence: How does dp[i] relate to smaller subproblems?
  3. Identify base cases: What are dp[0], dp[1]?
  4. Determine the order: Fill the table in the right order (usually left to right).
  5. Optimize space: Do you really need the entire DP array, or just the last few values?

⚠️ Common Mistakes

❌ Not identifying overlapping subproblems

DP only helps when the same subproblem is solved multiple times. If each subproblem is unique (like merge sort), DP won't help — that's just divide and conquer.

❌ Wrong state definition

If dp[i] doesn't clearly represent a complete subproblem, your recurrence will be wrong. Take time to define what dp[i] means precisely.

❌ Forgetting base cases
// ❌ dp[0] and dp[1] undefined → NaN propagates!
for (let i = 2; i <= n; i++) {
    dp[i] = dp[i-1] + dp[i-2]; // dp[0] = undefined!
}

// ✅ Always set base cases first
dp[0] = 0;
dp[1] = 1;

💼 Interview Perspective

  • Start with brute force recursion. Draw the recursion tree. Identify overlapping subproblems. Add memoization.
  • State the recurrence relation clearly before coding.
  • DP problems follow patterns: 1D DP (Fibonacci, stairs), 2D DP (grid paths, LCS), knapsack (subset sum).
  • If they ask to optimize space, replace the array with a few variables (prev1, prev2).
  • DP is the hardest interview topic. Practice recognizing the pattern rather than memorizing solutions.

📝 Quick Revision Notes

  • DP = recursion + memoization. Needs overlapping subproblems + optimal substructure.
  • Top-down (memoization): recursive + cache. Bottom-up (tabulation): iterative + table.
  • Framework: define state → find recurrence → base cases → fill order → optimize space.
  • Climbing Stairs = Fibonacci. House Robber = max(skip, take + prev2).
  • DP transforms exponential time (2ⁿ) into polynomial time (n, n²).

🃏 Flash Cards

What two properties must a problem have for DP to work?
1. Overlapping Subproblems: Same subproblems are solved multiple times. 2. Optimal Substructure: Optimal solution can be built from optimal solutions of subproblems.
What is the difference between memoization and tabulation?
Memoization (top-down): recursive, caches results as they're computed, solves only needed subproblems. Tabulation (bottom-up): iterative, fills a table from smallest subproblems up, solves all subproblems.
How does memoization turn O(2ⁿ) Fibonacci into O(n)?
Without memo, fib(n) recomputes the same values exponentially. With memo, each fib(k) is computed only ONCE and cached. There are n unique subproblems, each taking O(1) work → O(n) total.
What is the recurrence for the House Robber problem?
dp[i] = max(dp[i-1], dp[i-2] + nums[i]). At each house, choose the better option: skip it (dp[i-1]) or rob it and add the best from 2 houses back (dp[i-2] + nums[i]).
