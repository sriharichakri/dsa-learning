Stack Fundamentals

Phase 7 Beginner Friendly

📖 What is a Stack?

A Stack is a linear data structure that follows the LIFO principle — Last In, First Out. The last element added is the first one removed. You can only add or remove elements from the top.

💡 Key Insight

Stacks are deceptively simple but incredibly powerful. They're the secret behind function calls (call stack), undo/redo, browser history, expression evaluation, and DFS traversal. If you see "matching" or "nesting" in a problem, think stack!

🌍 Real-Life Analogy

🍽️ Stack of Plates

Imagine a stack of plates in a cafeteria. You can only take the top plate (pop) and can only add a new plate on top (push). You can't pull a plate from the middle without removing everything above it first. That's LIFO!

  • Undo button (Ctrl+Z): Each action is pushed onto a stack. Undo pops the last action.
  • Browser back button: Each visited page is pushed. Clicking back pops the current page.
  • Pringles can: First chip in, last chip out!
  • Stacking books: You always take the top book first.

⚙️ Implementation

JavaScript — Using Array
class Stack {
    constructor() {
        this.items = [];
    }
    
    push(val) {
        this.items.push(val);      // Add to top
    }
    
    pop() {
        if (this.isEmpty()) return undefined;
        return this.items.pop();    // Remove from top
    }
    
    peek() {
        if (this.isEmpty()) return undefined;
        return this.items[this.items.length - 1]; // Look at top
    }
    
    isEmpty() {
        return this.items.length === 0;
    }
    
    size() {
        return this.items.length;
    }
}

// Usage
const stack = new Stack();
stack.push(10);  // [10]
stack.push(20);  // [10, 20]
stack.push(30);  // [10, 20, 30]
stack.peek();    // 30 (doesn't remove)
stack.pop();     // 30 (removes and returns)
stack.size();    // 2
💡 In interviews, just use an array as a stack

In JavaScript/Python, arrays already have push() and pop(). In Java, use Deque<Integer> stack = new ArrayDeque<>(); (preferred over the legacy Stack class).

⭐ Classic: Valid Parentheses

The most famous stack problem — check if brackets are properly matched and nested.

JavaScript
function isValid(s) {
    const stack = [];
    const map = { ')': '(', '}': '{', ']': '[' };
    
    for (let char of s) {
        if (char === '(' || char === '{' || char === '[') {
            stack.push(char);  // Opening bracket → push
        } else {
            if (stack.pop() !== map[char]) return false;
            // Closing bracket → pop and check match
        }
    }
    return stack.length === 0; // Stack should be empty
}

isValid("({[]})"); // true
isValid("([)]");   // false

📊 Pattern: Next Greater Element (Monotonic Stack)

Find the next greater element for each element in the array.

JavaScript
function nextGreaterElement(arr) {
    const result = new Array(arr.length).fill(-1);
    const stack = []; // Stores indices
    
    for (let i = 0; i < arr.length; i++) {
        // Pop all elements smaller than current
        while (stack.length && arr[stack[stack.length - 1]] < arr[i]) {
            result[stack.pop()] = arr[i];
        }
        stack.push(i);
    }
    return result;
}

nextGreaterElement([4, 5, 2, 10, 8]);
// [5, 10, 10, -1, -1]

⚠️ Common Mistakes

❌ Popping from an empty stack
// ❌ Stack underflow!
stack.pop(); // What if stack is empty?

// ✅ Always check before popping
if (!stack.isEmpty()) stack.pop();
❌ Forgetting to check stack is empty after processing

In Valid Parentheses, an input like "(((" has unmatched opening brackets. You must check stack.length === 0 at the end.

❌ Using a Queue when you need a Stack (or vice versa)

Stack = LIFO (most recent first). Queue = FIFO (oldest first). Matching brackets needs LIFO → Stack. Processing in arrival order needs FIFO → Queue.

💼 Interview Perspective

  • Matching/nesting problems (brackets, HTML tags) → Stack.
  • Next greater/smaller element → Monotonic Stack.
  • Expression evaluation (infix, postfix, prefix) → Stack.
  • Undo/redo functionality → Two stacks.
  • The call stack is literally a stack — understand how recursion uses it.

📝 Quick Revision Notes

  • Stack = LIFO. Operations: push (add to top), pop (remove from top), peek (look at top).
  • All operations are O(1). Space is O(n).
  • Use cases: parenthesis matching, undo/redo, DFS, function call tracking.
  • Monotonic stack: maintain stack in sorted order to find next greater/smaller element in O(n).
  • In JS: use array with push() and pop(). In Java: use ArrayDeque, not Stack class.

🃏 Flash Cards

What does LIFO stand for?
Last In, First Out. The most recently added element is removed first. Think of a stack of plates — you take the top plate (last placed) first.
How do you solve the Valid Parentheses problem?
Push opening brackets onto a stack. When you encounter a closing bracket, pop from the stack and check if it matches. At the end, the stack must be empty. O(n) time, O(n) space.
What is a monotonic stack?
A stack that maintains elements in sorted (increasing or decreasing) order. When adding an element, pop all elements that violate the order. Used for "next greater/smaller element" problems. O(n) amortized.
