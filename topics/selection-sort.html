Selection Sort

Phase 4 Easy

📖 What is Selection Sort?

Selection Sort works by repeatedly finding the minimum element from the unsorted portion and placing it at the beginning. It "selects" the smallest element in each pass.

Think of it as: divide the array into sorted (left) and unsorted (right) portions. Each iteration, pick the smallest from the right and add it to the end of the left.

🌍 Real-Life Analogy

Imagine sorting a hand of playing cards. You scan all cards, find the smallest, pull it out and place it first. Then scan the remaining cards, find the next smallest, place it second. You keep "selecting" the minimum from what's left. That's selection sort!

⚙️ How It Works

JavaScript
function selectionSort(arr) {
    const n = arr.length;
    
    for (let i = 0; i < n - 1; i++) {
        // Find the minimum element in unsorted portion
        let minIdx = i;
        for (let j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx]) {
                minIdx = j;
            }
        }
        // Swap minimum with current position
        if (minIdx !== i) {
            [arr[i], arr[minIdx]] = [arr[minIdx], arr[i]];
        }
    }
    return arr;
}

// selectionSort([64, 25, 12, 22, 11]) → [11, 12, 22, 25, 64]

⏱️ Complexity Analysis

💡 Key Property

Selection sort always does O(n²) comparisons regardless of input. But it does at most O(n) swaps — one per pass. This makes it good when swaps are expensive (e.g., writing to flash memory).

⚠️ Common Mistakes

❌ Swapping inside the inner loop
// ❌ Wrong — swaps every time we find a smaller element
for (let j = i + 1; j < n; j++) {
    if (arr[j] < arr[i]) {
        [arr[i], arr[j]] = [arr[j], arr[i]]; // Too many swaps!
    }
}

// ✅ Correct — track minIdx, swap only once per pass
let minIdx = i;
for (let j = i + 1; j < n; j++) {
    if (arr[j] < arr[minIdx]) minIdx = j;
}
[arr[i], arr[minIdx]] = [arr[minIdx], arr[i]];
❌ Confusing with Bubble Sort

Selection sort finds the min and places it. Bubble sort compares adjacent elements and swaps. Selection sort does fewer swaps but always O(n²) comparisons.

💼 Interview Perspective

  • Selection sort is NOT stable — equal elements may change relative order.
  • It's useful when memory writes are expensive (minimum number of swaps).
  • Always O(n²) — doesn't benefit from partially sorted input (unlike insertion sort).
  • Know how it differs from insertion sort and bubble sort — common interview comparison.
  • In-place, but not adaptive and not stable.

📝 Quick Revision Notes

  • Repeatedly select the minimum from unsorted portion, swap to front.
  • Time: always O(n²). Space: O(1). Swaps: O(n) — at most one per pass.
  • NOT stable. In-place. NOT adaptive.
  • Good when swaps are expensive. Otherwise prefer insertion sort.

🃏 Flash Cards

What is the key advantage of selection sort over bubble sort?
Selection sort makes at most O(n) swaps (one per pass), while bubble sort can make O(n²) swaps. If writes/swaps are expensive, selection sort wins.
Is selection sort stable? Why or why not?
No, it's unstable. When swapping the minimum to the front, it can jump over equal elements. Example: [5a, 3, 5b] → after first pass [3, 5b, 5a] — the two 5s switched order.
