Longest Increasing Subsequence (LIS)

Phase 12 Medium

📖 What is LIS?

The Longest Increasing Subsequence finds the longest subsequence of an array where elements are in strictly increasing order. Elements don't need to be contiguous.

For [10, 9, 2, 5, 3, 7, 101, 18], the LIS is [2, 3, 7, 101] with length 4.

💡 Key Insight

Two approaches: O(n²) DP where dp[i] = LIS ending at index i, and O(n log n) Patience Sort using binary search on a "tails" array. The O(n log n) approach maintains the smallest possible tail element for each LIS length.

🌍 Real-Life Analogy

🃏 Patience Sorting (Card Game)

Deal cards left to right. Place each card on the leftmost pile whose top card is ≥ current card. If no pile works, start a new pile. The number of piles at the end equals the LIS length! This is actually how the O(n log n) algorithm works — binary search finds the right pile.

⚙️ LIS Implementations

O(n²) DP Approach

JavaScript — LIS O(n²)
function lengthOfLIS_DP(nums) {
    const n = nums.length;
    const dp = new Array(n).fill(1); // Each element is an LIS of length 1

    for (let i = 1; i < n; i++) {
        for (let j = 0; j < i; j++) {
            if (nums[j] < nums[i]) {
                dp[i] = Math.max(dp[i], dp[j] + 1);
            }
        }
    }
    return Math.max(...dp);
}
// dp[i] = length of LIS ending at index i

O(n log n) Binary Search Approach

JavaScript — LIS O(n log n)
function lengthOfLIS(nums) {
    const tails = []; // tails[i] = smallest tail element for LIS of length i+1

    for (const num of nums) {
        let lo = 0, hi = tails.length;

        // Binary search for the position to replace
        while (lo < hi) {
            const mid = Math.floor((lo + hi) / 2);
            if (tails[mid] < num) lo = mid + 1;
            else hi = mid;
        }

        tails[lo] = num; // Replace or extend
        // If lo === tails.length, we extended the LIS
    }

    return tails.length;
}
// [10,9,2,5,3,7,101,18] → tails grows: [10],[9],[2],[2,5],[2,3],[2,3,7],[2,3,7,101],[2,3,7,18]
// Length = 4

🔍 Dry Run: LIS on [10, 9, 2, 5, 3, 7, 101, 18]

1
num=10: tails empty → append. tails=[10].
tails = [10], LIS length = 1
2
num=9: 9 < 10 → replace tails[0]. tails=[9].
tails = [9], LIS length = 1
3
num=2: 2 < 9 → replace tails[0]. num=5: 5 > 2 → append. num=3: replace tails[1].
tails = [2, 3], LIS length = 2
4
num=7: 7 > 3 → append. num=101: append. num=18: replace 101.
tails = [2, 3, 7, 18], LIS length = 4 ✅

⚠️ Common Mistakes

❌ Thinking the tails array IS the LIS

The tails array gives the correct length of LIS but NOT the actual LIS sequence. To reconstruct, you need additional bookkeeping (parent pointers).

❌ Using <= instead of < for strictly increasing

LIS requires strictly increasing. Use nums[j] < nums[i], not <=. For non-decreasing, use <=.

💼 Interview Perspective

  • Know both O(n²) and O(n log n). The O(n²) is easier to explain; the O(n log n) shows mastery.
  • "Number of LIS" (LC 673) requires counting, not just length — needs modified O(n²) approach.
  • LIS connects to patience sorting, longest chain, and Russian doll envelopes problems.
  • Longest Non-decreasing Subsequence: change < to <=.

📝 Quick Revision Notes

  • LIS: longest strictly increasing subsequence (not contiguous).
  • O(n²): dp[i] = max(dp[j]+1) for all j < i where nums[j] < nums[i].
  • O(n log n): maintain "tails" array, binary search for insertion position.
  • tails array gives length, NOT the actual LIS.
  • Patience sorting = the O(n log n) algorithm in card game form.

🃏 Flash Cards

What does tails[i] represent in the O(n log n) LIS algorithm?
tails[i] stores the smallest possible tail element of all increasing subsequences of length i+1. By keeping tails as small as possible, we maximize the chance of extending the LIS with future elements.
What is the O(n²) DP recurrence for LIS?
dp[i] = max(dp[j] + 1) for all j < i where nums[j] < nums[i]. Each dp[i] represents the LIS length ending at index i. Answer = max(dp[0..n-1]).
