Radix Sort

Phase 4 Medium

📖 What is Radix Sort?

Radix Sort sorts numbers digit by digit, starting from the least significant digit (LSD) to the most significant. It uses a stable sort (like counting sort) as a subroutine for each digit position.

It achieves O(d × (n + k)) time, where d is the number of digits and k is the base (usually 10).

🌍 Real-Life Analogy

Imagine sorting a pile of checks by amount. You first sort by the ones digit into 10 piles (0-9). Collect them in order. Then sort by the tens digit, then hundreds. After processing every digit, the checks are fully sorted! Each pass is quick, and together they sort everything — without ever directly comparing two amounts.

⚙️ How It Works

JavaScript
function radixSort(arr) {
    if (arr.length === 0) return arr;
    const max = Math.max(...arr);
    
    // Sort by each digit position (1s, 10s, 100s, ...)
    for (let exp = 1; Math.floor(max / exp) > 0; exp *= 10) {
        countingSortByDigit(arr, exp);
    }
    return arr;
}

function countingSortByDigit(arr, exp) {
    const n = arr.length;
    const output = new Array(n);
    const count = new Array(10).fill(0);
    
    // Count occurrences of each digit
    for (let i = 0; i < n; i++) {
        const digit = Math.floor(arr[i] / exp) % 10;
        count[digit]++;
    }
    
    // Cumulative count
    for (let i = 1; i < 10; i++) {
        count[i] += count[i - 1];
    }
    
    // Build output (right to left for stability)
    for (let i = n - 1; i >= 0; i--) {
        const digit = Math.floor(arr[i] / exp) % 10;
        output[count[digit] - 1] = arr[i];
        count[digit]--;
    }
    
    // Copy back
    for (let i = 0; i < n; i++) arr[i] = output[i];
}

// radixSort([170, 45, 75, 90, 802, 24, 2, 66])
// → [2, 24, 45, 66, 75, 90, 170, 802]

💡 When is radix sort faster than O(n log n)?

When d (number of digits) is constant or small. For 32-bit integers, d=10 with base 10 (or d=4 with base 256). So it's effectively O(n) for fixed-width integers!

⚠️ Common Mistakes

❌ Using an unstable sort as subroutine

Radix sort MUST use a stable sort for each digit. If the subroutine is unstable, earlier digit orderings get destroyed. Counting sort (stable version) is the standard choice.

❌ Forgetting to handle negative numbers

Standard radix sort works only on non-negative integers. For negatives, separate them, sort the absolute values, reverse, then concatenate.

💼 Interview Perspective

  • Radix sort is O(d(n+k)) — effectively O(n) for fixed-width integers.
  • Non-comparison sort — breaks the Ω(n log n) barrier.
  • Used in practice for sorting large datasets of integers or fixed-length strings.
  • LSD (least significant digit first) is most common. MSD is used for strings.
  • Requires stable subroutine sort — always mention this in interviews.

📝 Quick Revision Notes

🃏 Flash Cards

Why must radix sort use a STABLE subroutine sort?
If the subroutine is unstable, sorting by a later digit could destroy the ordering from earlier digits. Stability ensures that elements with the same current digit remain in their previously-sorted order.
What is the time complexity of radix sort for 32-bit integers?
O(d × n) where d = number of digits (constant for fixed-width). For 32-bit integers using base 256: d = 4 passes. Effectively O(n) — linear time!
