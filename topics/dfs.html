Depth-First Search (DFS)

Phase 10 Medium

📖 What is DFS?

Depth-First Search (DFS) is a graph/tree traversal algorithm that explores as far as possible along each branch before backtracking. It uses a stack (or recursion, which uses the call stack) to keep track of nodes to visit.

💡 Key Insight

DFS goes deep before wide. Think of it as "fully explore one path before trying another." This makes it ideal for problems like finding connected components, cycle detection, and topological sorting.

🌍 Real-Life Analogy

🏰 Exploring a Maze

You're in a maze. At every fork, you always go as deep as possible down one path. If you hit a dead end, you backtrack to the last fork and try the next path. You keep doing this until you've explored every corridor. That's DFS — depth first, then backtrack!

  • Solving a puzzle: Try one possibility fully, backtrack if it fails, try next
  • File system search: Enter a folder, go into its subfolders, then come back and try the next folder
  • Family tree research: Follow one ancestor line all the way back, then trace another branch

⚙️ How DFS Works

DFS Traversal Order on a Graph
Graph: 1 --- 2 --- 5 | | 3 --- 4 --- 6 DFS from node 1 (visit order): 1 → 2 → 4 → 3 → (backtrack) → 6 → (backtrack) → 5 Stack trace: Push 1 → Visit 1 → Push neighbors [2, 3] Pop 3 → Visit 3 → Push neighbor [4] (1 already visited) Pop 4 → Visit 4 → Push neighbors [6] (2,3 visited) Pop 6 → Visit 6 → No unvisited neighbors Pop 2 → Visit 2 → Push neighbor [5] (1,4 visited) Pop 5 → Visit 5 → Done! Visit order: 1, 3, 4, 6, 2, 5 ✅ (Order depends on neighbor processing order)

Recursive DFS (Most Common)

JavaScript — DFS on Graph
// Graph represented as adjacency list
function dfs(graph, start) {
    const visited = new Set();
    const result = [];
    
    function traverse(node) {
        if (visited.has(node)) return;
        visited.add(node);
        result.push(node);
        
        for (const neighbor of graph[node]) {
            traverse(neighbor);
        }
    }
    
    traverse(start);
    return result;
}

// Example: graph = { 1: [2,3], 2: [1,4,5], 3: [1,4], 4: [2,3,6], 5: [2], 6: [4] }
// dfs(graph, 1) → [1, 2, 4, 3, 6, 5]

Iterative DFS (Using Explicit Stack)

JavaScript — Iterative DFS
function dfsIterative(graph, start) {
    const visited = new Set();
    const stack = [start];
    const result = [];
    
    while (stack.length > 0) {
        const node = stack.pop();
        if (visited.has(node)) continue;
        
        visited.add(node);
        result.push(node);
        
        // Push neighbors in reverse for same order as recursive
        for (let i = graph[node].length - 1; i >= 0; i--) {
            if (!visited.has(graph[node][i])) {
                stack.push(graph[node][i]);
            }
        }
    }
    return result;
}

🗺️ DFS on a 2D Grid (Number of Islands)

JavaScript — Number of Islands
function numIslands(grid) {
    let count = 0;
    const rows = grid.length, cols = grid[0].length;
    
    function dfs(r, c) {
        // Out of bounds or water or already visited
        if (r < 0 || r >= rows || c < 0 || c >= cols || grid[r][c] === '0') return;
        
        grid[r][c] = '0'; // Mark as visited (sink the island)
        
        // Explore all 4 directions
        dfs(r + 1, c);  // down
        dfs(r - 1, c);  // up
        dfs(r, c + 1);  // right
        dfs(r, c - 1);  // left
    }
    
    for (let r = 0; r < rows; r++) {
        for (let c = 0; c < cols; c++) {
            if (grid[r][c] === '1') {
                count++;
                dfs(r, c);  // Sink the entire island
            }
        }
    }
    return count;
}

⏱️ Time & Space Complexity

AspectComplexityWhy?
Time (graph)O(V + E)Visit each vertex once, traverse each edge once
Time (grid)O(R × C)Visit each cell at most once
Space (recursive)O(V) or O(R×C)Recursion stack depth = longest path
Space (iterative)O(V)Explicit stack + visited set

⚠️ Common Mistakes

❌ Forgetting to Mark Nodes as Visited

Without a visited set, DFS will loop forever in graphs with cycles. Always check and mark before recursing.

❌ Stack Overflow on Large Grids

Recursive DFS on a 1000×1000 grid can overflow the call stack. Use iterative DFS with an explicit stack for large inputs.

❌ Not Handling All Connected Components

A single DFS from one node only explores its connected component. For disconnected graphs, loop through all nodes and start DFS from each unvisited node.

💼 Interview Tips

  • DFS vs BFS: DFS for exploring paths, detecting cycles, topological sort. BFS for shortest path in unweighted graphs.
  • Grid DFS: "Number of Islands" pattern is the #1 DFS question. Master it!
  • Backtracking = DFS + pruning: Many backtracking problems are DFS with conditions
  • Pre/Post order: Preorder processes before children, postorder processes after — useful for different problems
  • Cycle detection: Use DFS with 3 states (white/gray/black) for directed graph cycle detection

🏗️ Where DFS Is Used

  • Web crawlers: Follow links deeply before backtracking to explore other branches
  • Maze solvers: Explore one path fully before trying alternatives
  • Compilers: Topological sorting of dependency graphs
  • Social networks: Finding connected communities
  • Game AI: Exploring game states deeply (minimax with alpha-beta pruning)

📝 Quick Revision Notes

  • DFS = go deep first, backtrack when stuck. Uses stack (implicit via recursion or explicit).
  • Time: O(V + E) for graphs, O(R × C) for grids
  • Always use a visited set to avoid infinite loops in graphs
  • Grid DFS: check bounds, mark visited, explore 4 directions
  • DFS for: connected components, cycle detection, topological sort, path finding

🃏 Flash Cards

What data structure does DFS use and why?
A Stack (LIFO). Recursion uses the call stack implicitly. LIFO ensures we explore the most recently discovered node first, going deep before wide.
When should you choose DFS over BFS?
Use DFS for: exploring all paths, cycle detection, topological sort, backtracking problems, connected components. BFS is better for shortest path in unweighted graphs.
How do you detect a cycle in a directed graph using DFS?
Use 3 states: White (unvisited), Gray (in current DFS path), Black (fully processed). If you visit a Gray node during DFS, there's a cycle (back edge found).
