Prefix Sum

Phase 2 Medium

📖 What is Prefix Sum?

A prefix sum (also called cumulative sum) is an array where each element at index i stores the sum of all elements from the start up to index i in the original array.

It's a preprocessing technique that trades O(n) setup time and O(n) space for O(1) range sum queries. Instead of summing elements each time, you compute the answer instantly using subtraction.

💡 Key Insight

Sum of elements from index l to r = prefix[r+1] - prefix[l]. This formula turns an O(n) operation into O(1). If you have many range sum queries, prefix sum is a game-changer!

🌍 Real-Life Analogy

💰 Running Total of Expenses

Imagine tracking your daily expenses: Mon=$10, Tue=$20, Wed=$15, Thu=$25. Your running total is: $10, $30, $45, $70. To find how much you spent from Tuesday to Thursday: Total through Thursday ($70) minus Total through Monday ($10) = $60. That's the prefix sum technique — precompute running totals for instant range queries!

  • Odometer reading: Distance between two cities = odometer at city B minus odometer at city A
  • Bank balance: Running balance after each transaction — subtract to find net deposits in a range
  • Sports stats: Cumulative points — subtract to find points scored in a specific quarter

⚙️ How Prefix Sum Works

Building a Prefix Sum Array
Original: [2, 4, 1, 3, 5] ↓ Index: 0 1 2 3 4 Prefix Sum: [0, 2, 6, 7, 10, 15] ↑ ↑ ↑ ↑ (0) (0+2) (0+2+4+1) (0+2+4+1+3+5) prefix[0] = 0 (empty sum) prefix[1] = 2 (sum of first 1 element) prefix[2] = 6 (sum of first 2 elements) prefix[3] = 7 (sum of first 3 elements) prefix[4] = 10 (sum of first 4 elements) prefix[5] = 15 (sum of all 5 elements) Range Sum Query: sum(index 1 to 3) = ? = prefix[4] - prefix[1] = 10 - 2 = 8 ✅ (4 + 1 + 3 = 8)
JavaScript — Prefix Sum
// Build prefix sum array
function buildPrefixSum(arr) {
    const prefix = new Array(arr.length + 1).fill(0);
    for (let i = 0; i < arr.length; i++) {
        prefix[i + 1] = prefix[i] + arr[i];
    }
    return prefix;
}

// Query: sum from index l to r (inclusive)
function rangeSum(prefix, l, r) {
    return prefix[r + 1] - prefix[l];
}

// Example
const arr = [2, 4, 1, 3, 5];
const prefix = buildPrefixSum(arr);  // [0, 2, 6, 7, 10, 15]
rangeSum(prefix, 1, 3);  // 8  (4 + 1 + 3)

⭐ Pattern: Subarray Sum Equals K

Find the number of subarrays with sum equal to K. This combines prefix sum with a HashMap.

JavaScript — Subarray Sum Equals K
function subarraySum(nums, k) {
    const prefixCount = new Map();
    prefixCount.set(0, 1);  // Empty subarray has sum 0
    
    let sum = 0;
    let count = 0;
    
    for (const num of nums) {
        sum += num;
        
        // If (sum - k) was a prefix sum before, then the subarray
        // from that point to here has sum k
        if (prefixCount.has(sum - k)) {
            count += prefixCount.get(sum - k);
        }
        
        prefixCount.set(sum, (prefixCount.get(sum) || 0) + 1);
    }
    return count;
}

// [1, 2, 3], k=3 → subarrays [1,2] and [3] → count = 2
Why sum - k Works
Array: [1, 2, 3], k = 3 Index: 0 1 2 Running: 1 3 6 At index 1: sum=3, sum-k=0. We saw prefix 0 once (start) → subarray [1,2]=3 ✅ At index 2: sum=6, sum-k=3. We saw prefix 3 once (index 1) → subarray [3]=3 ✅ Total count = 2 Logic: If prefix[j] - prefix[i] = k, then subarray (i, j] sums to k. So we look for prefix[i] = prefix[j] - k in our HashMap.

⭐ Pattern: Product of Array Except Self

JavaScript — Using Prefix and Suffix Products
function productExceptSelf(nums) {
    const n = nums.length;
    const result = new Array(n).fill(1);
    
    // Prefix products (left to right)
    let prefix = 1;
    for (let i = 0; i < n; i++) {
        result[i] = prefix;
        prefix *= nums[i];
    }
    
    // Suffix products (right to left)
    let suffix = 1;
    for (let i = n - 1; i >= 0; i--) {
        result[i] *= suffix;
        suffix *= nums[i];
    }
    
    return result;
}

// [1, 2, 3, 4] → [24, 12, 8, 6]
// Without division, O(n) time, O(1) extra space!

💡 When is Prefix Sum Worth It?

If you have Q range sum queries on an array of size n, brute force is O(Q × n). With prefix sum: O(n) build + O(Q × 1) queries = O(n + Q). Worth it when Q > 1!

⚠️ Common Mistakes

❌ Off-by-One in Prefix Array Size

Prefix array should be size n + 1 with prefix[0] = 0. This handles the case where the range starts at index 0. Without it, you need special cases.

❌ Forgetting to Initialize HashMap with (0, 1)

For "subarray sum = k" problems, you MUST initialize with map.set(0, 1) to handle subarrays starting from index 0.

❌ Confusing Prefix Sum with Sliding Window

Prefix sum handles any range query. Sliding window handles fixed-size or variable-size contiguous subarrays with specific properties. They solve different problems!

❌ Not Considering Negative Numbers

Prefix sums work with negative numbers! The running total can decrease. This is why "subarray sum = k" needs a HashMap, not just two pointers.

  • Prefix Sum + HashMap: The combination solves "count subarrays with sum = k" in O(n)
  • 2D Prefix Sum: Extend to 2D for rectangle sum queries. Uses inclusion-exclusion principle.
  • Difference Array: Related technique for range update queries. Inverse of prefix sum.
  • XOR Prefix: Replace sum with XOR for "subarray with XOR = k" problems

🏗️ Where Prefix Sum Is Used

  • Database queries: Aggregate functions (SUM, AVG) over ranges use prefix sums internally
  • Image processing: Summed-area tables (2D prefix sums) for fast blur and convolution
  • Finance: Cumulative returns, running averages of stock prices
  • Gaming: Damage calculations over regions, cumulative score tracking
  • Network traffic: Cumulative byte counts for bandwidth monitoring

📝 Quick Revision Notes

  • Prefix sum: prefix[i+1] = prefix[i] + arr[i]. prefix[0] = 0.
  • Range sum (l, r) = prefix[r+1] - prefix[l] → O(1)
  • Build time: O(n) | Query time: O(1) | Space: O(n)
  • Subarray Sum = K: use HashMap storing (prefixSum → count). Look for sum - k.
  • Always init HashMap with {0: 1} for subarrays starting at index 0

🃏 Flash Cards

How do you get the sum of elements from index l to r using prefix sum?
sum(l, r) = prefix[r + 1] - prefix[l]. The prefix array has size n+1 with prefix[0] = 0. This subtracts the sum before l from the sum through r.
How does the HashMap technique find subarrays with sum = k?
Track running prefix sum. If (currentSum - k) exists in the HashMap, a subarray with sum k exists between that earlier prefix and now. Count how many times (currentSum - k) appeared. Init map with {0: 1}.
What is the difference between Prefix Sum and Sliding Window?
Prefix Sum: precompute cumulative sums for O(1) range queries. Works for ANY range. Sliding Window: maintains a window that expands/shrinks to find optimal contiguous subarray. Best for fixed-size or monotonic conditions.
Why must you initialize the HashMap with {0: 1} in subarray sum problems?
It handles the case where a subarray starting from index 0 has sum = k. At that point, currentSum = k, and we look for currentSum - k = 0 in the map. Without the initial entry, we'd miss this subarray.
