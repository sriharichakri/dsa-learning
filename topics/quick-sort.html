Quick Sort

Phase 4 Medium

📖 What is Quick Sort?

Quick Sort is a divide and conquer sorting algorithm that works by selecting a pivot element and partitioning the array so that elements smaller than the pivot go left, and elements larger go right. Then it recursively sorts each partition.

It's the fastest general-purpose sorting algorithm in practice, averaging O(n log n) with small constant factors and excellent cache performance.

💡 Key Insight

Unlike Merge Sort, Quick Sort does its heavy work before the recursion (partitioning), not after (merging). This in-place partitioning gives it O(1) extra space — a major advantage.

🌍 Real-Life Analogy

📚 Organizing Books on a Shelf

Pick a random book (the pivot). Put all books that come before it alphabetically on the left, and all books that come after on the right. The pivot book is now in its correct final position. Repeat this process for the left pile and the right pile. Eventually, all books are sorted!

  • Sorting a deck of cards: Pick a card, put lower cards left, higher cards right. Repeat for each pile.
  • Organizer at an event: "Everyone shorter than me go left, taller go right." Repeat in each group.
  • Filing papers: Pick a reference letter, separate A-M vs N-Z, then subdivide each group.

Lomuto Partition Scheme

JavaScript — Quick Sort (Lomuto)
function quickSort(arr, low = 0, high = arr.length - 1) {
    if (low < high) {
        const pivotIndex = partition(arr, low, high);
        quickSort(arr, low, pivotIndex - 1);   // Sort left
        quickSort(arr, pivotIndex + 1, high);   // Sort right
    }
    return arr;
}

function partition(arr, low, high) {
    const pivot = arr[high];  // Choose last element as pivot
    let i = low - 1;          // Boundary of "smaller" region
    
    for (let j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            [arr[i], arr[j]] = [arr[j], arr[i]];  // Swap into smaller region
        }
    }
    
    [arr[i + 1], arr[high]] = [arr[high], arr[i + 1]];  // Place pivot
    return i + 1;  // Pivot's final position
}

🔄 Lomuto vs Hoare Partition

Hoare Partition Scheme

Hoare's scheme uses two pointers that move toward each other, making fewer swaps on average.

JavaScript — Hoare Partition
function hoarePartition(arr, low, high) {
    const pivot = arr[Math.floor((low + high) / 2)];
    let i = low - 1;
    let j = high + 1;
    
    while (true) {
        do { i++; } while (arr[i] < pivot);
        do { j--; } while (arr[j] > pivot);
        
        if (i >= j) return j;
        [arr[i], arr[j]] = [arr[j], arr[i]];
    }
}

// Note: With Hoare, recurse on (low, pivotIndex) and (pivotIndex+1, high)

⏱️ Time & Space Complexity

CaseTimeWhy?
BestO(n log n)Pivot splits array into equal halves
AverageO(n log n)Random pivots give balanced partitions on average
WorstO(n²)Already sorted + always picking min/max as pivot
Space (avg)O(log n)Recursion stack depth = log n for balanced splits
Space (worst)O(n)Skewed partitions = n recursive calls
💡 Avoiding Worst Case

Use randomized pivot selection or median-of-three (pick median of first, middle, last elements) to avoid O(n²) on sorted input.

⚠️ Common Mistakes

❌ Always Picking First/Last Element as Pivot

On sorted or nearly sorted arrays, this causes O(n²) worst case. Use random pivot or median-of-three.

❌ Not Handling Duplicates

Arrays with many duplicates cause Lomuto partition to degrade. Use 3-way partitioning (Dutch National Flag) for arrays with duplicates.

❌ Stack Overflow on Large Arrays

Worst-case O(n) recursion depth can overflow the stack. Use tail call optimization: recurse on the smaller partition, iterate on the larger one.

❌ Confusing Lomuto and Hoare Recursive Calls

Lomuto: recurse on (low, pi-1) and (pi+1, high). Hoare: recurse on (low, pi) and (pi+1, high). Mixing them causes bugs!

💼 Interview Tips

  • Quick Select: Use partitioning to find kth smallest/largest in O(n) average — very common interview question
  • In-place advantage: Quick Sort uses O(log n) space vs Merge Sort's O(n)
  • Not stable: Quick Sort is not stable — mention this when comparing sorts
  • 3-way partition: Know Dutch National Flag for handling duplicates efficiently
  • Real-world usage: C's qsort, Java's Arrays.sort for primitives both use Quick Sort variants

🏗️ Where Quick Sort Is Used

  • C standard library: qsort() is typically implemented as Quick Sort
  • Java: Arrays.sort() for primitive types uses dual-pivot Quick Sort
  • V8 engine (Chrome): JavaScript's Array.sort used Timsort with Quick Sort internals
  • Database query sorting: In-memory sorting often uses Quick Sort for its cache efficiency

📝 Quick Revision Notes

  • Quick Sort = pick pivot → partition (small left, big right) → recurse on each side
  • Average: O(n log n) | Worst: O(n²) | Space: O(log n)
  • Lomuto: simple, pivot at end | Hoare: fewer swaps, pivot at middle
  • Use randomized pivot to avoid worst case on sorted input
  • Not stable, but in-place — fastest in practice for random data

🃏 Flash Cards

What causes Quick Sort's O(n²) worst case?
When the pivot always lands at the extreme end (smallest or largest element), creating maximally unbalanced partitions. Happens with sorted input + first/last pivot choice.
How does Quick Select find the kth element in O(n)?
Partition once. If pivot index = k, done. If k < pivot index, recurse on left only. If k > pivot index, recurse on right only. Only recurse on ONE side → O(n) average.
What is 3-way partitioning?
Partition into three groups: elements < pivot, elements == pivot, elements > pivot. Handles many duplicates efficiently. Used in Dutch National Flag problem.
