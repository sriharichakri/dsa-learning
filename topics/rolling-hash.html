Rolling Hash

Phase 15 Hard

📖 What is a Rolling Hash?

A Rolling Hash (also called a polynomial hash) is a technique for computing hash values of sliding windows of a string in O(1) per slide after O(n) preprocessing. It converts substrings into numerical fingerprints, enabling fast string comparisons without character-by-character checks.

The core idea is that when you slide the window one position right, you can compute the new hash from the old one by removing the outgoing character and adding the incoming character — all in O(1) using modular arithmetic.

🔢 Real-Life Analogy: Odometer Reading

Think of a car odometer that shows the last 4 digits. When the car drives forward (slides the window), the leftmost digit "falls off" and a new digit appears on the right. You don't re-read all 4 digits from scratch — you shift left, drop the old leading digit, and add the new trailing one. Rolling hash works the same way with string characters as "digits" in a polynomial number system.

⚙️ How Rolling Hash Works

Polynomial Rolling Hash Formula
Hash of s[l..r] using base B and modulus M: h(s[l..r]) = s[l]×B^(r-l) + s[l+1]×B^(r-l-1) + ... + s[r]×B^0 (mod M) Prefix hash approach: prefixHash[i] = hash of s[0..i-1] prefixHash[0] = 0 prefixHash[i] = prefixHash[i-1] × B + s[i-1] (mod M) Substring hash s[l..r]: h(l, r) = (prefixHash[r+1] - prefixHash[l] × B^(r-l+1)) mod M Example: "abcab", B=31, M=1e9+7 a=1, b=2, c=3 prefixHash[0] = 0 prefixHash[1] = 0×31 + 1 = 1 ("a") prefixHash[2] = 1×31 + 2 = 33 ("ab") prefixHash[3] = 33×31 + 3 = 1026 ("abc") prefixHash[4] = 1026×31 + 1 = 31807 ("abca") prefixHash[5] = 31807×31 + 2 = 986019 ("abcab") h("ab") using indices [0,1]: = prefixHash[2] - prefixHash[0] × 31² = 33 - 0 = 33 h("ab") using indices [3,4]: = prefixHash[5] - prefixHash[3] × 31² = 986019 - 1026×961 = 986019 - 985986 = 33 ← Same hash! ✓

🔧 Implementation

JavaScript — Rolling Hash (Prefix Hash + Substring Query)
class RollingHash {
    constructor(s, base = 31, mod = 1_000_000_007) {
        this.n = s.length;
        this.BASE = base;
        this.MOD = mod;

        // Precompute prefix hashes and powers
        this.prefixHash = new Array(this.n + 1).fill(0);
        this.power = new Array(this.n + 1).fill(1);

        for (let i = 0; i < this.n; i++) {
            this.prefixHash[i + 1] = (this.prefixHash[i] * this.BASE +
                (s.charCodeAt(i) - 96)) % this.MOD;
            this.power[i + 1] = (this.power[i] * this.BASE) % this.MOD;
        }
    }

    // Get hash of substring s[l..r] (0-indexed, inclusive) — O(1)
    getHash(l, r) {
        const len = r - l + 1;
        let h = (this.prefixHash[r + 1] -
                 this.prefixHash[l] * this.power[len] % this.MOD + 
                 this.MOD * 2) % this.MOD;
        return h;
    }

    // Compare two substrings in O(1)
    isEqual(l1, r1, l2, r2) {
        return this.getHash(l1, r1) === this.getHash(l2, r2);
    }
}

// Double hashing for collision safety
class DoubleHash {
    constructor(s) {
        this.h1 = new RollingHash(s, 31, 1_000_000_007);
        this.h2 = new RollingHash(s, 37, 1_000_000_009);
    }

    getHash(l, r) {
        return this.h1.getHash(l, r) + "," + this.h2.getHash(l, r);
    }

    isEqual(l1, r1, l2, r2) {
        return this.h1.isEqual(l1, r1, l2, r2) &&
               this.h2.isEqual(l1, r1, l2, r2);
    }
}

// Usage
const rh = new RollingHash("abcabc");
console.log(rh.getHash(0, 2));  // hash of "abc"
console.log(rh.getHash(3, 5));  // hash of "abc" — same!
console.log(rh.isEqual(0, 2, 3, 5)); // true

// Find longest duplicate substring using binary search + rolling hash
function longestDupSubstring(s) {
    const rh = new DoubleHash(s);
    let lo = 1, hi = s.length - 1, result = "";

    while (lo <= hi) {
        const mid = (lo + hi) >> 1;
        const seen = new Set();
        let found = "";

        for (let i = 0; i + mid - 1 < s.length; i++) {
            const h = rh.getHash(i, i + mid - 1);
            if (seen.has(h)) { found = s.substring(i, i + mid); break; }
            seen.add(h);
        }

        if (found) { result = found; lo = mid + 1; }
        else hi = mid - 1;
    }
    return result;
}

⚠️ Common Mistakes

❌ Not handling negative modular arithmetic

When computing prefixHash[r+1] - prefixHash[l] × power[len], the result can be negative. Always add MOD before taking modulus: (result + MOD) % MOD. Some use + 2*MOD for extra safety.

❌ Using single hash for competitive programming

A single hash with one modulus can be hacked by adversarial inputs (anti-hash tests). Always use double hashing (two different base-mod pairs) to make collision probability negligible (~1/10^18).

❌ Integer overflow in multiplication

In languages without BigInt, hash * BASE can overflow. In JavaScript, numbers are safe up to 2^53, so with MOD ≈ 10^9, products up to ~10^18 fit. In C++/Java, use long long or long.

💼 Interview Perspective

  • Rolling hash is a versatile building block: pattern matching (Rabin-Karp), longest duplicate substring, palindrome detection.
  • Combine with binary search for problems like "longest common substring" or "longest repeated substring."
  • Key advantage over KMP/Z: substring comparison in O(1) after preprocessing. Useful when you need to compare many pairs of substrings.
  • Double hashing makes it practically collision-free — mention this to interviewers.
  • The prefix hash approach gives O(1) hash for any substring, not just sliding windows.

📝 Quick Revision Notes

  • Rolling Hash: polynomial hash where h(s) = Σ s[i] × B^(n-1-i) mod M.
  • Prefix hash: precompute cumulative hash, get any substring hash in O(1).
  • Sliding window: remove leading char, shift, add trailing char — O(1).
  • Double hashing: two (base, mod) pairs eliminates collisions practically.
  • Applications: Rabin-Karp, longest duplicate substring, string comparison, plagiarism detection.

🃏 Flash Cards

How do you get the hash of any substring in O(1)?
Precompute prefix hashes: pH[i] = hash(s[0..i-1]). Then: hash(s[l..r]) = pH[r+1] - pH[l] × B^(r-l+1) (mod M). The subtraction removes the prefix contribution, and the power scales the prefix hash to align with the substring.
Why use double hashing instead of single?
With a single (base, mod), collision probability ≈ 1/M ≈ 10^-9 per comparison. Adversarial inputs can force collisions. With double hashing (two independent hash functions), collision probability ≈ 1/(M₁ × M₂) ≈ 10^-18 — virtually impossible even with 10^6 comparisons.
