Bellman-Ford Algorithm

Phase 10 Hard

📖 What is the Bellman-Ford Algorithm?

Bellman-Ford finds the shortest path from a source vertex to all other vertices in a weighted graph, even when edges have negative weights. Unlike Dijkstra's, it can also detect negative-weight cycles.

The algorithm works by relaxing all edges V-1 times (where V is the number of vertices). After V-1 iterations, all shortest paths are found. A V-th iteration check detects negative cycles.

💡 Key Insight

Any shortest path has at most V-1 edges (otherwise it has a cycle). Each iteration of Bellman-Ford guarantees at least one more edge of the shortest path is finalized. So after V-1 iterations, ALL shortest paths are found. If a V-th pass still reduces a distance, a negative cycle exists.

🌍 Real-Life Analogy

💱 Currency Exchange

Imagine exchanging currencies: USD → EUR → GBP → USD. If after a round trip you end up with MORE money than you started, you've found a negative cycle (an arbitrage opportunity). Bellman-Ford can detect such cycles. It's also useful when some "exchange rates" might make you lose money (negative edges) — Dijkstra can't handle this.

  • Delivery routes with tolls and rebates: Some roads have tolls (positive weight), others give rebates (negative weight).
  • Game mechanics: Some paths have penalties and others have bonuses.

Graph: A --4→ B --(-2)→ C | ↗ 6 3 | / D -------- Edges: (A,B,4), (A,D,6), (D,C,3), (B,C,-2) Iteration 0 (init): dist = {A:0, B:∞, C:∞, D:∞} Iteration 1 (relax all edges): (A,B,4): dist[B] = min(∞, 0+4) = 4 (A,D,6): dist[D] = min(∞, 0+6) = 6 (D,C,3): dist[C] = min(∞, 6+3) = 9 (B,C,-2): dist[C] = min(9, 4+(-2)) = 2 ← Negative edge! dist = {A:0, B:4, C:2, D:6} Iteration 2: No changes → converged ✅ Iteration 3 (V-1=3): No changes → No negative cycle ✅
JavaScript — Bellman-Ford Algorithm
function bellmanFord(vertices, edges, source) {
    // Initialize distances
    const dist = new Array(vertices).fill(Infinity);
    dist[source] = 0;

    // Relax all edges V-1 times
    for (let i = 0; i < vertices - 1; i++) {
        for (const [u, v, weight] of edges) {
            if (dist[u] !== Infinity && dist[u] + weight < dist[v]) {
                dist[v] = dist[u] + weight;
            }
        }
    }

    // Check for negative-weight cycles (V-th iteration)
    for (const [u, v, weight] of edges) {
        if (dist[u] !== Infinity && dist[u] + weight < dist[v]) {
            return null; // Negative cycle detected!
        }
    }

    return dist;
}

// Cheapest Flights Within K Stops (modified Bellman-Ford)
function findCheapestPrice(n, flights, src, dst, k) {
    let prices = new Array(n).fill(Infinity);
    prices[src] = 0;

    for (let i = 0; i <= k; i++) {  // At most k+1 edges
        const temp = [...prices]; // Copy to avoid using updates from this round
        for (const [u, v, w] of flights) {
            if (prices[u] !== Infinity && prices[u] + w < temp[v]) {
                temp[v] = prices[u] + w;
            }
        }
        prices = temp;
    }
    return prices[dst] === Infinity ? -1 : prices[dst];
}

⚠️ Common Mistakes

❌ Running only 1 iteration instead of V-1

One iteration only relaxes edges reachable in 1 hop. You need V-1 iterations to propagate distances through the longest possible shortest path.

❌ Using Dijkstra's with negative edges

Dijkstra's greedy approach fails with negative edges because a "finalized" node's distance might decrease later via a negative edge. Always use Bellman-Ford for negative weights.

❌ K-stops variant: not copying the distance array

In "Cheapest Flights Within K Stops", you must use a copy of the distance array each iteration. Otherwise, updates from the current round cascade and allow more stops than k.

💼 Interview Perspective

  • "Cheapest Flights Within K Stops" (LC 787) is the most common Bellman-Ford interview problem.
  • Know when to use Bellman-Ford vs Dijkstra: negative edges → Bellman-Ford.
  • Bellman-Ford is simpler to code than Dijkstra (no priority queue needed).
  • The algorithm can be optimized with early termination if no relaxation occurs in an iteration.
  • SPFA (Shortest Path Faster Algorithm) is a queue-based optimization of Bellman-Ford.

📝 Quick Revision Notes

  • Bellman-Ford: relax ALL edges V-1 times. Handles negative weights.
  • V-th iteration check: if any distance decreases → negative cycle exists.
  • Time O(VE), Space O(V). Slower than Dijkstra but more general.
  • K-stops variant: run only k+1 iterations, use array copy per iteration.
  • Use Dijkstra for non-negative weights (faster). Use Bellman-Ford when negative edges exist.

🃏 Flash Cards

Why does Bellman-Ford relax edges V-1 times?
The shortest path between any two vertices has at most V-1 edges. Each iteration guarantees that shortest paths using at most i edges are correct. After V-1 iterations, all shortest paths (up to V-1 edges) are finalized.
How does Bellman-Ford detect negative cycles?
After V-1 iterations, run one more pass over all edges. If any distance can still be reduced, it means there's a negative cycle — because without a cycle, V-1 iterations would have found all shortest paths.
