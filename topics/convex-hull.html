Convex Hull

Phase 15 Hard

📖 What is a Convex Hull?

The Convex Hull of a set of points is the smallest convex polygon that contains all the points. It's like stretching a rubber band around all the points — the shape it forms is the convex hull. Only the outermost points form the hull; interior points are enclosed.

Two classic algorithms solve this: Graham Scan (O(n log n)) and Gift Wrapping / Jarvis March (O(nh), where h = number of hull points). Graham Scan is preferred for general use.

🔩 Real-Life Analogy: Rubber Band Around Nails

Imagine a board with nails hammered at random positions. Stretch a rubber band around all the nails and let it snap tight. The rubber band only touches the outermost nails, forming a convex shape. The nails it touches are the convex hull vertices, and the interior nails are enclosed but not part of the hull. Finding the convex hull is equivalent to finding which nails the rubber band touches!

⚙️ Key Concept: Cross Product

Cross Product and Turn Direction
Cross product of vectors OA and OB: cross(O, A, B) = (A.x - O.x) × (B.y - O.y) - (A.y - O.y) × (B.x - O.x) cross > 0 → Counter-clockwise turn (left turn) cross = 0 → Collinear (straight line) cross < 0 → Clockwise turn (right turn) B B \ / \ cross > 0 / cross < 0 O————A O————A LEFT TURN RIGHT TURN Graham Scan uses this to decide: - If we make a LEFT turn (or straight) → point is on the hull boundary - If we make a RIGHT turn → previous point is interior, remove it Points: (0,0) (1,0) (2,1) (1,2) (0,1) (-1,1) Convex Hull: (1,2) / \ (-1,1) (2,1) \ / (0,0)——(1,0)

🔧 Implementation

JavaScript — Graham Scan (Convex Hull)
// Cross product of vectors OA and OB
function cross(O, A, B) {
    return (A[0] - O[0]) * (B[1] - O[1]) - (A[1] - O[1]) * (B[0] - O[0]);
}

// Graham Scan — O(n log n)
function convexHull(points) {
    const n = points.length;
    if (n <= 1) return [...points];

    // Sort by x-coordinate, then y-coordinate
    points.sort((a, b) => a[0] - b[0] || a[1] - b[1]);

    // Build lower hull
    const lower = [];
    for (const p of points) {
        while (lower.length >= 2 &&
               cross(lower[lower.length - 2], lower[lower.length - 1], p) <= 0) {
            lower.pop(); // Remove point that makes a clockwise turn
        }
        lower.push(p);
    }

    // Build upper hull
    const upper = [];
    for (let i = points.length - 1; i >= 0; i--) {
        const p = points[i];
        while (upper.length >= 2 &&
               cross(upper[upper.length - 2], upper[upper.length - 1], p) <= 0) {
            upper.pop();
        }
        upper.push(p);
    }

    // Remove last point of each half (it's the first of the other)
    lower.pop();
    upper.pop();

    return lower.concat(upper);
}

// Gift Wrapping / Jarvis March — O(nh)
function jarvisMarch(points) {
    const n = points.length;
    if (n <= 1) return [...points];

    // Start from leftmost point
    let start = 0;
    for (let i = 1; i < n; i++) {
        if (points[i][0] < points[start][0] ||
            (points[i][0] === points[start][0] && points[i][1] < points[start][1])) {
            start = i;
        }
    }

    const hull = [];
    let current = start;
    do {
        hull.push(points[current]);
        let next = 0;
        for (let i = 1; i < n; i++) {
            if (i === current) continue;
            const c = cross(points[current], points[next], points[i]);
            if (next === current || c > 0 ||
                (c === 0 && dist(points[current], points[i]) >
                             dist(points[current], points[next]))) {
                next = i;
            }
        }
        current = next;
    } while (current !== start);

    return hull;
}

function dist(a, b) {
    return (a[0] - b[0]) ** 2 + (a[1] - b[1]) ** 2;
}

// Usage
const points = [[0,0], [1,0], [2,1], [1,2], [0,1], [-1,1], [1,1]];
const hull = convexHull(points);
console.log(hull);
// [[-1,1], [0,0], [1,0], [2,1], [1,2]] — counter-clockwise order

⚠️ Common Mistakes

❌ Wrong cross product sign convention

The cross product's sign determines turn direction. If you swap the order of subtraction, left and right turns are reversed. Double-check: cross > 0 = counter-clockwise (left turn) for standard x-right, y-up orientation.

❌ Not handling collinear points

When cross product = 0, points are collinear. Decide whether to include or exclude collinear boundary points. Using <= 0 excludes them (most common); using < 0 includes them.

❌ Forgetting to process both upper and lower hulls

Graham Scan (monotone chain variant) builds the hull in two halves. Forgetting the upper hull gives only the bottom boundary. Both passes are needed for a complete convex hull.

💼 Interview Perspective

  • Convex hull is a computational geometry classic. Know at least one algorithm (Graham Scan or Monotone Chain).
  • The cross product is fundamental — it determines turn direction and is used in many geometry problems.
  • Applications: collision detection, image processing, GIS, robotics path planning.
  • If asked "given n points, find the smallest enclosing polygon," the answer is convex hull.
  • Convex hull can also optimize DP problems (convex hull trick) — a different but related concept.

📝 Quick Revision Notes

  • Convex Hull: smallest convex polygon enclosing all points.
  • Cross product: determines left turn (>0), right turn (<0), or collinear (=0).
  • Graham Scan: sort by x, build lower hull + upper hull. O(n log n).
  • Jarvis March: start from leftmost, repeatedly find next CCW point. O(nh).
  • Monotone Chain (Andrew's) is a simpler variant of Graham Scan.

🃏 Flash Cards

What does the cross product tell you about three points?
For points O, A, B: cross(O, A, B) = (A-O) × (B-O). Positive: O→A→B makes a counter-clockwise (left) turn. Negative: clockwise (right) turn. Zero: points are collinear. This is the fundamental operation in convex hull algorithms.
How does Graham Scan decide which points to remove?
As points are added to the hull, if the last three points make a clockwise (right) turn (cross product ≤ 0), the middle point is interior — pop it from the stack. Keep popping until a left turn is made. This ensures only convex vertices remain.
When is Jarvis March better than Graham Scan?
Jarvis March is O(nh) where h = hull vertices. When h is very small (e.g., 3-4 hull points out of millions), it can be faster than Graham's O(n log n). But in the worst case (all points on hull), Jarvis is O(n²), so Graham is safer for general use.
