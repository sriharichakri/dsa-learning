Subsets (Power Set)

Phase 13 Medium

📖 What are Subsets?

The power set of an array is the collection of ALL possible subsets, including the empty set and the full set. For n elements, there are 2ⁿ subsets.

💡 Key Insight

At each element, you make a binary choice: include or exclude. This gives 2 choices per element → 2ⁿ total subsets. In backtracking, every node in the recursion tree (not just leaves) is a valid subset.

🌍 Real-Life Analogy

🍕 Pizza Toppings

With 3 toppings (cheese, pepperoni, mushroom), you can order: no toppings, cheese only, pepperoni only, mushroom only, cheese+pepperoni, cheese+mushroom, pepperoni+mushroom, or all three. That's 2³ = 8 possible pizzas — exactly the power set!

⚙️ Subset Generation

Subsets (Distinct Elements)

JavaScript — Subsets (LC 78)
function subsets(nums) {
    const result = [];

    function backtrack(start, path) {
        result.push([...path]); // Every path is a valid subset

        for (let i = start; i < nums.length; i++) {
            path.push(nums[i]);
            backtrack(i + 1, path);
            path.pop();
        }
    }

    backtrack(0, []);
    return result;
}
// subsets([1,2,3]) → [[],[1],[1,2],[1,2,3],[1,3],[2],[2,3],[3]]

Subsets II (With Duplicates)

JavaScript — Subsets II (LC 90)
function subsetsWithDup(nums) {
    const result = [];
    nums.sort((a, b) => a - b); // Sort to group duplicates

    function backtrack(start, path) {
        result.push([...path]);

        for (let i = start; i < nums.length; i++) {
            if (i > start && nums[i] === nums[i - 1]) continue; // Skip duplicates
            path.push(nums[i]);
            backtrack(i + 1, path);
            path.pop();
        }
    }

    backtrack(0, []);
    return result;
}

Iterative Approach

JavaScript — Iterative Subsets
function subsetsIterative(nums) {
    let result = [[]]; // Start with empty set

    for (const num of nums) {
        // Add current number to all existing subsets
        const newSubsets = result.map(subset => [...subset, num]);
        result = [...result, ...newSubsets];
    }
    return result;
}
// [1,2,3]: [[]] → [[],[1]] → [[],[1],[2],[1,2]] → [[],[1],[2],[1,2],[3],[1,3],[2,3],[1,2,3]]

⚠️ Common Mistakes

❌ Only collecting at leaf nodes

Unlike permutations/combinations that collect when path.length equals target, subsets collect at EVERY node. Add to result BEFORE the loop.

❌ Not sorting before deduplication

For Subsets II, you MUST sort first. The skip condition nums[i] === nums[i-1] only works on adjacent elements.

💼 Interview Perspective

  • Subsets is the simplest backtracking problem — great warm-up in interviews.
  • Subsets = combinations of all sizes (k=0,1,...,n combined).
  • Three approaches: backtracking, iterative (cascading), and bitmask. Know at least two.
  • Bitmask approach: iterate 0 to 2ⁿ-1, each bit determines include/exclude.

📝 Quick Revision Notes

  • Power set: 2ⁿ subsets for n elements.
  • Backtracking: add to result at every node, not just leaves.
  • Duplicates: sort + skip if nums[i] === nums[i-1] && i > start.
  • Iterative: for each num, add it to all existing subsets.

🃏 Flash Cards

How does subsets differ from combinations in backtracking?
Subsets: add the current path to results at EVERY recursion level (every node is a valid subset). No target size.
Combinations: only add when path reaches exactly k elements (only leaves are valid).
How does the iterative (cascading) approach generate subsets?
Start with [[]] (empty set). For each number, create new subsets by adding that number to all existing subsets. Merge new subsets with existing ones. After processing all numbers, you have all 2ⁿ subsets.
