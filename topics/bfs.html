Breadth-First Search (BFS)

Phase 10 Medium

📖 What is BFS?

Breadth-First Search (BFS) is a graph/tree traversal algorithm that explores all neighbors at the current depth before moving to the next level. It uses a queue (FIFO) to process nodes level by level.

BFS is the go-to algorithm for finding the shortest path in unweighted graphs — every edge has the same "cost," so the first time BFS reaches a node is guaranteed to be via the shortest path.

💡 Key Insight

BFS goes wide before deep. It explores nodes in order of their distance from the source. Level 0 → Level 1 → Level 2... This is why BFS naturally finds shortest paths!

🌍 Real-Life Analogy

🌊 Ripples in Water

Drop a stone in a pond. The ripples spread outward in concentric circles — first the closest water is affected, then the next ring, then the next. BFS works the same way: it visits all nodes at distance 1, then distance 2, then distance 3, and so on. Every point on the same ripple ring is at the same distance from the stone.

  • GPS navigation: Explore all routes of length 1, then length 2, etc. — find shortest first
  • Social network "People you may know": Friends of friends (distance 2), then friends of friends of friends (distance 3)
  • Fire spreading: Fire reaches all adjacent cells simultaneously, then spreads outward level by level

⚙️ How BFS Works

Visit order: 1, 2, 3, 5, 4, 6 ✅
JavaScript — BFS on Graph
function bfs(graph, start) {
    const visited = new Set([start]);
    const queue = [start];
    const result = [];
    
    while (queue.length > 0) {
        const node = queue.shift();  // Dequeue from front
        result.push(node);
        
        for (const neighbor of graph[node]) {
            if (!visited.has(neighbor)) {
                visited.add(neighbor);
                queue.push(neighbor);  // Enqueue
            }
        }
    }
    return result;
}

🛤️ BFS for Shortest Path

JavaScript — Shortest Path in Unweighted Graph
function shortestPath(graph, start, end) {
    const visited = new Set([start]);
    const queue = [[start, [start]]]; // [node, path]
    
    while (queue.length > 0) {
        const [node, path] = queue.shift();
        
        if (node === end) return path; // Found shortest!
        
        for (const neighbor of graph[node]) {
            if (!visited.has(neighbor)) {
                visited.add(neighbor);
                queue.push([neighbor, [...path, neighbor]]);
            }
        }
    }
    return null; // No path exists
}

🗺️ BFS on Grid — Rotting Oranges Pattern

JavaScript — Multi-source BFS (Rotting Oranges)
function orangesRotting(grid) {
    const rows = grid.length, cols = grid[0].length;
    const queue = [];
    let fresh = 0;
    
    // Find all rotten oranges (multiple sources) and count fresh
    for (let r = 0; r < rows; r++) {
        for (let c = 0; c < cols; c++) {
            if (grid[r][c] === 2) queue.push([r, c]);
            if (grid[r][c] === 1) fresh++;
        }
    }
    
    if (fresh === 0) return 0;
    
    const dirs = [[1,0], [-1,0], [0,1], [0,-1]];
    let minutes = 0;
    
    while (queue.length > 0 && fresh > 0) {
        const size = queue.length;  // Process level by level!
        minutes++;
        
        for (let i = 0; i < size; i++) {
            const [r, c] = queue.shift();
            for (const [dr, dc] of dirs) {
                const nr = r + dr, nc = c + dc;
                if (nr >= 0 && nr < rows && nc >= 0 && nc < cols && grid[nr][nc] === 1) {
                    grid[nr][nc] = 2;
                    fresh--;
                    queue.push([nr, nc]);
                }
            }
        }
    }
    
    return fresh === 0 ? minutes : -1;
}

⏱️ Time & Space Complexity

AspectComplexityWhy?
Time (graph)O(V + E)Visit each vertex once, traverse each edge once
Time (grid)O(R × C)Visit each cell at most once
SpaceO(V) or O(R×C)Queue can hold up to all nodes at widest level
💡 BFS Space is Typically More than DFS

BFS stores an entire level in the queue. For a balanced tree, the widest level has ~n/2 nodes. DFS stack only holds the current path depth (log n for balanced trees).

⚠️ Common Mistakes

❌ Marking Visited When Dequeuing Instead of Enqueuing
// ❌ WRONG: Mark when dequeuing — causes duplicates in queue
const node = queue.shift();
visited.add(node);

// ✅ RIGHT: Mark when enqueuing — prevents duplicates
if (!visited.has(neighbor)) {
    visited.add(neighbor);  // Mark HERE
    queue.push(neighbor);
}
❌ Not Processing Level by Level

For problems like "minimum steps" or "rotting oranges," you MUST process the queue level by level using const size = queue.length and a for loop.

❌ Using BFS for Weighted Graphs

BFS finds shortest path only in unweighted graphs. For weighted graphs, use Dijkstra's algorithm with a priority queue.

💼 Interview Tips

  • Shortest path in unweighted graph: Always think BFS first
  • Multi-source BFS: Start with ALL sources in the queue (rotting oranges, walls and gates)
  • Level-order traversal: Binary tree BFS = level-order traversal
  • 0-1 BFS: For graphs with edge weights 0 and 1, use deque (push 0-weight edges to front)
  • BFS vs DFS: BFS for shortest path and level-by-level processing; DFS for exhaustive exploration

🏗️ Where BFS Is Used

  • GPS/Maps: Finding shortest route between two locations
  • Social networks: "Degrees of separation" — BFS from person A to find how far person B is
  • Web crawlers: Crawl all pages at depth 1, then depth 2, etc.
  • Network broadcasting: Spreading messages to all nodes level by level
  • Garbage collection: Mark-and-sweep algorithms use BFS to find reachable objects

📝 Quick Revision Notes

  • BFS = explore level by level using a Queue (FIFO)
  • Guarantees shortest path in unweighted graphs
  • Time: O(V + E) | Space: O(V) (queue can hold entire level)
  • Mark visited when ENQUEUING, not when dequeuing
  • Multi-source BFS: start with all sources in queue simultaneously

🃏 Flash Cards

Why does BFS guarantee the shortest path in unweighted graphs?
BFS explores nodes in order of distance from the source. Level k nodes are all at distance k. So the first time BFS reaches a node, it's via the shortest path (fewest edges).
What is multi-source BFS and when do you use it?
Start BFS with multiple nodes in the queue simultaneously. Used when measuring distance from ANY of several sources (e.g., rotting oranges, nearest gate). Simulates spreading from all sources at once.
Why should you mark nodes visited when enqueuing, not dequeuing?
If you mark on dequeue, the same node can be added to the queue multiple times by different neighbors. This wastes time and memory. Mark on enqueue to prevent duplicates.
