Heap Sort

Phase 4 Medium

📖 What is Heap Sort?

Heap Sort leverages the max-heap data structure to sort an array. It first builds a max-heap from the data, then repeatedly extracts the maximum element and places it at the end of the array.

It guarantees O(n log n) in ALL cases — no worst-case degradation like quicksort.

🌍 Real-Life Analogy

Imagine a tournament bracket. The winner (largest) always floats to the top. You crown the champion (extract max), remove them, run the tournament again to find the runner-up, and repeat. Each "tournament" takes O(log n) rounds because the bracket is a binary tree. After n tournaments, everyone is ranked!

⚙️ How It Works

  1. Build a max-heap from the array (in-place).
  2. Swap the root (max element) with the last element.
  3. Reduce heap size by 1 (last element is now sorted).
  4. Heapify the root to restore the max-heap property.
  5. Repeat steps 2-4 until heap size is 1.
JavaScript
function heapSort(arr) {
    const n = arr.length;
    
    // Step 1: Build max-heap (start from last non-leaf)
    for (let i = Math.floor(n / 2) - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }
    
    // Step 2: Extract max one by one
    for (let i = n - 1; i > 0; i--) {
        [arr[0], arr[i]] = [arr[i], arr[0]]; // Swap max to end
        heapify(arr, i, 0);                   // Restore heap
    }
    return arr;
}

function heapify(arr, size, root) {
    let largest = root;
    let left = 2 * root + 1;
    let right = 2 * root + 2;
    
    if (left < size && arr[left] > arr[largest]) {
        largest = left;
    }
    if (right < size && arr[right] > arr[largest]) {
        largest = right;
    }
    
    if (largest !== root) {
        [arr[root], arr[largest]] = [arr[largest], arr[root]];
        heapify(arr, size, largest); // Recursively fix subtree
    }
}

// heapSort([4, 10, 3, 5, 1]) → [1, 3, 4, 5, 10]

💡 Heap Sort vs Quick Sort

Heap sort guarantees O(n log n) always. Quick sort averages O(n log n) but degrades to O(n²). However, quicksort is usually faster in practice due to better cache performance. Heap sort wins when you need a guaranteed worst-case bound.

⚠️ Common Mistakes

❌ Wrong starting index for build-heap

Start heapify from Math.floor(n/2) - 1 (last non-leaf node), not from n-1 or 0. Leaf nodes don't need heapifying.

❌ Forgetting to reduce heap size after each extraction

After swapping max to position i, call heapify(arr, i, 0) not heapify(arr, n, 0). The size parameter must shrink each iteration!

❌ Building min-heap for ascending sort

To sort in ascending order, build a max-heap (not min-heap). Extract max → place at end → builds sorted array from right to left.

💼 Interview Perspective

  • Heap sort is NOT stable — swapping elements can change relative order.
  • O(n log n) guaranteed — no worst-case degradation unlike quicksort.
  • In-place with O(1) extra space — unlike merge sort's O(n).
  • Build-heap is O(n), not O(n log n) — this is a common interview question!
  • Know when to use: when you need guaranteed O(n log n) and O(1) space.

📝 Quick Revision Notes

  • Build max-heap → swap root with last → shrink heap → heapify → repeat.
  • Time: O(n log n) always. Space: O(1). NOT stable.
  • Build-heap is O(n). Each extraction is O(log n). Total: O(n log n).
  • Heapify: compare root with children, swap with larger, recurse down.

🃏 Flash Cards

Why is build-heap O(n) and not O(n log n)?
Most nodes are near the bottom (leaves). Leaves need 0 work, nodes one level up need 1 swap, etc. The sum is: n/2·0 + n/4·1 + n/8·2 + ... = O(n). More nodes do less work!
For ascending sort, do you build a max-heap or min-heap?
Max-heap. You extract the maximum, place it at the end. Then extract next max, place before it. This builds the sorted array from right to left in ascending order.
