Sliding Window

Phase 5 Medium

📖 What is the Sliding Window Technique?

The Sliding Window technique is a method for solving problems that involve contiguous subarrays or substrings. Instead of recalculating everything from scratch for each position, you "slide" a window across the data, adding a new element and removing an old one — updating the answer incrementally.

This converts an O(n × k) brute force into an O(n) solution.

💡 Key Insight

Sliding Window works when the problem asks about contiguous sequences (subarrays, substrings) and you need to find a maximum, minimum, or count. The "window" maintains a running state that you update cheaply as it moves.

🌍 Real-Life Analogy

🚂 Looking Through a Train Window

Imagine you're on a train, looking through a window of fixed width. As the train moves forward, you see new scenery on the right side while old scenery disappears on the left. You don't have to look at the entire landscape — just what's visible in your window. The window slides along, and you only track what enters and exits.

  • Security camera frame: The camera shows a fixed-width view that pans across a scene. At each position, only the new and disappearing parts change.
  • 7-day moving average: Stock analysts look at the average of the last 7 days. Each new day: add today, remove the day from 8 days ago.
  • Conveyor belt with a sensor: The sensor monitors k items at a time. As the belt moves, one item enters the view and one leaves.

📐 Fixed-Size Sliding Window

The window size k stays constant. You slide it one position at a time.

Sliding Window: Max sum of 3 consecutive elements in [2, 1, 5, 1, 3, 2]
Array: [2, 1, 5, 1, 3, 2] k = 3 Step 1: [2, 1, 5] 1, 3, 2 sum = 8 └──────┘ Step 2: 2 [1, 5, 1] 3, 2 sum = 8 - 2 + 1 = 7 └──────┘ Step 3: 2, 1 [5, 1, 3] 2 sum = 7 - 1 + 3 = 9 ← MAX! └──────┘ Step 4: 2, 1, 5 [1, 3, 2] sum = 9 - 5 + 2 = 6 └──────┘ Answer: 9 (window [5, 1, 3])
JavaScript
function maxSumSubarray(arr, k) {
    // Step 1: Calculate sum of first window
    let windowSum = 0;
    for (let i = 0; i < k; i++) {
        windowSum += arr[i];
    }
    let maxSum = windowSum;
    
    // Step 2: Slide the window
    for (let i = k; i < arr.length; i++) {
        windowSum += arr[i];       // Add new element (right)
        windowSum -= arr[i - k];   // Remove old element (left)
        maxSum = Math.max(maxSum, windowSum);
    }
    return maxSum;
}

maxSumSubarray([2, 1, 5, 1, 3, 2], 3); // 9

windowSum=8, maxSum=8

3

📏 Variable-Size Sliding Window

The window expands and contracts based on a condition. Two pointers (left, right) define the boundaries.

Longest Substring Without Repeating Characters

JavaScript
function lengthOfLongestSubstring(s) {
    const charSet = new Set();
    let left = 0, maxLen = 0;
    
    for (let right = 0; right < s.length; right++) {
        // Shrink window while duplicate exists
        while (charSet.has(s[right])) {
            charSet.delete(s[left]);
            left++;
        }
        // Expand window
        charSet.add(s[right]);
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}

lengthOfLongestSubstring("abcabcbb"); // 3 ("abc")
Variable Window: "abcabcbb" → Find longest substring without repeats
String: a b c a b c b b L R set={a,b,c} len=3 L R 'a' repeats → shrink L R set={c,a,b} len=3 L R 'c' repeats → shrink L R set={b,c} len=2 Best: length 3 ("abc")

⏱️ Complexity Analysis

ApproachTimeSpace
Brute force (check all subarrays)O(n × k) or O(n²)O(1)
Fixed sliding windowO(n)O(1)
Variable sliding windowO(n)O(k) for set/map
💡 Why is variable window O(n)?

Even though there's a while loop inside the for loop, each element is added and removed at most once. The left pointer never moves backward. So total operations across all iterations = 2n = O(n).

🎯 When to Use Sliding Window

Look for these keywords in the problem:

  • "Contiguous subarray/substring"
  • "Maximum/minimum sum/length of subarray of size k"
  • "Longest/shortest substring with some property"
  • "At most k distinct characters"
  • "Find a substring containing all characters of another string"

⚠️ Common Mistakes

❌ Forgetting to handle window smaller than k
// ❌ If array has fewer than k elements, no valid window exists
function maxSum(arr, k) {
    if (arr.length < k) return -1; // Handle edge case!
    // ...
}
❌ Off-by-one in window boundaries

Window size = right - left + 1, not right - left. A window from index 2 to 4 has 3 elements, not 2.

❌ Using sliding window on non-contiguous problems

Sliding window only works for contiguous subarrays/substrings. If the problem asks for subsequences (non-contiguous), you likely need DP or two pointers.

💼 Interview Perspective

  • If you see "subarray of size k" → immediately think fixed sliding window.
  • If you see "longest/shortest substring with condition" → think variable sliding window.
  • Always start with brute force, then explain how sliding window optimizes it.
  • Know how to combine sliding window with HashMap (frequency counting inside the window).
  • Practice the variable window template until it's muscle memory.

📝 Quick Revision Notes

  • Fixed window: size k stays constant. Slide by adding right, removing left. O(n).
  • Variable window: expand right, shrink left when condition violated. O(n).
  • Window size = right - left + 1.
  • Keywords: contiguous, subarray, substring, maximum/minimum, at most k.
  • Each element enters and exits the window at most once → amortized O(n).

🃏 Flash Cards

What is the difference between fixed and variable sliding window?
Fixed: window size k is constant, slide one position at a time. Variable: window size changes dynamically — expand right pointer, shrink left pointer based on a condition.
Why is the variable sliding window O(n) despite having a while loop inside a for loop?
Each element is added (by right pointer) at most once and removed (by left pointer) at most once. Total operations = 2n = O(n). The left pointer never moves backward.
When should you use sliding window vs two pointers?
Sliding window: contiguous subarrays/substrings with a running state. Two pointers: sorted arrays, searching pairs, or problems where elements aren't necessarily contiguous.
