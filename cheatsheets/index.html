📋 Cheat Sheets

Quick-reference checklists for every major DSA pattern. Print these or review before interviews.

🔍 Binary Search Checklist

  • Is the input sorted or has a monotonic property?
  • Use left + (right - left) / 2 for mid (avoid overflow)
  • Decide: left <= right (inclusive) or left < right (exclusive)?
  • Move left = mid + 1 or right = mid - 1 (never leave mid in range)
  • For lower/upper bound: use right = mid (not mid-1)
  • Binary search on answer: define a feasibility function
  • Edge cases: empty array, single element, target not present
  • Time: O(log n) | Space: O(1) iterative, O(log n) recursive
Template: left=0, right=n-1, while left≤right, mid=(left+right)/2, compare, adjust

🪟 Sliding Window Checklist

  • Problem involves contiguous subarray or substring
  • Fixed window: maintain window of size K, slide right, add new, remove old
  • Variable window: expand right pointer, shrink left when condition violated
  • Track window state with HashMap, counter, or running sum
  • Update answer when window is valid (max/min)
  • For string problems: use frequency array (size 26 for lowercase)
  • Edge cases: empty input, window larger than input, all same characters
  • Time: O(n) — each element enters and leaves window once | Space: O(k) or O(1)
Template: left=0, for right in range(n): add arr[right], while invalid: remove arr[left], left++, update answer

👆👆 Two Pointers Checklist

  • Opposite direction: left=0, right=n-1, converge based on condition
  • Same direction: slow and fast pointer for in-place operations
  • Usually requires sorted array for pair-finding problems
  • For 3Sum: fix one element, use two pointers for remaining
  • Skip duplicates to avoid duplicate results
  • Edge cases: empty array, two elements, all same values
  • Time: O(n) with sort O(n log n) | Space: O(1)

🌊 DFS Checklist

  • Use for: explore all paths, connected components, cycle detection, topological sort
  • Implement with: recursion (implicit stack) or explicit stack
  • Always mark visited nodes to avoid infinite loops
  • For grids: check bounds and visited before exploring neighbors
  • For trees: no need for visited array (no cycles)
  • Preorder: process node, then recurse children
  • Postorder: recurse children, then process node
  • Time: O(V + E) | Space: O(V) for visited + O(V) for stack

🌐 BFS Checklist

  • Use for: shortest path (unweighted), level-order traversal, minimum steps
  • Implement with: queue (FIFO)
  • Add to visited WHEN ADDING to queue (not when popping) to avoid duplicates
  • For level-by-level: process queue size at each level
  • Multi-source BFS: start with all sources in queue simultaneously
  • For grids: 4-directional → dx=[0,0,1,-1], dy=[1,-1,0,0]
  • Time: O(V + E) | Space: O(V) for queue + visited

🧩 Dynamic Programming Checklist

  • Step 1: Can I define the problem in terms of smaller subproblems?
  • Step 2: Define state — what variables describe a subproblem? (dp[i], dp[i][j])
  • Step 3: Write recurrence relation (transition formula)
  • Step 4: Identify base cases
  • Step 5: Choose top-down (memoization) or bottom-up (tabulation)
  • Step 6: Optimize space if possible (rolling array, two rows)
  • Common patterns: Fibonacci, Knapsack, LCS, LIS, Grid traversal, Interval DP
  • If problem says "count ways" or "min/max cost" → likely DP
  • Time: depends on states × transitions | Space: depends on state dimensions
Template: dp[i] = best answer considering first i elements. Transition: dp[i] = f(dp[i-1], dp[i-2], ...)

🔙 Backtracking Checklist

  • Use for: generating all valid combinations, permutations, subsets
  • Structure: choose → explore → unchoose (backtrack)
  • Prune early: skip invalid branches before recursing
  • For subsets: at each element, choose to include or exclude
  • For permutations: try each unused element at current position
  • For combinations: use start index to avoid duplicates
  • Sort input first to skip duplicates easily
  • Time: usually O(2ⁿ) or O(n!) | Space: O(n) for recursion depth

🕸️ Graph Checklist

  • Representation: Adjacency list (sparse), Adjacency matrix (dense)
  • Traversal: DFS (stack/recursion), BFS (queue)
  • Shortest path: BFS (unweighted), Dijkstra (weighted, no negative), Bellman-Ford (negative edges)
  • Cycle detection: DFS with coloring (white/gray/black)
  • Topological sort: Kahn's (BFS) or DFS-based
  • Connected components: DFS/BFS or Union-Find
  • MST: Kruskal's (sort edges + Union-Find) or Prim's (priority queue)
  • Always consider: directed vs undirected, weighted vs unweighted, cyclic vs acyclic

🏔️ Heap Checklist

  • Use for: Top K elements, Kth largest/smallest, merge K sorted, median of stream
  • Min-heap: root is minimum → use for "K largest" (keep K smallest out)
  • Max-heap: root is maximum → use for "K smallest"
  • Two heaps pattern: max-heap for lower half, min-heap for upper half → median
  • Insert: O(log n) | Extract: O(log n) | Peek: O(1)
  • Build heap from array: O(n) (not O(n log n)!)
  • Heap sort: O(n log n) time, O(1) space (in-place)

📚 Stack Checklist

  • LIFO: Last In, First Out
  • Use for: matching brackets, expression evaluation, undo operations
  • Monotonic stack: maintain increasing/decreasing order for next greater/smaller
  • For expression evaluation: two stacks (operators + operands)
  • For parentheses: push opening, pop on closing, check match
  • All operations O(1): push, pop, peek
  • Stack overflow: watch for deep recursion

🗺️ HashMap Checklist

  • Use for: O(1) lookup, counting frequency, grouping by key
  • Two Sum pattern: store complement as you iterate
  • Frequency count: count occurrences, check if matches condition
  • Group by: map from key → list of values (anagram grouping)
  • Sliding window + HashMap: track character frequencies in window
  • HashSet for "have I seen this before?" queries
  • Time: O(1) average for get/set | Space: O(n)
  • Time-space tradeoff: HashMap gives O(1) lookups at the cost of O(n) space

🌳 Tree Checklist

  • Traversals: Inorder (left-root-right), Preorder (root-left-right), Postorder (left-right-root), Level-order (BFS)
  • Inorder of BST gives sorted order
  • Height/depth: max(leftHeight, rightHeight) + 1
  • Balanced check: |leftHeight - rightHeight| ≤ 1 at every node
  • LCA: if p and q on different sides of root → root is LCA
  • Path problems: DFS with backtracking, pass accumulated value
  • Most tree problems are solved with DFS (recursion)
  • Time: O(n) — visit each node | Space: O(h) — recursion depth = height
