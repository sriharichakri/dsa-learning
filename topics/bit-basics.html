Bit Manipulation Basics

Phase 14 Medium

📖 What is Bit Manipulation?

Bit manipulation is the act of algorithmically manipulating bits — the most basic unit of data in computing (0 or 1). Using bitwise operators, we can perform operations that are incredibly fast because they work directly on the binary representation of numbers.

💡 Key Insight

Bitwise operations are the fastest operations a CPU can perform — they execute in a single clock cycle. Understanding bits lets you write O(1) solutions to problems that seem to require O(n) approaches.

🌍 Real-Life Analogy

💡 Light Switches (On/Off)

Imagine a row of 8 light switches. Each switch is either ON (1) or OFF (0). The pattern "10110001" means switches 1, 3, 4, and 8 are on. AND is like keeping switches on only where BOTH panels have them on. OR turns on a switch if EITHER panel has it on. XOR flips switches that differ. NOT flips all switches.

  • Permission flags: Read=4(100), Write=2(010), Execute=1(001). rwx = 7(111)
  • Presence/absence: Tracking which items in a set are present using bits
  • Binary counting: A 3-bit counter goes 000, 001, 010, 011, 100, 101, 110, 111

JavaScript — Bitwise Operators
let a = 12;  // 1100
let b = 10;  // 1010

a & b;   // 8   (1000) — AND
a | b;   // 14  (1110) — OR
a ^ b;   // 6   (0110) — XOR
~a;      // -13        — NOT (flips all bits)
a << 1;  // 24  (11000) — Left shift (×2)
a >> 1;  // 6   (110)   — Right shift (÷2)

🎯 Essential Bit Tricks

JavaScript — Must-Know Bit Tricks
// 1. Check if number is odd or even
function isOdd(n) { return (n & 1) === 1; }
// n & 1 checks the last bit: 1 = odd, 0 = even

// 2. Check if number is power of 2
function isPowerOfTwo(n) { return n > 0 && (n & (n - 1)) === 0; }
// Powers of 2 have exactly one bit set: 1000 & 0111 = 0

// 3. Count set bits (number of 1s)
function countBits(n) {
    let count = 0;
    while (n > 0) {
        count += n & 1;
        n >>= 1;
    }
    return count;
}

// 4. Swap two numbers without temp variable
function swap(a, b) {
    a ^= b; b ^= a; a ^= b;
    return [a, b];
}

// 5. Get, Set, Clear, Toggle a specific bit
function getBit(n, i)    { return (n >> i) & 1; }
function setBit(n, i)    { return n | (1 << i); }
function clearBit(n, i)  { return n & ~(1 << i); }
function toggleBit(n, i) { return n ^ (1 << i); }

// 6. Turn off rightmost set bit
function turnOffRightmost(n) { return n & (n - 1); }

// 7. Isolate rightmost set bit
function isolateRightmost(n) { return n & (-n); }

// 8. Check if two numbers have opposite signs
function oppositeSign(a, b) { return (a ^ b) < 0; }

⭐ XOR — The Magic Operator

XOR Properties
Property Example ───────────────────────────────────────────── a ^ 0 = a 5 ^ 0 = 5 a ^ a = 0 5 ^ 5 = 0 a ^ b = b ^ a Commutative (a ^ b) ^ c = a ^ (b^c) Associative The "Single Number" Trick: [4, 1, 2, 1, 2] 4 ^ 1 ^ 2 ^ 1 ^ 2 = 4 ^ (1^1) ^ (2^2) — Rearrange (commutative) = 4 ^ 0 ^ 0 — Pairs cancel out = 4 — Only unique number remains! ✅
JavaScript — Single Number (XOR)
function singleNumber(nums) {
    let result = 0;
    for (const num of nums) {
        result ^= num;  // Pairs cancel, single remains
    }
    return result;
}
// [2, 3, 2] → 0^2=2, 2^3=1, 1^2=3 → 3 ✅

⚠️ Common Mistakes

❌ Operator Precedence Issues
// ❌ WRONG: == has higher precedence than &
if (n & 1 == 1)    // Parsed as: n & (1 == 1) → n & 1

// ✅ RIGHT: Use parentheses!
if ((n & 1) === 1)
❌ Signed vs Unsigned Right Shift

In JavaScript, >> preserves the sign bit (arithmetic shift). >>> fills with zeros (logical shift). For negative numbers, they give different results!

❌ Forgetting 32-bit Limit in JavaScript

JavaScript bitwise operators work on 32-bit signed integers. For numbers larger than 2³¹-1, bitwise operations may give unexpected results.

💼 Interview Tips

  • XOR for "find the unique": When elements appear in pairs except one, XOR all elements
  • n & (n-1) trick: Turns off the rightmost set bit. Useful for counting bits and power-of-2 checks
  • Bit masking for subsets: Enumerate all subsets of n elements using 0 to 2ⁿ-1
  • Missing number: XOR all numbers 0 to n, then XOR with array elements. Missing number remains
  • Practice converting: Be comfortable converting between decimal and binary in your head

🏗️ Where Bit Manipulation Is Used

  • Unix file permissions: rwxr-xr-- = 754 in octal (111 101 100 in binary)
  • Graphics: Color channels packed into 32 bits (ARGB: 8 bits each)
  • Networking: Subnet masks, IP addressing (AND operation to check network membership)
  • Cryptography: XOR is fundamental to many encryption algorithms
  • Compression: Huffman coding uses variable-length bit patterns
  • Game development: Flag-based state management using bitmasks

📝 Quick Revision Notes

  • AND (&): both 1 → 1 | OR (|): either 1 → 1 | XOR (^): different → 1
  • Left shift (<<): multiply by 2 | Right shift (>>): divide by 2
  • a ^ a = 0 | a ^ 0 = a | n & (n-1) removes rightmost set bit
  • Check odd: n & 1 | Power of 2: n & (n-1) === 0
  • XOR all elements: pairs cancel, unique remains

🃏 Flash Cards

How does n & (n-1) work and what does it do?
It turns off the rightmost set bit. Example: 12 (1100) & 11 (1011) = 8 (1000). n-1 flips all bits from the rightmost 1 to the end. AND cancels the rightmost 1. Used for power-of-2 check and counting bits.
Why does XORing all elements find the single unique number?
XOR is commutative and associative. a^a=0 and a^0=a. So duplicate pairs cancel to 0, and the unique element XORed with 0 gives itself. One pass, O(1) space!
How do you check, set, clear, and toggle the ith bit?
Check: (n >> i) & 1. Set: n | (1 << i). Clear: n & ~(1 << i). Toggle: n ^ (1 << i). The mask (1 << i) has only the ith bit set.
How to find the missing number from [0, 1, ..., n] using XOR?
XOR all numbers from 0 to n, then XOR with all elements in the array. The missing number remains because all present numbers cancel out in pairs.
