🔍 Pattern: When to Use Binary Search

Binary search isn't just for sorted arrays! Use it whenever you can define a monotonic condition:

  • ✅ Sorted array search
  • ✅ Find boundary (first true / last false)
  • ✅ "Minimum X such that condition is satisfied" → Binary search on answer
  • ✅ Search in rotated sorted array
  • ✅ Find peak element
💡 The Key Test

If you can define a function f(x) that returns false for all x < answer and true for all x ≥ answer (or vice versa), binary search works!

🔍 Pattern: When to Use Sliding Window

Sliding window works when you need to find an optimal contiguous sequence:

  • ✅ Fixed window: "Max sum of subarray of size K"
  • ✅ Variable window: "Smallest subarray with sum ≥ S"
  • ✅ String pattern: "Longest substring with at most K distinct characters"
💡 The Key Test

If the problem asks about a contiguous subarray/substring and you can maintain valid state by expanding right and shrinking left, use sliding window!

🔍 Pattern: Optimization Strategy

When you have a brute force solution, optimize using this progression:

O(n³) → Sort + Two Pointers → O(n²) O(n²) → HashMap/HashSet → O(n) O(n²) → Sort + Binary Search → O(n log n) O(n²) → Sliding Window → O(n) O(n²) → Prefix Sum → O(n) O(n²) → Monotonic Stack → O(n) O(2ⁿ) → Memoization (DP) → O(n·k)
🎯 The Golden Rule

If you're doing redundant work (checking things you've already checked), there's likely a data structure or technique that can eliminate that redundancy.

🎓 Practice: Pattern Identification

For each problem below, identify the pattern before looking at the answer:

"Given a sorted array, find two numbers that sum to target" ▼
Pattern: Two Pointers — Array is sorted, looking for pair → put one pointer at start, one at end, converge based on sum comparison.
"Find the longest substring with at most 2 distinct characters" ▼
Pattern: Sliding Window (Variable) — Contiguous substring + condition on distinct characters → expand right, shrink left when condition violated.
"Given n courses with prerequisites, find a valid order to take them" ▼
Pattern: Topological Sort — Dependencies/prerequisites = directed edges → topological ordering.
"Find minimum number of coins to make amount X" ▼
Pattern: Dynamic Programming (Unbounded Knapsack) — Overlapping subproblems (many ways to make sub-amounts), optimal substructure (min coins for X = 1 + min coins for X-coin).
"Count the number of islands in a 2D grid" ▼
Pattern: DFS/BFS (Graph traversal on grid) — Connected components on a grid → traverse each island, mark visited.
"Given daily temperatures, find days until a warmer day" ▼
Pattern: Monotonic Stack — "Next greater element" pattern → maintain a decreasing stack, pop when we find something greater.
"Find if there's a cycle in a linked list" ▼
Pattern: Fast & Slow Pointer (Floyd's) — Cycle detection in linked structure → two pointers at different speeds will meet if cycle exists.
"Generate all valid combinations of n pairs of parentheses" ▼
Pattern: Backtracking — Generate all valid combinations → decision tree with pruning (open count must ≥ close count).
