Cycle Detection

Phase 6 Medium

📖 What is Cycle Detection?

Cycle Detection determines whether a linked list contains a loop — where a node's next pointer points back to a previous node, creating an infinite cycle. This is critical because cycles cause infinite loops in traversal.

🌍 Real-Life Analogy

Imagine driving on a highway following road signs. If you keep seeing the same landmark again and again, you know you're going in circles! One approach: drop breadcrumbs (hash set) at each landmark. If you find a breadcrumb already there → cycle! Better approach: send two cars at different speeds. If the road loops, the faster car will catch up to the slower one.

⚙️ How It Works

Approach 1: Hash Set — O(n) time, O(n) space

JavaScript
function hasCycleHashSet(head) {
    const visited = new Set();
    let curr = head;
    
    while (curr) {
        if (visited.has(curr)) return true; // Seen before = cycle!
        visited.add(curr);
        curr = curr.next;
    }
    return false;  // Reached null = no cycle
}

Approach 2: Floyd's Algorithm — O(n) time, O(1) space ⭐

JavaScript
function hasCycle(head) {
    let slow = head, fast = head;
    
    while (fast && fast.next) {
        slow = slow.next;         // 1 step
        fast = fast.next.next;     // 2 steps
        if (slow === fast) return true;
    }
    return false;
}

// Find WHERE the cycle starts
function detectCycle(head) {
    let slow = head, fast = head;
    
    while (fast && fast.next) {
        slow = slow.next;
        fast = fast.next.next;
        
        if (slow === fast) {
            // Phase 2: Find cycle entry
            slow = head;
            while (slow !== fast) {
                slow = slow.next;
                fast = fast.next;
            }
            return slow;  // Cycle start node
        }
    }
    return null;  // No cycle
}

// Find cycle LENGTH
function cycleLength(head) {
    let slow = head, fast = head;
    
    while (fast && fast.next) {
        slow = slow.next;
        fast = fast.next.next;
        
        if (slow === fast) {
            let count = 1;
            let runner = slow.next;
            while (runner !== slow) {
                count++;
                runner = runner.next;
            }
            return count;
        }
    }
    return 0;
}

⚠️ Common Mistakes

❌ Comparing node VALUES instead of references
// ❌ Two different nodes can have the same value!
if (slow.val === fast.val) return true;

// ✅ Compare node REFERENCES (identity)
if (slow === fast) return true;
❌ Not checking fast AND fast.next

If fast is null or fast.next is null, there's no cycle. Skipping this check causes a null reference error.

💼 Interview Perspective

  • Floyd's algorithm is the gold standard — O(n) time, O(1) space.
  • Three variants: (1) detect cycle, (2) find cycle start, (3) find cycle length.
  • The math behind finding cycle start: if distance to cycle start is a, and slow traveled a + b inside cycle, then fast traveled a + b + c + b. Since fast = 2×slow: a = c. So resetting slow to head and moving both by 1 makes them meet at cycle start.
  • Related: "Find Duplicate Number" (LC 287) — treat array as linked list with cycle.

📝 Quick Revision Notes

  • Floyd's: slow (1 step) + fast (2 steps). If they meet → cycle.
  • Cycle start: after meeting, reset slow to head, move both by 1.
  • Cycle length: after meeting, count steps until they meet again.
  • Time: O(n), Space: O(1). Compare REFERENCES, not values.

🃏 Flash Cards

Why does resetting slow to head and moving both by 1 find the cycle start?
Let a = distance from head to cycle start, b = distance from cycle start to meeting point, c = remaining cycle length. Fast traveled 2× slow's distance. Math gives us: a = c. So a pointer from head and one from the meeting point both need 'a' steps to reach the cycle start.
How is "Find the Duplicate Number" (LC 287) related to cycle detection?
Treat the array as a linked list: index i points to arr[i]. A duplicate value means two indices point to the same node → a cycle exists! Use Floyd's to find where the cycle starts = the duplicate number.
