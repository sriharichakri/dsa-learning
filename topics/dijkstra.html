Dijkstra's Algorithm

Phase 10 Medium

рҹ“– What is Dijkstra's Algorithm?

Dijkstra's Algorithm finds the shortest path from a source node to all other nodes in a weighted graph with non-negative edges. It uses a greedy approach: always process the node with the smallest known distance next.

рҹ’Ў Key Insight

Dijkstra's is essentially BFS with a priority queue instead of a regular queue. Instead of processing nodes level by level, it processes them in order of their distance from the source. This handles weighted edges correctly.

рҹҢҚ Real-Life Analogy

рҹ—әпёҸ GPS Finding the Fastest Route

Your GPS doesn't just count intersections вҖ” it considers actual travel time on each road. It starts at your location, checks all nearby intersections, and always expands to the closest reachable point next. It keeps updating shorter routes as it discovers them. When it reaches your destination, it's guaranteed to have found the fastest route!

  • Flight routing: Find cheapest flights considering different ticket prices per segment
  • Network routing: Internet packets find the fastest path through network nodes
  • Game pathfinding: Characters finding the cheapest path across terrain with varying costs

JavaScript вҖ” Dijkstra's with Min-Heap
function dijkstra(graph, start) {
    // graph: { node: [[neighbor, weight], ...] }
    const dist = {};
    for (const node in graph) dist[node] = Infinity;
    dist[start] = 0;
    
    // Min-heap: [distance, node]
    const pq = [[0, start]];  // Priority queue
    const visited = new Set();
    
    while (pq.length > 0) {
        // Get node with minimum distance
        pq.sort((a, b) => a[0] - b[0]);  // Simple sort (use heap for efficiency)
        const [d, u] = pq.shift();
        
        if (visited.has(u)) continue;  // Already processed
        visited.add(u);
        
        for (const [v, weight] of graph[u]) {
            const newDist = d + weight;
            if (newDist < dist[v]) {
                dist[v] = newDist;
                pq.push([newDist, v]);
            }
        }
    }
    return dist;
}

// Example
const graph = {
    'A': [['B',4], ['C',2]],
    'B': [['A',4], ['D',1], ['E',3]],
    'C': [['A',2], ['D',5]],
    'D': [['B',1], ['C',5], ['F',2]],
    'E': [['B',3], ['F',6]],
    'F': [['D',2], ['E',6]]
};
console.log(dijkstra(graph, 'A'));
// { A: 0, B: 4, C: 2, D: 5, E: 7, F: 7 }

рҹ’Ў When to Use Which

For sparse graphs (E вүҲ V), use min-heap вҶ’ O(V log V). For dense graphs (E вүҲ VВІ), simple array is fine вҶ’ O(VВІ). In interviews, always use min-heap.

вҡ пёҸ Common Mistakes

вқҢ Using Dijkstra's with Negative Edges

Dijkstra's does NOT work with negative edge weights. Once a node is finalized, its distance is locked вҖ” but a negative edge could offer a shorter path later. Use Bellman-Ford for negative edges.

вқҢ Not Skipping Already-Visited Nodes

A node can be added to the priority queue multiple times. Always check if (visited.has(u)) continue; after dequeuing to avoid reprocessing.

вқҢ Using BFS Instead of Dijkstra's for Weighted Graphs

BFS treats all edges as equal weight. For weighted graphs, BFS gives wrong shortest paths. Always use Dijkstra's (or Bellman-Ford) for weighted graphs.

рҹ’ј Interview Tips

  • BFS vs Dijkstra's: Unweighted вҶ’ BFS. Non-negative weighted вҶ’ Dijkstra's. Negative weights вҶ’ Bellman-Ford.
  • Modified Dijkstra's: Many problems modify the "distance" вҖ” e.g., maximum probability, minimum cost with constraints
  • Reconstruct path: Maintain a prev map alongside dist to trace the actual shortest path
  • K stops constraint: "Cheapest Flights Within K Stops" needs modified BFS/Bellman-Ford, not pure Dijkstra's
  • In JavaScript: Use array + sort as simple PQ, or implement a proper min-heap for efficiency

рҹҸ—пёҸ Where Dijkstra's Is Used

  • Google Maps / Waze: Finding fastest driving routes considering road speeds
  • Network routing (OSPF): Internet routers find optimal paths using Dijkstra's
  • Social networks: Finding closest connections weighted by interaction frequency
  • Game pathfinding: A* algorithm (Dijkstra's with heuristic) for game character movement
  • Airline route planning: Finding cheapest flight combinations

рҹ“қ Quick Revision Notes

  • Dijkstra's = BFS with priority queue. Greedy: always process closest unvisited node.
  • Works only with non-negative edge weights
  • Time: O((V+E) log V) with min-heap | Space: O(V)
  • Negative edges вҶ’ use Bellman-Ford instead
  • Skip already-visited nodes when dequeuing from priority queue

рҹғҸ Flash Cards

Why doesn't Dijkstra's work with negative edge weights?
Dijkstra's "finalizes" a node's distance once processed (greedy property). A negative edge could later provide a shorter path to an already-finalized node, violating this assumption.
What's the difference between BFS and Dijkstra's?
BFS uses a regular queue (FIFO) вҖ” all edges have equal weight. Dijkstra's uses a priority queue (min-heap) вҖ” processes the closest node first, handling different edge weights correctly.
How do you reconstruct the actual shortest path in Dijkstra's?
Maintain a `prev` map: when updating dist[v] = dist[u] + w, set prev[v] = u. After algorithm finishes, trace back from destination using prev until you reach the source.
