Heap / Priority Queue

Phase 9 Medium

📖 What is a Heap?

A Heap is a complete binary tree stored in an array that satisfies the heap property:

  • Max-Heap: Every parent node is greater than or equal to its children
  • Min-Heap: Every parent node is less than or equal to its children

A Priority Queue is an abstract data type that always gives you the highest (or lowest) priority element first. A heap is the most common implementation of a priority queue.

💡 Key Insight

A heap is not fully sorted — only the root is guaranteed to be the min/max. But this partial ordering allows O(log n) insert and O(log n) extract, making it perfect for "find the kth largest" type problems.

🌍 Real-Life Analogy

🏥 Hospital Emergency Room

Patients don't get treated in arrival order — they're treated by severity. A heart attack patient jumps ahead of someone with a sprained ankle. The most critical patient is always seen first (max priority). When a new patient arrives, they're placed based on severity, and when one is treated, the next most critical patient moves up. That's a priority queue!

  • Task scheduler: OS processes with highest priority run first
  • Print queue: Urgent documents print before regular ones
  • Airline boarding: First class → Business → Premium Economy → Economy
  • Event-driven simulation: Process the nearest future event first

💡 Why Use an Array?

A complete binary tree maps perfectly to an array with no wasted space. Parent-child relationships are computed with simple math — no pointers needed!

🔧 Heap Operations

Insert (Bubble Up / Sift Up)

Add element at the end, then bubble it up until the heap property is restored.

JavaScript — Min-Heap Implementation
class MinHeap {
    constructor() {
        this.heap = [];
    }
    
    insert(val) {
        this.heap.push(val);
        this._bubbleUp(this.heap.length - 1);
    }
    
    _bubbleUp(i) {
        while (i > 0) {
            const parent = Math.floor((i - 1) / 2);
            if (this.heap[parent] <= this.heap[i]) break;
            [this.heap[parent], this.heap[i]] = [this.heap[i], this.heap[parent]];
            i = parent;
        }
    }
    
    extractMin() {
        if (this.heap.length === 0) return null;
        const min = this.heap[0];
        const last = this.heap.pop();
        if (this.heap.length > 0) {
            this.heap[0] = last;
            this._sinkDown(0);
        }
        return min;
    }
    
    _sinkDown(i) {
        const n = this.heap.length;
        while (true) {
            let smallest = i;
            const left = 2 * i + 1;
            const right = 2 * i + 2;
            
            if (left < n && this.heap[left] < this.heap[smallest]) smallest = left;
            if (right < n && this.heap[right] < this.heap[smallest]) smallest = right;
            
            if (smallest === i) break;
            [this.heap[i], this.heap[smallest]] = [this.heap[smallest], this.heap[i]];
            i = smallest;
        }
    }
    
    peek() { return this.heap[0]; }
    size() { return this.heap.length; }
}

🔨 Build Heap (Heapify)

Building a heap from an unsorted array. Surprisingly, this is O(n), not O(n log n)!

JavaScript — Build Heap
function buildMinHeap(arr) {
    // Start from last non-leaf node and sift down each
    for (let i = Math.floor(arr.length / 2) - 1; i >= 0; i--) {
        siftDown(arr, i, arr.length);
    }
    return arr;
}

function siftDown(arr, i, n) {
    while (true) {
        let smallest = i;
        const left = 2 * i + 1, right = 2 * i + 2;
        if (left < n && arr[left] < arr[smallest]) smallest = left;
        if (right < n && arr[right] < arr[smallest]) smallest = right;
        if (smallest === i) break;
        [arr[i], arr[smallest]] = [arr[smallest], arr[i]];
        i = smallest;
    }
}
💡 Why O(n) and not O(n log n)?

Most nodes are near the bottom and need few swaps. Only the root sinks log n levels. The math works out to O(n) total operations. This is a classic interview question!

⚠️ Common Mistakes

❌ Confusing Min-Heap and Max-Heap

Min-heap gives you the minimum at the root. Max-heap gives the maximum. For "kth largest," use a min-heap of size k (the root is the kth largest).

❌ Wrong Index Formulas
// For 0-indexed arrays:
parent(i) = Math.floor((i - 1) / 2)
left(i)   = 2 * i + 1
right(i)  = 2 * i + 2

// ❌ WRONG (1-indexed formulas on 0-indexed array):
parent(i) = Math.floor(i / 2)  // Off by one!
❌ Forgetting to Handle Edge Cases in Extract

When extracting from a heap with 1 element, after popping there's nothing to sink. Always check heap.length > 0 before placing last element at root.

❌ Using a Heap When a Simpler Structure Works

If you only need the overall min/max once, just scan the array in O(n). Heaps shine when you need repeated min/max extractions.

💼 Interview Tips

  • "Top K" pattern: Use a heap of size k. Min-heap for k largest, max-heap for k smallest.
  • Two heaps for median: Max-heap for lower half, min-heap for upper half. Classic interview problem!
  • Merge K sorted: Use a min-heap to efficiently merge k sorted lists/arrays
  • In JavaScript: No built-in heap. Implement one or use sorted insertion (small n). In Python, use heapq.
  • Heap vs BST: Heap gives O(1) peek of min/max. BST gives O(log n) search for any element.

🏗️ Where Heaps Are Used

  • Dijkstra's algorithm: Priority queue selects the closest unvisited node
  • OS task scheduling: Processes with highest priority run first
  • Huffman coding: Build optimal prefix codes using a min-heap
  • Event-driven simulations: Process events in chronological order
  • Median maintenance: Streaming data median with two heaps

📝 Quick Revision Notes

  • Heap = complete binary tree stored in array, satisfying heap property
  • Min-heap: root = smallest | Max-heap: root = largest
  • Insert: add at end, bubble up — O(log n)
  • Extract: remove root, move last to root, sink down — O(log n)
  • Build heap from array: O(n) using bottom-up heapify
  • Parent = (i-1)/2 | Left child = 2i+1 | Right child = 2i+2

🃏 Flash Cards

How do you find the kth largest element using a heap?
Maintain a min-heap of size k. For each element, if it's larger than the heap's root, replace the root and sift down. At the end, the root is the kth largest. O(n log k).
Why is building a heap O(n) instead of O(n log n)?
Half the nodes are leaves (0 sinks). Quarter need 1 sink. Eighth need 2 sinks. Sum: n/2·0 + n/4·1 + n/8·2 + ... converges to O(n), not O(n log n).
How to find the running median of a stream using heaps?
Use two heaps: max-heap for lower half, min-heap for upper half. Balance sizes (differ by at most 1). Median = root of larger heap, or average of both roots.
What are the index formulas for a 0-indexed heap array?
Parent of i = floor((i-1)/2). Left child of i = 2i+1. Right child of i = 2i+2. A complete binary tree maps perfectly to a contiguous array.
