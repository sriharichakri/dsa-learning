Combinations

Phase 13 Medium

📖 What are Combinations?

Combinations select k elements from n elements where order doesn't matter. Unlike permutations (order matters), {1,2} and {2,1} are the same combination. There are C(n,k) = n! / (k!(n-k)!) combinations.

💡 Key Insight

To avoid duplicates, always iterate forward from the current start index. In combinations, the loop starts from start (not 0), ensuring we never pick an earlier element — this naturally prevents duplicate combinations.

🌍 Real-Life Analogy

🎰 Lottery Numbers

Picking 6 numbers from 49 for a lottery ticket — order doesn't matter (1-2-3 is the same ticket as 3-2-1). That's a combination: C(49,6) = 13,983,816 possible tickets.

⚙️ Combination Problems

Combinations (Choose k from 1..n)

JavaScript — Combinations (LC 77)
function combine(n, k) {
    const result = [];

    function backtrack(start, path) {
        if (path.length === k) {
            result.push([...path]);
            return;
        }

        // Pruning: need k-path.length more elements, so stop early
        for (let i = start; i <= n - (k - path.length) + 1; i++) {
            path.push(i);
            backtrack(i + 1, path); // i+1: don't reuse
            path.pop();
        }
    }

    backtrack(1, []);
    return result;
}

Combination Sum (reuse allowed)

JavaScript — Combination Sum (LC 39)
function combinationSum(candidates, target) {
    const result = [];

    function backtrack(start, path, remaining) {
        if (remaining === 0) { result.push([...path]); return; }
        if (remaining < 0) return;

        for (let i = start; i < candidates.length; i++) {
            path.push(candidates[i]);
            backtrack(i, path, remaining - candidates[i]); // i (not i+1): can reuse
            path.pop();
        }
    }

    backtrack(0, [], target);
    return result;
}

Combination Sum II (no reuse, has duplicates)

JavaScript — Combination Sum II (LC 40)
function combinationSum2(candidates, target) {
    const result = [];
    candidates.sort((a, b) => a - b);

    function backtrack(start, path, remaining) {
        if (remaining === 0) { result.push([...path]); return; }
        if (remaining < 0) return;

        for (let i = start; i < candidates.length; i++) {
            if (i > start && candidates[i] === candidates[i - 1]) continue; // Skip dupes
            path.push(candidates[i]);
            backtrack(i + 1, path, remaining - candidates[i]); // i+1: no reuse
            path.pop();
        }
    }

    backtrack(0, [], target);
    return result;
}

⏱️ Complexity Analysis

ProblemTimeSpace

⚠️ Common Mistakes

❌ Using i+1 when elements can be reused

Combination Sum allows reuse → recurse with i. Combination Sum II doesn't → recurse with i+1.

❌ Not pruning in combinations

In C(n,k), if you need k-path.length more elements, stop the loop at n-(k-path.length)+1. This prunes many unnecessary branches.

💼 Interview Perspective

  • Combinations vs Permutations: combinations start loop from start, permutations from 0 with used array.
  • Know all three: Combination Sum (reuse), Combination Sum II (no reuse, dupes), Combinations (fixed k).
  • The duplicate-skipping pattern if (i > start && nums[i] === nums[i-1]) continue is reused everywhere.

📝 Quick Revision Notes

  • Combinations: order doesn't matter. Use start index to avoid duplicates.
  • Reuse allowed: recurse with i. No reuse: recurse with i+1.
  • Handle duplicate elements: sort + skip if nums[i] === nums[i-1] && i > start.
  • Pruning: stop loop early when not enough elements remain.

🃏 Flash Cards

How do combinations differ from permutations in backtracking?
Combinations: loop from start to n, pass start=i+1 (only pick forward). Order doesn't matter.
Permutations: loop from 0 to n with a used[] array. Order matters. Each element can appear at any position.
How do you handle duplicate candidates in Combination Sum II?
1. Sort the candidates. 2. At each recursion level, skip candidate i if i > start && candidates[i] === candidates[i-1]. This prevents picking the same value multiple times at the same decision level.
