💡 Tips & Tricks

Proven strategies to solve problems faster, optimize solutions, and ace interviews.

🧠 How to Think During an Interview

  1. Understand the problem (2-3 min): Repeat it back. Ask clarifying questions. Identify input/output format, constraints, edge cases.
  2. Work through examples (2-3 min): Trace through 2-3 examples by hand. Include edge cases.
  3. Brute force first (1-2 min): State the obvious O(n²) or O(2ⁿ) solution. This shows you understand the problem.
  4. Optimize (5-10 min): Apply pattern recognition. "Can I use a HashMap? Two Pointers? Sorting?" State the improved complexity.
  5. Code it (10-15 min): Write clean, modular code. Use meaningful variable names.
  6. Test it (3-5 min): Walk through your code with the examples. Check edge cases.
🎯 The #1 Mistake

Jumping straight into coding without understanding the problem. Spend 5 minutes thinking BEFORE writing any code. The interviewer wants to see your thought process, not just code.

⚡ How to Reduce O(n²) to O(n)

Strategy 1: HashMap for Lookup

If you're using a nested loop to find a match, use a HashMap instead.

// ❌ O(n²) - nested loop
for (let i = 0; i < n; i++)
    for (let j = i+1; j < n; j++)
        if (arr[i] + arr[j] === target) return [i, j];

// ✅ O(n) - HashMap
const map = {};
for (let i = 0; i < n; i++) {
    if (map[target - arr[i]] !== undefined) return [map[target - arr[i]], i];
    map[arr[i]] = i;
}

Strategy 2: Sorting + Two Pointers

Sort first (O(n log n)), then use two pointers (O(n)). Total: O(n log n).

Strategy 3: Prefix Sum for Range Queries

Precompute prefix sums to answer any range sum query in O(1).

// Instead of summing arr[l..r] each time: O(n)
// Precompute: prefix[i] = arr[0] + arr[1] + ... + arr[i-1]
// Then: sum(l, r) = prefix[r+1] - prefix[l]  → O(1)!

Strategy 4: Sliding Window

Instead of checking all subarrays (O(n²)), maintain a window that slides (O(n)).

Strategy 5: Monotonic Stack

For "next greater element" patterns, stack processes each element at most twice → O(n).

🎩 Common Interview Tricks

XOR to find the single unique number ▼

XOR of a number with itself = 0. XOR of any number with 0 = itself. So XOR all numbers and duplicates cancel out!

// Find the one number that appears once (all others appear twice)
let result = 0;
for (let num of arr) result ^= num;
// result = the unique number
Use sorting to simplify problems ▼

Many problems become dramatically easier after sorting: finding duplicates, merging intervals, 3Sum, closest pair. Always ask: "Would sorting help?"

Reverse an array in-place with swaps ▼
function reverse(arr, l, r) {
    while (l < r) {
        [arr[l], arr[r]] = [arr[r], arr[l]];
        l++; r--;
    }
}
// Used in: rotate array, reverse words, palindrome checks
Dummy node for linked list operations ▼

Create a dummy node before the head. This eliminates edge cases for operations at the beginning of the list.

let dummy = { next: head };
// ... do operations ...
return dummy.next; // new head
Fast power: O(log n) exponentiation ▼
function power(base, exp, mod) {
    let result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp % 2 === 1) result = (result * base) % mod;
        exp = Math.floor(exp / 2);
        base = (base * base) % mod;
    }
    return result;
}
Check if number is power of 2 ▼
// n & (n-1) === 0 means only one bit is set = power of 2
function isPowerOf2(n) {
    return n > 0 && (n & (n - 1)) === 0;
}
// 8 = 1000, 7 = 0111 → 1000 & 0111 = 0000 ✓

🔎 Recognizing Hidden Patterns

  • "Maximum/Minimum subarray" → Kadane's Algorithm or Sliding Window
  • "K-th element" → Heap or QuickSelect
  • "Number of ways" → Dynamic Programming or Backtracking
  • "Can you partition" → DP (subset sum variant)
  • "Schedule meetings/tasks" → Greedy (sort by end time)
  • "Matrix traversal" → BFS/DFS (treat as graph)
  • "Parentheses matching" → Stack
  • "Detect cycle" → Fast-Slow Pointer (linked list) or DFS coloring (graph)
  • "Merge K sorted" → Min Heap
  • "String search/match" → Trie, KMP, or Rolling Hash

🐛 Debugging Techniques

  1. Print the state: Add console.log for key variables at each step
  2. Trace by hand: Walk through your code line-by-line with a simple input
  3. Check boundaries: Off-by-one errors are the #1 bug. Check loop bounds carefully
  4. Simplify the input: If failing on large input, try the smallest failing case
  5. Check your base case: For recursion/DP, verify base cases handle all terminal conditions
  6. Rubber duck debugging: Explain your code out loud — you'll often spot the bug while explaining

💎 How to Derive the Optimal Solution

Follow this mental framework:

  1. What work am I repeating? → Can I cache it? (HashMap, Memoization)
  2. Am I looking at all pairs? → Can I use Two Pointers or HashMap to avoid nested loops?
  3. Am I recomputing sums? → Prefix Sum
  4. Am I checking all subarrays? → Sliding Window
  5. Am I searching linearly in sorted data? → Binary Search
  6. Can I make a greedy choice? → Prove it's always optimal, then use Greedy
  7. Does the problem have optimal substructure + overlapping subproblems? → DP
💡 The Master Question

"What information am I throwing away between iterations that I could keep?" — This question alone can lead you to most optimizations.

🔢 Useful Formulas & Facts

  • Sum of 1 to n: n * (n + 1) / 2
  • Number of subsets of n elements: 2ⁿ
  • Number of subarrays of length n: n * (n + 1) / 2
  • Height of balanced binary tree with n nodes: log₂(n)
  • Nodes in complete binary tree of height h: 2^(h+1) - 1
  • Max nodes at level k: 2^k
  • For HashMap: load factor > 0.75 → rehash (double size)
  • Master theorem for divide & conquer: T(n) = aT(n/b) + O(nᵈ)
