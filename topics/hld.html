Heavy-Light Decomposition

Phase 15 Hard

📖 What is Heavy-Light Decomposition?

Heavy-Light Decomposition (HLD) is a technique to decompose a tree into chains (paths) such that any root-to-leaf path crosses at most O(log n) chains. This allows us to answer path queries (sum, max, min along a path between two nodes) and perform path updates in O(log² n) by combining HLD with a Segment Tree.

Each node's heaviest child (the one with the largest subtree) continues the current chain, while other children start new chains. This ensures chains are long and there aren't too many of them along any path.

🚂 Real-Life Analogy: Railway Express Lines

Think of a railway network shaped like a tree. The busiest routes (most passengers) become express lines — long, uninterrupted stretches. Less-traveled branch lines start their own segments. To travel between any two stations, you take at most O(log n) express lines, switching at junction points. HLD decomposes the tree the same way: heavy edges form express chains, and you traverse at most O(log n) chains between any two nodes.

🔧 Implementation

JavaScript — Heavy-Light Decomposition (Path Sum Query)
class HLD {
    constructor(n, edges, values, root = 0) {
        this.n = n;
        this.adj = Array.from({ length: n }, () => []);
        for (const [u, v] of edges) {
            this.adj[u].push(v);
            this.adj[v].push(u);
        }

        this.parent = new Array(n).fill(-1);
        this.depth = new Array(n).fill(0);
        this.subtreeSize = new Array(n).fill(1);
        this.heavy = new Array(n).fill(-1);    // Heavy child
        this.head = new Array(n).fill(0);       // Chain head
        this.pos = new Array(n).fill(0);        // Position in segment tree
        this.flat = new Array(n);               // Flattened values

        this.dfs(root);               // Compute sizes & heavy children
        this.decompose(root, root);   // Assign chains & positions
        
        // Map values to flattened order
        for (let i = 0; i < n; i++) {
            this.flat[this.pos[i]] = values[i];
        }
        this.seg = new SegTree(this.flat); // Segment tree on flattened array
    }

    dfs(root) {
        const stack = [[root, -1, false]];
        const order = [];
        while (stack.length) {
            const [u, par, processed] = stack.pop();
            if (processed) {
                let maxSize = 0, heavyChild = -1;
                for (const v of this.adj[u]) {
                    if (v === par) continue;
                    this.subtreeSize[u] += this.subtreeSize[v];
                    if (this.subtreeSize[v] > maxSize) {
                        maxSize = this.subtreeSize[v];
                        heavyChild = v;
                    }
                }
                this.heavy[u] = heavyChild;
                continue;
            }
            stack.push([u, par, true]);
            this.parent[u] = par;
            this.depth[u] = par === -1 ? 0 : this.depth[par] + 1;
            for (const v of this.adj[u]) {
                if (v !== par) stack.push([v, u, false]);
            }
        }
    }

    decompose(root, h) {
        let timer = 0;
        const stack = [[root, h]];
        while (stack.length) {
            const [u, head] = stack.pop();
            this.head[u] = head;
            this.pos[u] = timer++;
            // Process light children first (pushed last, processed first won't work)
            // We need heavy child processed first in DFS order
            const lightChildren = [];
            for (const v of this.adj[u]) {
                if (v !== this.parent[u] && v !== this.heavy[u]) {
                    lightChildren.push(v);
                }
            }
            // Push light children (they'll be processed after heavy subtree)
            for (const v of lightChildren.reverse()) {
                stack.push([v, v]); // Light children start new chains
            }
            // Push heavy child last (processed first — DFS order)
            if (this.heavy[u] !== -1) {
                stack.push([this.heavy[u], head]); // Continue chain
            }
        }
    }

    // Path query: sum of values on path from u to v
    pathQuery(u, v) {
        let result = 0;
        while (this.head[u] !== this.head[v]) {
            // Move the deeper chain head up
            if (this.depth[this.head[u]] < this.depth[this.head[v]]) {
                [u, v] = [v, u];
            }
            result += this.seg.query(this.pos[this.head[u]], this.pos[u]);
            u = this.parent[this.head[u]];
        }
        // Now u and v are in the same chain
        if (this.depth[u] > this.depth[v]) [u, v] = [v, u];
        result += this.seg.query(this.pos[u], this.pos[v]);
        return result;
    }

    // Point update: set value of node u
    pointUpdate(u, val) {
        this.seg.update(this.pos[u], val);
    }
}

// Minimal Segment Tree for sum queries
class SegTree {
    constructor(arr) {
        this.n = arr.length;
        this.tree = new Array(4 * this.n).fill(0);
        this._build(arr, 1, 0, this.n - 1);
    }
    _build(arr, nd, s, e) {
        if (s === e) { this.tree[nd] = arr[s]; return; }
        const m = (s + e) >> 1;
        this._build(arr, 2*nd, s, m);
        this._build(arr, 2*nd+1, m+1, e);
        this.tree[nd] = this.tree[2*nd] + this.tree[2*nd+1];
    }
    update(i, v, nd=1, s=0, e=this.n-1) {
        if (s===e) { this.tree[nd]=v; return; }
        const m=(s+e)>>1;
        if(i<=m) this.update(i,v,2*nd,s,m);
        else this.update(i,v,2*nd+1,m+1,e);
        this.tree[nd]=this.tree[2*nd]+this.tree[2*nd+1];
    }
    query(l, r, nd=1, s=0, e=this.n-1) {
        if(r>1;
        return this.query(l,r,2*nd,s,m)+this.query(l,r,2*nd+1,m+1,e);
    }
}

⚠️ Common Mistakes

❌ Heavy child must be visited first in DFS

During decomposition, the heavy child must get the next consecutive position in the segment tree. If light children are visited first, the chain won't be contiguous and segment tree queries break.

❌ Confusing node values vs edge values

If weights are on edges (not nodes), you assign each edge's weight to the child node. Then in path queries, exclude the LCA node (it doesn't correspond to any edge on the path).

❌ Moving the wrong chain up

In the path query loop, always move the node whose chain head is deeper. Moving the shallower one can cause infinite loops or skip nodes.

💼 Interview Perspective

  • HLD is an advanced competitive programming technique, rarely asked in standard interviews but important for ICPC-level contests.
  • If asked "how to do path queries on a tree efficiently," HLD + Segment Tree is the standard answer.
  • Alternative: Euler Tour + Segment Tree handles subtree queries (not path queries) more easily.
  • The O(log² n) can be reduced to O(log n) using Euler Tour + LCA for certain problems.
  • Core insight: any root-to-leaf path crosses at most O(log n) light edges → at most O(log n) chains.

📝 Quick Revision Notes

  • HLD decomposes tree into chains via heavy edges (to child with largest subtree).
  • Any root-to-leaf path crosses ≤ O(log n) chains.
  • Chains are contiguous in a flattened array → use Segment Tree for queries.
  • Path query: climb chains until same chain, querying segment tree at each step.
  • Time: O(log² n) per path query. Space: O(n).

🃏 Flash Cards

Why at most O(log n) chains on any root-to-leaf path?
Each time you cross a light edge, the subtree size at least halves (the heavy child has the larger subtree). Since the subtree starts at n and halves each time, you can halve at most log(n) times. So at most O(log n) light edges = O(log n) chain switches.
What data structure is used on top of HLD chains?
A Segment Tree built on the flattened chain order. Each chain occupies a contiguous range, so range queries/updates on a chain become segment tree operations. This gives O(log n) per chain operation, and O(log n) chains → O(log² n) total.
