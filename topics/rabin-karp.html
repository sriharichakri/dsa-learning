Rabin-Karp Algorithm

Phase 15 Hard

📖 What is the Rabin-Karp Algorithm?

The Rabin-Karp algorithm is a string pattern matching algorithm that uses hashing to find pattern occurrences. Instead of comparing characters one by one, it computes a hash of the pattern and a rolling hash of each window in the text. When hashes match, it verifies with a character-by-character comparison to handle collisions.

Average time is O(n + m), making it practical and especially powerful for multi-pattern matching — searching for multiple patterns simultaneously.

🏷️ Real-Life Analogy: Checking Price Tags

Imagine you're looking for a specific book in a library shelf. Instead of reading every title (naive approach), you first check if the book's spine color and thickness match (quick hash). Only when these "features" match do you actually read the title to confirm (character verification). Rabin-Karp does the same: use a quick hash to filter candidates, then verify only the promising ones.

⚙️ Rolling Hash Mechanism

Rolling Hash Example: text="abcda", pattern="bcd", base=31, mod=1e9+7
Hash formula: h(s) = s[0]×base^(m-1) + s[1]×base^(m-2) + ... + s[m-1] Pattern hash: h("bcd") = 2×31² + 3×31 + 4 = 1922 + 93 + 4 = 2019 Window 1: h("abc") = 1×31² + 2×31 + 3 = 961 + 62 + 3 = 1026 ✗ Window 2: h("bcd") = ? Use rolling: Remove 'a': 1026 - 1×31² = 65 Shift left: 65 × 31 = 2015 Add 'd': 2015 + 4 = 2019 ✓ Hash matches! Verify: "bcd" = "bcd" → MATCH at index 1 Window 3: h("cda") = ? Remove 'b': 2019 - 2×31² = 97 Shift left: 97 × 31 = 3007 Add 'a': 3007 + 1 = 3008 ✗ Rolling: O(1) per window instead of O(m)!

🔧 Implementation

JavaScript — Rabin-Karp Pattern Matching
function rabinKarp(text, pattern) {
    const n = text.length, m = pattern.length;
    if (m > n) return [];

    const BASE = 31;
    const MOD = 1_000_000_007;
    const results = [];

    // Compute base^(m-1) % MOD
    let basePow = 1;
    for (let i = 0; i < m - 1; i++) {
        basePow = (basePow * BASE) % MOD;
    }

    // Hash function: char -> (charCode - 96) for 'a'=1, 'b'=2, etc.
    const val = ch => ch.charCodeAt(0) - 96;

    // Compute pattern hash and first window hash
    let patHash = 0, winHash = 0;
    for (let i = 0; i < m; i++) {
        patHash = (patHash * BASE + val(pattern[i])) % MOD;
        winHash = (winHash * BASE + val(text[i])) % MOD;
    }

    for (let i = 0; i <= n - m; i++) {
        // Check if hashes match
        if (winHash === patHash) {
            // Verify character by character (avoid false positives)
            if (text.substring(i, i + m) === pattern) {
                results.push(i);
            }
        }

        // Roll the hash: remove text[i], add text[i + m]
        if (i < n - m) {
            winHash = (winHash - val(text[i]) * basePow % MOD + MOD) % MOD;
            winHash = (winHash * BASE + val(text[i + m])) % MOD;
        }
    }
    return results;
}

// Usage
console.log(rabinKarp("ababcababcab", "ababc"));
// [0, 5] — found at indices 0 and 5

console.log(rabinKarp("aaaaaaa", "aaa"));
// [0, 1, 2, 3, 4] — overlapping matches

⚠️ Common Mistakes

❌ Skipping character verification after hash match

Hash collisions can produce false positives. Always verify with actual string comparison when hashes match. This is critical for correctness.

❌ Negative modular arithmetic

When removing the leading character: (hash - char * basePow) can go negative. Always add MOD before taking modulus: (hash - char * basePow % MOD + MOD) % MOD.

❌ Using a small or easily exploitable modulus

Small modulus causes many collisions. Use a large prime like 1e9+7. For adversarial inputs, use double hashing (two different mod values) to virtually eliminate collisions.

💼 Interview Perspective

  • Rabin-Karp is ideal when asked to search for multiple patterns simultaneously — compute all pattern hashes upfront.
  • For single pattern matching, KMP is generally preferred (guaranteed O(n+m)). Rabin-Karp's worst case is O(nm).
  • The rolling hash concept is widely applicable: plagiarism detection, longest duplicate substring, string comparison.
  • Double hashing (two mods) makes collision probability negligible: ~1/10^18.
  • Key insight: rolling hash converts string comparison to integer comparison in O(1).

📝 Quick Revision Notes

  • Rabin-Karp: hash-based pattern matching, O(n+m) average.
  • Rolling hash: remove leading char, shift, add trailing char — O(1) per slide.
  • Always verify on hash match (handle collisions).
  • Excels at multi-pattern search — hash all patterns, check window against all.
  • Use large prime MOD (1e9+7) or double hash to minimize collisions.

🃏 Flash Cards

How does the rolling hash update work?
Remove the outgoing character's contribution: hash -= char[i] × base^(m-1). Shift: hash *= base. Add the incoming character: hash += char[i+m]. All modulo MOD. This converts an O(m) recomputation to O(1).
When is Rabin-Karp preferred over KMP?
Multi-pattern matching: searching for k patterns simultaneously. With Rabin-Karp, compute all k pattern hashes in a set, then slide one window and check if its hash is in the set. KMP would need k separate passes. Also preferred for 2D pattern matching.
