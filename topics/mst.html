Minimum Spanning Tree (MST)

Phase 10 Medium

📖 What is a Minimum Spanning Tree?

A Minimum Spanning Tree (MST) of a connected, undirected, weighted graph is a subset of edges that connects all vertices with the minimum total edge weight and no cycles. It has exactly V-1 edges for V vertices.

Two classic algorithms: Kruskal's (sort edges, add cheapest non-cycle-forming edge) and Prim's (grow tree from a starting vertex using cheapest available edge).

💡 Key Insight

Both algorithms are greedy and produce the same MST (or one of equal weight if ties exist). Kruskal's works globally (sort all edges), Prim's works locally (expand from current tree). Kruskal's uses Union-Find, Prim's uses a Priority Queue.

🌍 Real-Life Analogy

🏙️ Connecting Cities with Minimum Cable

Imagine you need to connect several cities with fiber optic cable. Each possible connection has a different cost. You want ALL cities connected using the least total cable. You don't need every possible connection — just enough to reach every city from every other city (a spanning tree), at minimum cost.

  • Road network: Build the cheapest road network connecting all towns.
  • Water pipeline: Connect all neighborhoods with minimum piping.
  • Electrical grid: Wire all buildings using least cable.

JavaScript — Kruskal's Algorithm with Union-Find
class UnionFind {
    constructor(n) {
        this.parent = Array.from({length: n}, (_, i) => i);
        this.rank = new Array(n).fill(0);
    }
    find(x) {
        if (this.parent[x] !== x) this.parent[x] = this.find(this.parent[x]);
        return this.parent[x];
    }
    union(x, y) {
        const px = this.find(x), py = this.find(y);
        if (px === py) return false; // Already connected
        if (this.rank[px] < this.rank[py]) this.parent[px] = py;
        else if (this.rank[px] > this.rank[py]) this.parent[py] = px;
        else { this.parent[py] = px; this.rank[px]++; }
        return true;
    }
}

function kruskal(n, edges) {
    // edges: [[u, v, weight], ...]
    edges.sort((a, b) => a[2] - b[2]); // Sort by weight
    const uf = new UnionFind(n);
    const mst = [];
    let totalWeight = 0;

    for (const [u, v, w] of edges) {
        if (uf.union(u, v)) { // No cycle? Add to MST
            mst.push([u, v, w]);
            totalWeight += w;
            if (mst.length === n - 1) break; // MST complete
        }
    }

    return { mst, totalWeight };
}

🔧 Prim's Algorithm

JavaScript — Prim's Algorithm
function prims(n, edges) {
    // Build adjacency list
    const adj = Array.from({length: n}, () => []);
    for (const [u, v, w] of edges) {
        adj[u].push([v, w]);
        adj[v].push([u, w]);
    }

    const inMST = new Array(n).fill(false);
    // Min-heap: [weight, node]
    const pq = [[0, 0]]; // Start from node 0
    let totalWeight = 0;
    let edgesAdded = 0;

    while (pq.length > 0 && edgesAdded < n) {
        pq.sort((a, b) => a[0] - b[0]); // Simple sort (use real heap for efficiency)
        const [w, u] = pq.shift();

        if (inMST[u]) continue;
        inMST[u] = true;
        totalWeight += w;
        edgesAdded++;

        for (const [v, weight] of adj[u]) {
            if (!inMST[v]) {
                pq.push([weight, v]);
            }
        }
    }

    return totalWeight;
}

// Min Cost to Connect All Points
function minCostConnectPoints(points) {
    const n = points.length;
    const edges = [];
    for (let i = 0; i < n; i++) {
        for (let j = i + 1; j < n; j++) {
            const dist = Math.abs(points[i][0]-points[j][0]) + Math.abs(points[i][1]-points[j][1]);
            edges.push([i, j, dist]);
        }
    }
    return kruskal(n, edges).totalWeight;
}

⚠️ Common Mistakes

❌ Forgetting to check for cycles in Kruskal's

Without Union-Find cycle detection, you'll add edges that create cycles, resulting in a graph that isn't a tree.

❌ Not marking visited nodes in Prim's

Without the inMST check, you'll process the same node multiple times and get incorrect total weight.

❌ Confusing MST with shortest path

MST minimizes TOTAL edge weight. Shortest path minimizes the path between two specific nodes. They solve different problems!

💼 Interview Perspective

  • "Min Cost to Connect All Points" (LC 1584) is the most common MST interview problem.
  • Know both algorithms: Kruskal's is easier to code, Prim's is better for dense graphs.
  • Kruskal's needs Union-Find — make sure you know path compression and union by rank.
  • MST has exactly V-1 edges. If you can't add V-1 edges, the graph is disconnected.
  • MST edge weight is unique if all edge weights are distinct.

📝 Quick Revision Notes

  • MST: connects all vertices with minimum total edge weight, no cycles, V-1 edges.
  • Kruskal's: sort edges + Union-Find. O(E log E). Good for sparse graphs.
  • Prim's: grow tree with min-heap. O(E log V). Good for dense graphs.
  • Both are greedy algorithms and produce optimal results.
  • MST ≠ shortest path. MST minimizes total weight, not individual paths.

🃏 Flash Cards

What's the difference between Kruskal's and Prim's algorithms?
Kruskal's: Global approach — sort ALL edges, greedily add cheapest non-cycle edge. Uses Union-Find. Better for sparse graphs (E ≈ V).
Prim's: Local approach — grow MST from one vertex, always add cheapest edge connecting tree to non-tree vertex. Uses min-heap. Better for dense graphs (E ≈ V²).
How many edges does an MST have?
Exactly V - 1 edges, where V is the number of vertices. This is because a tree with V nodes always has V-1 edges (adding one more would create a cycle, removing one would disconnect).
