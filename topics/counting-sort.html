Counting Sort

Phase 4 Medium

📖 What is Counting Sort?

Counting Sort is a non-comparison sort that works by counting occurrences of each value, then placing elements in the correct position based on those counts. It sorts in O(n + k) time where k is the range of values.

It doesn't compare elements at all — it breaks the O(n log n) lower bound of comparison-based sorts!

🌍 Real-Life Analogy

Imagine sorting exam papers by grade (A, B, C, D, F). You don't compare papers against each other. Instead, you set up 5 piles labeled A through F. Walk through all papers, placing each in its pile. Then collect piles in order: all F's, then D's, C's, B's, A's. Done! You counted where each goes without comparing.

⚙️ How It Works

JavaScript
function countingSort(arr) {
    if (arr.length === 0) return arr;
    
    const max = Math.max(...arr);
    const min = Math.min(...arr);
    const range = max - min + 1;
    
    // Step 1: Count occurrences
    const count = new Array(range).fill(0);
    for (let num of arr) {
        count[num - min]++;
    }
    
    // Step 2: Build sorted output
    let idx = 0;
    for (let i = 0; i < range; i++) {
        while (count[i] > 0) {
            arr[idx++] = i + min;
            count[i]--;
        }
    }
    return arr;
}

// countingSort([4, 2, 2, 8, 3, 3, 1]) → [1, 2, 2, 3, 3, 4, 8]

Stable Version (preserves order of equal elements)

JavaScript
function countingSortStable(arr) {
    const max = Math.max(...arr);
    const min = Math.min(...arr);
    const range = max - min + 1;
    
    const count = new Array(range).fill(0);
    const output = new Array(arr.length);
    
    // Count occurrences
    for (let num of arr) count[num - min]++;
    
    // Cumulative count (prefix sum)
    for (let i = 1; i < range; i++) count[i] += count[i - 1];
    
    // Build output (traverse RIGHT to LEFT for stability)
    for (let i = arr.length - 1; i >= 0; i--) {
        output[count[arr[i] - min] - 1] = arr[i];
        count[arr[i] - min]--;
    }
    return output;
}

💡 When is counting sort efficient?

When k (range) is O(n) or smaller. If values range from 0 to 1,000,000 but you only have 10 elements, counting sort wastes memory. Use it for small-range integers like ages, grades, or ASCII characters.

⚠️ Common Mistakes

❌ Not handling negative numbers

If the array has negatives, shift all values by -min to make them non-negative. The range is max - min + 1, not just max + 1.

❌ Using counting sort on large ranges

Array of [1, 1000000000] — the count array would need a billion entries! Only use counting sort when the range k is reasonable (k ≈ O(n)).

💼 Interview Perspective

  • Counting sort is O(n+k) — faster than O(n log n) comparison sorts when k is small.
  • It's a non-comparison sort — doesn't use <, >, or comparisons at all.
  • Used as a subroutine in radix sort.
  • Can only sort integers (or items that map to integers). Not for floating-point or strings directly.
  • Stability matters when counting sort is used inside radix sort.

📝 Quick Revision Notes

  • Count occurrences → write back in order. No comparisons needed.
  • Time: O(n + k). Space: O(n + k). Stable (with proper implementation).
  • Best for small-range integers (ages, grades, ASCII chars).
  • Not suitable when range k ≫ n.

🃏 Flash Cards

What's the time complexity of counting sort and when does it beat O(n log n)?
O(n + k) where k is the range of values. It beats O(n log n) when k = O(n) or smaller. For example, sorting ages (k=150) of a million people: O(n) vs O(n log n).
Why is counting sort called a "non-comparison" sort?
It never compares two elements against each other. It works by counting how many times each value appears, then placing them in order. This breaks the Ω(n log n) lower bound of comparison sorts.
