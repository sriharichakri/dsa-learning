Functions

Phase 1 Beginner Friendly

📖 What Are Functions?

A function is a reusable block of code that performs a specific task. You give it input (parameters), it does some work, and gives you back output (return value).

Functions are the building blocks of organized, readable, and maintainable code. Without them, your programs would be long, repetitive, and impossible to debug.

💡 Key Insight

In DSA, every algorithm you write will be a function. Sorting? A function. Searching? A function. Every LeetCode problem? You're writing a function. Master how functions work — parameters, return values, scope — and everything else becomes easier.

🌍 Real-Life Analogy

🥤 Vending Machine

A vending machine is a perfect function: you insert coins (input/parameters), press a button (call the function), and it dispenses a drink (return value). You don't need to know how the machine works internally — you just use the interface. That's abstraction!

  • Recipe: A recipe is a function — it takes ingredients (parameters) and produces a dish (return value). You can follow the same recipe many times.
  • Calculator: Press the √ button with input 16 → get output 4. The sqrt() function.
  • ATM: Insert card + PIN (inputs) → get cash (output). The withdraw() function.
  • Google Translate: Input text in English → output text in Spanish. The translate() function.

⚙️ Function Syntax

JavaScript
// Function Declaration
function greet(name) {         // "name" is a PARAMETER
    return "Hello, " + name;   // Return value
}
greet("Alice");  // "Alice" is an ARGUMENT → returns "Hello, Alice"

// Function Expression
const add = function(a, b) {
    return a + b;
};

// Arrow Function (modern JS)
const multiply = (a, b) => a * b;

// Default Parameters
function power(base, exp = 2) {
    return Math.pow(base, exp);
}
power(3);     // 9  (exp defaults to 2)
power(3, 3);  // 27

📋 Parameters vs Arguments

These terms are often confused but have distinct meanings:

  • Parameters: Variables listed in the function definition. They're placeholders.
  • Arguments: Actual values passed when you call the function.
JavaScript
function greet(name, age) {  // name, age = PARAMETERS
    return `Hi ${name}, you're ${age}!`;
}

greet("Bob", 25);  // "Bob", 25 = ARGUMENTS

🔒 Scope: Where Variables Live

Scope determines where a variable can be accessed. This is critical for avoiding bugs.

JavaScript
let globalVar = "I'm global";  // Accessible everywhere

function myFunc() {
    let localVar = "I'm local";  // Only inside myFunc
    console.log(globalVar);  // ✅ Can access global
    console.log(localVar);   // ✅ Can access local
}

console.log(globalVar);  // ✅ Works
// console.log(localVar); // ❌ ReferenceError! Not accessible
┌─────────────────────────────────┐ │ GLOBAL SCOPE │ │ globalVar = "I'm global" │ │ │ │ ┌───────────────────────────┐ │ │ │ FUNCTION SCOPE (myFunc) │ │ │ │ localVar = "I'm local" │ │ │ │ Can see: globalVar ✅ │ │ │ └───────────────────────────┘ │ │ │ │ Can see localVar? ❌ │ └─────────────────────────────────┘
⚡ Closure: Functions Remember Their Scope
function outer() {
    let count = 0;
    return function inner() {
        count++;
        return count;
    };
}
const counter = outer();
counter(); // 1
counter(); // 2 — remembers count!

✨ Pure Functions

A pure function has two properties:

  1. Same input always produces the same output
  2. No side effects (doesn't modify external state)
JavaScript
// ✅ Pure function — same input → same output, no side effects
function add(a, b) {
    return a + b;
}

// ❌ Impure — modifies external state
let total = 0;
function addToTotal(n) {
    total += n;  // Side effect: changes external variable
    return total;
}

// ❌ Impure — different outputs for same input
function getRandom(max) {
    return Math.floor(Math.random() * max);
}
🎯 Why Pure Functions Matter

Pure functions are easier to test, debug, and reason about. In DSA problems, your solution function should almost always be pure — take input, return output, don't touch anything else.

🔍 Dry Run: Function Call Execution

Let's trace how a function call works internally.

JavaScript
function square(n) {
    return n * n;
}

function sumOfSquares(a, b) {
    let sq1 = square(a);
    let sq2 = square(b);
    return sq1 + sq2;
}

let result = sumOfSquares(3, 4);
1
Call sumOfSquares(3, 4) → a=3, b=4. Push frame onto call stack.
Stack: [sumOfSquares]
2
Inside: call square(3) → n=3, returns 3*3=9. Pop frame.
sq1 = 9, Stack: [sumOfSquares]
3
Inside: call square(4) → n=4, returns 4*4=16. Pop frame.
sq2 = 16, Stack: [sumOfSquares]
4
Return sq1 + sq2 = 9 + 16 = 25. Pop sumOfSquares frame.
result = 25 ✅

💡 Passing an Array is O(1)

When you pass an array to a function, only the reference (address) is copied, not the entire array. So even a million-element array is passed in O(1). But if you copy the array inside the function, that's O(n).

⚠️ Common Mistakes

❌ Forgetting to return a value
// ❌ No return statement — returns undefined!
function add(a, b) {
    let sum = a + b;
    // forgot: return sum;
}
let result = add(3, 5); // undefined!

// ✅ Always return the result
function add(a, b) {
    return a + b;
}
❌ Modifying input array unintentionally
// ❌ Mutates the original array!
function sortArray(arr) {
    return arr.sort(); // .sort() mutates in-place
}
let original = [3, 1, 2];
sortArray(original);
console.log(original); // [1, 2, 3] — changed!

// ✅ Create a copy first
function sortArray(arr) {
    return [...arr].sort();
}
❌ Confusing function reference with function call
function greet() { return "Hi"; }

let a = greet;   // a holds the FUNCTION itself
let b = greet(); // b holds "Hi" (the RETURN VALUE)

💼 Interview Perspective

  • Every LeetCode solution is a function — know how to define, call, and test them quickly.
  • Understand pass by value vs pass by reference — interviewers test this with tricky problems.
  • Know the concept of callback functions and higher-order functions (functions that take/return functions).
  • Be able to explain closures in one sentence: "A function that remembers variables from its outer scope."
  • Helper functions are your friend — break complex problems into smaller functions during interviews.

📝 Quick Revision Notes

  • Function = reusable code block. Takes parameters, returns a value.
  • Parameters = placeholders in definition. Arguments = actual values in call.
  • Scope = where variables are accessible. Inner scope can see outer, not vice versa.
  • Pure functions: same input → same output, no side effects.
  • Arrays/objects are passed by reference (shared). Primitives are passed by value (copied).

🃏 Flash Cards

What is the difference between parameters and arguments?
Parameters are variables in the function definition (placeholders). Arguments are the actual values passed when calling the function.
What is a pure function?
A function that (1) always returns the same output for the same input, and (2) has no side effects (doesn't modify external state). Example: add(a, b) → a + b
What is a closure?
A function that "remembers" variables from its enclosing scope, even after the outer function has returned. Used for data privacy, counters, and callbacks.
