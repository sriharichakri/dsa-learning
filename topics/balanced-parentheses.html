Balanced Parentheses

Phase 7 Easy

📖 What is the Balanced Parentheses Problem?

Given a string containing brackets — (), {}, [] — determine if every opening bracket has a matching closing bracket in the correct order. This is the quintessential stack problem!

🌍 Real-Life Analogy

Think of nesting Russian dolls (matryoshka). Each doll you open must be closed before the doll that contains it. If you open a big doll, then a medium, then a small, you must close the small first, then medium, then big. Opening in wrong order or leaving one unclosed = imbalanced!

⚙️ How It Works

JavaScript
function isValid(s) {
    const stack = [];
    const map = { ')': '(', '}': '{', ']': '[' };
    
    for (let ch of s) {
        if (ch === '(' || ch === '{' || ch === '[') {
            stack.push(ch);  // Opening → push
        } else {
            // Closing → check if matches top of stack
            if (stack.length === 0 || stack[stack.length - 1] !== map[ch]) {
                return false;
            }
            stack.pop();
        }
    }
    return stack.length === 0;  // Stack must be empty!
}

// isValid("()")       → true
// isValid("()[]{}")   → true
// isValid("(]")       → false
// isValid("([)]")     → false
// isValid("{[]}")     → true

Generate All Valid Parentheses

JavaScript
function generateParenthesis(n) {
    const result = [];
    
    function backtrack(current, open, close) {
        if (current.length === 2 * n) {
            result.push(current);
            return;
        }
        if (open < n) backtrack(current + '(', open + 1, close);
        if (close < open) backtrack(current + ')', open, close + 1);
    }
    
    backtrack('', 0, 0);
    return result;
}

// generateParenthesis(3) →
// ["((()))","(()())","(())()","()(())","()()()"]

⚠️ Common Mistakes

❌ Not checking stack empty at the end
// ❌ "(((" has no mismatches but stack is not empty!
return true; // Wrong — unclosed brackets remain

// ✅ Must verify stack is empty
return stack.length === 0;
❌ Not checking stack empty before popping

If you encounter a closing bracket and the stack is empty, it's invalid. Don't try to pop from an empty stack — check stack.length === 0 first.

💼 Interview Perspective

  • This is the classic stack problem — often the first stack question in interviews.
  • Use a hash map for bracket matching: { ')': '(', ']': '[', '}': '{' }.
  • "Generate Parentheses" is a classic backtracking problem — know both validation and generation.
  • Variations: minimum removals to make valid, longest valid parentheses substring.
  • For only '(' and ')': can solve with a counter instead of a stack (O(1) space).

📝 Quick Revision Notes

  • Opening bracket → push. Closing bracket → check top matches, pop.
  • Return true only if stack is empty at the end.
  • Use hash map for O(1) bracket matching.
  • Time: O(n). Space: O(n) for the stack.

🃏 Flash Cards

Why is a stack the perfect data structure for bracket matching?
Brackets follow LIFO order — the most recently opened bracket must be closed first. This exactly matches stack behavior: push on open, pop on close. If the popped bracket matches, it's valid.
Can you check balanced parentheses without a stack?
Only for single type brackets like '(' and ')'. Use a counter: increment on '(', decrement on ')'. If counter goes negative or isn't 0 at the end → invalid. For multiple types, you NEED a stack.
