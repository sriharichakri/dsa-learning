Greedy Fundamentals

Phase 11 Medium

📖 What is a Greedy Algorithm?

A greedy algorithm makes the locally optimal choice at each step, hoping to find the globally optimal solution. It never looks back or reconsiders previous choices.

Greedy algorithms don't always work — they only work when the problem has the greedy choice property (local optimum leads to global optimum) and optimal substructure (optimal solution contains optimal solutions to subproblems).

💡 Key Insight

The hardest part of greedy isn't coding — it's proving the greedy choice is correct. If you can argue that the greedy choice is never worse than any alternative, the algorithm works. If not, you need DP or another approach.

🌍 Real-Life Analogy

🪙 Making Change with Coins

You need to give $0.67 in change using the fewest coins. Your strategy: always pick the largest denomination that fits. $0.67 → 25¢ + 25¢ + 10¢ + 5¢ + 1¢ + 1¢ = 6 coins. This greedy approach works for standard US denominations! (But fails for some arbitrary coin systems.)

  • Eating at a buffet: Always grab the dish you want most next (might miss a better combination!)
  • Scheduling meetings: Always pick the meeting that ends earliest to fit the most meetings
  • Filling a truck: Load the lightest items first to fit the most items (not always optimal for value)
  • Driving route: Always take the next nearest gas station (works for fuel, not always for shortest path)

💡 The Two Properties

1. Greedy Choice Property: A locally optimal choice leads to a globally optimal solution.
2. Optimal Substructure: The optimal solution to the problem contains optimal solutions to its subproblems.

⚙️ Classic Greedy Problems

1. Activity Selection (Interval Scheduling)

JavaScript — Maximum Non-overlapping Intervals
// Pick the maximum number of non-overlapping activities
function activitySelection(activities) {
    // Sort by end time (GREEDY: pick earliest ending first)
    activities.sort((a, b) => a[1] - b[1]);
    
    const selected = [activities[0]];
    let lastEnd = activities[0][1];
    
    for (let i = 1; i < activities.length; i++) {
        if (activities[i][0] >= lastEnd) {  // No overlap
            selected.push(activities[i]);
            lastEnd = activities[i][1];
        }
    }
    return selected;
}
// activities = [[1,3], [2,5], [3,6], [5,7], [8,9]]
// Selected: [1,3], [5,7], [8,9] → 3 activities

2. Jump Game

JavaScript — Can you reach the last index?
function canJump(nums) {
    let maxReach = 0;
    
    for (let i = 0; i < nums.length; i++) {
        if (i > maxReach) return false;  // Can't reach this index
        maxReach = Math.max(maxReach, i + nums[i]);
    }
    return true;
}
// [2,3,1,1,4] → true (2→3→4 or 2→1→1→4)
// [3,2,1,0,4] → false (stuck at index 3)

3. Gas Station

JavaScript — Circular Route Gas Station
function canCompleteCircuit(gas, cost) {
    let totalSurplus = 0;
    let currentSurplus = 0;
    let start = 0;
    
    for (let i = 0; i < gas.length; i++) {
        totalSurplus += gas[i] - cost[i];
        currentSurplus += gas[i] - cost[i];
        
        if (currentSurplus < 0) {
            start = i + 1;      // Can't start from any station before i+1
            currentSurplus = 0;
        }
    }
    
    return totalSurplus >= 0 ? start : -1;
}

🔍 Dry Run: Jump Game II — Minimum Jumps

JavaScript
function jump(nums) {
    let jumps = 0, currentEnd = 0, farthest = 0;
    
    for (let i = 0; i < nums.length - 1; i++) {
        farthest = Math.max(farthest, i + nums[i]);
        if (i === currentEnd) {
            jumps++;
            currentEnd = farthest;
        }
    }
    return jumps;
}

Array: [2, 3, 1, 1, 4]

jumps=1, currentEnd=2, farthest=2
2

jumps=1, currentEnd=2, farthest=4
3

jumps=2, currentEnd=4, farthest=4
4
i=3: farthest = max(4, 3+1) = 4. i ≠ currentEnd → continue. Loop ends (i < n-1)
Answer: 2 jumps ✅ (index 0 → 1 → 4)

⚠️ Common Mistakes

❌ Assuming Greedy Always Works

Greedy fails for many problems (0/1 knapsack, coin change with arbitrary coins). Always verify the greedy choice property before using it!

❌ Wrong Sorting Order

Activity selection needs sorting by end time, not start time. Fractional knapsack sorts by value/weight ratio. The sorting criterion is everything!

❌ Confusing Greedy with DP

If a greedy solution doesn't work, the problem likely needs DP. Greedy makes ONE choice per step and never revisits. DP considers ALL choices and picks the best.

❌ Not Proving Correctness

In interviews, just saying "use greedy" isn't enough. You should explain why the greedy choice is correct, even if briefly.

💼 Interview Tips

  • Pattern recognition: "Minimum/maximum number of..." often suggests greedy or DP. Try greedy first — it's simpler.
  • Sort first: Many greedy problems start with sorting by some criterion.
  • Exchange argument: To prove greedy works, show that swapping a greedy choice for a non-greedy one never improves the solution.
  • Greedy vs DP: If greedy fails on a counterexample, switch to DP. Many problems have both solutions.
  • Interval problems: Sort by end time for maximum non-overlapping, by start time for merging.

🏗️ Where Greedy Is Used

  • Huffman coding: Data compression (ZIP, JPEG) uses greedy to build optimal prefix codes
  • Dijkstra's algorithm: Greedy choice of nearest unvisited node
  • Kruskal's/Prim's MST: Greedy edge selection for minimum spanning tree
  • CPU scheduling: Shortest Job First algorithm is greedy
  • Coin dispensers: ATMs use greedy to dispense fewest bills

📝 Quick Revision Notes

  • Greedy = make the best local choice at each step, never look back
  • Works only with greedy choice property + optimal substructure
  • Activity selection: sort by end time, pick non-overlapping
  • Jump Game: track max reachable index in one pass
  • If greedy fails (counterexample exists), switch to DP

🃏 Flash Cards

What two properties must a problem have for greedy to work?
1) Greedy Choice Property: A locally optimal choice leads to a globally optimal solution. 2) Optimal Substructure: The optimal solution contains optimal solutions to subproblems.
Why does greedy fail for 0/1 Knapsack?
Greedy picks items with best value/weight ratio, but you can't take fractions. A less "efficient" item might fit the remaining capacity better. Example: items (60, 10kg), (100, 20kg), (120, 30kg), capacity 50kg. Greedy misses optimal combo.
How does the "exchange argument" prove greedy correctness?
Assume an optimal solution that differs from the greedy solution. Show that swapping the differing choice to the greedy choice produces a solution that is at least as good. This proves greedy is optimal.
