Bubble Sort

Phase 4 Easy

📖 What is Bubble Sort?

Bubble Sort is the simplest sorting algorithm. It works by repeatedly stepping through the list, comparing adjacent elements, and swapping them if they're in the wrong order. This process repeats until the list is sorted.

After each full pass through the array, the largest unsorted element "bubbles up" to its correct position at the end — just like air bubbles rising to the surface of water!

💡 Key Insight

Bubble Sort is rarely used in production (it's O(n²)), but it's essential to understand because it teaches the concept of pairwise comparisons and the idea of "sorted suffixes" — concepts that appear in many advanced algorithms.

🌍 Real-Life Analogy

🫧 Bubbles Rising in Water

Drop a handful of differently-sized bubbles into a glass of water. The largest bubbles rise to the top first, followed by smaller ones. Each bubble "competes" with its neighbor and the bigger one floats up. That's exactly how Bubble Sort works — larger elements bubble up to the end of the array!

More real-world examples:

  • Sorting students by height: Compare adjacent pairs and swap if the taller one is in front. Repeat until sorted.
  • Organizing playing cards: Sweep left to right, swapping out-of-order neighbors.
  • Ranking race times: Compare consecutive entries and push slower times down the list.

⚙️ How Bubble Sort Works

The algorithm makes multiple passes through the array. In each pass, it compares every pair of adjacent elements and swaps them if the left is greater than the right.

Bubble Sort: Sorting [5, 3, 8, 1, 2]
Pass 1: Largest element (8) bubbles to the end [5, 3, 8, 1, 2] Compare 5,3 → Swap! [3, 5, 8, 1, 2] Compare 5,8 → OK [3, 5, 8, 1, 2] Compare 8,1 → Swap! [3, 5, 1, 8, 2] Compare 8,2 → Swap! [3, 5, 1, 2, 8] ✅ 8 is in place! Pass 2: Next largest (5) bubbles up [3, 5, 1, 2, 8] Compare 3,5 → OK [3, 5, 1, 2, 8] Compare 5,1 → Swap! [3, 1, 5, 2, 8] Compare 5,2 → Swap! [3, 1, 2, 5, 8] ✅ 5 is in place! Pass 3: Next largest (3) bubbles up [3, 1, 2, 5, 8] Compare 3,1 → Swap! [1, 3, 2, 5, 8] Compare 3,2 → Swap! [1, 2, 3, 5, 8] ✅ 3 is in place! Pass 4: Check remaining [1, 2, 3, 5, 8] Compare 1,2 → OK. No swaps! ✅ Array is fully sorted!
JavaScript — Basic Bubble Sort
function bubbleSort(arr) {
    let n = arr.length;
    for (let i = 0; i < n - 1; i++) {
        for (let j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                // Swap adjacent elements
                [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
            }
        }
    }
    return arr;
}
💡 Why n - 1 - i?

After each pass i, the last i elements are already sorted. So we don't need to compare them again. This is the "sorted suffix" property.

⚡ Optimized Bubble Sort (Early Termination)

If during a complete pass we make no swaps, the array is already sorted — we can stop early!

JavaScript — Optimized with Flag
function bubbleSortOptimized(arr) {
    let n = arr.length;
    for (let i = 0; i < n - 1; i++) {
        let swapped = false;  // Flag to detect if any swap happened
        
        for (let j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
                swapped = true;
            }
        }
        
        if (!swapped) break;  // No swaps = already sorted!
    }
    return arr;
}

// Best case: [1, 2, 3, 4, 5] → Only 1 pass, O(n)!
🎯 When Does Optimization Help?

The optimized version makes Bubble Sort O(n) for already sorted arrays. It also helps for nearly sorted arrays, terminating early once everything is in place.

AverageO(n²)Roughly n²/2 comparisons on average
Worst (reverse sorted)O(n²)Every pair needs swapping: n(n-1)/2 swaps
SpaceO(1)In-place — only uses a temp variable for swapping
💡 Stability

Bubble Sort is a stable sort — equal elements maintain their relative order. This matters when sorting objects by multiple criteria.

⚠️ Common Mistakes

❌ Forgetting the Optimization Flag

Without the swapped flag, Bubble Sort always does n-1 passes, even for a sorted array. This turns best case from O(n) to O(n²).

❌ Wrong Inner Loop Bound
// ❌ WRONG: Goes out of bounds
for (let j = 0; j < n; j++) { arr[j] vs arr[j+1] }

// ✅ RIGHT: Stop at n - 1 - i
for (let j = 0; j < n - 1 - i; j++) { ... }
❌ Using Bubble Sort for Large Data

Bubble Sort is O(n²). For n = 10,000 that's 100 million operations. Use Merge Sort or Quick Sort for large datasets.

❌ Confusing Bubble Sort with Selection Sort

Bubble Sort swaps adjacent elements repeatedly. Selection Sort finds the minimum and places it at the front. Different mechanisms!

💼 Interview Tips

  • Know when NOT to use it: Interviewers may ask you to sort — don't use Bubble Sort! Use it only if asked specifically.
  • Explain the optimization: Always mention the swapped flag. It shows you understand best-case improvements.
  • Compare with other sorts: Be ready to explain why Merge Sort (O(n log n)) and Quick Sort are preferred.
  • Stable sort: If asked about stability, Bubble Sort preserves order of equal elements.
  • In-place: Bubble Sort uses O(1) extra space — good when memory is constrained.

🏗️ Where Bubble Sort Appears

  • Teaching: First sorting algorithm taught in CS courses worldwide
  • Nearly sorted data: Optimized Bubble Sort is efficient for data that's almost sorted (like a leaderboard with one position change)
  • Embedded systems: Tiny memory footprint, simple to implement in hardware
  • Detection of sorted arrays: One pass with no swaps = sorted (useful as a check)

📝 Quick Revision Notes

  • Bubble Sort = compare & swap adjacent elements, largest bubbles to end each pass
  • Worst/Average: O(n²) | Best (optimized): O(n) | Space: O(1)
  • Optimized version uses a swapped flag — stops early if no swaps in a pass
  • Stable sort — equal elements keep their original relative order
  • After pass i, the last i+1 elements are in their final positions

🃏 Flash Cards

What is the best case time complexity of optimized Bubble Sort?
O(n) — when the array is already sorted. The swapped flag detects no swaps in the first pass and terminates.
Is Bubble Sort stable? What does that mean?
Yes, it's stable. Equal elements maintain their original relative order. It only swaps when left > right, not when left == right.
After pass i in Bubble Sort, what is guaranteed?
The last (i+1) elements are in their final sorted positions. The largest i+1 elements have "bubbled up" to the end.
How does the swapped flag optimization work?
Set swapped = false before each pass. Set to true on any swap. If still false after a complete pass, the array is sorted — break out of the loop.
