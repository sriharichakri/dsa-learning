Reverse Linked List

Phase 6 Medium

üìñ What is Reversing a Linked List?

Reversing a linked list means making every node point to its previous node instead of its next, effectively flipping the direction of all pointers. The old tail becomes the new head.

üåç Real-Life Analogy

Imagine a conga line where each person has their hands on the shoulders of the person in front. To reverse it, each person turns around and puts their hands on the person who was behind them. The last person in the old line becomes the leader of the new line!

‚öôÔ∏è How It Works

Iterative Approach (3 pointers)

JavaScript
function reverseList(head) {
    let prev = null;
    let curr = head;
    
    while (curr) {
        let next = curr.next;  // Save next
        curr.next = prev;      // Reverse pointer
        prev = curr;           // Advance prev
        curr = next;           // Advance curr
    }
    return prev;  // New head
}

// 1‚Üí2‚Üí3‚Üí4‚Üí5 becomes 5‚Üí4‚Üí3‚Üí2‚Üí1

Recursive Approach

JavaScript
function reverseListRecursive(head) {
    if (!head || !head.next) return head;
    
    const newHead = reverseListRecursive(head.next);
    head.next.next = head;  // Make next node point back to us
    head.next = null;       // Remove old forward pointer
    return newHead;
}

Reverse a Sublist (between positions left and right)

JavaScript
function reverseBetween(head, left, right) {
    const dummy = { val: 0, next: head };
    let prev = dummy;
    
    // Move to node before left
    for (let i = 1; i < left; i++) prev = prev.next;
    
    // Reverse from left to right
    let curr = prev.next;
    for (let i = 0; i < right - left; i++) {
        let next = curr.next;
        curr.next = next.next;
        next.next = prev.next;
        prev.next = next;
    }
    return dummy.next;
}

‚öÝÔ∏è Common Mistakes

‚ùå Not saving the next pointer before reversing
// ‚ùå Loses the rest of the list!
curr.next = prev;    // Now curr.next points backward
curr = curr.next;    // Goes backward, not forward!

// ‚úÖ Save next BEFORE reversing
let next = curr.next;  // Save forward reference
curr.next = prev;       // Reverse
curr = next;            // Move forward using saved reference
‚ùå Returning head instead of prev

After reversal, head is now the last node (pointing to null). The new head is prev, which is the old tail. Return prev!

üíº Interview Perspective

  • This is arguably the most frequently asked linked list question.
  • Master the iterative version ‚Äî O(1) space, no stack overflow risk.
  • The pattern: save next ‚Üí reverse pointer ‚Üí advance prev ‚Üí advance curr.
  • Reverse sublist (LC 92) is a common follow-up ‚Äî uses the same core logic.
  • "Reverse in groups of K" (LC 25) is the hard variant ‚Äî reverses K nodes at a time.

üìù Quick Revision Notes

  • 3 pointers: prev, curr, next. Pattern: save ‚Üí reverse ‚Üí advance √ó 2.
  • Iterative: O(n) time, O(1) space. Recursive: O(n) time, O(n) space.
  • Return prev (not head!) ‚Äî prev becomes the new head.
  • Always save curr.next BEFORE reversing curr.next = prev.

üÉè Flash Cards

What are the 4 steps in each iteration of iterative linked list reversal?
1. next = curr.next (save next node)
2. curr.next = prev (reverse the pointer)
3. prev = curr (advance prev)
4. curr = next (advance curr)
Why return prev instead of head after reversal?
After reversal, the original head is now the tail (its next is null). The variable prev points to the last node processed, which is the old tail ‚Äî now the new head of the reversed list.
