Binary Tree

Phase 9 Medium

📖 What is a Binary Tree?

A Binary Tree is a hierarchical data structure where each node has at most two children, called the left child and right child. It's called "binary" because each node branches into at most 2 paths.

Trees are non-linear — unlike arrays and linked lists, data branches out, allowing for efficient searching, sorting, and hierarchical representation.

💡 Key Insight

Trees are the most recursion-friendly data structure. Almost every tree problem has an elegant recursive solution: solve for the left subtree, solve for the right subtree, combine the results. Once you internalize this pattern, tree problems become formulaic.

🌍 Real-Life Analogy

👨‍👩‍👧‍👦 Family Tree

A family tree is the perfect analogy. One person (the root) has children, who have their own children (grandchildren), and so on. People with no children are leaves. The number of generations is the height. You can trace lineage from root to any leaf — that's a path.

  • Company org chart: CEO (root) → VPs → Directors → Managers → Employees (leaves).
  • File system: Root folder contains subfolders, which contain files (leaves).
  • Tournament bracket: Each match has two participants — winner advances up.
  • Decision tree: Each node is a yes/no question, branches lead to outcomes.

🚶 Tree Traversals

Tree: 1 / \ 2 3 / \ 4 5 Inorder (Left, Root, Right): 4, 2, 5, 1, 3 Preorder (Root, Left, Right): 1, 2, 4, 5, 3 Postorder (Left, Right, Root): 4, 5, 2, 3, 1 Level Order (BFS): 1, 2, 3, 4, 5
JavaScript
// Inorder: Left → Root → Right
function inorder(node, result = []) {
    if (!node) return result;
    inorder(node.left, result);
    result.push(node.val);      // Process root
    inorder(node.right, result);
    return result;
}

// Preorder: Root → Left → Right
function preorder(node, result = []) {
    if (!node) return result;
    result.push(node.val);       // Process root first
    preorder(node.left, result);
    preorder(node.right, result);
    return result;
}

// Level Order (BFS) — uses Queue
function levelOrder(root) {
    if (!root) return [];
    const queue = [root];
    const result = [];
    
    while (queue.length > 0) {
        const levelSize = queue.length;
        const level = [];
        for (let i = 0; i < levelSize; i++) {
            const node = queue.shift();
            level.push(node.val);
            if (node.left) queue.push(node.left);
            if (node.right) queue.push(node.right);
        }
        result.push(level);
    }
    return result; // [[1], [2, 3], [4, 5]]
}

⭐ Classic: Maximum Depth

JavaScript
function maxDepth(root) {
    if (!root) return 0;  // Base case: empty tree
    
    let leftDepth = maxDepth(root.left);
    let rightDepth = maxDepth(root.right);
    
    return 1 + Math.max(leftDepth, rightDepth);
}
// The pattern: solve left, solve right, combine!
1
maxDepth(1): needs max of left subtree and right subtree
Recurse into node 2 and node 3
2
maxDepth(2): left=maxDepth(4)=1, right=maxDepth(5)=1 → 1+max(1,1) = 2
Left subtree depth = 2
3
maxDepth(3): left=null→0, right=null→0 → 1+max(0,0) = 1
Right subtree depth = 1
4
maxDepth(1) = 1 + max(2, 1) = 3
Answer: 3 ✅

Invert Binary Tree

JavaScript
function invertTree(root) {
    if (!root) return null;
    
    // Swap left and right children
    [root.left, root.right] = [root.right, root.left];
    
    // Recursively invert subtrees
    invertTree(root.left);
    invertTree(root.right);
    
    return root;
}

💡 Balanced vs Skewed

A balanced tree has height O(log n), giving fast operations. A skewed tree (every node has only one child) degrades to a linked list with height O(n). This is why self-balancing trees (AVL, Red-Black) exist.

⚠️ Common Mistakes

❌ Forgetting the base case (null check)
// ❌ Crashes when node is null
function height(node) {
    return 1 + Math.max(height(node.left), height(node.right));
    // node.left throws if node is null!
}

// ✅ Always check for null first
function height(node) {
    if (!node) return 0;  // BASE CASE
    return 1 + Math.max(height(node.left), height(node.right));
}
❌ Confusing height vs depth

Height: measured from a node DOWN to the farthest leaf. Depth: measured from the root DOWN to the node. The root has depth 0 and the tree's height equals the root's height.

❌ Using BFS when DFS is needed (or vice versa)

DFS (recursion/stack): depth-related problems, path problems. BFS (queue): level-related problems, shortest path in unweighted graphs.

💼 Interview Perspective

  • The recursive template: handle null → recurse left → recurse right → combine. This solves 80% of tree problems.
  • Know all four traversals: inorder, preorder, postorder, level-order.
  • Inorder traversal of a BST gives sorted order — this is a common trick.
  • Tree problems test recursion skills. If recursion is weak, practice tree problems.
  • Common follow-up: "Can you do it iteratively?" — use an explicit stack for DFS.

📝 Quick Revision Notes

  • Binary tree: each node has at most 2 children (left, right).
  • Traversals: Inorder (L,Root,R), Preorder (Root,L,R), Postorder (L,R,Root), Level-order (BFS).
  • Balanced tree height = O(log n). Skewed tree height = O(n).
  • Recursive pattern: base case (null) → recurse left → recurse right → combine.
  • DFS uses recursion/stack. BFS uses queue.

🃏 Flash Cards

What are the four tree traversal orders?
Inorder (Left→Root→Right), Preorder (Root→Left→Right), Postorder (Left→Right→Root), Level-order (BFS, level by level). Inorder on BST gives sorted order.
What is the height of a balanced binary tree with n nodes?
O(log n). A perfect binary tree with height h has 2^(h+1) - 1 nodes. So height ≈ log₂(n). This is why balanced BSTs give O(log n) search.
What is the recursive template for tree problems?
1. Base case: if node is null, return default value. 2. Recurse on left subtree. 3. Recurse on right subtree. 4. Combine results (e.g., max, sum, and/or). This solves 80% of tree problems.
