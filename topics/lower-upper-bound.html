Lower & Upper Bound

Phase 3 Medium

📖 What are Lower & Upper Bound?

In a sorted array, the lower bound of a target is the index of the first element ≥ target, and the upper bound is the index of the first element > target. These are the building blocks for many binary search variations.

Together they let you find: first occurrence, last occurrence, count of a value, insertion position, and range queries — all in O(log n).

🌍 Real-Life Analogy

Think of a library shelf with books sorted by page count. Lower bound of 200 pages = "Where does the 200-page section start?" (first book with ≥ 200 pages). Upper bound of 200 pages = "Where does the 200-page section end?" (first book with > 200 pages). The books between lower and upper bound are exactly the 200-page books!

⚙️ How It Works

Lower Bound (First element ≥ target)

JavaScript
function lowerBound(arr, target) {
    let lo = 0, hi = arr.length;
    
    while (lo < hi) {
        let mid = Math.floor((lo + hi) / 2);
        if (arr[mid] < target) {
            lo = mid + 1;   // Too small, search right
        } else {
            hi = mid;       // Could be the answer, search left
        }
    }
    return lo;  // Index of first element >= target
}

// [1, 3, 3, 5, 8, 8, 10]
// lowerBound(arr, 3) → 1  (first 3 is at index 1)
// lowerBound(arr, 4) → 3  (5 is first element >= 4)
// lowerBound(arr, 8) → 4  (first 8 is at index 4)

Upper Bound (First element > target)

JavaScript
function upperBound(arr, target) {
    let lo = 0, hi = arr.length;
    
    while (lo < hi) {
        let mid = Math.floor((lo + hi) / 2);
        if (arr[mid] <= target) {
            lo = mid + 1;   // Still ≤ target, search right
        } else {
            hi = mid;       // First element > target
        }
    }
    return lo;  // Index of first element > target
}

// [1, 3, 3, 5, 8, 8, 10]
// upperBound(arr, 3) → 3  (5 is first element > 3)
// upperBound(arr, 8) → 6  (10 is first element > 8)
// upperBound(arr, 10) → 7 (past end = all elements ≤ 10)

Applications Built on Lower/Upper Bound

JavaScript
// First occurrence of target
function firstOccurrence(arr, target) {
    let idx = lowerBound(arr, target);
    return (idx < arr.length && arr[idx] === target) ? idx : -1;
}

// Last occurrence of target
function lastOccurrence(arr, target) {
    let idx = upperBound(arr, target) - 1;
    return (idx >= 0 && arr[idx] === target) ? idx : -1;
}

// Count occurrences of target
function countOccurrences(arr, target) {
    return upperBound(arr, target) - lowerBound(arr, target);
}

// Insertion position (where target should be inserted)
function insertionIndex(arr, target) {
    return lowerBound(arr, target);  // Same as lower bound!
}

⚠️ Common Mistakes

❌ Confusing lower and upper bound conditions
// Lower bound: first element >= target
if (arr[mid] < target) lo = mid + 1;   // < not <=

// Upper bound: first element > target
if (arr[mid] <= target) lo = mid + 1;  // <= not <

// The ONLY difference is < vs <=
❌ Setting hi = arr.length - 1 instead of arr.length

If the target is larger than all elements, the answer should be arr.length (past the end). Using arr.length - 1 would miss this case. Always use hi = arr.length (exclusive).

❌ Forgetting to verify the result

Lower bound returns the insertion point, not necessarily the target. Always check if arr[result] === target when looking for exact matches.

💼 Interview Perspective

  • Master these two functions — they solve 90% of binary search interview problems.
  • "Find first/last occurrence" is a classic interview question — use lower/upper bound.
  • The difference between lower and upper bound is just < vs <= in the condition.
  • Count of a value = upperBound(target) - lowerBound(target).
  • JavaScript's built-in doesn't have these, but Python has bisect_left (lower) and bisect_right (upper).

📝 Quick Revision Notes

  • Lower bound: first index where arr[i] ≥ target. Condition: arr[mid] < target → go right.
  • Upper bound: first index where arr[i] > target. Condition: arr[mid] ≤ target → go right.
  • Both use hi = arr.length (can return past-end index).
  • First occurrence = lowerBound, Last occurrence = upperBound - 1.
  • Count = upperBound - lowerBound.

Just < vs <=. Everything else is identical.
How do you count occurrences of a value using lower and upper bound?
count = upperBound(target) - lowerBound(target). Upper bound gives the index after the last occurrence, lower bound gives the first occurrence. The difference is the count.
