Memoization (Top-Down DP)

Phase 12 Medium

📖 What is Memoization?

Memoization is the technique of caching the results of expensive recursive function calls and returning the cached result when the same inputs occur again. It transforms a recursive solution into a top-down DP solution.

Formula: Recursion + Cache = Memoization. Write the naive recursive solution first, then add a memo object/array to store computed results.

💡 Key Insight

Memoization is the easiest way to convert brute force into DP. You don't need to figure out the iteration order or table dimensions — just add a cache to your recursive solution. It only computes subproblems that are actually needed (unlike tabulation which computes all).

🌍 Real-Life Analogy

📒 Student's Formula Sheet

Instead of re-deriving the quadratic formula every time you need it, you write it on a formula sheet (cache). First time: derive it (compute). Every subsequent time: look at the sheet (cached lookup). That's memoization — compute once, look up forever.

  • Browser cache: First visit downloads the page. Subsequent visits load from cache.
  • Cooking: Calculate the recipe for 1 batch, then scale — don't re-derive each time.

⚙️ Memoization Patterns

Pattern 1: Climbing Stairs

JavaScript — Climbing Stairs with Memoization
// Step 1: Naive recursion — O(2^n) ❌
function climbNaive(n) {
    if (n <= 2) return n;
    return climbNaive(n - 1) + climbNaive(n - 2);
}

// Step 2: Add memoization — O(n) ✅
function climbStairs(n, memo = {}) {
    if (n <= 2) return n;
    if (memo[n] !== undefined) return memo[n];

    memo[n] = climbStairs(n - 1, memo) + climbStairs(n - 2, memo);
    return memo[n];
}

Pattern 2: House Robber

JavaScript — House Robber with Memoization
function rob(nums) {
    const memo = {};

    function dp(i) {
        if (i >= nums.length) return 0;
        if (memo[i] !== undefined) return memo[i];

        // Choice: rob this house (skip next) OR skip this house
        memo[i] = Math.max(
            nums[i] + dp(i + 2),  // Rob house i
            dp(i + 1)             // Skip house i
        );
        return memo[i];
    }

    return dp(0);
}

Pattern 3: Coin Change

JavaScript — Coin Change with Memoization
function coinChange(coins, amount) {
    const memo = {};

    function dp(remaining) {
        if (remaining === 0) return 0;
        if (remaining < 0) return Infinity;
        if (memo[remaining] !== undefined) return memo[remaining];

        let minCoins = Infinity;
        for (const coin of coins) {
            const result = dp(remaining - coin);
            minCoins = Math.min(minCoins, result + 1);
        }

        memo[remaining] = minCoins;
        return minCoins;
    }

    const result = dp(amount);
    return result === Infinity ? -1 : result;
}
// coinChange([1,5,10,25], 30) → 2 (25+5)

⚠️ Common Mistakes

❌ Stack overflow on deep recursion

Memoization is still recursive. For n > ~10,000, you may hit the call stack limit. Convert to tabulation (bottom-up) for large inputs.

❌ Wrong memo key

The memo key must capture ALL state that affects the result. If your function depends on both index and remaining capacity, the key should be `${index},${capacity}`, not just index.

❌ Mutating shared state without restoring

If you modify arrays/objects during recursion (like backtracking), make sure to undo changes before returning — or the memoized result will be based on corrupted state.

💼 Interview Perspective

  • Start with brute force recursion. Draw the recursion tree. Add memoization. This is the standard DP interview workflow.
  • Memoization is easier to write than tabulation — use it as your first DP attempt.
  • If interviewer asks to optimize space, convert to tabulation (bottom-up).
  • Memo keys: for 1D DP use index, for 2D use (i, j), for knapsack use (index, capacity).
  • Python's @lru_cache / @cache gives you memoization for free.

📝 Quick Revision Notes

  • Memoization = recursion + cache. Top-down DP approach.
  • Workflow: write recursion → identify overlapping subproblems → add memo.
  • Memo key must capture all state variables.
  • Advantage: only computes needed subproblems. Disadvantage: stack overflow risk.
  • Convert to tabulation for space optimization or to avoid stack limits.

🃏 Flash Cards

What is the difference between memoization and tabulation?
Memoization (top-down): recursive, caches results lazily, only computes needed subproblems, risk of stack overflow.
Tabulation (bottom-up): iterative, fills table eagerly, computes all subproblems, no stack risk, easier to optimize space.
How do you convert brute-force recursion to memoized DP?
1. Write the recursive solution. 2. Identify the state (function parameters that change). 3. Add a memo object/map. 4. Before computing, check if result is already cached. 5. After computing, store the result in memo. 6. Return cached result.
