Trie (Prefix Tree)

Phase 9 Medium

📖 What is a Trie?

A Trie (pronounced "try") is a tree-like data structure used for storing strings efficiently. Each node represents a single character, and paths from root to marked nodes form complete words.

Tries are the backbone of autocomplete, spell checkers, and IP routing. They allow O(L) lookup where L is the length of the word — independent of how many words are stored!

💡 Key Insight

Words that share a common prefix share the same path in the trie. "car", "card", "care" all share the path c → a → r. This is why it's called a prefix tree.

🌍 Real-Life Analogy

📱 Autocomplete on Your Phone Keyboard

When you type "hel" on your phone, it suggests "hello", "help", "helmet". How does it know? It stores all words in a trie! After following the path h → e → l, it explores all branches from that node to show you possible completions. Each keystroke narrows down the suggestions instantly.

  • Phone directory: Navigate through digits to reach a contact — each digit narrows results
  • File system: Folders inside folders — path from root to file is like a trie path
  • Dictionary index: Look up words by first letter, then second, then third...
  • Domain Name System: com → google → www follows a trie-like structure

⚙️ Trie Structure

Trie storing: ["app", "apple", "apt", "bat", "bad"]
(root) / \ a b | | p a / \ / \ p t* t* d* | l | e* * = isEndOfWord (marks a complete word) Paths: root → a → p → p = "app" ✗ (not marked, unless we mark it) root → a → p → p → l → e = "apple" ✅ root → a → p → t = "apt" ✅ root → b → a → t = "bat" ✅ root → b → a → d = "bad" ✅

🔧 Trie Implementation

JavaScript — Trie
class TrieNode {
    constructor() {
        this.children = {};      // char → TrieNode
        this.isEndOfWord = false;
    }
}

class Trie {
    constructor() {
        this.root = new TrieNode();
    }
    
    // Insert a word — O(L)
    insert(word) {
        let node = this.root;
        for (const char of word) {
            if (!node.children[char]) {
                node.children[char] = new TrieNode();
            }
            node = node.children[char];
        }
        node.isEndOfWord = true;
    }
    
    // Search for exact word — O(L)
    search(word) {
        let node = this.root;
        for (const char of word) {
            if (!node.children[char]) return false;
            node = node.children[char];
        }
        return node.isEndOfWord;
    }
    
    // Check if any word starts with prefix — O(L)
    startsWith(prefix) {
        let node = this.root;
        for (const char of prefix) {
            if (!node.children[char]) return false;
            node = node.children[char];
        }
        return true;  // Node exists = prefix exists
    }
    
    // Delete a word — O(L)
    delete(word) {
        this._deleteHelper(this.root, word, 0);
    }
    
    _deleteHelper(node, word, depth) {
        if (depth === word.length) {
            if (!node.isEndOfWord) return false;
            node.isEndOfWord = false;
            return Object.keys(node.children).length === 0;
        }
        const char = word[depth];
        if (!node.children[char]) return false;
        
        const shouldDelete = this._deleteHelper(node.children[char], word, depth + 1);
        if (shouldDelete) {
            delete node.children[char];
            return !node.isEndOfWord && Object.keys(node.children).length === 0;
        }
        return false;
    }
}

// Usage
const trie = new Trie();
trie.insert("apple");
trie.search("apple");     // true
trie.search("app");       // false
trie.startsWith("app");   // true
trie.insert("app");
trie.search("app");       // true

🔍 Dry Run: Insert "cat" and Search "car"

1
insert("cat"): At root, 'c' not in children → create node for 'c'
root → {c: {}}
2
Move to 'c' node, 'a' not in children → create node for 'a'
root → {c: {a: {}}}
3
Move to 'a' node, 't' not in children → create node for 't'
root → {c: {a: {t: {}}}}
4
Mark 't' node as isEndOfWord = true
"cat" inserted ✅
5
search("car"): Follow root → c → a. Next char 'r' not in 'a' node's children
Return false ❌ ("car" not found)

⏱️ Time & Space Complexity

OperationTimeWhy?
InsertO(L)L = length of word. Traverse/create L nodes
SearchO(L)Follow L edges, check isEndOfWord
startsWithO(L)Follow L edges, return true if path exists
DeleteO(L)Traverse L nodes, clean up unused nodes
SpaceO(N × L)N words of average length L. Shared prefixes save space
💡 Trie vs HashMap

HashMap search is O(L) too (hashing the key). But tries support prefix queries and autocomplete that HashMaps cannot do efficiently.

⚠️ Common Mistakes

❌ Confusing search() and startsWith()

search("app") returns true only if "app" was inserted as a complete word. startsWith("app") returns true if ANY word starts with "app" (even if "app" itself wasn't inserted).

❌ Not Marking End of Word

If you insert "apple" but forget to mark 'e' as end-of-word, search("apple") will fail. Always set isEndOfWord = true for the last character.

❌ Memory-Inefficient for Sparse Data

With 26 children per node (for lowercase English), a trie can use a lot of memory. Consider using a HashMap for children instead of a fixed-size array when the alphabet is large.

📝 Quick Revision Notes

  • Trie = tree of characters. Each path = a word or prefix.
  • All operations are O(L) where L = word length
  • Each node has children (map of char → node) and isEndOfWord flag
  • search checks isEndOfWord; startsWith just checks if path exists
  • Perfect for autocomplete, spell check, prefix matching

🃏 Flash Cards

What is the difference between search() and startsWith() in a Trie?
search("app") → true only if "app" is a complete word (isEndOfWord=true). startsWith("app") → true if ANY word begins with "app" (just checks path exists).
What is the time complexity of Trie operations and why?
O(L) for insert, search, delete — where L is the word length. Each character = one node traversal. Independent of how many words are stored (unlike hash collisions).
How would you implement wildcard search (e.g., "h.llo") in a Trie?
Use DFS/recursion. For regular characters, follow the specific child. For '.', recursively try ALL children at that level. Return true if any path leads to a match.
