Sparse Table

Phase 15 Hard

📖 What is a Sparse Table?

A Sparse Table is a data structure that answers Range Minimum Queries (RMQ) — or any idempotent function like min, max, GCD — on a static array in O(1) per query after O(n log n) preprocessing.

The key idea is to precompute answers for all ranges whose length is a power of 2. Any range [l, r] can then be covered by overlapping two precomputed ranges of equal power-of-2 length. Since min/max/GCD are idempotent (overlapping answers don't cause errors), this overlap is fine.

📚 Real-Life Analogy: Library Floor Maps

Imagine a library where each floor has maps showing the shortest route to the exit for sections of 1, 2, 4, 8... rooms. To find the shortest exit path across any range of rooms, you just overlap two precomputed maps. No recalculation needed — you look up two maps and pick the shorter route. That's a sparse table: precompute power-of-2 sized answers, combine two to answer any range instantly.

⚙️ How Sparse Table Works

Sparse Table for array [3, 1, 4, 1, 5, 9, 2, 6] — Range Min
Array: index 0 1 2 3 4 5 6 7 value 3 1 4 1 5 9 2 6 Precompute table[k][i] = min of 2^k elements starting at index i: k=0 (len 1): [3, 1, 4, 1, 5, 9, 2, 6] ← each element itself k=1 (len 2): [1, 1, 1, 1, 5, 2, 2, -] ← min(pair) k=2 (len 4): [1, 1, 1, 1, 2, 2, -, -] ← min(quadruple) k=3 (len 8): [1, -, -, -, -, -, -, -] ← min(entire array) Query min(2, 6): range length = 5, k = floor(log2(5)) = 2 Overlap: table[2][2] = min(2..5) = 1 table[2][3] = min(3..6) = 1 Answer: min(1, 1) = 1 ✅ The two ranges [2,5] and [3,6] overlap at [3,5] — but since min is idempotent, the overlap doesn't matter!

🔧 Implementation

JavaScript — Sparse Table (Range Min Query)
class SparseTable {
    constructor(arr) {
        this.n = arr.length;
        this.LOG = Math.floor(Math.log2(this.n)) + 1;
        // table[k][i] = min of 2^k elements starting at i
        this.table = Array.from({ length: this.LOG }, () => new Array(this.n));
        this.buildLog = new Array(this.n + 1).fill(0);

        // Precompute floor(log2(i)) for all i
        for (let i = 2; i <= this.n; i++) {
            this.buildLog[i] = this.buildLog[i >> 1] + 1;
        }

        // Base case: k = 0, ranges of length 1
        for (let i = 0; i < this.n; i++) {
            this.table[0][i] = arr[i];
        }

        // Fill table using DP: table[k][i] = min(table[k-1][i], table[k-1][i + 2^(k-1)])
        for (let k = 1; k < this.LOG; k++) {
            for (let i = 0; i + (1 << k) - 1 < this.n; i++) {
                this.table[k][i] = Math.min(
                    this.table[k - 1][i],
                    this.table[k - 1][i + (1 << (k - 1))]
                );
            }
        }
    }

    // Query min in range [l, r] — O(1)
    query(l, r) {
        const len = r - l + 1;
        const k = this.buildLog[len];
        return Math.min(this.table[k][l], this.table[k][r - (1 << k) + 1]);
    }
}

// Usage
const st = new SparseTable([3, 1, 4, 1, 5, 9, 2, 6]);
console.log(st.query(0, 7)); // 1 (min of entire array)
console.log(st.query(2, 6)); // 1 (min of [4,1,5,9,2])
console.log(st.query(4, 7)); // 2 (min of [5,9,2,6])

⚠️ Common Mistakes

❌ Using Sparse Table for non-idempotent functions

Sparse Table's O(1) query only works for idempotent functions (min, max, GCD, OR, AND). For sum, the overlapping ranges would double-count elements. Use prefix sums for static range sum.

❌ Off-by-one in second range index

The second range starts at r - 2^k + 1, not r - 2^k. Getting this wrong shifts the range and gives incorrect answers.

❌ Trying to update a Sparse Table

Sparse Tables are static. Any element change requires rebuilding in O(n log n). If updates are needed, use a Segment Tree instead.

💼 Interview Perspective

  • Sparse Table is the optimal answer when asked for O(1) range min/max queries on a static array.
  • Often used as a building block in LCA algorithms (via Euler tour + RMQ).
  • Know the limitation: no updates. If the interviewer asks about updates, pivot to Segment Tree.
  • Understand idempotency: f(a, a) = a. This is why overlapping ranges work for min/max but not sum.
  • Range GCD queries are another common application.

📝 Quick Revision Notes

  • Sparse Table: static RMQ in O(1) query, O(n log n) preprocessing.
  • table[k][i] = answer for range [i, i + 2^k - 1].
  • Query: find k = floor(log2(len)), combine table[k][l] and table[k][r - 2^k + 1].
  • Only works for idempotent operations (min, max, GCD, OR, AND).
  • No update support — use Segment Tree if updates are needed.

🃏 Flash Cards

Why does Sparse Table achieve O(1) queries?
Any range [l, r] can be covered by exactly two overlapping power-of-2 ranges. Since the query function is idempotent (min, max, GCD), overlapping doesn't cause errors. We just look up two precomputed values and combine them.
What does "idempotent" mean in the context of Sparse Table?
An operation f is idempotent if f(a, a) = a. Examples: min(3,3)=3, max(5,5)=5, GCD(6,6)=6. This means overlapping ranges don't corrupt the answer. Sum is NOT idempotent: sum(3,3) = 6 ≠ 3.
