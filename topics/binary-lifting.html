Binary Lifting

Phase 15 Hard

📖 What is Binary Lifting?

Binary Lifting is a technique for efficiently finding the k-th ancestor of any node in a tree in O(log n) time. It also provides an elegant solution for Lowest Common Ancestor (LCA) queries in O(log n).

The idea is to precompute, for every node, its 1st, 2nd, 4th, 8th, ... (powers of 2) ancestor. Then, to jump k levels up, decompose k into binary and chain the precomputed jumps — just like binary representation of numbers.

🏢 Real-Life Analogy: Express Elevator

Imagine a skyscraper where regular elevators go 1 floor at a time, but express elevators jump 1, 2, 4, 8, 16... floors. To go up 13 floors, you take the 8-floor express, then the 4-floor express, then the 1-floor regular elevator (13 = 8 + 4 + 1). Instead of 13 individual hops, you make just 3 jumps. Binary Lifting uses the same trick in trees!

⚙️ How Binary Lifting Works

Binary Lifting Table for a Tree (0-indexed)
Tree: 0 / | \ 1 2 3 / \ | 4 5 6 / 7 up[k][v] = 2^k-th ancestor of node v v: 0 1 2 3 4 5 6 7 up[0] (1): -1 0 0 0 1 1 3 4 ← parent up[1] (2): -1 -1 -1 -1 0 0 0 1 ← grandparent up[2] (4): -1 -1 -1 -1 -1 -1 -1 0 ← 4th ancestor up[3] (8): -1 -1 -1 -1 -1 -1 -1 -1 ← too far kth-ancestor(7, 3): k=3 → binary 011 Jump 2^0=1: up[0][7] = 4 Jump 2^1=2: up[1][4] = 0 Result: node 0 ✅ (7→4→1→0, 3 hops) LCA(7, 6): Depths: depth[7]=3, depth[6]=2 Equalize: lift 7 by 1 → node 4. Now both at depth 2. Binary search: up[1][4]=0, up[1][6]=0 → same, try smaller up[0][4]=1, up[0][6]=3 → different, lift both Now at nodes 1 and 3. LCA = parent of 1 (or 3) = 0 ✅

🔧 Implementation

JavaScript — Binary Lifting (Kth Ancestor + LCA)
class BinaryLifting {
    constructor(n, edges, root = 0) {
        this.n = n;
        this.LOG = Math.ceil(Math.log2(n)) + 1;
        this.depth = new Array(n).fill(0);
        // up[k][v] = 2^k-th ancestor of v (-1 if none)
        this.up = Array.from({ length: this.LOG }, () => new Array(n).fill(-1));

        // Build adjacency list
        const adj = Array.from({ length: n }, () => []);
        for (const [u, v] of edges) {
            adj[u].push(v);
            adj[v].push(u);
        }

        // BFS to set parent and depth
        const visited = new Array(n).fill(false);
        const queue = [root];
        visited[root] = true;
        while (queue.length > 0) {
            const u = queue.shift();
            for (const v of adj[u]) {
                if (!visited[v]) {
                    visited[v] = true;
                    this.up[0][v] = u;       // Parent
                    this.depth[v] = this.depth[u] + 1;
                    queue.push(v);
                }
            }
        }

        // Fill binary lifting table
        for (let k = 1; k < this.LOG; k++) {
            for (let v = 0; v < n; v++) {
                const mid = this.up[k - 1][v];
                this.up[k][v] = mid === -1 ? -1 : this.up[k - 1][mid];
            }
        }
    }

    // Find k-th ancestor of node v — O(log n)
    kthAncestor(v, k) {
        for (let i = 0; i < this.LOG && v !== -1; i++) {
            if ((k >> i) & 1) {
                v = this.up[i][v];
            }
        }
        return v;
    }

    // Find LCA of nodes u and v — O(log n)
    lca(u, v) {
        // Ensure u is deeper
        if (this.depth[u] < this.depth[v]) [u, v] = [v, u];

        // Equalize depths
        let diff = this.depth[u] - this.depth[v];
        for (let i = 0; i < this.LOG; i++) {
            if ((diff >> i) & 1) u = this.up[i][u];
        }

        if (u === v) return u;

        // Binary lift both until just below LCA
        for (let i = this.LOG - 1; i >= 0; i--) {
            if (this.up[i][u] !== this.up[i][v]) {
                u = this.up[i][u];
                v = this.up[i][v];
            }
        }
        return this.up[0][u]; // Parent is LCA
    }
}

// Usage
const edges = [[0,1],[0,2],[0,3],[1,4],[1,5],[3,6],[4,7]];
const bl = new BinaryLifting(8, edges, 0);
console.log(bl.kthAncestor(7, 3)); // 0 (7→4→1→0)
console.log(bl.lca(7, 6));          // 0
console.log(bl.lca(7, 5));          // 1

⚠️ Common Mistakes

❌ Not equalizing depths before LCA binary search

The binary search phase assumes u and v are at the same depth. If you skip the equalization step, the algorithm produces wrong answers.

❌ Wrong LOG value

LOG should be ceil(log2(n)) + 1 to handle all possible ancestor jumps. Using floor(log2(n)) may miss the highest jump needed.

❌ Forgetting the u === v check after equalization

After equalizing depths, if u === v, the LCA is u itself. Skipping this check and proceeding to binary search causes incorrect results.

💼 Interview Perspective

  • Binary Lifting is the standard technique for LCA queries in coding interviews and competitive programming.
  • LeetCode 1483 "Kth Ancestor of a Tree Node" directly tests binary lifting.
  • Also useful for path queries: distance(u,v) = depth[u] + depth[v] − 2×depth[LCA(u,v)].
  • Alternative LCA methods: Euler Tour + Sparse Table (O(1) query), Tarjan's offline LCA.
  • The same "power of 2 jumps" idea appears in skip lists and binary indexed trees.

📝 Quick Revision Notes

  • up[k][v] = 2^k-th ancestor of v. up[k][v] = up[k-1][up[k-1][v]].
  • Kth ancestor: decompose k in binary, chain power-of-2 jumps.
  • LCA: equalize depths, then binary search upward until ancestors match.
  • Preprocessing O(n log n), queries O(log n).
  • Space: O(n log n) for the lifting table.

🃏 Flash Cards

How is the binary lifting table built?
up[0][v] = parent(v) (base case). For k ≥ 1: up[k][v] = up[k-1][up[k-1][v]]. Jump 2^k = two jumps of 2^(k-1). Like squaring in fast exponentiation.
What are the two phases of LCA via binary lifting?
Phase 1: Equalize depths — lift the deeper node up until both nodes are at the same level. Phase 2: Binary search — try jumps from largest to smallest; if ancestors differ, take the jump; at the end, the LCA is the parent of the current positions.
