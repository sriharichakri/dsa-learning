Interval Scheduling

Phase 11 Medium

📖 What is Interval Scheduling?

Interval Scheduling problems involve selecting, merging, or counting intervals (ranges with start and end points). The classic problem: given a set of intervals, find the maximum number of non-overlapping intervals.

The greedy strategy: sort by end time, then greedily select intervals that don't overlap with the last selected one. This is provably optimal.

💡 Key Insight

Sorting by end time (not start time!) is the key. By always picking the interval that ends earliest, you leave the most room for future intervals. This greedy choice leads to the global optimum.

🌍 Real-Life Analogy

📅 Conference Room Booking

You're a conference room manager with many meeting requests. Each meeting has a start and end time. To fit the most meetings in one room, always schedule the meeting that ends earliest — it frees up the room soonest for the next meeting.

  • TV show scheduling: Watch the most shows by picking those ending earliest.
  • Job interviews: Schedule the most candidates by picking shortest/earliest-ending slots.
  • Classroom allocation: Assign classes to rooms with minimum rooms needed.

⚙️ Key Interval Problems

1. Merge Intervals

JavaScript — Merge Intervals (LC 56)
function merge(intervals) {
    intervals.sort((a, b) => a[0] - b[0]); // Sort by start time
    const merged = [intervals[0]];

    for (let i = 1; i < intervals.length; i++) {
        const last = merged[merged.length - 1];
        if (intervals[i][0] <= last[1]) {
            // Overlapping — extend the end
            last[1] = Math.max(last[1], intervals[i][1]);
        } else {
            // Non-overlapping — add new interval
            merged.push(intervals[i]);
        }
    }
    return merged;
}
// merge([[1,3],[2,6],[8,10],[15,18]]) → [[1,6],[8,10],[15,18]]

2. Non-overlapping Intervals (Maximum intervals to keep)

JavaScript — Erase Overlap Intervals (LC 435)
function eraseOverlapIntervals(intervals) {
    intervals.sort((a, b) => a[1] - b[1]); // Sort by END time
    let count = 0;
    let lastEnd = -Infinity;

    for (const [start, end] of intervals) {
        if (start >= lastEnd) {
            // No overlap — keep this interval
            lastEnd = end;
        } else {
            // Overlap — remove this interval (greedy: keep earlier ending)
            count++;
        }
    }
    return count;
}
// eraseOverlapIntervals([[1,2],[2,3],[3,4],[1,3]]) → 1 (remove [1,3])

3. Meeting Rooms II (Minimum rooms needed)

JavaScript — Meeting Rooms II (LC 253)
function minMeetingRooms(intervals) {
    const starts = intervals.map(i => i[0]).sort((a, b) => a - b);
    const ends = intervals.map(i => i[1]).sort((a, b) => a - b);
    
    let rooms = 0, maxRooms = 0;
    let s = 0, e = 0;

    while (s < starts.length) {
        if (starts[s] < ends[e]) {
            rooms++;  // A meeting starts before another ends → need new room
            s++;
        } else {
            rooms--;  // A meeting ended → free up a room
            e++;
        }
        maxRooms = Math.max(maxRooms, rooms);
    }
    return maxRooms;
}
// minMeetingRooms([[0,30],[5,10],[15,20]]) → 2

⚠️ Common Mistakes

❌ Sorting by start time for non-overlapping intervals

For maximum non-overlapping, sort by end time. Sorting by start time with a greedy approach doesn't give the optimal solution.

❌ Using < instead of <= for overlap check

Are [1,2] and [2,3] overlapping? Depends on the problem! Some treat touching endpoints as non-overlapping (>=), others as overlapping (>). Read carefully.

❌ Not handling merge correctly

When merging, take max(last[1], curr[1]) for the end, not just curr[1]. The current interval might end before the last one.

💼 Interview Perspective

  • Interval problems are extremely common in interviews — know all three patterns.
  • First instinct: sort the intervals. Then decide: by start or end?
  • Merge → sort by start. Non-overlapping max → sort by end. Min rooms → event sweepline.
  • "Meeting Rooms" has I (can attend all?) and II (how many rooms?). Know both.
  • Line sweep (event-based) approach is powerful: +1 at start, -1 at end, max prefix sum = max overlap.

📝 Quick Revision Notes

  • Merge intervals: sort by start, extend end if overlap.
  • Max non-overlapping: sort by END, greedily keep non-overlapping.
  • Min rooms: separate starts/ends, two-pointer sweep.
  • All interval problems start with sorting — O(n log n) dominates.
  • Always clarify: are touching endpoints [1,2],[2,3] considered overlapping?

🃏 Flash Cards

Why sort by end time for maximum non-overlapping intervals?
Sorting by end time ensures we pick the interval that finishes earliest, leaving the most room for subsequent intervals. This greedy choice is provably optimal — it never excludes a better solution.
How do you find the minimum number of meeting rooms?
Separate start times and end times, sort both. Use two pointers: if next start < next end → need a new room (rooms++). Otherwise → a room freed up (rooms--). Track the maximum rooms at any point.
