Monotonic Queue

Phase 8 Hard

📖 What is a Monotonic Queue?

A Monotonic Queue is a deque that maintains elements in monotonically increasing or decreasing order. Unlike a monotonic stack (LIFO), a monotonic queue supports removing from both ends: remove from front when elements expire (leave the window), remove from back to maintain monotonicity.

It's the key to solving sliding window min/max in O(n).

🌍 Real-Life Analogy

Imagine a VIP line where only increasingly important people can stand. When a new VIP arrives, everyone less important behind them leaves (back removal). When the first person's ticket expires, they leave from the front (front removal). The front of the line is always the most important active person!

⚙️ How It Works

JavaScript
// Sliding Window Maximum — O(n)
function maxSlidingWindow(nums, k) {
    const deque = [];  // Stores indices, values are decreasing
    const result = [];
    
    for (let i = 0; i < nums.length; i++) {
        // 1. Remove expired front (out of window)
        if (deque.length && deque[0] < i - k + 1) {
            deque.shift();
        }
        
        // 2. Remove from back: smaller elements will never be max
        while (deque.length && nums[deque[deque.length - 1]] <= nums[i]) {
            deque.pop();
        }
        
        // 3. Add current
        deque.push(i);
        
        // 4. Record answer when window is full
        if (i >= k - 1) {
            result.push(nums[deque[0]]);  // Front = max
        }
    }
    return result;
}

// Sliding Window Minimum — same idea, flip the comparison
function minSlidingWindow(nums, k) {
    const deque = [];
    const result = [];
    
    for (let i = 0; i < nums.length; i++) {
        if (deque.length && deque[0] < i - k + 1) deque.shift();
        while (deque.length && nums[deque[deque.length - 1]] >= nums[i]) {
            deque.pop();
        }
        deque.push(i);
        if (i >= k - 1) result.push(nums[deque[0]]);
    }
    return result;
}

DP Optimization with Monotonic Queue

JavaScript
// Jump Game VI: max score path, can jump 1..k steps
// dp[i] = nums[i] + max(dp[i-k], ..., dp[i-1])
function maxResult(nums, k) {
    const n = nums.length;
    const dp = new Array(n).fill(0);
    dp[0] = nums[0];
    const deque = [0]; // Monotonic decreasing on dp values
    
    for (let i = 1; i < n; i++) {
        // Remove expired
        if (deque[0] < i - k) deque.shift();
        
        dp[i] = nums[i] + dp[deque[0]]; // Best in window
        
        // Maintain decreasing monotonicity
        while (deque.length && dp[deque[deque.length - 1]] <= dp[i]) {
            deque.pop();
        }
        deque.push(i);
    }
    return dp[n - 1];
}

⚠️ Common Mistakes

❌ Confusing monotonic stack vs monotonic queue

Monotonic stack: removes from back only (next greater/smaller element). Monotonic queue: removes from BOTH ends (front for expiry, back for monotonicity). Queue is for sliding window problems.

❌ Storing values instead of indices

Store indices so you can check if elements have expired (left the window). Condition: deque[0] < i - k + 1.

💼 Interview Perspective

  • Monotonic queue = deque + monotonic property. It's the O(n) solution for sliding window min/max.
  • Key signal: "sliding window" + "maximum/minimum" → monotonic queue.
  • Also optimizes certain DP problems: dp[i] = f(max/min over dp[i-k..i-1]).
  • Each element enters and leaves the deque at most once → amortized O(1) per element.
  • If interviewer asks about sliding window max, start with heap O(n log n), then optimize to deque O(n).

📝 Quick Revision Notes

  • Monotonic queue = deque maintaining decreasing (for max) or increasing (for min) order.
  • Remove from front: expired elements. Remove from back: maintain monotonicity.
  • O(n) time, O(k) space for sliding window max/min.
  • Store indices, not values. Front = answer for current window.

🃏 Flash Cards

What's the difference between monotonic stack and monotonic queue?
Monotonic stack: removes from back only. Used for next greater/smaller element (no window constraint). Monotonic queue: removes from BOTH ends — back for monotonicity, front for window expiry. Used for sliding window problems.
Why is monotonic queue O(n) for sliding window max?
Each element is pushed once (at most) and popped once (at most) across the entire algorithm. Even though there's a while loop inside the for loop, the total pops across all iterations is bounded by n. Amortized O(1) per element.
