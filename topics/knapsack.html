0/1 Knapsack

Phase 12 Medium

📖 What is the 0/1 Knapsack Problem?

Given a set of items, each with a weight and a value, determine the maximum value you can carry in a knapsack with a weight capacity. Each item can either be taken whole (1) or left (0) — no fractions allowed.

This is the foundational Dynamic Programming problem. Understanding it unlocks a whole family of DP problems: subset sum, partition, coin change, and many more.

💡 Key Insight

For each item, you have exactly two choices: include it or exclude it. The optimal solution is the better of these two choices. This "include/exclude" pattern is the essence of the 0/1 Knapsack and appears in countless DP problems.

🌍 Real-Life Analogy

🧳 Packing a Suitcase with Weight Limit

You're going on a trip with a suitcase that can hold at most 15 kg. You have a laptop (3kg, value 9), camera (4kg, value 7), books (5kg, value 4), clothes (2kg, value 5), and shoes (6kg, value 6). You can't take everything — which combination gives you the most value without exceeding 15 kg?

  • Investment portfolio: Limited budget, different returns — choose the best combination
  • Cargo shipping: Truck capacity limit, maximize the value of goods shipped
  • Course selection: Limited time slots (weight), different credit values

⚙️ State Definition and Transition

State: dp[i][w] = maximum value achievable using items 0 to i with capacity w.

Transition:

If item i is too heavy (weight[i] > w):
    dp[i][w] = dp[i-1][w]              // Can't include item i

Otherwise, take the better of:
    dp[i][w] = max(
        dp[i-1][w],                     // Exclude item i
        dp[i-1][w - weight[i]] + value[i]  // Include item i
    )
DP Table: Items [(wt:2, val:3), (wt:3, val:4), (wt:4, val:5), (wt:5, val:6)], Capacity = 5
Capacity → 0 1 2 3 4 5 Item 0 (w=2, v=3) [ 0, 0, 3, 3, 3, 3] Item 1 (w=3, v=4) [ 0, 0, 3, 4, 4, 7] ← max(3, 0+4)=4 at w=3 Item 2 (w=4, v=5) [ 0, 0, 3, 4, 5, 7] max(3, dp[0][2]+4)=7 at w=5 Item 3 (w=5, v=6) [ 0, 0, 3, 4, 5, 7] Answer: dp[3][5] = 7 (Take items 0 and 1: weight=2+3=5, value=3+4=7) ✅
JavaScript — 0/1 Knapsack (2D DP)
function knapsack(weights, values, capacity) {
    const n = weights.length;
    // dp[i][w] = max value using items 0..i with capacity w
    const dp = Array.from({ length: n + 1 }, () => new Array(capacity + 1).fill(0));
    
    for (let i = 1; i <= n; i++) {
        for (let w = 0; w <= capacity; w++) {
            dp[i][w] = dp[i - 1][w]; // Don't take item i-1
            
            if (weights[i - 1] <= w) {
                dp[i][w] = Math.max(
                    dp[i][w],
                    dp[i - 1][w - weights[i - 1]] + values[i - 1]  // Take item i-1
                );
            }
        }
    }
    return dp[n][capacity];
}

⚡ Space-Optimized (1D Array)

Since each row only depends on the previous row, we can use a single 1D array. But we must iterate capacity in reverse to avoid using updated values.

JavaScript — Space-Optimized O(capacity)
function knapsackOptimized(weights, values, capacity) {
    const dp = new Array(capacity + 1).fill(0);
    
    for (let i = 0; i < weights.length; i++) {
        // Iterate BACKWARDS to avoid using item i twice!
        for (let w = capacity; w >= weights[i]; w--) {
            dp[w] = Math.max(dp[w], dp[w - weights[i]] + values[i]);
        }
    }
    return dp[capacity];
}
💡 Why Iterate Backwards?

If we iterate forwards, dp[w - weight[i]] might already be updated in this round, effectively allowing us to use item i multiple times (that's the Unbounded Knapsack variant). Backwards ensures each item is used at most once.

💡 Pseudo-polynomial Time

O(n × W) is called "pseudo-polynomial" because W is not the input size but the input value. If W is huge (like 10⁹), DP is too slow and you might need approximation algorithms.

⚠️ Common Mistakes

❌ Iterating Forward in 1D 0/1 Knapsack

Forward iteration reuses updated values, turning it into Unbounded Knapsack. For 0/1, always iterate capacity backwards.

❌ Off-by-One in Table Indices

dp has n+1 rows and W+1 columns. Item i in the table corresponds to weights[i-1] and values[i-1] in the input arrays.

❌ Confusing 0/1 with Unbounded Knapsack

If items can be reused → Unbounded (inner loop forward). If each item used once → 0/1 (inner loop backward). This is the #1 source of bugs.

💼 Interview Tips

  • Recognize the pattern: "Choose/don't choose", "subset with constraint", "partition into groups" → 0/1 Knapsack variant
  • State clearly: Define what dp[i][w] represents before coding
  • Optimize space: Always mention the 1D optimization — shows depth of understanding
  • Subset Sum = Knapsack: "Can we find a subset with sum X?" is knapsack with values = weights
  • Target Sum: Count of ways = change dp transition from max to sum

🏗️ Where Knapsack Is Used

  • Resource allocation: Allocate budget across projects for maximum return
  • Cargo loading: Maximize value of goods in a container with weight limit
  • Portfolio optimization: Select investments within budget for best returns
  • Cutting stock: Cut material to fill orders with minimum waste
  • Cryptography: Knapsack-based encryption schemes

📝 Quick Revision Notes

  • 0/1 Knapsack: for each item, include or exclude. dp[i][w] = max value
  • Transition: dp[i][w] = max(dp[i-1][w], dp[i-1][w-wt[i]] + val[i])
  • Time: O(n × W) | Space: O(W) with 1D optimization
  • 1D optimization: iterate capacity BACKWARDS for 0/1, FORWARDS for unbounded
  • Subset Sum = Knapsack where value equals weight, target = desired sum

🃏 Flash Cards

Why must we iterate capacity backwards in 1D 0/1 Knapsack?
To prevent using an item multiple times. Forward iteration would use already-updated dp values (from the current item's row), effectively allowing reuse. Backwards ensures we use values from the "previous row" (before current item).
How is Partition Equal Subset Sum related to Knapsack?
Calculate total sum. If odd, return false. If even, it becomes: "Can we find a subset with sum = totalSum/2?" — which is exactly the Subset Sum problem (a 0/1 Knapsack variant with boolean dp).
What is the transition formula for 0/1 Knapsack?
dp[i][w] = max(dp[i-1][w], dp[i-1][w-weight[i]] + value[i]). First term: skip item i. Second term: take item i (subtract its weight, add its value). Pick the maximum.
