Singly Linked List

Phase 6 Medium

📖 What is a Singly Linked List?

A Singly Linked List is a linear data structure where each element (called a node) contains a value and a pointer to the next node. Unlike arrays, nodes are NOT stored in contiguous memory — they can be scattered anywhere, connected by pointers.

💡 Key Insight

Linked lists trade random access (arrays' strength) for efficient insertion/deletion. Adding or removing a node takes O(1) if you have a reference to the position — no shifting needed. But finding that position takes O(n) since you must traverse from the head.

🌍 Real-Life Analogy

🗺️ Treasure Hunt with Clues

Imagine a treasure hunt where each location has a clue pointing to the next location. You start at clue #1 (the head). Each clue tells you where to go next. The final clue says "STOP" (points to null). You can't jump to clue #5 directly — you must follow the chain from the beginning.

  • Train carriages: Each carriage is connected to the next. You walk through them in order. You can detach a carriage (delete) or insert a new one without moving the others.
  • Scavenger hunt: Each clue leads to the next clue's location.
  • Conga line: Each person holds the shoulders of the person in front. Adding someone = they grab the person ahead.

🔧 Common Operations

Insert at Head — O(1)

JavaScript
insertAtHead(value) {
    const newNode = new Node(value);
    newNode.next = this.head;  // Point new node to current head
    this.head = newNode;       // Update head to new node
    this.size++;
}

Insert at Tail — O(n)

JavaScript
insertAtTail(value) {
    const newNode = new Node(value);
    if (!this.head) {
        this.head = newNode;
    } else {
        let current = this.head;
        while (current.next) {    // Walk to the last node
            current = current.next;
        }
        current.next = newNode;   // Link last node to new node
    }
    this.size++;
}

Delete a Node — O(n) to find, O(1) to remove

JavaScript
delete(value) {
    if (!this.head) return;
    if (this.head.value === value) {
        this.head = this.head.next;  // Remove head
        this.size--;
        return;
    }
    let current = this.head;
    while (current.next && current.next.value !== value) {
        current = current.next;
    }
    if (current.next) {
        current.next = current.next.next; // Skip the node
        this.size--;
    }
}

Traversal — O(n)

JavaScript
traverse() {
    let current = this.head;
    const values = [];
    while (current) {
        values.push(current.value);
        current = current.next;
    }
    return values; // [10, 20, 30]
}

⭐ Classic: Reverse a Linked List

The most commonly asked linked list interview question.

JavaScript
function reverseList(head) {
    let prev = null;
    let current = head;
    
    while (current) {
        let next = current.next;  // Save next
        current.next = prev;      // Reverse pointer
        prev = current;           // Move prev forward
        current = next;           // Move current forward
    }
    return prev; // New head
}

⚠️ Common Mistakes

❌ Losing reference to the list (forgetting to save next)
// ❌ When reversing, if you change current.next 
//    without saving current.next first, you lose the rest!
current.next = prev;  // ← rest of list is now lost!

// ✅ Save next BEFORE reversing
let next = current.next;  // Save reference
current.next = prev;      // Now safe to reverse
❌ Not handling null/empty list
// ❌ NullPointerException!
head.next.value; // What if head is null? 💥

// ✅ Always check for null
if (head && head.next) { ... }
❌ Forgetting to update head after operations

When deleting the head node or reversing the list, you must update the head pointer to the new first node.

💼 Interview Perspective

  • Reversing a linked list is asked in ~70% of linked list interviews. Know it cold.
  • Use a dummy node before the head to simplify edge cases (inserting/deleting at head).
  • Fast-slow pointers: Find middle, detect cycle, find cycle start.
  • Draw the nodes on paper during the interview — pointer manipulation is error-prone without visualization.
  • Know the trade-offs: linked lists are better for frequent insertions/deletions; arrays are better for random access.

📝 Quick Revision Notes

  • Node = {value, next}. Head points to first node. Last node.next = null.
  • Insert at head: O(1). Insert at tail: O(n) without tail pointer.
  • No random access — must traverse from head to reach any node.
  • Reverse: use three pointers (prev, current, next). Save next before reversing.
  • Dummy node simplifies edge cases. Fast-slow finds middle in O(n).

🃏 Flash Cards

Why is insertion at the head of a linked list O(1)?
Create a new node, point its next to the current head, update head to the new node. Only 2 pointer operations — no elements need to be shifted (unlike arrays).
How do you reverse a singly linked list?
Use three pointers: prev (starts null), current (starts at head), next (saved each iteration). For each node: save next, reverse pointer (current.next = prev), advance prev and current. Return prev as new head.
How do you find the middle of a linked list in one pass?
Use fast-slow pointers. Slow moves 1 step, fast moves 2 steps. When fast reaches the end, slow is at the middle. O(n) time, O(1) space.
