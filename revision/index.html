🎯 Pattern → Algorithm Quick Map

  • "Sorted array + search" → Binary Search
  • "Contiguous subarray" → Sliding Window or Prefix Sum
  • "Find pair with sum" → Two Pointers (sorted) or HashMap
  • "All subsets/permutations" → Backtracking
  • "Shortest path" → BFS (unweighted), Dijkstra (weighted)
  • "Connected components" → DFS/BFS or Union-Find
  • "Overlapping subproblems" → Dynamic Programming
  • "Top K / Kth element" → Heap or QuickSelect
  • "Next greater element" → Monotonic Stack
  • "Frequency / existence" → HashMap / HashSet
  • "Prefix search" → Trie
  • "Cycle detection" → Fast-Slow Pointer

📐 Key Formulas

  • Sum 1 to n: n(n+1)/2
  • Number of subarrays: n(n+1)/2
  • Number of subsets: 2ⁿ
  • Permutations of n: n!
  • Combinations C(n,k): n! / (k!(n-k)!)
  • Height of balanced BST: ⌊log₂(n)⌋
  • Nodes in complete binary tree: 2^(h+1) - 1
  • Amortized push for dynamic array: O(1)
  • log₂(10⁶) ≈ 20, log₂(10⁹) ≈ 30

🃏 Flash Cards — Key Concepts

When to use HashMap vs TreeMap?
HashMap: O(1) lookup, unordered.
TreeMap: O(log n) lookup, sorted keys.
Use TreeMap when you need sorted iteration.
DFS vs BFS: when to use which?
DFS: path finding, cycle detection, topological sort, backtracking.
BFS: shortest path (unweighted), level-order, minimum steps.
Memoization vs Tabulation?
Memoization: top-down, recursive, stores computed results, only computes needed states.
Tabulation: bottom-up, iterative, fills table systematically, no recursion overhead.
Stable vs Unstable sort?
Stable: preserves relative order of equal elements (Merge, Insertion, Counting sort).
Unstable: may change order of equal elements (Quick, Heap, Selection sort).
